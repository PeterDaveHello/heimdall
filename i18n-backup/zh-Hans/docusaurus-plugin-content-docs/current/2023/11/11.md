# 2023-11-11

## [定制工具链：打造人工智能产品的更好方法](https://www.builder.io/blog/build-ai)

- 现有的人工智能模型（如 ChatGPT）存在局限性和缺点，包括缺乏差异化、成本高、处理速度慢以及定制化程度有限。
- 作者建议创建一个定制工具链，将微调模型、定制编译器和定制训练模型结合起来，用于构建人工智能产品。
- 重要的是，要从正常的编程实践开始，只在必要时才纳入人工智能模型。
- 作者分享了他们构建 Visual Copilot（一种将 Figma 设计转换为高质量代码的人工智能产品）的经验，并强调了控制自己的模型并不断加以改进的好处。
- 隐私控制也是定制方法的一个优势。
- 作者建议谨慎使用人工智能，并将重点放在它能提供最大价值的领域。

### [反应](https://news.ycombinator.com/item?id=38221552)

- 重点是利用人工智能技术，特别是聊天机器人，来解决用户的实际问题，而不是为了使用新兴技术而使用这些技术。
- 关于聊天机器人在客户服务中的有效性和局限性，人们众说纷纭。
- 开发成功的人工智能产品需要差异化、专业技术和用户反馈，同时还要避免过度依赖人工智能技术，并考虑产品的其他方面。

## [掌握浏览器调试的高级技巧](https://alan.norbauer.com/articles/browser-debugging-tricks)

- 文章提供了一系列有用的小技巧，以最大限度地提高浏览器调试器的效率。
- 这些技巧包括使用高级条件断点、日志点/跟踪点、观察窗格、跟踪调用堆栈以及改变程序行为。

### [反应](https://news.ycombinator.com/item?id=38226743)

- 文章探讨了浏览器中的调试技术，如猴子修补和访问局部变量。
- 它讨论了当前浏览器工具的局限性，并提出了可能的改进措施。
- 作者提到了 Replay.io 工具及其团队为开发 Chromium fork 所做的努力，Chromium fork 与当前的 Firefox fork 具有类似的功能，同时还为浏览器提供了新的时空旅行功能。

## [苹果公司将支付 2500 万美元解决美国招聘歧视指控：司法部](https://arstechnica.com/tech-policy/2023/11/apple-discriminated-against-us-citizens-in-hiring-doj-says/)

- 美国司法部（DOJ）指控苹果公司在招聘过程中歧视美国公民，阻止他们申请向外国工人开放的职位。
- 苹果公司已同意支付高达 2500 万美元的拖欠工资和民事罚款，以了结指控，这是根据《移民和国籍法》反歧视条款收取的最大一笔赔偿金。
- 苹果公司没有认罪，但承认没有遵守司法部的标准，并已实施了一项符合政府要求的计划，包括改变招聘方式，对永久劳工证计划（PERM）下的所有职位进行更广泛的招聘。

### [反应](https://news.ycombinator.com/item?id=38224950)

- 美国司法部指控苹果公司在招聘过程中偏袒外国工人而非美国公民。
- 苹果公司被指控设立虚假职位，以雇用外国工人，绕过移民法。
- 苹果公司已与司法部达成和解，同意支付罚款。

## [Nordic 采用 RISC-V 架构实现高能效物联网应用](https://blog.nordicsemi.com/getconnected/why-nordic-is-getting-involved-in-risc-v)

- Nordic 已加入一个联盟，以促进 RISC-V 这一开源芯片架构的采用。
- 此举并不意味着 Nordic 与 Arm 关系的中断，因为他们认为 RISC-V 是一种互补的替代方案，尤其是在功耗关键型移动和物联网应用中。
- RISC-V 提供了定制硬件的灵活性，可实现极低功耗，有利于传感器和本地化机器学习中使用的更简单嵌入式芯片。它为 Nordic 客户提供了更多的设计选择，促进了物联网市场的创新。

### [反应](https://news.ycombinator.com/item?id=38218667)

- 蓝牙芯片的领先供应商 Nordic Semiconductor 正面临来自中国廉价竞争对手的竞争，这促使他们考虑采用 RISC-V 技术。
- 对中国芯片安全漏洞的担忧依然存在，但由于成本较低，许多物联网设备已经在使用中国芯片。
- Nordic Semiconductor 正计划开发在同一芯片上同时采用 Arm 和 RISC-V 内核的处理器，以提高功耗和性能，同时强调 RISC-V 架构在功耗效率和简单性方面的优势。

## [Dan's Tools 提供的 Unix 时间戳和网络开发转换工具](https://www.unixtimestamp.com/)

- 本周二，Unix 时间戳将达到 17 的里程碑，这引发了关于过去和未来重要时间戳的讨论。
- 用户正在分享他们在里程碑时间戳方面的经验和记忆，并分享用于显示和计算时间戳的网站和工具链接。
- 对话探讨了 Unix 时代系统中这些里程碑的魅力和意义，以及关于使用人类可读格式和时间戳变化局限性的争论。

## [Github 字体系统的纹理修复功能解决了空格字体的设计问题](https://github.com/githubnext/monaspace/blob/main/docs/Texture%20Healing.md)

- Github Type System 的空格字体中的纹理愈合功能通过引入可调整间距的备用字形，解决了空格字体的设计问题。
- 这是通过 OpenType 功能代码实现的，允许在间距要求发生冲突时交换字形。
- 这一过程包括创建备用字形版本和根据邻近字符定义查找，代码和文档均可公开访问。

### [反应](https://news.ycombinator.com/item?id=38221379)

- "纹理愈合 "是一个概念，它涉及调整单空格字体中字母的宽度，以提高视觉吸引力，但对于它是否真的提高了可读性还是增加了阅读难度还存在争议。
- 一些用户对 "纹理愈合 "一词是否恰当及其对编码和可读性的影响表示担忧。
- 讨论了 Monaspace 字体，对其可读性有不同看法，一些用户对字符的移动和对齐表示担忧。

## [苹果要求亚马逊屏蔽竞争对手广告，引发对不公平优势的担忧](https://www.businessinsider.com/amazon-gives-apple-special-treatment-while-others-suffer-junk-ads-2023-11)

- 据称，苹果公司得到了亚马逊的优待，导致苹果相关页面上的竞争对手广告受到限制，产品页面更加整洁，没有非苹果产品的推荐。

### [反应](https://news.ycombinator.com/item?id=38224217)

- 苹果公司以担心假冒产品为由，要求亚马逊在其平台上屏蔽竞争对手公司的广告。
- 苹果和亚马逊之间的协议旨在消除假冒产品，特别是苹果产品。
- 有指控称亚马逊和苹果串通删除假冒广告，联邦贸易委员会正在调查潜在的版权侵权行为。
- 一些客户在向亚马逊退货苹果产品时遇到了困难。
- 与三星等竞争对手相比，亚马逊对苹果的优待引发了人们对市场公平性的担忧。
- 亚马逊因允许假冒商品在其平台上销售而受到批评，用户呼吁采取更积极的措施打击假冒商品。
- 亚马逊对第三方卖家的处理及其责任的担忧也正在得到解决。
