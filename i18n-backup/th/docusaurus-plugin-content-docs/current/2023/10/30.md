# 2023-10-30

## [Anti-Adblock และ uBlock Origin ของ YouTube](https://andadinosaur.com/youtube-s-anti-adblock-and-ublock-origin)

- YouTube ได้เพิ่มความพยายามในการต่อต้าน adblockers โดยเริ่มสงครามชักเย่อระหว่างตัวเองกับนักพัฒนา adblocker รวมถึงการกําหนดเป้าหมายส่วนขยายความเป็นส่วนตัวอื่น ๆ และการตั้งค่าการป้องกันการติดตาม
- ผู้ใช้ Reddit ประสบปัญหาในการแสวงหาข้อมูลที่เป็นประโยชน์โดยบางคนหันไปใช้การแชร์ตัวกรองที่เป็นอันตรายซึ่งอาจส่งผลให้ adblockers ของพวกเขาถูกบล็อกอย่างสมบูรณ์
- แรงกดดันต่อนักพัฒนา adblocker ทําให้เกิดการขัดสีสมาชิกในทีมและการลบบัญชีซึ่งบ่งบอกถึงอนาคตที่เป็นไปได้ที่ adblockers อาจได้รับ

- มันเน้นย้ําถึงอุปสรรคที่พบรวมถึงหนี้ทางเทคนิคกระบวนการจัดทํางบประมาณที่ไม่เพียงพอและการต่อต้านของแผนกแม้จะประหยัดต้นทุนได้อย่างมาก
- ผู้เขียนแสดงความไม่พอใจต่อการขาดการยอมรับหรือรางวัลแม้จะมีการสนับสนุนทางการเงินจํานวนมากให้กับ บริษัท

## [π ในจักรวาลอื่น ๆ](https://azeemba.com/posts/pi-in-other-universes.html)

- บทความนี้กล่าวถึงการกําหนดค่าของ pi ซึ่งเป็นค่าคงที่ทางคณิตศาสตร์พื้นฐานและได้รับอิทธิพลจากคําจํากัดความของระยะทางอย่างไร
- มันแนะนําแนวคิดเช่นระยะทางแบบยุคลิดและแมนฮัตตันเมตริกที่มีอิทธิพลต่อการคํานวณ pi
- ระบุว่าค่าต่ําสุดที่เป็นไปได้ของ pi คือ 3.14159 ในทุกเมตริก แต่เมตริกอื่น ๆ นับไม่ถ้วนเห็น pi ตั้งแต่ 3 ถึง 4 สิ่งนี้ชี้ให้เห็นว่าค่าของ pi ไม่สัมบูรณ์ แต่ขึ้นอยู่กับปัจจัยต่างๆเช่นเมตริกที่เลือก

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38063536)

- การอภิปรายหมุนรอบหลายแง่มุมของคณิตศาสตร์ที่มีการตีความเป็นเกมตรรกะและการบังคับใช้และความยืดหยุ่น
- การสนทนาให้ความกระจ่างเกี่ยวกับต้นกําเนิดของคณิตศาสตร์และความสัมพันธ์ที่เกี่ยวพันกับวิทยาศาสตร์ตลอดจนการนําทางของแนวคิดทางคณิตศาสตร์ที่แปลกใหม่และลึกลับ
- บทสนทนายังสะท้อนให้เห็นถึงลักษณะดุลยพินิจของสัจพจน์ทางคณิตศาสตร์ความหมายของ pi ความคิดของค่าคงที่วงกลมที่หลากหลายและการพิจารณาอ่านบทความที่ครอบคลุมเกี่ยวกับทฤษฎีการวัด

## [บลูโอเชียนของ Apple](https://hypercritical.co/2023/10/29/apples-blue-ocean)

- ผู้เขียนแนะนํากลยุทธ์ "มหาสมุทรสีฟ้า" ซึ่งแสดงให้เห็นผ่านชัยชนะของ Nintendo ด้วยคอนโซล Wii กลยุทธ์มหาสมุทรสีฟ้าหมายถึงการสร้างพื้นที่ตลาดใหม่ที่ทําให้การแข่งขันไม่เกี่ยวข้อง
- ผู้เขียนแนะนําโอกาสที่ยิ่งใหญ่ต่อไปของ Apple หรือ "มหาสมุทรสีฟ้า" อาจนําแบตเตอรี่แบบถอดได้ที่ผู้ใช้สามารถเข้าถึงได้กลับมาใช้ใหม่ในแกดเจ็ตเนื่องจากความก้าวหน้าทางเทคโนโลยีและความต้องการของผู้บริโภคที่เปลี่ยนแปลงไป
- แนวคิดที่เสนอนี้นําเสนอประโยชน์ที่อาจเกิดขึ้น เช่น อายุการใช้งานแบตเตอรี่ที่เพิ่มขึ้นและขยะอิเล็กทรอนิกส์ที่ลดลง ทําให้ Apple ซึ่งเป็นผู้นําด้านการออกแบบอุปกรณ์อิเล็กทรอนิกส์สําหรับผู้บริโภคสามารถเอาชนะความท้าทายที่คาดการณ์ได้ซึ่งเชื่อมโยงกับโครงการนี้

## [ซินธิไซเซอร์ Web FM ทําด้วย HTML5](https://www.taktech.org/takm/WebFMSynth/)

- นอกจากนี้ยังอ้างอิงเครื่องมือและทรัพยากรอื่น ๆ ที่เกี่ยวข้องโดยเน้นความกว้างของการสนับสนุนและความพร้อมใช้งานในสาขา
- ข้อความสัมผัสกับซินธิไซเซอร์ DXi FM ที่สร้างขึ้นสําหรับ iPhone/iPad ระหว่างปี 2010-2014 โดย Takashi Mizuhiki และสตูดิโอสร้างสรรค์ CUE เผยให้เห็นแบบอย่างทางประวัติศาสตร์ในเทคโนโลยีเพลงมือถือ

## [DIY IP-KVM ขึ้นอยู่กับ Raspberry Pi](https://github.com/pikvm/pikvm)

- PiKVM เป็นระบบ DIY IP-KVM แบบโอเพนซอร์สที่คุ้มค่าซึ่งใช้ Raspberry Pi สําหรับการจัดการเซิร์ฟเวอร์หรือเวิร์กสเตชันจากระยะไกล ช่วยในการแก้ไขปัญหาการกําหนดค่า BIOS และการติดตั้งระบบปฏิบัติการใหม่
- โครงการมีอุปกรณ์เวอร์ชันต่างๆพร้อมคําแนะนําและความช่วยเหลือที่สามารถเข้าถึงได้ผ่านเว็บไซต์และชุมชน Discord
- PiKVM มีข้อกําหนดเฉพาะสําหรับ Raspberry Pi บางรุ่นและส่วนประกอบพิเศษที่จําเป็นสําหรับการกําหนดค่าเฉพาะ ความคิดริเริ่มรับบริจาคและให้การปรับแต่งสําหรับการใช้งานเชิงพาณิชย์

## [กฎของ Carl Sagan สําหรับการพล่ามและการคิดเชิงวิพากษ์](https://www.themarginalian.org/2014/01/03/baloney-detection-kit-carl-sagan/)

- ผู้เขียน The Marginalian ซึ่งเป็นเว็บไซต์ฟรีและไม่มีโฆษณากําลังมองหาการสนับสนุนจากผู้อ่านเพื่อรักษาเว็บไซต์
- ผู้เขียนกล่าวถึงหนังสือของ Carl Sagan ซึ่งเน้นย้ําถึงความสําคัญของการคิดอย่างมีวิจารณญาณและเสนอวิธีการแยกแยะความจริงจากความเท็จ
- เพื่อนําทางโลกที่เต็มไปด้วยการโฆษณาชวนเชื่อและ pseudoscience ผู้เขียนเน้นย้ําถึงความจําเป็นในการประเมินการอ้างสิทธิ์อย่างมีวิจารณญาณการหลีกเลี่ยงความผิดพลาดเชิงตรรกะและการใช้เครื่องมือเช่น "ชุดตรวจจับ baloney"

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38060912)

- บทความนี้กล่าวถึงหัวข้อต่างๆเช่นหลักการของ Carl Sagan สําหรับการคิดอย่างมีวิจารณญาณความสําคัญของการปลอมแปลงในวิทยาศาสตร์และอุปสรรคในการแยกความแตกต่างของวิทยาศาสตร์ในยุคหลังความจริง
- มันนํามาซึ่งการอภิปรายเกี่ยวกับแบบจําลองการเปลี่ยนแปลงสภาพภูมิอากาศการดํารงอยู่ของพระเจ้าและความเชื่อทางศาสนาทฤษฎีการตัดสินใจการทดสอบสมมติฐานทางสถิติและความจําเป็นของการคิดเชิงวิพากษ์ในวารสารศาสตร์

- บล็อกกระจายข้อดีของ Gateway API มากกว่า Ingress - กลยุทธ์ที่มุ่งเน้นบทบาทการพกพาการแสดงออกและการขยาย
- นอกจากนี้ยังมีบทช่วยสอนเกี่ยวกับการติดตั้งและการใช้ ingress2gateway สําหรับการโยกย้าย บล็อกสรุปโดยยอมรับการพัฒนาอย่างต่อเนื่องของ Gateway API และชุมชนที่สนับสนุน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38062269)

- Kubernetes ได้เปิดตัวเครื่องมือ ingress2gateway ซึ่งเป็นส่วนหนึ่งของการเตรียมการสําหรับความพร้อมใช้งานทั่วไป (GA) ของ Gateway API โดยมีเป้าหมายเพื่อทําให้การใช้งาน API ง่ายขึ้นสําหรับผู้เชี่ยวชาญด้าน DevOps และ SRE
- เครื่องมือนี้พัฒนาโดย Kong ซึ่งเป็นผู้ให้บริการ Ingress ที่รู้จักรองรับทั้ง Ingress และ Gateway API ซึ่งให้การเปลี่ยนแปลงที่ราบรื่นระหว่างทั้งสอง
- การย้ายนี้เกิดจากข้อจํากัดที่พบกับ Ingress API ในการจัดการคุณลักษณะขั้นสูง เช่น การกําหนดเส้นทางสตริงการสืบค้นและการปรับสมดุลโหลดแบบถ่วงน้ําหนัก ซึ่งเชื่อว่าได้รับการแก้ไขโดย Gateway API

## [การย้ายแบ็กเอนด์ของเราจาก Vercel ไปยัง Fly.io](https://www.openstatus.dev/blog/migration-backend-from-vercel-to-fly)

- บทความนี้เล่าถึงการย้ายแบ็กเอนด์ของ OpenStatus จาก Vercel ไปยัง Fly.io โดยได้รับแรงบันดาลใจจากความต้องการเซิร์ฟเวอร์ที่มีน้ําหนักเบากว่าและความกังวลเกี่ยวกับราคาของ Vercel
- การเปลี่ยนแปลงต้องเผชิญกับความท้าทาย เช่น การปรับอิมเมจ Docker ให้เหมาะสมใน monorepo การจัดการกับการหมดเวลาการปรับใช้บน Fly.io และการแก้ไขปัญหาข้อบกพร่องด้วยรันไทม์ Bun
- แม้จะมีอุปสรรคเหล่านี้ แต่การโยกย้ายไปยัง Fly.io ก็ประสบความสําเร็จและผู้เขียนยังคงชื่นชมผลิตภัณฑ์ของ Vercel

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38062649)

- การอภิปรายเกี่ยวกับประสบการณ์และความคิดเห็นของผู้ใช้ในการโยกย้ายแบ็กเอนด์จาก Vercel ไปยัง Fly.io โดยเน้นถึงปัญหาเกี่ยวกับการปรับใช้ฟังก์ชันการทํางานและราคาที่ไม่น่าเชื่อถือของ Vercel
- ผลกระทบของ Vercel ต่อ React ความคุ้มค่าของฟังก์ชัน Edge และ Lambda และความเสถียรของแพลตฟอร์ม Fly.io เป็นหัวข้อย่อยที่สําคัญภายในวาทกรรม
- การอภิปรายยังรวมถึงการกล่าวถึงแพลตฟอร์มทางเลือกเช่น DigitalOcean, Render และ Hetzner และหารือเกี่ยวกับการแลกเปลี่ยนระหว่างกําไรระยะสั้นและค่าบํารุงรักษาระยะยาวในการปรับใช้โครงสร้างพื้นฐาน
