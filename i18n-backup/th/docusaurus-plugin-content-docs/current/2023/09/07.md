# 2023-09-07

## [สหราชอาณาจักรถอนตัวจากการปะทะกับ Big Tech เกี่ยวกับการส่งข้อความส่วนตัว](https://www.ft.com/content/770e58b1-a299-4b7b-a129-bded8649a43b)

- รัฐบาลสหราชอาณาจักรได้ตัดสินใจที่จะถอนตัวจากความขัดแย้งกับ บริษัท Big Tech เกี่ยวกับการเข้าถึงการส่งข้อความส่วนตัว
- การตัดสินใจนี้เป็นไปตามกฎระเบียบที่เสนอเพื่อปกป้องเด็กจากเนื้อหาออนไลน์ที่เป็นอันตรายโดยเน้นถึงการปะทะกันที่อาจเกิดขึ้นระหว่างความกังวลด้านความปลอดภัยและความเป็นส่วนตัว
- การย้อนรอยของรัฐบาลสะท้อนให้เห็นถึงความกังวลเกี่ยวกับผลกระทบที่อาจเกิดขึ้นกับการเข้ารหัสแบบ end-to-end และความเป็นส่วนตัวของข้อความ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37408196)

- รัฐบาลสหราชอาณาจักรถูกวิพากษ์วิจารณ์ว่าบิดเบือนจุดยืนในการเข้าถึงข้อความที่เข้ารหัสโดยนักวิจารณ์แย้งว่าร่างกฎหมายความปลอดภัยออนไลน์ที่เสนอนั้นบ่อนทําลายการเข้ารหัส นี่เป็นการเสนอแพลตฟอร์มการส่งข้อความที่ปลอดภัยเพื่อพิจารณาออกจากสหราชอาณาจักรเพื่อรักษามาตรการรักษาความปลอดภัย
- รัฐบาลปฏิเสธที่จะถอยกลับในข้อพิพาทการเข้ารหัสโดยยืนยันว่าจะทุ่มเทให้กับความปลอดภัยของเด็ก อย่างไรก็ตามสิ่งนี้ได้จุดประกายการอภิปรายเกี่ยวกับการสแกนฝั่งไคลเอ็นต์ผลกระทบด้านความเป็นส่วนตัวการทําเหมืองข้อมูลการเข้ารหัสแบบ end-to-end และการเข้าถึงแบบลับๆ
- แผนการของรัฐบาลที่จะชะลอการสแกนแอปพลิเคชันการส่งข้อความเพื่อหาเนื้อหาที่เป็นอันตรายจนกว่าจะเป็นไปได้ทางเทคนิคนั้นได้รับการวิพากษ์วิจารณ์ ผลกระทบที่อาจเกิดขึ้นจากการประมวลผลควอนตัมต่อการเข้ารหัสและช่องโหว่ของวิธีการเข้ารหัสที่แตกต่างกันก็เป็นหัวข้อที่ถกเถียงกันในหมู่สาธารณชน

- รุ่น 1.7 นําเสนอการอัปเดตและการปรับปรุงที่หลากหลายรวมถึงเอกสารที่ได้รับการปรับปรุงความเข้ากันได้ของแพลตฟอร์มการเปลี่ยนแปลงอินเทอร์เฟซบรรทัดคําสั่ง (CLI) และการปรับปรุงภาษา
- โพสต์รับทราบผู้มีส่วนร่วมในโครงการและให้ลิงก์สําหรับการตรวจสอบการเปลี่ยนแปลงทั้งหมดในการอัปเดตนี้ผ่านบันทึกการเปลี่ยนแปลง

## [ผลการตรวจสอบทางเทคนิคสําหรับการเข้าซื้อกิจการที่สําคัญ Storm-0558](https://msrc.microsoft.com/blog/2023/09/results-of-major-technical-investigations-for-storm-0558-key-acquisition/)

- Microsoft ได้แบ่งปันผลการสอบสวนนักแสดง Storm-0558 ที่ใช้คีย์ผู้บริโภคเพื่อละเมิด OWA และ Outlook.com
- ผลการวิจัยพบว่าการถ่ายโอนข้อมูลผิดพลาดในเดือนเมษายน 2021 มีคีย์การลงนามซึ่งต่อมาเปลี่ยนเป็นสภาพแวดล้อมการดีบักภายในเครือข่ายขององค์กร ต่อจากนี้บัญชีองค์กรของวิศวกรของ Microsoft ถูกบุกรุกโดยนักแสดงเพื่อรับคีย์นี้
- Microsoft ยังพบข้อผิดพลาดของไลบรารีที่ช่วยให้ระบบอีเมลยอมรับคําขออีเมลขององค์กรที่ลงนามด้วยคีย์ผู้บริโภคซึ่ง บริษัท ได้แก้ไขแล้ว

- เหตุการณ์ดังกล่าวได้จุดประกายการถกเถียงเกี่ยวกับมาตรการรักษาความปลอดภัยและปัญหาเชิงโครงสร้างของ Microsoft โดยตั้งคําถามถึงความเสี่ยงและผลกระทบที่อาจเกิดขึ้นจากการโจมตีทางไซเบอร์
- โดยเน้นย้ําถึงความจําเป็นในการเสริมระบบวิศวกรรมความปลอดภัยของข้อมูลนโยบายการเก็บรักษาบันทึกและผู้ตรวจสอบบุคคลที่สามการอภิปรายวิพากษ์วิจารณ์การจัดการของ Microsoft เกี่ยวกับเหตุการณ์เหล่านี้

## [โปรแกรม Erlang ที่ฉันชอบ (2013)](https://joearms.github.io/#2013-11-21%20My%20favorite%20Erlang%20Program)

- ผู้เขียนได้พัฒนาโปรแกรมชื่อ "Universal Server" ในภาษา Erlang ซึ่งแสดงให้เห็นถึงความเก่งกาจเนื่องจากสามารถกําหนดค่าให้เป็นเซิร์ฟเวอร์เฉพาะได้
- พวกเขาขยายสิ่งนี้โดยการสร้างเซิร์ฟเวอร์แฟกทอเรียลและรวมเซิร์ฟเวอร์เหล่านี้ในโปรแกรมทดสอบซึ่งเป็นตัวอย่างเพิ่มเติมเกี่ยวกับประสิทธิภาพของ Erlang
- การใช้เครือข่ายการวิจัย Planet Lab ผู้เขียนใช้อัลกอริธึมซุบซิบทําให้เครือข่ายสามารถแปลงเป็นเซิร์ฟเวอร์ต่างๆได้แสดงให้เห็นถึงลักษณะแบบไดนามิกของภาษา Erlang และยูทิลิตี้ในแอปพลิเคชันเครือข่าย

- การอภิปรายครอบคลุมหัวข้อทางเศรษฐกิจในวงกว้าง ได้แก่ สังคมนิยมทุนนิยมแรงงานและการว่างงานสัมผัสกับความท้าทายของการใช้ลัทธิสังคมนิยมและทฤษฎีคุณค่าของมาร์กซ์
- บทสนทนารวมถึงปฏิสัมพันธ์ระหว่างแรงงานและทุนผลกระทบของการว่างงานต่อระบบทุนนิยมและการวิพากษ์วิจารณ์ระบบเศรษฐกิจที่แตกต่างกัน
- นอกจากนี้การสนทนายังรับทราบถึงความซับซ้อนโดยธรรมชาติภายในทฤษฎีทางเศรษฐกิจความท้าทายของระบบการวัดมูลค่าที่มีประสิทธิภาพและมุมมองที่แตกต่างกันต่อการแก้ปัญหาที่อาจเกิดขึ้น

## [37signals แนะนํา "ครั้งเดียว" - ซื้อซอฟต์แวร์ครั้งเดียว](https://once.com/)

- ผู้เขียนชี้ให้เห็นถึงการเปลี่ยนแปลงในรูปแบบซอฟต์แวร์ธุรกิจโดยเปลี่ยนจากการเป็นเจ้าของซอฟต์แวร์เป็นการสมัครสมาชิกภายใต้รูปแบบ Software as a Service (SaaS)
- พวกเขายืนยันว่าในขณะที่ SaaS ได้รับรางวัลทางการเงินสําหรับผู้จําหน่ายซอฟต์แวร์แนวโน้มนี้ดูเหมือนจะอุดหนุนด้วยการปรับปรุงเทคโนโลยีการโฮสต์ด้วยตนเองและความต้องการขององค์กรในการควบคุมที่มากขึ้น
- ผู้เขียนแนะนํา "ONCE" ซึ่งเป็นชุดเครื่องมือซอฟต์แวร์ที่กําลังจะมาถึงซึ่งสามารถเป็นเจ้าของได้เรื่อย ๆ ด้วยการชําระเงินเพียงครั้งเดียวซึ่งแสดงถึงยุคหลัง SaaS ที่อาจเกิดขึ้นและวางแผนที่จะเปิดตัวในปลายปี 2023

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37408929)

- มีการถกเถียงกันอย่างต่อเนื่องระหว่างรูปแบบการกําหนดราคาการสมัครสมาชิกและการชําระเงินแบบครั้งเดียวสําหรับซอฟต์แวร์ซึ่งกระตุ้นโดยความกังวลเกี่ยวกับการยกเลิกที่ยากลําบากค่าใช้จ่ายสะสมและความต้องการการอัปเดตอย่างต่อเนื่อง
- การประกาศรูปแบบการกําหนดราคา "ครั้งเดียว" โดย 37signals กระตุ้นให้เกิดปฏิกิริยาที่หลากหลายโดยผู้ใช้บางคนชื่นชมตัวเลือกการชําระเงินแบบครั้งเดียวในขณะที่คนอื่นแสดงความสงสัยและวิพากษ์วิจารณ์ภาษาที่ใช้ในการประกาศ
- การอภิปรายยังมีข้อกังวลเกี่ยวกับการสนับสนุนซอฟต์แวร์ระยะยาวความเข้ากันได้ข้อ จํากัด ของรูปแบบ SaaS (Software as a Service) และประโยชน์ของการโฮสต์ด้วยตนเอง

## [หน้านี้มีอยู่เฉพาะเมื่อมีคนกําลังดูอยู่เท่านั้น](http://ephemeralp2p.durazo.us/2bbbf21959178ef2f935e90fc60e5b6e368d27514fe305ca7dcecc32c0134838)

- Ephemeral P2P เป็นแพลตฟอร์มการแบ่งปันเนื้อหาแบบเพียร์ทูเพียร์ ผู้ใช้ลงทะเบียนแฮชของเนื้อหาที่ต้องการแบ่งปันและเนื้อหานี้จะถูกดึงและส่งต่อโดยเซิร์ฟเวอร์เมื่อผู้อื่นเยี่ยมชมลิงก์ที่เกี่ยวข้อง
- แพลตฟอร์มนี้ใช้เฟรมเวิร์ก Elixir และ Phoenix สํารวจศักยภาพของธุรกรรมแบบเพียร์ทูเพียร์ผ่านเว็บซ็อคเก็ตของเบราว์เซอร์และแนวคิดของเว็บที่กล่าวถึงเนื้อหา
- แอปพลิเคชันนี้ไม่ได้ฆ่าเชื้อเนื้อหา HTML ที่ใช้ร่วมกัน อย่างไรก็ตามไม่มีข้อมูลส่วนตัวรวมอยู่ในการแลกเปลี่ยน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37410630)

- การอภิปรายมุ่งเน้นไปที่แอปของเล่นที่เน้นการเรียนรู้สําหรับ Elixir, Phoenix และ websockets ซึ่งนําศักยภาพของความสามารถแบบเพียร์ทูเพียร์ของเบราว์เซอร์สําหรับเครือข่ายแบบกระจายอย่างแท้จริง
- ผู้เข้าร่วมถกเถียงกันถึงข้อดีและความท้าทายของการแบ่งปันเนื้อหาแบบเพียร์ทูเพียร์ชั่วคราวบนเว็บรวมถึงไคลเอนต์โอเพ่นซอร์สที่ปรับแต่งได้ศักยภาพของ WebRTC และแพลตฟอร์มส่วนกลางที่แพร่หลายในปัจจุบัน
- แนวคิดสําหรับการปรับปรุงการแคชเซิร์ฟเวอร์การจัดเตรียมสําหรับลูกค้าที่ไม่ใช่ JS ที่เปิดใช้งานเพื่อดูหน้าเว็บการจัดการข้อบกพร่องของเครือข่ายและศักยภาพในการใช้เทคโนโลยีนี้ในเว็บโฮสติ้งก็เป็นที่ถกเถียงกันโดยเน้นทั้งข้อ จํากัด และประโยชน์

## [Exa เลิกใช้แล้ว](https://github.com/ogham/exa)

- โครงการ exa เป็นเครื่องมือโอเพ่นซอร์สที่ออกแบบมาเพื่อทดแทนคําสั่ง "ls" ร่วมสมัยโดยนําเสนอคุณสมบัติเพิ่มเติมและค่าเริ่มต้นที่ได้รับการปรับปรุง
- เครื่องมือนี้ใช้การเข้ารหัสสีเพื่อแยกประเภทไฟล์และทราบดีเกี่ยวกับ symlinks (ลิงก์สัญลักษณ์) แอตทริบิวต์เพิ่มเติมและ Git ซึ่งเป็นระบบควบคุมเวอร์ชันแบบกระจาย
- เครื่องมือ exa โดดเด่นด้วยขนาดที่เล็กความเร็วสูงและเข้ากันได้กับระบบ macOS และ Linux โพสต์รวมถึงข้อมูลเกี่ยวกับขั้นตอนการติดตั้งและวิธีการพัฒนาและทดสอบเครื่องมือโดยใช้ Vagrant ซึ่งเป็นเครื่องมือสําหรับการสร้างและบํารุงรักษาสภาพแวดล้อมการพัฒนาซอฟต์แวร์เสมือนแบบพกพา

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37416430)

- ผู้ใช้กําลังมีบทสนทนาเกี่ยวกับยูทิลิตี้บรรทัดคําสั่งต่างๆโดยเน้นที่คําสั่ง" ls" และทางเลือกอื่น ๆ เช่น "exa" และ" lsd" โดยถกเถียงกันถึงจุดแข็งและจุดอ่อนของพวกเขา
- การสนทนายังดึงดูดความสําคัญของนักพัฒนาโอเพ่นซอร์สในการถ่ายทอดสถานะของพวกเขาสํารวจความเป็นไปได้สําหรับส้อมและเขียนใหม่ในซอฟต์แวร์
- พวกเขายังเจาะลึกความสมดุลระหว่างความเสถียรและนวัตกรรมในการพัฒนาซอฟต์แวร์ซึ่งแสดงให้เห็นถึงการแลกเปลี่ยนที่เกี่ยวข้อง
