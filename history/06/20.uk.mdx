### [PostgreSQL переглядає свою модель, засновану на процесах](https://lwn.net/SubscriberLink/934940/3abb2d4086680b78/)

- PostgreSQL, популярна система управління базами даних, розглядає можливість фундаментальних змін, щоб відійти від своєї процесно-орієнтованої моделі.
- Розробники вважають, що ця зміна може усунути обмеження продуктивності у великих системах і зменшити кількість дубльованого коду, але є занепокоєння щодо потенційних змін, які можуть призвести до збоїв і втрати надійності системи.
- Перехід на потоки може спричинити такі проблеми, як переробка глобальних змінних та забезпечення сумісності з існуючими розширеннями, але попередня версія PostgreSQL з потоками дає певне уявлення про проект.

#### [Реакція галузі](http://news.ycombinator.com/item?id=36393030)

- PostgreSQL розглядає можливість переходу від моделі, заснованої на процесах, до моделі, заснованої на потоках.
- Цей перехід може мати значні наслідки для стабільності, продуктивності та процесів розробки.
- Спільнота розділилася в думках щодо того, чи необхідний цей перехід, чи вартий він зусиль.

### [OpenLLM](https://github.com/bentoml/OpenLLM)

- OpenLLM - це відкрита платформа для роботи з великими мовними моделями (LLM) у виробництві.
- Вона підтримує широкий спектр найсучасніших LLM і надає гнучкі API для обслуговування та розгортання LLM.
- Користувачі можуть тонко налаштовувати LLM відповідно до своїх потреб і легко створювати додатки зі штучним інтелектом, комбінуючи LLM з іншими моделями та сервісами.

#### [Реакція галузі](http://news.ycombinator.com/item?id=36388219)

- OpenLLM - це відкрита платформа для роботи з великими мовними моделями (LLM) у виробництві.
- Вона дозволяє користувачам запускати LLM на власному обладнанні замість того, щоб покладатися на комерційно доступні API.
- Вона надає фреймворк для обслуговування LLM і підтримує різні типи моделей і модальностей, включаючи текст, зображення і відео.

- Процес реєстрації квитків зіткнувся з різними проблемами, включаючи перевантаження серверів і недосконалу функцію скидання пароля. Крім того, було виявлено чорний хід, який дозволяв людям генерувати коди і завершувати процес замовлення квитків навіть після його офіційного закриття.
- Витік даних також поширився на подібну програму під назвою DiscoverEU, яка пропонує безкоштовні проїзні Interrail для молодих європейців. Було виявлено, що понад 245 000 реєстраційних записів з DiscoverEU були легко доступні в Інтернеті. З того часу проблема була вирішена, і наразі впроваджуються подальші заходи безпеки.

### [Slint - інструментарій декларативного інтерфейсу користувача для вбудованих і десктопних систем](https://slint.dev)

- Slint - це інструментарій декларативного інтерфейсу користувача, який дозволяє розробникам створювати нативні користувацькі інтерфейси для вбудованих систем та десктопних додатків.
- Він надає такі функції, як Live-Preview, що дозволяє розробникам швидко ітерації та вносити зміни в дизайн інтерфейсу.
- Slint є легким і продуктивним, займає мало пам'яті, має оптимальні методи рендерингу графіки та підтримує декілька мов програмування.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=36388821)

- Slint - це декларативний інструментарій графічного інтерфейсу, написаний переважно мовою Rust, призначений для настільного та вбудованого використання.
- Інструментарій пропонує підтримку API для багатьох мов програмування, включаючи C++ та JavaScript.
- Користувачі високо оцінили дизайн Slint, простоту використання та зворотній зв'язок з графічним інтерфейсом у реальному часі при використанні розширення VS Code. Дехто порівнює його з Windows Presentation Foundation від Microsoft.
- Один користувач висловив занепокоєння щодо оптимізації для двійкового розміру на мікроконтролері з обмеженим обсягом флеш-пам'яті.
- Команда Slint визнала це занепокоєння і повідомила про плани оптимізувати згенерований код для подальшого зменшення розміру двійкового коду.
- Інший користувач висловив зацікавленість у бібліотеці графічного інтерфейсу, призначеній для невеликих вбудованих систем, які не споживають багато флеш-пам'яті або оперативної пам'яті.
- Команда Slint відповіла, що саме це було їхньою метою при створенні інструментарію.
- Користувачі рекомендували альтернативні бібліотеки графічного інтерфейсу, такі як fltk-rs та imgui, які можуть краще відповідати конкретним потребам.
- Користувачі обговорювали оптимізацію розміру коду та продуктивності для мікроконтролерів, з пропозиціями на кшталт коригування налаштувань кодогенератора у Rust та експериментів з пороговими значеннями інлайнів.
- Користувачі обговорювали GRPC, Rust як бекенд для інтерфейсу користувача та інтеграцію Flutter з Rust.
- Користувачі висловили зацікавленість у кращих порівняльних розділах на сайті Slint і запропонували інші фреймворки інтерфейсу, які варто розглянути.
- Користувачі обговорили структуру ціноутворення Slint і попросили більше ясності в інформації про ціни.
- Користувачі запропонували додати на сайт Slint посібник для початківців і покращити приклади галереї віджетів.
- Користувачі ставили питання про обробку тексту, підтримку доступності, інтеграцію віджетів та плани щодо API на Python.
- Користувачі висловили свою вдячність за назву "Slint" та її зв'язок з гуртом.
