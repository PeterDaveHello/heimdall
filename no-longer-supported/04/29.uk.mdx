### Пригоди водія для веб-камери 1999 року

#### [Новини](https://blog.benjojo.co.uk/post/quickcam-usb-userspace-driver)

### Чи вбиває Gmail незалежну електронну пошту?

#### [Новини](https://tutanota.com/blog/posts/gmail-independent-email)

- Фільтрація спаму в Gmail призводить до того, що деякі листи, розміщені на власному хостингу, потрапляють до папок "Спам", що обмежує можливості для людей та компаній, які хочуть, щоб їхні листи надійно отримувалися Gmail.
- Розміщення власного поштового сервера можливе, але вимагає достатніх технічних знань і належної ретельності, щоб гарантувати, що ваші листи не будуть позначені як спам провайдерами.
- Можливість Google вирішувати, чи надсилає компанія спам, за допомогою фільтрації електронної пошти підкреслює потенційне зловживання ринковою владою з боку великих технологічних компаній, які обмежують можливості менших сервісів.

#### [Реакція галузі](http://news.ycombinator.com/item?id=35744888)

- Спадкові культурні проблеми та глибокі кишені провайдерів електронної пошти вважаються основною проблемою інтероперабельності, а створення власної поштової інфраструктури все ще є життєздатним і бажаним.
- У дописі обговорюються різні стратегії управління особистими поштовими серверами та труднощі уникнення маркування як спаму, в тому числі використання зовнішніх поштових сервісів та впровадження явної згоди на розсилку.

### Керівник Microsoft заявив, що ядро Windows 11 незабаром буде завантажуватися з Rust всередині

#### [Новини](https://www.neowin.net/news/senior-microsoft-exec-says-windows-11-kernel-will-soon-be-booting-with-rust-inside/)

Віце-президент Microsoft Джейсон Зандер (Jason Zander) повідомив, що компанія планує включити Rust в ядро Windows 11. Rust - це мова програмування, відома своєю безпекою пам'яті та безпекою потоків, що робить її привабливим вибором для кодування на системному рівні. Використання Rust спрямоване на підвищення безпеки та надійності операційної системи Windows. Цей крок є частиною зусиль Microsoft, спрямованих на посилення безпеки та зменшення кількості вразливостей у Windows 11.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=35738829)

- Функції захисту пам'яті в Rust можуть покращити продуктивність в певних областях та підвищити рівень безпеки.
- В той час як одні висловлюють захоплення перевагами Rust, інші припускають, що його одного буде недостатньо, щоб запобігти всім вразливостям. Windows також впроваджує суворіші заходи безпеки для підвищення надійності.

### Чудовий безгілковий двійковий пошук

#### [Новини](https://probablydance.com/2023/04/27/beautiful-branchless-binary-search/)

- У статті обговорюється алгоритм пошуку без розгалужень, який називається "алгоритм Шара", описаний у книзі Джона Бентлі у 1982 році.
- Алгоритм використовує цикл з інструкцією умовного переходу замість оператора розгалуження, що робить його швидшим на GCC порівняно з std::lower_bound.
- Змінна "step" стрибає в степені двійки, щоб знайти середній елемент для пошуку, і алгоритм виконує більше порівнянь, ніж традиційний двійковий пошук.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=35737862)

- Деякі коментатори пропонують методи оптимізації для різних апаратних конфігурацій, такі як використання SIMD інструкцій та попередньої вибірки, а також використання оптимізації на основі профілю з інструкціями CMOV.
- Розробники обговорюють проблеми продуктивності бінарного пошуку для даних розміром, близьким до степеня 2, а також демонструють різні погляди на низькорівневу оптимізацію та компроміси між переносимістю мови та специфікою апаратного забезпечення.

### Gpt4free repo отримав повідомлення про припинення роботи від OpenAI

#### [Новини](https://github.com/xtekky/gpt4free)

- Юридична команда OpenAI опублікувала повідомлення про видалення репозиторію Gpt4free на GitHub.
- Репозиторій містить API сторонніх сайтів мовних моделей, а сам проект призначений виключно для освітніх цілей.
- Автор репозиторію не несе відповідальності за будь-які наслідки, що виникли в результаті використання або неправильного використання репозиторію або контенту, наданого сторонніми API.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=35740836)

- Суперечки навколо права власності на навчальні дані GPT-4 та інтелектуальної власності продовжують викликати занепокоєння, зокрема дискусії точаться навколо дотримання прав інтелектуальної власності, етичності використання контенту, створеного ШІ, для навчання моделей ШІ без дозволу творців контенту, а також динаміки співвідношення сил між OpenAI та його платними клієнтами.
- Деякі коментатори припускають вплив Microsoft і GitHub у цій суперечці, тоді як інші пропонують альтернативи нині видаленому репозиторію Gpt4free, як-от створене користувачами дзеркало або пожертвування на користь оригінального творця.
