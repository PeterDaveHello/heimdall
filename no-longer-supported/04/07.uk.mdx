### Якщо ми втратимо інтернет-архів, нам кінець.

[Оригінал](https://www.sbstatesman.com/2023/04/04/if-we-lose-the-internet-archive-were-screwed/).

Однак Інтернет-архів подав апеляцію на це рішення, яке, на думку деяких, є, по суті, страйком корпорацій і приватних осіб проти публічних послуг, що фінансуються платниками податків. Якщо апеляція не матиме успіху, люди втратять більше знань, ніж коли-небудь містила Бібліотека Олександри.

### Chrome постачає WebGPU

[Оригінал](https://developer.chrome.com/blog/webgpu-release/).
Chrome запускає WebGPU, новий API веб-графіки, що пропонує покращену 3D-графіку та паралельні обчислення на ChromeOS, macOS та Windows, щоб забезпечити доступ до розширених можливостей GPU та ефективне програмування на веб-платформі. WebGPU розроблений з ідіоматичним JavaScript API, інтеграцією з обіцянками та чудовими повідомленнями про помилки, і є будівельним блоком для майбутніх удосконалень, таких як доступ до ядер шейдерів для більшої оптимізації машинного навчання та більшої ергономічності WGSL. WebGPU є результатом 6-річної спільної роботи групи спільноти W3C "GPU для Інтернету", включаючи внесок Mozilla, Apple, Intel та Microsoft. Платформи ChromeOS, Windows та macOS підтримують WebGPU, а Linux, Android та інші платформи розширять підтримку в найближчому майбутньому. Популярні бібліотеки WebGL, такі як Babylon.js, PlayCanvas та TensorFlow.js, вже пропонують певну підтримку WebGPU або працюють над цим. Ресурси, щоб дізнатися більше про WebGPU, включають специфікації W3C, документацію MDN, приклади, обчислення на GPU, серед іншого.

[Discussion Service](http://news.ycombinator.com/item?id=35465729).
Chrome випустив WebGPU, який обіцяє покращення порівняно з WebGL. WebGPU змінює правила гри з позитивним внеском; думки розходяться щодо того, чи слід надавати пріоритет настільним чи мобільним графічним процесорам. Користувачі обговорюють способи обмеження функцій браузерів, що виточують інформацію, та запобігання збору відбитків пальців. Існує занепокоєння щодо потенційного зловмисного використання для майнінгу криптовалют. Web3DSurvey відстежує функції та обмеження, пов'язані з WebGPU. Існує хвилювання з приводу потенціалу широкого впровадження WebGPU, незважаючи на занепокоєння з приводу обмежень у порівнянні з більш потужними графічними технологіями.

### Показати Tabby - самостійний копілот GitHub

[Оригінал](https://github.com/TabbyML/tabby).
TabbyML випустила Tabby, саморозміщену альтернативу GitHub Copilot з відкритим вихідним кодом та попередньо встановленим програмним забезпеченням. Він відрізняється автономністю без потреби в СУБД або хмарному сервісі, веб-інтерфейсом для візуалізації та конфігурації моделей і MLOps, інтерфейсом OpenAPI і легкою інтеграцією з існуючою інфраструктурою. Розробники можуть використовувати докер-образ для легкого розгортання, а TabbyML підтримує графічний процесор споживчого рівня з ваговим завантаженням FP-16 та іншими оптимізаціями. Сервер FastAPI містить документацію OpenAPI для HTTP API.

[Discussion Service](http://news.ycombinator.com/item?id=35470915).
Tabby, самостійна альтернатива GitHub Copilot, пропонує повний контроль над даними та конфіденційністю під час точного налаштування моделей. Це економить час, але викликає занепокоєння щодо конфіденційності. Проблеми конфіденційності GitHub перебільшені, а Copilot має обмеження. TabbyML генерує шаблонний код і викликає питання щодо захисту інтелектуальної власності на код. Альфа-версія Tabby популярна, незважаючи на відсутність професійного оформлення та підтверджуючих доказів. Copilot точно прогнозує код, але має обмеження і може запропонувати поганий код. Деякі користувачі пропонують розмістити версію Copilot на власному хостингу та змінити назву для кращого SEO.

### Працівники Tesla поділилися чутливими зображеннями, знятими автомобілями клієнтів

[Оригінал](https://www.reuters.com/technology/tesla-workers-shared-sensitive-images-recorded-by-customer-cars-2023-04-06/).
Працівники Tesla поділилися чутливими відео, знятими камерами автомобілів клієнтів у період з 2019 по 2022 рік, згідно з інтерв'ю Reuters з дев'ятьма колишніми працівниками Tesla. Аварії, агресивна поведінка на дорозі та незручні ситуації були серед відео, якими ділилися через внутрішню систему обміну повідомленнями Tesla, деякі з них публічно. У повідомленні компанії про конфіденційність клієнтів підкреслюється гарантія анонімності записів з камер, які не пов'язані з клієнтами або їхніми автомобілями, але деякі колишні працівники назвали це "порушенням приватності". У відповідь на занепокоєння щодо захисту даних Tesla внесла зміни до режиму Sentry Mode, включивши пульсуючі фари на припаркованих автомобілях, щоб попередити перехожих про те, що за ними може вестися спостереження. Reuters стверджує, що всі котирування будуть відкладені, і надає посилання для виправлення і зворотного зв'язку на сайті.

[Discussion Service](http://news.ycombinator.com/item?id=35468855).
Працівники Tesla поділилися чутливими зображеннями, записаними автомобілями клієнтів, без гарантій конфіденційності. Необхідно забезпечити дотримання правил конфіденційності з серйозними наслідками, і компанії повинні будувати конфіденційність з нуля. Анекдоти підкреслюють брак конфіденційності в різних компаніях і стартапах, а відсутність в Індії законів про цифрову конфіденційність піддається критиці. Конфіденційність даних не є повністю захищеною, і працівники можуть бачити і зловживати приватними даними. Німецькі стандарти конфіденційності не є досконалими, але органи захисту даних будуть зацікавлені в будь-яких порушеннях даних автовиробниками у зв'язку з виконанням GDPR. Існують проблеми з конфіденційністю щодо підключених транспортних засобів, але дехто зазначає, що подібні можливості відстеження існують і в інших транспортних засобах, крім електромобілів. Звіти свідчать, що деякі кредитори встановили пристрої для відстеження на автомобілях, хоча незрозуміло, чи продають вони ці дані.

### Buck2: Наша система збірки з відкритим вихідним кодом

[Оригінал](https://engineering.fb.com/2023/04/06/open-source/buck2-open-source-large-scale-build-system/).
Meta випускає Buck2, систему збірки з відкритим вихідним кодом на GitHub, написану на Rust. Buck2 розділяє основні та специфічні для мови правила, а внутрішні тести показують, що збірка вдвічі швидша, ніж у Buck1, збільшений паралелізм та перероблений консольний вивід. Buck2 може підійти для багатомовних проектів середнього розміру, розроблених з розширеними можливостями для підвищення продуктивності та виразними, динамічними функціями залежностей. Meta ділиться технологічними проектами з відкритим вихідним кодом, включаючи ШІ, дані, розробку, фронтенд, мови, платформи, безпеку і VR, без будь-яких помітних оновлень або релізів.

[Discussion Service](http://news.ycombinator.com/item?id=35470371).
Система збірки з відкритим вихідним кодом Buck2 від Facebook привертає увагу завдяки своєму механізму інкрементальних обчислень і підтримці Windows.
