### [Якщо PEP 703 буде прийнято, Мета може виділити три інженерних роки на no-GIL CPython](https://discuss.python.org/t/a-fast-free-threading-python/27903/99)

- Оператор "pmf = sum(counters) # чи працює це? не пам'ятаю" працюватиме, якщо надано початковий порожній лічильник.
- Без початкового порожнього лічильника оператор почнеться з 0 і видасть помилку.
- Ця інформація важлива для розуміння того, як правильно використовувати інструкцію у програмуванні.

- Перехід на Python без GIL вимагатиме перебудови та оновлення розширень C-API, що може стати серйозною проблемою для кодових баз, які значною мірою на них покладаються.
- Facebook (Meta) зобов'язався інвестувати роки роботи інженерів у вдосконалення інтерпретатора Python і зробити можливим відключення GIL.
