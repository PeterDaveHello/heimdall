### [Google Photorealistic 3D Tiles and Unreal Engine](https://nilsbakker.nl/portfolio/3d-tiles/)

- Розробник використав Google Photorealistic 3D tiles API та Unreal Engine 5.1, щоб створити імерсивний досвід, який поєднує дані про місцезнаходження з динамічними відповідями від ChatGPT API.
- Додаток дозволяє користувачам вводити підказки у певному форматі та отримувати інформацію про місцезнаходження, включаючи цікаві факти, які відображаються у віртуальному світі у вигляді 3D-тексту.
- Розробник використав Google Maps API та платформу Cesium для отримання даних про місцезнаходження та геометрію, а також створив розумний механізм порталу з використанням системи частинок Ніагари для переходу користувача з поточного місцезнаходження до нового місця.

#### [Реакція галузі](http://news.ycombinator.com/item?id=36000631)

- Cesium та Unreal Engine тепер підтримують цей API, що потенційно уможливлює 3D проїзд рідними містами користувачів в іграх.
- У бета-версії API має обмеження на квоти, і дехто висловлює занепокоєння щодо законності використання даних Google Maps та правил використання дронів для картографування.

### [Інструмент Linux для показу прогресу для cp, mv, dd](https://github.com/Xfennec/progress)

- Progress - це інструмент, який працює на Linux, FreeBSD і macOS і показує відсоток скопійованих даних для основних команд, таких як cp, mv, dd, tar, gzip/gunzip і cat.
- Він може показувати приблизний час, пропускну здатність і має режим "зверху" для моніторингу.
- Progress легко встановити і запустити на deb-, archlinux-, rpm-системах, macOS з homebrew або MacPorts, а також можна зібрати з коду за допомогою бібліотеки ncurses.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=36000407)

- Обговорення використання індикаторів виконання у командах Linux, включно з пропозиціями щодо таких інструментів, як pv та Pipe Watch
- Обговорюються альтернативи dd, такі як ddrescue або rsync, а також переваги сповільнення передачі даних для запобігання перевантаженню спільних з'єднань.

### [FreeBSD витрачає 7% часу завантаження на сортування бульбашок у SYSINIT'ах](https://twitter.com/cperciva/status/1659558311920914432)

- Коли ядро FreeBSD завантажується у Firecracker (1 процесор, 128 МБ оперативної пам'яті), воно витрачає 7% часу на перебір своїх SYSINIT'ів.
- O(N^2) може сильно кусатися, коли ви сортуєте більше тисячі елементів. Час замінити bubblesort на щось швидше.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=36002574)

- Алгоритми сортування складно реалізувати на мові програмування C та C++, Rust може бути кращою альтернативою для загального програмування або легкого створення загальних структур даних та алгоритмів сортування
- Amazon тестує нову техніку сортування для оптимізації часу завантаження екземплярів EC2 при холодному старті, а FreeBSD почала використовувати швидший метод qsort для перерахування SATA дисків у драйвері AHCI, щоб зменшити час завантаження ядра. Було обговорено найповільнішу частину часу завантаження ядра та потенційні оптимізації, які можна зробити.

### [Перехід з Supabase](https://blog.val.town/blog/migrating-from-supabase)

- Вал Таун перейшов з Supabase на Render через проблеми з масштабуванням та локальними особливостями розробки.
- Відсутність документації Supabase та зламаний інструментарій для CLI ускладнювали локальну розробку, а обмеженість продукцією була основною проблемою.
- Зрештою, Вал Таун перейшов на простіший "ванільний" сервіс Postgres в Render, усунувши складні функції та залежності для більш оптимізованої архітектури бази даних.

#### [Реакція галузі](http://news.ycombinator.com/item?id=36004925)

- Мета Supabase - зробити Postgres простішим, але не обов'язково; рекомендується краща документація та більше прикладів
- Пропозиції щодо покращення інструментів CLI, попереднього перегляду та впровадження міграції, а також ставлення до бази даних як до рівня персистентності, а не як до додатку

### [Venture Predation](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=4437360)

### [Супутники виявили повсюдне скорочення запасів води в озерах світу](https://www.science.org/doi/10.1126/science.abo2812)

- Супутникові спостереження, кліматичні та гідрологічні моделі показують, що понад 50% великих природних озер і водосховищ зазнали втрати обсягу води за останні 30 років через людську діяльність та кліматичні фактори.
- Зменшення об'єму води в цих озерах і водосховищах становить значну загрозу для основних екосистемних послуг, включаючи зберігання прісної води, постачання продовольства, місця проживання водоплавних птахів, кругообіг забруднюючих речовин і поживних речовин, а також рекреацію.
- Висновки Яо та ін. підкреслюють нагальну потребу в удосконаленні стратегій управління водними ресурсами для збереження та захисту цих життєво важливих прісноводних ресурсів.

#### [Реакція промисловості](http://news.ycombinator.com/item?id=35999438)

- Стаття закликає суспільства до відповідальності за стале управління своїми природними ресурсами.
- Користувачі дискусійного сервісу обговорюють можливі шляхи вирішення проблеми нестачі води, зокрема опріснення води на основі чистої енергії, збереження та зменшення відходів, лісонасадження, кращий розподіл води та перенесення водомістких промислових процесів до водних джерел.
