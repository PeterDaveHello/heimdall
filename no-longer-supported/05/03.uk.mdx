### [Mojo - нова мова програмування для розробників ШІ](https://www.modular.com/mojo)

- Mojo - це нова мова програмування, розроблена спеціально для розробників штучного інтелекту (ШІ), з функціями, які спрощують процес розробки ШІ та полегшують читання і написання коду.
- Розроблена дослідниками Вашингтонського університету, Mojo оптимізована для роботи на графічних процесорах (GPU) та іншому спеціалізованому обладнанні, яке зазвичай використовується для досліджень і розробок ШІ.
- Творці Mojo сподіваються, що він допоможе демократизувати розробку ШІ, зробивши її більш доступною та легшою для програмістів усіх рівнів для створення складних додатків ШІ.

#### [Реакція галузі](http://news.ycombinator.com/item?id=35790367)

- Технічні переваги Mojo, такі як власність і відсутність збирання сміття, дають йому перевагу над конкурентами, водночас підтримуючи поточну екосистему Python
- Mojo стикається з конкуренцією з боку Triton, TVM, OpenXLA та ONNX Runtime, але пропонує новий варіант для розробників машинного навчання та ШІ

### [Посібник з програмування модулів ядра Linux](https://sysprog21.github.io/lkmpg/)

- Посібник з програмування модулів ядра Linux - це безкоштовна книга, яка навчає читачів, як писати модулі ядра, які розширюють функціональність ядра без необхідності перезавантаження системи.
- Книга охоплює різноманітні теми, включаючи заголовки, драйвери пристроїв, файлові системи, системні виклики, обробники переривань та планування завдань.
- Автор рекомендує витягувати, компілювати та завантажувати всі приклади з консолі. Книга знайомить з новими способами керування налаштуваннями модулів ядра, включаючи використання макросів module_init та module_exit, а також пояснює варіанти ліцензування модулів ядра. Крім того, модулі можуть приймати аргументи командного рядка за допомогою глобальних змінних і макросу module_param().

#### [Реакція індустрії](http://news.ycombinator.com/item?id=35782630)

- eBPF є новішою і дещо простішою/безпечнішою альтернативою деяким завданням модулів ядра.
- Посібник є гарною відправною точкою для вивчення модулів ядра, але може виявитися недостатнім для написання більш складних драйверів.

### [Microsoft зламала функцію Chrome для просування Edge](https://gizmodo.com/microsoft-windows-google-chrome-feature-broken-edge-1850392901)

- Microsoft випустила оновлення для Windows, яке зламало функцію Chrome, що ускладнило зміну браузера за замовчуванням і дратувало користувачів Chrome спливаючими вікнами, повідомляє Gizmodo.

- Цей крок є частиною моделі поведінки Microsoft, яка веде війну проти веб-браузерів, що не належать до Windows, та людей, які ними користуються.

#### [Реакція індустрії](http://news.ycombinator.com/item?id=35787707)

- Google розробляє пісочницю конфіденційності, але пропозиція була відхилена через компроміс між обслуговуванням різноманітного населення та захистом ідентичності користувачів.
- Користувачі Windows на корпоративних версіях зіткнулися з проблемою відкриття системних налаштувань за замовчуванням щоразу, коли вони відкривали Chrome через проблему з API, що призвело до занепокоєння щодо конфіденційності користувачів і потенційної антимонопольної поведінки.

- AMD вже випустила оновлення прошивки для усунення цієї проблеми, і користувачам рекомендується оновити свої системи якнайшвидше.
