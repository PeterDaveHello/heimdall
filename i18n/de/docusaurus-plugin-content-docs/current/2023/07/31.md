## [Was ist los, Python? Die GIL entfernt, ein neuer Compiler, optparse veraltet](https://www.bitecode.dev/p/whats-up-python-the-gil-removed-a)

- Im Juli 2023 wurden mehrere Aktualisierungen und Entwicklungen für Python angekündigt.
- Das Global Interpreter Lock (GIL) in Python wurde entfernt, was die Leistung verbessern und eine bessere Nutzung mehrerer Kerne ermöglichen kann.
- Python hat jetzt einen neuen Compiler namens LPython.
- Pydantic 2 wurde mit Verbesserungen bei Geschwindigkeit und Stabilität veröffentlicht.
- Die Module getopt und optparse wurden sanft veraltet.
- Cython 3.0 bietet jetzt eine bessere Unterstützung für reines Python.
- Ein neuer Vorschlag mit der Bezeichnung PEP 722 wurde für die Angabe von Abhängigkeiten in Skripten mit nur einer Datei gemacht.
- Die Unterstützung von Python VSCode wurde verbessert, um die Leistung zu erhöhen.
- Es wurde eine neue terminalbasierte Malanwendung namens textual-paint eingeführt.

#### [Reaktionen](https://news.ycombinator.com/item?id=36935041)

- Die Diskussion dreht sich um die Optimierung von Python-Code für Multithreading-Leistung und die mögliche Abschaffung des Global Interpreter Lock (GIL) in Python.
- Es werden verschiedene Perspektiven aufgezeigt, einschließlich Vorschlägen für den Einsatz von Tools wie `gunicorn` und `gevent` für Multiprocessing und Multithreading.
- Alternativen wie die Verwendung von Sprachen wie C++ oder Rust zur Verbesserung der Leistung werden in Betracht gezogen.
- Es werden Vorschläge zur Optimierung gemacht, wie z. B. die Verwendung von Caching oder Shared Memory mit Redis oder memcached.
- Es werden Bedenken zu Themen wie Deadlocks und Ressourcennutzung geäußert und Vorschläge für alternative Sprachen oder Technologien gemacht.
- Die Leistungsnachteile von Python und mögliche Lösungen wie die Verwendung anderer Sprachen oder die Nutzung von JIT-Compilern werden diskutiert.
- Die Kompatibilität der verschiedenen Programmiersprachen, die Herausforderungen des Übergangs und die Komplexität des Designs und der Verwendung von Programmiersprachen werden erwähnt.
- Die potenziellen Vor- und Nachteile der Entfernung der GIL in Python werden diskutiert, wobei es unterschiedliche Meinungen zu Leistungsverbesserungen und potenziellen Gleichzeitigkeitsproblemen gibt.
