# 2024-01-21

## [Spotube: Cliente de Spotify de código abierto con integración en YouTube](https://github.com/KRTirtho/spotube)

- Spotube es un cliente de Spotify de código abierto que se puede utilizar sin necesidad de suscribirse a Spotify Premium.
- Utiliza la API de datos de Spotify y ofrece fuentes de audio alternativas como YouTube, Piped.video o JioSaavn.
- Entre sus características se incluyen: sin anuncios, canciones descargables, compatibilidad con varias plataformas, tamaño reducido, bajo consumo de datos, inicio de sesión anónimo, letras sincronizadas con el tiempo y rendimiento nativo. No recopila datos de usuario ni es compatible con Spotify Shows & Podcasts.

### [Reacciones](https://news.ycombinator.com/item?id=39066136)

- El debate abarca varios aspectos de las plataformas de streaming de música, como las claves API, las limitaciones de Spotify para los podcasts, los problemas de DRM, la calidad del audio, la comparación entre Spotify y YouTube, las frustraciones con la reproducción aleatoria, las repercusiones en los ingresos de los artistas, plataformas alternativas como Bandcamp y sugerencias para apoyar económicamente a los músicos.
- La conversación hace hincapié en la importancia de mejorar la remuneración de los artistas y la exploración de modelos alternativos de streaming.
- Este amplio debate arroja luz sobre los retos actuales y las posibles soluciones en la industria del streaming musical.

### [Reacciones](https://news.ycombinator.com/item?id=39074062)

## [Presentación de Bun Shell: Simplificación de las secuencias de comandos Shell en JavaScript](https://bun.sh/blog/the-bun-shell)

- Ejecutar scripts de shell en JavaScript puede resultar complicado debido a las diferencias de sintaxis y comandos entre plataformas.
- Bun Shell es un lenguaje e intérprete integrado en Bun que permite la creación de scripts shell multiplataforma en JavaScript y TypeScript.
- Bun Shell pretende ofrecer una solución ágil y eficaz para ejecutar scripts de shell en JavaScript, abordando las complejidades e incoherencias de los shells existentes.

### [Reacciones](https://news.ycombinator.com/item?id=39071688)

- Los usuarios están discutiendo el nuevo shell llamado Bun Shell, expresando sus preocupaciones sobre la compatibilidad, la sustitución de los comandos existentes, y coreutils hinchado.
- La herramienta es alabada por su sencillez, innovación y utilidad en tareas de scripting.
- En la conversación también se habla de la disponibilidad de diferentes carcasas y se recomiendan opciones alternativas.
- El desarrollador de Bun responde a preguntas y comentarios, resolviendo dudas sobre los nombres de las importaciones y la estabilidad.
- El debate se amplía al lenguaje de programación Zig, con usuarios que comparten experiencias y discuten su estabilidad.
- Otros temas tratados incluyen scripting en JavaScript, problemas de rendimiento con comandos shell en Node.js, plantillas etiquetadas, Lit Element y la biblioteca rimraf.
- Preocupan los riesgos de seguridad y las limitaciones del uso de Bun Shell.
- La discusión también explora ZSH, lenguajes de plantillas, archivos de configuración JSON y YAML, y compatibilidad entre plataformas.
- Surge un debate sobre la ejecución síncrona frente a la asíncrona y las experiencias negativas con un proveedor de alojamiento.
