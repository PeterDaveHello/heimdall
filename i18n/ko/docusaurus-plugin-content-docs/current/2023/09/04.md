## [콜라: 실시간 공동 편집을 위한 텍스트 CRDT](https://nomad.foo/blog/cola)

- 이 블로그 게시물에서는 실시간 공동 편집을 위해 Rust로 작성된 "콜라"라는 텍스트 충돌 없는 복제 데이터 유형(CRDT)을 적용하는 방법을 소개합니다.
- 일반 텍스트 문서를 표현할 때 시퀀스 CRDT의 기술적 설계를 살펴봅니다. 툼스톤, 버전 벡터, 실행 길이 인코딩, 링크된 목록, B-트리, G-트리와 같은 전략과 데이터 구조를 사용하여 공동 편집을 효과적으로 구현할 때 발생하는 문제와 해결책을 다룹니다.
- 상대적으로 '콜라'는 다른 CRDT에 비해 여전히 빠르지만, 게시물에서 개선해야 할 중요한 부분으로 강조한 삭제 취소를 지원하지 않습니다.

### [반응](https://news.ycombinator.com/item?id=37373796)

- Cola는 실시간 공동 편집을 위해 설계된 데이터 구조인 텍스트 충돌 없는 복제 데이터 유형(CRDT)으로, 노드를 저장하는 동적 배열과 다른 노드를 참조하기 위한 인덱스를 활용합니다.
- 이 구현은 다른 CRDT와 비교하여 벤치마킹한 결과 작동 속도 측면에서 만족스러운 성능을 보여주었습니다. 하지만 현재 서식 있는 텍스트 서식을 지원하지 않습니다.
- 콜라(Cola)는 Rust로 작성되었으며 MIT 라이선스에 따라 GitHub에서 공개적으로 사용 가능하며, 접근성과 향후 개선 가능성을 인정받고 있습니다.
