- 인제뉴이티는 NASA의 퍼시버런스 탐사선의 기술 시연 및 공중 정찰 역할을 수행하며 화성의 겨울철 전력 문제와 같은 어려움에도 불구하고 화성에서 동력 및 제어 비행이 가능하다는 것을 보여주었습니다. 이제 인제뉴이티 팀은 최종 테스트를 수행하고 남은 데이터를 다운로드하여 임무를 공식적으로 마무리할 예정입니다.

## [rhboot / shim 소프트웨어의 범위 외 쓰기 취약점 수정](https://github.com/rhboot/shim/commit/0226b56513b2b8bd5fd281bce77c40c9bf07c66d)

- 이 요약은 소프트웨어 프로젝트 rhboot / shim의 코드 문제를 다룹니다.
- 이 문제는 파일 검색 중 HTTP 헤더에 대한 잘못된 신뢰로 인해 발생하며, 이로 인해 범위를 벗어난 쓰기 취약점이 발생합니다.
- 버퍼가 할당된 크기를 초과하지 않도록 하는 패치가 적용되어 취약점이 해결되었습니다.
- 이 문제는 Microsoft 보안 대응 센터의 빌 데미르카피가 발견했습니다.

### [반응](https://news.ycombinator.com/item?id=39132300)

- Linux 배포판에서 보안 부팅을 위해 사용하는 shim EFI 부트로더에서 심각한 취약점(CVE-2023-40547)이 발견되었습니다.
- 공격자는 로컬, 인접 네트워크 또는 HTTP 부팅을 통해 원격으로 이 버그를 악용하여 보안 부팅을 우회하고 무단 액세스 권한을 얻을 수 있습니다.
- 이 취약점으로 인해 보안 부팅의 한계, Microsoft의 보안 부팅 구현, GPLv3 라이선스의 호환성, 암호화 및 서명 사용, HTTP 부팅과 관련된 잠재적 위험에 대한 논의가 촉발되었습니다. 이 버그로 인해 보안 취약점이 노출되고 보안 부팅의 효과에 대한 우려가 제기되고 있습니다.
