- Eric Froemling의 버그 보고는 앱 스토어 검토 과정을 수용하는 방법에 대해 Python 개발자들 사이에서 논의를 촉발시켰으며, 난독화 및 배포 수준의 패치와 같은 해결책을 고려하게 되었습니다.

### [반응](https://news.ycombinator.com/item?id=40815130)

- Python 앱이 바이너리에 'itms-services' URL 스킴이 포함되어 있다는 이유로 Apple App Store에서 거부되고 있으며, PyInstaller로 빌드된 경우 Windows Defender에서도 이러한 앱을 플래그합니다.
- 개발자들은 주요 운영 체제 플랫폼이 비용이 많이 드는 코드 서명 인증서를 요구하여 소규모 개발자들이 앱 배포에 어려움을 겪고 있다고 주장합니다.
- 대안으로는 Mac 앱을 위한 Homebrew 사용이나 Windows 빌드를 위한 SignPath의 무료 인증서를 사용하는 것이 제안되었으며, 이는 앱 배포에서 보안 관련 거부 문제의 더 넓은 문제를 강조합니다.
