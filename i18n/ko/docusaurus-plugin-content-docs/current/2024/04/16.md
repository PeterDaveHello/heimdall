## [NIST P521 개인 키가 노출되는 치명적인 PuTTY 취약점](https://www.chiark.greenend.org.uk/~sgtatham/putty/wishlist/vuln-p521-bias.html)

- PuTTY 버전 0.68~0.80에는 편향된 서명 생성을 통해 NIST P521 개인 키를 노출하는 심각한 취약점이 포함되어 있어 공격자가 키를 복구하고 서명을 위조할 수 있어 SSH 서버 인증이 위험해질 수 있습니다.
- 사용자는 영향을 받는 키를 해지하고 새 키를 생성한 후 버전 0.81로 업그레이드하여 문제를 해결해야 합니다. 이 취약점은 PuTTY의 DSA 서명에 대한 무작위성 생성으로 인해 발생하며, 솔루션은 보다 안전한 접근 방식을 구현합니다.

### [반응](https://news.ycombinator.com/item?id=40044665)

- PuTTY는 ECDSA의 편향된 논스 생성으로 인한 취약점을 가지고 있어 보안 문제가 제기되고 있으며 EdDSA 및 Schnorr와 같은 대안이 제시되고 있습니다.
- 암호화의 명확한 통신, 보안 인증을 위한 하드웨어 토큰, Windows 시스템에서 SSH의 문제 해결에 중점을 두며 PuTTY와 같은 GUI 도구를 선호합니다.
- 권장 사항에는 손상된 서버에 연결할 위험을 고려하여 더 안전한 알고리즘으로 전환하고, 기업 SSH 사용 시 보안 수준을 위해 RSA와 EC 키 크기를 비교하는 것이 포함됩니다.
