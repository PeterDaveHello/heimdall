## [Redka를 소개합니다: SQLite로 재탄생한 Redis](https://github.com/nalgeon/redka)

- Redka는 내구성 있는 데이터 저장소, ACID 트랜잭션, SQL 보기, Redis API와의 호환성 등의 기능을 제공하는 SQLite를 활용하여 Redis를 재구성하는 프로젝트입니다.
- 문자열 관련 명령에 중점을 두고 주요 Redis 데이터 유형과 명령을 지원하며, 독립형 서버 옵션과 Go API를 사용한 인프로세스 서버 옵션을 모두 제공합니다.
- Redis보다 2~6배 느리지만 Redka는 상당한 읽기 및 쓰기 작업을 관리할 수 있으며, 향후 잠재적인 개선 사항과 BSD-3-Clause 라이선스에 따른 오픈 기여를 약속합니다.

### [반응](https://news.ycombinator.com/item?id=40030746)

- GitHub 토론은 Redis와 SQLite를 결합하여 작은 메모리 공간, ACID 트랜잭션 및 SQL 인터페이스와 같은 이점을 제공하는 사용자 친화적인 API를 목표로 하는 프로젝트인 Redka를 중심으로 진행됩니다.
- 사용자들은 Redka의 단순성을 높이 평가하지만, Redis와 비교할 때 Lua 스크립팅 지원과 같은 기능이 누락된 것에 대해 우려를 표합니다.
- 효율적인 캐싱, 샤딩 및 성능을 위해 Garnet, Postgres, SSDB 및 KVrocks와 같은 대안이 논의되며, SQLite를 통한 프로덕션 워크로드에는 TiKV 또는 FoundationDB를 제안합니다.
