## [AST-grep：用于代码结构搜索、语义分析和重写的 CLI 工具](https://github.com/ast-grep/ast-grep)

- AST-grep 是一款命令行工具，可利用抽象语法树（AST）进行代码结构搜索、过滤和重写。
- 它的功能类似于 grep，但不是匹配文本，而是匹配 AST 节点。
- 用户可以编写类似代码的模式，以匹配具有相同语法结构的代码。
- AST-grep 提供高级代码操作、直观的模式匹配和类似 jQuery 的 AST 遍历 API。
- 它可以通过软件包管理器安装，也可以从源代码构建。
- 该工具旨在简化 AST 编程，对开源库作者、技术负责人和安全研究人员特别有用。

### [反应](https://news.ycombinator.com/item?id=38590984)

- Ast-grep, grep-ast 和 syntax-searcher 作为代码结构搜索、内衬和重写的有效工具，提供了查找和显示匹配代码的功能。
- 对这些工具的文件和复杂性提出了关切，指出了可以改进的领域。
- 此外，还提到了 semgrep、Conscat 和 OpenRewrite 等其他工具，作为代码重构和识别语义相似性的选择，从而扩大了开发人员可用工具的范围。
- 语言建模的局限性和确定语义等效性的挑战得到了承认，凸显了这些任务的复杂性。
- 讨论了这些工具的潜在增强和改进措施，展示了在代码分析和完善领域不断发展和创新的决心。

## [Krasue：隐形 Linux Rootkit 入侵泰国电信两年未被发现](https://arstechnica.com/security/2023/12/stealthy-linux-rootkit-found-in-the-wild-after-going-undetected-for-2-years/)

### [反应](https://news.ycombinator.com/item?id=38590827)

- 对话讨论了与 Linux 安全有关的各种话题，包括 Krasue rootkit 的发现和对 Linux 系统脆弱性的担忧。
- 它探讨了防病毒软件的有效性以及加强 Linux 桌面安全措施的必要性。
- 对话还探讨了在 Linux 上实施运行时权限系统的可能性，并将 Linux 的安全功能与其他操作系统进行了比较。

## [提高网络无障碍程度：鼓励网站整合 RSS](https://rknight.me/please-expose-your-rss/)

- 作者主张网站所有者加入 RSS 链接，以支持 RSS 并提高网站的可访问性。他们建议使用标准链接和 RSS 自动发现方法，使浏览器和软件更容易找到网站的 RSS 源。作者还建议为多个 RSS 源加入多个链接标签，并强调显示可识别 RSS 按钮的重要性。

### [反应](https://news.ycombinator.com/item?id=38595855)

- 由于缺乏可见性和浏览器的支持，在网站上发现和访问 RSS 源存在问题。
- Google Reader 的消失影响了 RSS 源的可用性和普及。
- RSS 源缺乏对广告的本地支持，因此对于那些喜欢无广告体验的人来说，RSS 源是首选。
- RSS 订阅内容不一致，格式和质量参差不齐。
- 使用 RSS 阅读器的优势在于方便和个性化的内容聚合。
- 关于以编程的方式刮取内容、出售 RSS 源中的数据以及公开订阅者人数的辩论。
- 建议将 RSS 扩展到论坛帖子，并将静态网站与 fediverse feed 整合。
- 讨论网站设计、RSS 源的设置和维护，以及 RSS 阅读器和社交聚合器之间的比较。
- 在 RSS 源中出现广告是一个争论不休的话题。

## [提高互联网性能：引入低延迟、低损耗和可扩展吞吐量的 L4S 架构](https://datatracker.ietf.org/doc/rfc9330/)

- RFC 9330 引入了 L4S 架构，旨在通过减少队列延迟、实现低延迟和可扩展吞吐量来增强互联网应用。
- L4S 架构提出了新的拥塞控制机制，这些机制与现有的控制机制一起工作，并使用经过修改的 "显式拥塞通知"（ECN）。
- 目标是为游戏、流媒体和视频会议等互动应用提供性能更高的高质量互联网服务。
- 该架构可逐步部署，并兼容不同的网络类型。
- 文中还讨论了部署注意事项、交通治安和安全注意事项。
- 它强调了拥塞控制算法中准确的 ECN 反馈和可扩展性的重要性。
- 该文件提供了有关拥塞控制、网络协议和差异化服务的其他资源参考。

### [反应](https://news.ycombinator.com/item?id=38597744)

- 低延迟、低损耗和可扩展吞吐量（L4S）互联网服务旨在减少延迟，提高视频通话等应用的性能。
- L4S 消除了缓冲区膨胀，改进了拥塞信号，从而减轻了数据传输的延迟和中断。
- L4S 可通过各种传输协议（包括 TCP 和 QUIC）实现，如果被互联网服务提供商采用，有可能提高网络性能。

## [杀虫剂中的惰性成分可能对蜜蜂造成更大威胁](https://theconversation.com/inert-ingredients-in-pesticides-may-be-more-toxic-to-bees-than-scientists-thought-218005)

- 新研究发现，以前被认为无毒的杀虫剂中的惰性成分对蜜蜂的危害可能比想象的更大。
- 一项研究将蜜蜂暴露在含有活性成分和惰性成分的杀真菌剂中，结果发现整个配方会损害蜜蜂的记忆，而单独的活性成分则不会。
- 惰性成分与活性成分功能不同，监管方式也不同，因此很难确定其毒性。需要对农药配方进行全面测试，以了解它们对蜜蜂健康的影响。通过在标签上标明惰性成分，并要求对长期影响进行更多测试，农药法规的改变有助于保护蜜蜂。

### [反应](https://news.ycombinator.com/item?id=38593008)

- 杀虫剂中的 "惰性 "成分通常被认为是无毒的，但它们对蜜蜂的危害可能比以前认为的更大。
- 一项研究显示，杀真菌剂的完整配方会损害蜜蜂的记忆，而单独的活性成分则不会。
- 杀虫剂的使用会对蜜蜂产生直接的负面影响，并间接影响它们的食物来源，使它们更容易受到寄生虫和疾病的侵害。围绕农药中惰性成分的监管和透明度引发了人们对潜在危害的担忧。
