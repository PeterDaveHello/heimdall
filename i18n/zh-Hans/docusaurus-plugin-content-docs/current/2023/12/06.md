# 2023-12-06

## [迷你蜂鸣器安卓系统获得端到端加密的 iMessage](https://www.beeper.com/)

- Beeper Mini 是安卓系统的第三方 iMessage 客户端，可让安卓用户直接向苹果服务器发送和接收端到端加密信息，而无需 Mac 服务器中继。
- 用户可以加入 iMessage 群聊，使用所有聊天功能，并使用 Beeper Mini 在 iMessage 上注册自己的 Android 手机号码。
- 开发人员计划在未来增加对 SMS/RCS、WhatsApp 和 Signal 等其他聊天网络的支持，并指出第三方 iMessage 客户端以前就存在，如 iChat 等多协议聊天应用。

### [反应](https://news.ycombinator.com/item?id=38531759)

- 讨论围绕着信息应用与不同平台的兼容性以及逆向工程苹果协议的合法性展开。
- 此外，还讨论了信息平台之间的互操作性及其对垃圾邮件和网络钓鱼的影响。
- 此外，还讨论了苹果 iMessage 的局限性和实现跨设备标准化的愿望，揭示了开发人员在实现兼容性和安全性方面面临的挑战。

## [火狐浏览器使用率下降威胁到美国政府网站的支持](https://www.brycewray.com/posts/2023/11/firefox-brink/)

- 由于市场份额较低，Mozilla Firefox 浏览器对美国政府网站的支持可能会减少。
- 美国网页设计系统只支持使用率超过 2% 的浏览器，目前火狐浏览器的使用率为 2.2%。
- 移动设备上 Chrome 浏览器和 Safari 浏览器的崛起导致了火狐浏览器使用率的下降。
- 如果火狐浏览器低于 2% 的门槛，政府网站可能不再支持它，这可能会导致多米诺骨牌效应，企业也会停止支持。
- 如果火狐浏览器继续下滑，作者对其未来的相关性表示担忧。

### [反应](https://news.ycombinator.com/item?id=38531104)

- 人们担心火狐浏览器的潜在衰落及其对 Mozilla 的影响。
- 在政府技术中支持 Firefox 的使用率和必要性是人们争论的话题。
- 讨论内容包括用户隐私、政府资助、网络标准、用户代理字符串以及 Firefox 的性能和兼容性等方面的考虑。

## [了解 iMessage了解苹果的信息协议](https://jjtech.dev/reverse-engineering/imessage-explained/)

- 本博文全面介绍了 iMessage 的内部工作原理，尤其侧重于两个关键组件：苹果推送通知服务 (APN) 和 IDentity 服务 (IDS)。
- Apple 推送通知服务 (APN) 负责促进 iMessage 中推送通知的发送和接收。
- 身份识别服务（IDS）在加密对话中充当密钥服务器，允许交换公开密钥。

### [反应](https://news.ycombinator.com/item?id=38532167)

- 摘要涵盖与苹果 iMessage 有关的各种主题：安全性、信息应用偏好、设备耐用性、加密标准、逆向工程和互操作性。
- 共同的主题包括对 iMessage 安全漏洞的担忧，以及与 Signal 等其他信息应用的比较。
- 讨论还涉及 Mac 与 PC 相比的耐用性和使用寿命，以及实验和原型开发在软件开发中的重要性。

## [人工智能驱动的大规模间谍活动：隐私的侵蚀](https://www.schneier.com/blog/archives/2023/12/the-internet-enabled-mass-surveillance-ai-will-enable-mass-spying.html)

- 人工智能（AI）正在彻底改变监控和间谍活动，以前所未有的规模跟踪活动和了解对话。
- 现在，由人工智能驱动的系统可以总结会议内容，组织数以百万计的对话，实现大规模间谍活动，所有数据都可以大量保存、搜索和理解。
- 大规模间谍活动可以揭示个人的复杂细节，包括他们的关系、联盟和对话，而 Siri 和 Alexa 等无处不在的麦克风则为间谍活动提供了助力。
- 政府、公司和技术垄断企业已经在进行大规模监控，而大规模间谍活动将使个性化广告更加恶化，并侵蚀隐私。
- 限制大规模间谍活动需要强有力的数据隐私法规，尽管迄今为止在解决大规模监控方面几乎没有采取任何措施。

### [反应](https://news.ycombinator.com/item?id=38530880)

- 文章和讨论围绕大规模监控的政治影响、人工智能的潜在负面影响以及对隐私和数据收集的担忧展开。
- 涉及的主题包括政府监控、技术的社会影响、人工智能系统中的偏见以及人工智能和监控使用中保障措施的重要性。
- 对话强调了这些问题的复杂性以及深思熟虑和监管的必要性。

## [SQLite 引入 JSONB 以加快 JSON 处理速度](https://sqlite.org/forum/forumpost/fa6f64e3dc1a5d97)

- SQLite 引入了一项名为 JSONB 的新功能，它是对 SQLite JSON 函数的重写。
- JSONB 可以比原始 JSON 函数快很多，具体取决于使用模式。
- JSONB 将 JSON 的内部二进制表示转换为连续的字节数组，使其变得更小，并有可能减少数据库的大小。

### [反应](https://news.ycombinator.com/item?id=38540421)

- SQLite 引入了一种名为 JSONB 的新数据类型，可对以二进制格式存储的 JSON 数据进行高效操作。
- JSON 和 JSONB 之间的选择取决于具体要求和用例，讨论内容包括 JSON 对象中键的排序，以及 JSON 和 JSONB 在 Postgres 和 SQLite 等数据库中的区别。
- 作者表示不喜欢基于文档的数据库，并讨论了迁移数据所面临的挑战，同时还提到了使用嵌入式 SQLite 数据库作为记录的好处和缺点。
- 然后，重点转向在 SQLite 中采用二进制 JSON 格式，并强调了可移植性和向后兼容性的重要性。大家对 SQLite 中 JSONB 的稳定性和可读性表示赞赏，同时也对该项目的长期可持续性表示担忧。
- 讨论简要涉及 SQLite 软件项目领导层的过渡，并表达了对 JSON 和 SQL 注入安全漏洞的担忧。

## [安卓 14 将引入 AVF：虚拟化安卓工作负载以增强功能](https://android-developers.googleblog.com/2023/12/virtual-machines-as-core-android-primitive.html)

- Android 虚拟化框架（AVF）将在 Android 14 设备中推出，为平台开发人员提供新的功能。
- AVF 可在 Android 上实现虚拟化，从而隔离工作负载和操作系统。
- 开发人员可以选择单向隔离（Android 控制虚拟机）或双向隔离（Android 和虚拟机之间完全隔离）。

### [反应](https://news.ycombinator.com/item?id=38538100)

- 谷歌正在安卓系统中引入虚拟机技术，以提高应用程序的安全性和隔离性。
- 讨论探讨了虚拟化在 Android 中的潜在用途、漏洞和优势。
- 它还研究了其他虚拟化选项以及在安卓设备上运行非安卓虚拟机的能力。
- 面对操作系统的信息泄露或恶意行为，信任和隔离是重点。
- 话题延伸到可信计算技术在机器学习中的应用、数据隐私问题以及数字身份证和远程认证的有效性。
- 对于敏感的身份验证目的，首选 Root 手机和隔离实施。
- 是否需要银行电话进行交易验证受到质疑。
- 讨论还涉及对智能手机的依赖、虚拟机中的保密计算以及欧盟为让个人对其设备拥有更多控制权所做的努力。
- 作者认为，控制自己的设备和选择供应商可以提高技术工具的实用性。
