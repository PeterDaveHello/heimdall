## [研究发现：改善深度睡眠可预防痴呆症](https://www.monash.edu/news/articles/improving-deep-sleep-may-prevent-dementia,-study-finds)

- 莫纳什大学最近的一项研究发现，缺乏深度睡眠（即慢波睡眠）与老年人患痴呆症的风险增加之间存在联系。
- 研究表明，60 岁以上的人即使每年深度睡眠减少 1%，患痴呆症的风险也会增加 27%。
- 这项研究对 346 名 60 岁及以上的参与者进行了长达 17 年的痴呆症发病情况监测，结果表明，改善或维持慢波睡眠可能是一种预防痴呆症的措施。

### [反应](https://news.ycombinator.com/item?id=38097184)

- 在线讨论的重点是通过关注咖啡因的影响，尤其是对具有慢代谢基因 CYP1A2 的人的影响，来改善深度睡眠和预防痴呆症。
- 会上提出了各种策略，包括减少咖啡因摄入量、定期锻炼、保持睡眠卫生和服用特定的补充剂。不过，也有人提出了基因检测公司的隐私问题和咖啡因的健康风险。
- 与会者一致认为，不同的人可能因其独特的生理因素而需要不同的睡眠策略，但所有人都能从锻炼、良好的营养和保持睡眠卫生中受益。

## [罗伯-派克的《编程规则》（1989 年）](https://users.ece.utexas.edu/~adnan/pike.html)

- 著名程序员罗布-派克（Rob Pike）概述了软件开发的五条值得注意的规则，这些规则强调简单性，不鼓励过早优化。
- 这些规则包括使用简单明了的算法和数据结构，以减少错误和便于实施，重点是数据结构，因为它们在编程中起着至关重要的作用。
- Pike 还敦促不要预测程序将在哪些方面耗费大量时间，从而避免早期优化，并强调在选择速度调整之前进行测量的必要性。

### [反应](https://news.ycombinator.com/item?id=38097031)

- Hacker News 的讨论强调了 Rob Pike 1989 年的编程规则，主要重点是数据结构对优化代码的重要性，而不是算法。
- 与会者批评了当前主要集中于算法的面试趋势，认为这无法反映真实世界的编程任务。他们主张有效使用和理解数据结构，这可以使算法更加明显。
- 其他讨论的主题包括性能优化、错误处理策略、对象关系映射（ORM）的使用、SQL 知识以及编程中的浪费概念。与会者建议在优化之前先进行测量，并在实施理论上快速的解决方案之前考虑硬件限制。

## [朝日 Linux 从苹果硅移植项目变为 macOS 漏洞猎手](https://www.theregister.com/2023/11/01/asahi_linux_mac_black_screen/)

- 旨在为苹果硅Mac电脑改造Linux的Asahi Linux公司发现了苹果MacOS系统中的漏洞，这些漏洞可能导致某些型号的MacBook Pro无法启动。
- 这些错误与较新的 MacOS 版本管理刷新率的方式有关，主要影响配备 ProMotion 显示器的 MacBook Pro 机型（14 英寸和 16 英寸版本）；当合并到升级和启动序列中时，可能会导致黑屏，必须进行设备固件更新 (DFU) 恢复。
- Asahi Linux 澄清说，数据没有受到威胁，问题仅限于特定的 MacOS 版本（Sonoma 14.0+ 和 Ventura 13.6+）；他们的安装程序已经更新，如果刷新率不是 ProMotion，则无法安装。

### [反应](https://news.ycombinator.com/item?id=38101328)

- Asahi Linux 是一个由志愿者组成的团队，它发现了一个可导致 Mac 无法启动的错误，从而超越了苹果公司的专业团队，证明了自己的能力。
- 该小组的调查结果引发了关于苹果公司是否需要投入更多资源修复漏洞以及参与这些调查的志愿者应如何获得补偿的讨论。
- 人们对装有 Apple Silicone 或 T2 安全芯片的 Mac 上的自动数据加密、由此可能造成的数据丢失以及随后的定期数据备份和更新延迟建议的担忧得到了强调。

## [C++23：移除垃圾回收支持](https://www.sandordargo.com/blog/2023/11/01/cpp23-garbage-collection)

- C++23 打算完全取消对垃圾回收的支持，而垃圾回收功能在大多数情况下都是未实现和未支持的。
- 这一决定被认为是精简语言的积极措施，并将导致从 std::库中删除特定名称，如 declare_reachable、undeclare_reachable 等。
- 尽管有这些变化，但现有的垃圾收集器，主要是那些用 C++ 编程的机器和运行其他语言的机器，不会受到影响，因为它们并不依赖于该标准。

### [反应](https://news.ycombinator.com/item?id=38097984)

- 即将发布的 C++23 标准可能不再支持垃圾回收，这导致开发者社区对自动内存管理的要求众说纷纭。
- 辩论延伸到 C++ 的复杂性、其不可预见的编码行为以及取消过多功能的潜在后果，语言障碍给非英语与会者造成了困难。
- 对 C++ 和 Rust 进行了比较，表明 Rust 可以替代通常由 C++ 执行的任务，从而突出了理解各种系统功能的重要性。

- 开发人员和研究人员发现，在更新 macOS 的同时确保数据分析流程的完整性（通常与遗留代码相关）是一项挑战。
- 有人提出了 Docker、MacPorts 和 Nix 等解决方案，以及从 Homebrew 过渡到 MacPorts 来管理 macOS 软件包的方案，但发现这些方案具有挑战性，而且会增加工作量。
- 一些人建议使用 Linux 发行版，如 RedHat Enterprise Linux、Rocky Linux 和 Almalinux，因为 macOS 的更新、工具和过时的应用程序存在问题。用户对苹果过时系统的不满以及对更好的开发人员支持的渴望也浮出水面。
