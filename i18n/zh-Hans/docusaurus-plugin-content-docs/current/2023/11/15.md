# 2023-11-15

## [Blender 4.0：令人兴奋的更新和兼容性变更](https://wiki.blender.org/wiki/Reference/Release_Notes/4.0)

- Blender 4.0 于 2023 年 11 月 14 日发布，在动画、装配、建模等多个领域进行了重大更新和改进。
- 该版本包括对网格格式的更改和对显卡的新兼容性要求。
- 此次更新还增强了几何节点、着色、纹理、循环和用户界面等方面的功能。

### [反应](https://news.ycombinator.com/item?id=38262315)

- 用户们深入探讨了与 Blender 有关的各种话题，如发布说明、动画行业的就业前景，以及线性代数和编程在渲染和角色动画中的应用。
- 会议还讨论了 Blender 作为一个开源项目的成功及其用户界面的开发。
- 用户将 Blender 与 Adobe 软件进行了比较，并探讨了内置资源共享功能的潜力。

## [欧盟议会采取措施保护儿童免受网络性虐待](https://www.europarl.europa.eu/news/en/press-room/20231110IPR10118/child-sexual-abuse-online-effective-measures-no-mass-surveillance)

- 公民自由委员会批准了保护儿童免受网上儿童性虐待的新措施。
- 将要求互联网提供商评估其服务被滥用的风险，并采取措施防止对儿童的性虐待。
- 如果预防措施失效，可以使用法院批准的侦查令来查找非法材料，并建立欧盟儿童保护中心来执行相关规则。

### [反应](https://news.ycombinator.com/item?id=38261415)

- 欧盟议会公民自由委员会在拟议的聊天控制法规中删除了聊天控制要求和受保护的加密技术，但最终定稿的谈判仍有待进行。
- 有人对扫描信息可能被滥用于儿童性虐待材料表示关切。
- 欧盟议会否决加密后门和扫描私人信息被视为隐私权倡导者的胜利，但科技公司可能会游说豁免。

## [谷歌的 GraphCast 人工智能模型彻底改变了天气预报方法，其效果优于传统方法](https://deepmind.google/discover/blog/graphcast-ai-model-for-faster-and-more-accurate-global-weather-forecasting/)

- 谷歌开发了一种名为 GraphCast 的人工智能模型，与传统方法相比，它能更准确、更快速地做出中程天气预报。
- 该模型可提前 10 天预测天气状况，从而对气旋和洪水等极端天气事件发出预警。
- GraphCast 利用机器学习和图神经网络对历史天气数据进行训练，在准确性方面优于行业黄金标准天气模拟系统，并能更好地预测未来天气。
- 该模型的代码已经开源，使全世界的科学家和预测人员都能从中受益，并有可能拯救生命，为各行各业和社会的重要决策提供支持。

### [反应](https://news.ycombinator.com/item?id=38264641)

- 谷歌 DeepMind 开发了一种名为 GraphCast 的新型人工智能天气预报模型，该模型利用机器学习将多个数值天气模型的数据结合起来，以提高准确性。
- 讨论围绕现有应用程序接口的局限性、历史天气数据的可用性以及极端事件数据的整合展开。
- 在天气预报中使用机器学习既有潜在的好处，也有挑战，同时还存在行业竞争。这项技术可能会对依赖准确天气预报的各行各业产生影响。

## [某些处理器中的漏洞可导致故障状态，给云提供商带来安全风险](https://lock.cmpxchg8b.com/reptar.html)

- 谷歌的多个研究小组发现了某些处理器中的一个漏洞，该漏洞可导致处理器进入故障状态，造成不可预测的行为和系统停止。
- 该漏洞涉及使用多余的 rex 前缀和快速短重复移动 (FSRM) 功能。
- 英特尔发布了更新的微码和解决方法，以解决这个对云提供商有严重安全影响的漏洞。

### [反应](https://news.ycombinator.com/item?id=38266773)

- "Reptar "是一个 CPU 漏洞，会引发机器检查异常和处理器停机，谷歌和其他研究团队已经发现了这个漏洞。
- 文章讨论了 "Reptar "对 AMD CPU 的影响，并强调了 CPU 设计所面临的挑战。
- 文章还提到了另一个影响英特尔处理器的漏洞，该漏洞可能导致系统崩溃。文章还探讨了在 CPU 设计中使用形式化方法以及共享计算资源可能造成的破坏。文章还提到了现代 CPU 的复杂性以及人工智能对 CPU 设计的潜在影响。

## [车头更高、更垂直的车辆会增加行人的危险：研究](https://www.iihs.org/news/detail/vehicles-with-higher-more-vertical-front-ends-pose-greater-risk-to-pedestrians)

- 根据美国公路安全保险协会（IIHS）的研究发现，车头较高、较垂直的车辆对行人构成的风险较高。
- 与发动机罩高度在 30 英寸或以下且外形倾斜的车辆相比，发动机罩高度超过 40 英寸的车辆在行人碰撞事故中造成死亡的几率要高出约 45%。
- 引擎盖高度在 30 至 40 英寸之间、前端较钝的车辆也会增加行人的危险。该研究强调了设计前端较低和格栅有角度的车辆以减轻对行人的危险的重要性。

### [反应](https://news.ycombinator.com/item?id=38267588)

- 讨论涉及车辆安全、法规、消费者偏好和城市规划等多个领域。
- 主要议题包括车辆设计对行人安全的影响、呼吁制定更严格的法规和事故处罚措施、法规与汽车尺寸/燃油效率之间的相关性、SUV 和卡车的流行以及不同交通方式的优缺点。
- 对话展示了各种不同的观点，突出了驾驶和运输的复杂性。

## [Backblaze 报告 2023 年第三季度硬盘故障率下降](https://www.backblaze.com/blog/backblaze-drive-stats-for-q3-2023/)

- 2023 年第三季度，Backblaze 在其数据中心监控了超过 263,992 块硬盘（HDD）和固态硬盘（SSD）。
- 与上一季度相比，所有硬盘的季度故障率从 2.2% 降至 1.47%。
- Backblaze 退役了 4,585 块老化的 4TB 硬盘，并将密切监控少数超过最高温度的硬盘，将其故障率与符合制造商规格的硬盘进行比较。所有硬盘的总体使用寿命故障率始终保持在 1.4% 左右。

### [反应](https://news.ycombinator.com/item?id=38263435)

- 文章讨论了温度对硬盘寿命的影响，并建议将温度稳定的硬盘与温度波动的硬盘进行比较。
- 强调了数据处理和加密安全的重要性，并指出有必要对硬盘驱动器进行物理销毁。
- 用户的评论涉及硬盘可靠性、备份软件和用户体验等主题，其中既有正面的体验，也有对固件错误和某些备份解决方案可信度的担忧。

## [减少对 crates.io 的依赖：生锈打包的另一种方法](https://thomask.sdf.org/blog/2023/11/14/rust-without-crates-io.html)

- 作者对仅仅依赖 crates.io 作为 Rust 代码的存储库表示担忧，并提出了一种类似于 Debian 等 Linux 发行版的替代发行方法。
- 作者称赞 Debian 的 Rust 打包团队收集了大量 Rust 库，从而降低了仅依赖 crates.io 所带来的依赖风险。
- 这种替代方法增强了作者今后使用 Rust 的信心。

### [反应](https://news.ycombinator.com/item?id=38261539)

- 文章探讨了在 C 和 C++ 等语言中使用第三方库和管理依赖关系所面临的挑战，包括构建设置困难、维护问题和版本兼容性问题。
- Rust 的打包系统 crates.io 被认为是应对这些挑战的解决方案，它能更好地控制依赖关系并确保兼容性。
- 强调了改进安全措施的必要性，如防范供应链攻击，以及安全管理依赖关系、利用扫描工具和精心策划的资料库的重要性。

## [开发编程语言：挑战、技巧和见解](https://yorickpeterse.com/articles/a-decade-of-developing-a-programming-language/)

- 作者讨论了他们开发 Inko 编程语言的经验和挑战，包括改用静态类型的决定以及自托管和代码生成方面的困难。
- 在性能、平台支持、语法和用户获取等方面面临挑战的情况下，建立一个可靠的语言和生态系统至关重要。
- 建议在原型设计时，在特定资源和社区的指导下，重点关注语义并使用现有语法。
- 发展用户群和图书馆需要时间和投资，最初应优先考虑功能而不是性能。
- 开发一种广泛使用的成功语言至少需要 10-15 年的时间。

### [反应](https://news.ycombinator.com/item?id=38261982)

- 在开发编程语言时，了解目标用户和问题至关重要。
- 实施类型检查可能具有挑战性，建议从较简单的语言开始。
- 渐进式类型化和静态类型化之间存在争论，有些人主张渐进式类型化具有捕捉错误和记录文档的优势，而另一些人则更喜欢静态类型化。
- 在静态类型和动态类型之间做出选择，应基于个人偏好和工作效率。
- 此外，还讨论了将动态类型整合到静态类型语言中的问题。
- 讨论强调了语言设计和类型检查的各种方法、它们带来的挑战以及不同类型系统的优缺点。
