## [Just Simply - Stop saying how simple things are in our docs](https://justsimply.dev/)

- 在技术文档中使用 "简单 "这个词，对于那些正在挣扎的读者来说，可能是一种居高临下的态度和不屑。
- 当有人阅读技术文档时，这通常意味着他们正在努力学习一个新的概念，所以通过暗示它应该很简单而让他们感觉更糟是没有帮助的。
- 在文档中分享对代码的兴奋和热情是很重要的，但可以不使用可能被解释为居高临下的语言来这样做。

#### [行业反应](http://news.ycombinator.com/item?id=35759449)

- 清晰简洁的文档很重要，但友好的语气也可以是有效的
- 使用 "简单 "和 "容易 "这样的夸张术语会抑制清晰度，并产生误导。

## [工程师们开发出能去除 "永远的化学品 "的水过滤装置](https://news.ubc.ca/2023/03/22/new-ubc-water-treatment-zaps-forever-chemicals-for-good/)

- 不列颠哥伦比亚大学的工程师们已经开发出一种有效的水处理方法，可以去除饮用水中的 PFAS，也被称为 "永远的化学品"，使用一种独特的吸附材料，可以捕捉和破坏存在于供水中的所有 PFAS 颗粒。
- 由 Madjid Mohseni 博士和他的团队创造的技术可以捕获高达 99%的 PFAS 颗粒，并且可以再生和潜在地重复使用，消除了有毒废物的环境挑战。
- 这种新的水过滤技术对于那些缺乏资源来实施昂贵的解决方案以消除 PFAS 的小社区的人们来说特别有利，而且在分散的和家庭的水处理中会特别有用。

#### [行业反应](http://news.ycombinator.com/item?id=35764476)

- 永远的化学品具有很强的抗分解能力，可以在环境中积累，并与人类和动物的健康问题有关
- 新系统使用等离子体技术实时处理大流量的水，可用于处理饮用水、废水和农业径流。

## [纯 Sh 圣经](https://github.com/dylanaraps/pure-sh-bible)

- "Pure sh Bible "是一本关于外部进程的 POSIX sh 替代品的集合，旨在从脚本中去除不需要的依赖性，并使其更快。
- 该书涵盖了关于字符串、文件、文件路径、循环、变量、条件表达式、算术运算符的主题，并提供了常见的 Unix 命令的替代品的例子。
- 本书提倡在文本格式和颜色方面使用原始转义序列而不是 tput。

#### [行业反应](http://news.ycombinator.com/item?id=35765707)

- 一些用户建议使用 Python 或 Perl 来编写更复杂的脚本，而其他用户则认为 shell 脚本是一种可行的选择。
- 提到了避免潜在的危险做法的重要性，如使用 eval 或依赖 shell 特定的转义序列。

## [开发过程中我最喜欢的 C 编译器标志](https://nullprogram.com/blog/2023/04/29/)

- 本文介绍了 GCC、Clang 和 MSVC 中对开发者来说重要的 C 编译器标志，重点是警告、静态和动态运行时检查，以及调试信息。
- 启用-Wall -Wextra 和-Wdouble-promotion 等标志可以帮助静态地检测错误。
- 使用像 Address Sanitizer 和 Undefined Behavior Sanitizer 这样的净化器可以帮助检测动态运行时错误和数据竞赛，而用-g3 正确配置调试信息对于有效调试至关重要。

#### [行业反应](http://news.ycombinator.com/item?id=35758898)

- 一些标志旨在提高性能，重新输出符号，并解开 C 编译器的束缚
- 开发人员建议在开发过程中尽早和经常使用净化器，以发现 C 和 C++程序中的内存错误，具体的标志和设置被强调了
