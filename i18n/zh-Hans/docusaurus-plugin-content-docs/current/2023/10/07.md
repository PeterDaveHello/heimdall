## [如果您拒绝创建账户，Postman 更新会删除您的所有内容](https://news.ycombinator.com/item?id=37792690)

- 一名用户报告说，在没有账户的情况下离线使用 Postman，结果在提示时拒绝创建账户，导致其所有收藏和设置被删除。
- 尽管用户随后创建了一个账户，但被抹掉的数据并没有恢复，尽管用户设法从特定位置的备份中手动恢复了数据。
- 这一事件导致用户对 Postman 失去信任，现在他们正在寻找其他工具来过渡。

### [反应](https://news.ycombinator.com/item?id=37792690)

- API 客户端工具 Postman 最近的更新引发了用户的不满，因为它现在要求创建一个账户，从而导致用户担心数据丢失。
- 用户正在探索 Postman 的替代方案，其中包括 Insomnia、VSCode 扩展、HTTPie、Bruno、Kreya、Paw/Rapid API、Hopscotch 和 IntelliJ HTTP 请求等工具。
- 这些用户中出现了分歧，一些人倾向于使用 curl 或命令行替代工具，而另一些人则主张使用 Postman 等图形用户界面工具，尽管他们对价格和兼容性表示担忧。

## [23andMe 称用户数据在凭证填充攻击中被盗](https://www.bleepingcomputer.com/news/security/genetics-firm-23andme-says-user-data-stolen-in-credential-stuffing-attack/)

- 基因公司 23andMe 报告了一起通过凭证填充攻击造成的数据泄露事件，导致用户数据被盗。
- 被泄露的数据包含用户的完整姓名、用户名、个人资料图像、基因祖先结果和地理位置。
- 为防止此类攻击，该公司建议用户使用双因素身份验证，并为自己的在线账户设置强大、独特的密码。

### [反应](https://news.ycombinator.com/item?id=37794379)

- 讨论的中心主题是与 23andMe 等基因检测公司相关的安全和隐私问题，以及数据泄露、密码安全和数据保护责任等话题。
- 一些与会者主张采取更严格的安全措施和加强企业责任，而另一些与会者则强调用户对数据的所有权，并质疑某些安全策略的有效性。
- 对话强调了处理个人和基因数据所面临的复杂挑战和可能产生的影响。

## [每核线程](https://without.boats/blog/thread-per-core/)

- Rust 社区正在讨论异步编程中多线程执行器的默认用法，一些人主张采用 "每核线程 "架构。
- 文章深入探讨了 "偷工减料 "和 "无共享 "架构的概念，探讨了它们对系统性能的影响。文章引用了 Pekka Enberg 的论文，说明了无共享架构的优势。
- 与传统方法相比，作者对实施恩伯格架构的可行性和简易性提出了质疑，并得出结论认为，对于具有共享状态的系统来说，偷工减料可能是有益的。

### [反应](https://news.ycombinator.com/item?id=37790745)

- 课文围绕编程语言、并发模型、异步编程以及调试和故障排除的难点展开讨论。
- 重点介绍的具体主题包括每核线程架构、工作窃取执行器、堆栈式例行程序、数据处理、高速网络接口卡和 Rust 编程语言。
- 强调资源管理效率、性能权衡考虑、强大的调试工具的重要性，以及软件开发领域需要更严格的技术批判。

### [反应](https://news.ycombinator.com/item?id=37792507)

- Rolepad 是一款求职申请跟踪器，旨在提高招聘过程的透明度和效率，允许用户跟踪求职申请、积累公司信息，并通过图表直观显示进展情况。
- 用户还可以就申请、面试和录用程序的体验提供反馈。Rolepad 采用 React 和 C#，托管在 AWS 上，使用 Google Firebase 进行身份验证。

- 它建议在 Java 中使用虚拟线程，将其作为大流量应用的一种小众但潜在高效的解决方案，并回顾了在软件开发中使用 Clojure、ClojureScript、Kotlin 和 Java 的利弊。

## [虚幻引擎将不再免费提供](https://www.creativebloq.com/news/epic-games-unreal-engine-charge)

- Epic Games 宣布其图形引擎 "虚幻引擎"（Unreal Engine）将转为基于订阅的定价模式。
- 以前的版税模式只适用于收入超过 100 万美元的项目，与此不同的是，这种新模式可能会对独立制片人和非专业用户产生潜在影响。
- 虽然价格细节尚未公布，但 Epic Games 首席执行官 Tim Sweeney 保证新的价格不会过高或异常便宜；学生和教育工作者对虚幻引擎的访问将继续免费。

### [反应](https://news.ycombinator.com/item?id=37792030)

- 虚幻引擎修改了其许可模式，不再对所有用户免费。它现在纳入了游戏开发以外的大型商业用途的收入门槛。
- 文章认为，尽管在公平性和对行业的影响方面存在广泛争议，但变化并不像看起来那么剧烈，而且可以更早地预见到。
- 作者结合 Epic Games 的财务状况，讨论了订阅模式的利弊以及可持续商业模式的必要性。
