## [OpenSign - DocuSign 的开源替代方案](https://github.com/OpenSignLabs/OpenSign)

- OpenSign 是商业文档电子签名平台（如 DocuSign、PandaDoc 和 SignNow）的开源替代平台。
- OpenSign 提供的功能包括安全文件签署、用户友好界面、多平台支持、协作设施、审计跟踪、完成证书和 API 集成。
- OpenSign 由 OpenSignLabs 开发，采用 AGPL 许可，欢迎社区贡献，并获得了用户和贡献者的支持和反馈。

### [反应](https://news.ycombinator.com/item?id=38052344)

- OpenSign 实验室推出了 OpenSign，这是 DocuSign 的开源替代方案，引发了关于数字签名合法性和 OpenSign 法律研究深度的讨论。
- 演讲还以印度的《2000 年信息技术法》和欧洲的 eIDAS 为例，重点阐述了各地区数字签名统一要求的必要性。
- 此外，会议还重点讨论了 OpenSign 的安全性和可靠性、开源软件的利弊，以及信任和长期可行性在管理已签署文件的商业产品中的作用。

## [联邦通信委员会希望加强业余无线电](https://www.radioworld.com/news-and-business/business-and-law/fcc-wants-to-bolster-amateur-radio)

- 美国联邦通信委员会 (FCC) 打算用带宽限制取代波特率（数据传输速度）限制，从而实现业余无线电频段的现代化，目的是促进创新和加强紧急情况下的通信。
- 业余无线电爱好者机构美国无线电中继联盟 (ARRL) 支持这一倡议。
- FCC 还在考虑取消对其他频段的波特率限制，并愿意听取有关设定适当带宽限制的反馈意见。预计这些更新将极大地惠及业余无线电运营商，并促进数据通信。

### [反应](https://news.ycombinator.com/item?id=38052577)

- 在美国无线电中继联盟 (ARRL) 提出申请后，联邦通信委员会 (FCC) 正在修订业余无线电管理条例，这可能会导致更多的实验和新的调制技术。
- FCC 审议的重要议题包括莫尔斯电码、波特率限制、加密、业余无线电加密的潜在益处，以及扩大运营商的频率和数据传输速率。
- 虽然这些改进给社区带来了兴奋，但也有人对可能的干扰以及对非商业和实验性无线电活动的限制表示担忧。

## [电流对氨气的效率几乎达到 100% 的氮电还原（2022 年）](https://www.nature.com/articles/s41586-022-05108-y)

- 最近发表在《自然》（Nature）杂志上的一项研究重点关注将氮气电还原成氨气，该技术适用于可再生能源、运输和化肥生产等领域。
- 莫纳什大学的研究人员发现，高浓度亚胺基锂盐电解质能够实现高产氨率，电流对氨的效率接近 100%。
- 这些创新可能会产生一种更高效、更环保的合成氨生产方法，Jupiter Ionics 公司可能会将其应用于商业领域。

### [反应](https://news.ycombinator.com/item?id=38053586)

- 一篇新的研究论文强调了氮电还原领域的一项突破，即通过激光诱导氧化锂的多光子解离实现高效氨生产。
- 尽管存在高能耗和安全问题，但这种方法为在当地生产合成氨和无碳燃料替代品铺平了道路。
- 该研究还探讨了如何将电能高效转化为氨，用于能源储存、肥料和燃料生产，同时解决了实用性和安全性方面的问题，讨论了氨的可扩展性、毒性以及用于炸药生产的可能性。

## [英国备受争议的《网络安全法》正式生效](https://www.wired.com/story/the-uks-controversial-online-safety-act-is-now-law/)

- 英国政府的《在线安全法》现已成为法律，旨在保护互联网上的个人，尤其是儿童。

- 英国的《在线安全法》引发了围绕隐私、数据保护、信息自由和年龄验证方法有效性的辩论。
- 批评者称，该立法可能导致基于虚假信息的通信指控和不均衡的法律辩护；一些人甚至考虑阻止英国访问者访问其网站，以示抗议。
- 该法引发了关于处理网络虚假信息及其对表达自由的影响的更广泛讨论；还出现了一场涉及城市规划、公共交通和专制主义的外围辩论。

## [ImageMagick 的 Windows 安装程序将不再签名](https://github.com/ImageMagick/ImageMagick/discussions/6826)

- 由于代码签名证书已过期，ImageMagick 的 Windows 安装程序将不再具有签名，因此无法在 GitHub 操作中导出和使用该证书。
- 该团队目前正在考虑采用云解决方案，Digicert 是目前唯一的选择，一年的收费为 629 美元。
- 虽然他们邀请赞助商来获取代码签名证书，但一些用户建议使用 SignPath 和 Azure KeyVault 等替代方案。

### [反应](https://news.ycombinator.com/item?id=38055816)

- 对话围绕开源项目在为其软件获取和更新代码签名证书时所面临的困难展开，并对其高成本和低效率提出了批评。
- 关于财务可持续性、改进供资和安全系统的要求以及 Azure Key Vault（一种用于安全存储和访问加密密钥的云服务）等替代解决方案的讨论也很普遍。
- 重点是身份验证的重要性、代码的可信度以及对专有软件和自签名证书的担忧。此外，还介绍了 ImageMagick 这一用于图像编辑的开源软件套件的价值和弱点。
