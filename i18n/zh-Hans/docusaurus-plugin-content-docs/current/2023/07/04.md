## [Goodreads 是书评的未来，然后亚马逊收购了它](https://www.washingtonpost.com/technology/2023/07/01/amazon-goodreads-elizabeth-gilbert/)

- Goodreads 是亚马逊旗下的图书评论网站，因其过时的基础设施和缺乏保护功能而面临批评，导致了有针对性的骚扰和 "评论轰炸 "事件。
- 据报道，亚马逊没有投资更新或维护 Goodreads，主要将其用于用户生成的数据。
- 2013 年亚马逊收购 Goodreads，被认为是该网站创新和为读者和作者创造新服务的一个有希望的机会，但自那时以来几乎没有变化，导致用户和作者都感到失望。

### [行业反应](http://news.ycombinator.com/item?id=36575003)

- 亚马逊对 Goodreads 的收购被认为是反竞争的，是防止竞争的防御性举措。
- 一些精通技术的人的看法是，初创企业经常被收购以防止它们成为竞争对手。
- 亚马逊对 Goodreads 的收购让一些用户感到失望，他们期待该平台有更多的增长和创新。

## [加入 13 种方式](https://justinjaffray.com/joins-13-ways/?a=b)

- 连接操作可以通过各种方式实现，例如查找、嵌套循环、图路径或类型检查，这取决于数据的背景和要求。

### [行业反应](http://news.ycombinator.com/item?id=36575784)

- 文章讨论了在更高的非空间维度上的连接的概念，例如增加时间。
- 文章介绍了最坏情况下的最优连接的概念，它允许连接三个或更多的表而不产生临时结果。

## [利用 Nginx 漏洞泄露 Bitwarden 的 Vault](https://labs.hakaioffsec.com/nginx-alias-traversal/)

- Nginx，一个流行的网络服务器，有潜在的漏洞，与 location 和 alias 指令的错误配置有关。
- 这些漏洞可能导致数据暴露，允许未经授权访问敏感文件和目录。
- 像 NavGix 这样的自动化工具可以帮助识别和测试这些漏洞，使开发人员了解和实施安全配置变得非常重要。

### [行业反应](http://news.ycombinator.com/item?id=36580417)

- Bitwarden 的金库可以通过其配置语言中的一个 Nginx 漏洞被泄露。
- 该漏洞允许目录遍历攻击，暴露了敏感文件。
- Bitwarden 已经承认并修复了这个问题，甚至为其发现发布了赏金。

- 该书强调了正确构造和组织数据以实现高效处理的意义，包括规范化数据、不同类型的处理、组件和管理器的使用以及优化性能等主题。
- 它强调了了解数据库技术的必要性，数据处理和操作的各种技术，以及调试和维护用面向数据方法构建的软件的重要性。

### [行业反应](http://news.ycombinator.com/item?id=36571110)

- 面向数据的设计(DOD)是一种范式，专注于有效地组织和处理数据以获得高性能的代码。

## [SVG 路径的互动指南](https://www.nan.fyi/svg-paths)

- SVG 代码中的 d 属性是一系列的命令，它告诉浏览器如何绘制路径。
- d 属性中的命令由一个单字母代码和一个或多个数字组成。
- 大写的命令是绝对的，而小写的命令是相对于前一个命令的端点而言的。

### [行业反应](http://news.ycombinator.com/item?id=36574645)

- 文章中提到的 SVG 路径编辑器是手工构建或编辑 SVG 的一个有用工具。
- Inkscape 是另一个可以用来编辑 SVG 的工具，尽管它可能需要手工编辑以获得最佳优化。
- SVG 路径编辑器 "剖析 "了 SVG 路径的 d 参数，使其更容易理解和绘制参数化。
- SVG 是一个强大的网络开发工具，但许多开发者对它的了解有限。
- SVG 中缺少一些功能，比如有角度的渐变，这些功能在 Photoshop 等程序中存在。
- SVG 2 是对 SVG 标准的拟议更新，多年来一直停留在草案阶段，限制了对 SVG 支持的进步。
- SVG 2 旨在为 SVG 带来 CSS 支持，并解决当前标准的局限性，如 Z-index 和文本渲染。
- 在更新 SVG 标准方面缺乏进展，是由于这一过程的复杂性、来自实施者(主要是浏览器)的阻力，以及需要考虑像 Adobe Creative 这样的专业创作工具。
- 像文章中提到的那种交互式的、具有视觉吸引力的指南，是学习 SVG 和建立对贝塞尔曲线等概念的直觉的宝贵资源。
- 其他的 SVG 路径编辑器和教程，比如评论中提到的那些，都是值得探索的，可以从不同的角度和方法来处理 SVG。
