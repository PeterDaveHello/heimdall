# 2023-08-16

## [我们将在 Twitter 规模上构建 Mastodon 的成本降低了 100 倍](https://blog.redplanetlabs.com/2023/08/15/how-we-reduced-the-cost-of-building-twitter-at-twitter-scale-by-100x/)

- Red Planet Labs 使用 Rama 平台构建 Twitter 规模的 Mastodon 实例，成本降低了 100 倍。
- 与 Twitter 的 100 万行代码相比，该实例只用了 1 万行代码。
- Rama 具有可扩展性、性能和容错性，可简化后端开发。该团队计划发布 Rama 供公众下载，并开源他们的 Mastodon 实现。Mastodon 中的 PState 用于跟踪数据和提高效率。讨论了优化技术，并提到了 Rama 文档和 API 的发布。

### [反应](https://news.ycombinator.com/item?id=37137110)

- 文章重点介绍 Rama，这是一个新的后端框架，旨在简化和降低构建可扩展应用程序的成本。
- 评论者讨论了与 Twitter 的比较，并质疑 Rama 的可扩展性。
- 讨论还深入探讨了去中心化平台所面临的挑战、在游戏开发中使用实体组件系统的利弊以及安全和节制的重要性。

## [火狐浏览器在 SunSpider 中的表现终于超过了谷歌浏览器](https://www.phoronix.com/news/Firefox-Faster-SunSpider)

- 火狐浏览器在 SunSpider JavaScript 基准测试中超过了谷歌 Chrome 浏览器，这是 Mozilla 开发人员取得的一项重大成就。
- 尽管取得了这一成功，Chrome 浏览器在要求更高的 JetStream 2.0 基准测试中的表现仍然优于 Firefox 浏览器。
- Firefox 专注于提高上传速度，并引入其他增强功能以提高其整体性能。

### [反应](https://news.ycombinator.com/item?id=37134092)

- 讨论主题侧重于与 Firefox、Chrome 和 Safari 等网络浏览器相关的各种话题。
- 用户就性能基准、定制选项、内存使用情况和用户界面展开辩论。
- 人们对安全性、密码管理器和 Chrome 浏览器的主导地位表示担忧，并围绕火狐浏览器的衰落和谷歌对网络控制的潜在影响展开讨论。

## [开放信托基金宣言](https://opentf.org/)

- OpenTF 宣言提出了对开源平台 Terraform 许可证变更的担忧，并主张要么恢复变更，要么建立一个基金会来维护开源版本。
- 各种公司和个人积极参与开源社区活动，以加强项目和促进合作。
- 2023 年 8 月 15 日，一群人参与了个人开发和开源社区工作，如测试、文档编制、咨询和使用开源工具。

### [反应](https://news.ycombinator.com/item?id=37133054)

- OpenTF 宣言是由开发人员发起的一场讨论，由于担心当前所有者 Hashicorp 缺乏维护和支持，他们主张将 Terraform 分叉为一个非营利基金会。
- 一些开发者主张分叉以改进语言并优先考虑社区需求，而另一些开发者则希望 Hashicorp 能改变主意。
- 讨论还涉及秘密管理产品、声明式编程的局限性、对开源社区的影响、提供商生态系统、Hashicorp 的创收、替代许可选项、项目重命名和重塑、Terraform 中缺乏测试框架、所有权问题以及对新许可的担忧。
- 用户对开源 Terraform 表示支持，并探索替代解决方案。
- 此外，还讨论了有关转换许可证、对收入增长的影响、限制、兼容性以及云供应商对开源工具的影响等问题。

## [LLaMa.cpp 是如何实现的？](https://finbarr.ca/how-is-llama-cpp-possible/)

- 作者研究了在不同硬件（包括 Raspberry Pi 等内存有限的设备）上以原始 C++ 运行 LLaMa 推理代码的可行性。
- 讨论了深度学习中内存带宽的重要性，以及使用低精度模型来减少内存使用量的好处。
- 本书提供了在各种硬件上运行 LLaMa 的计算和示例，重点强调了从变压器采样时内存带宽的重要性。此外，作者还强调了对较小模型进行蒸馏和扩展训练以提高效率的重要性。不过，他们也承认计算中可能存在错误，并鼓励读者提出反馈意见。

### [反应](https://news.ycombinator.com/item?id=37140013)

- 讨论围绕量化模型、Llama 语言模型的能力、基于 CPU 的推理、内存带宽的限制、人工智能与人类表现相匹配的潜力、用于 AGI 的语言模型的局限性和潜力，以及在获取人造文本方面的挑战。
- 学员将学习具体的平台、方法和硬件配置。
- 关于人工智能模型的可行性和未来发展，人们表达了不同的观点。

## [Opendream：基于层的稳定扩散用户界面](https://github.com/varunshenoy/opendream)

- Opendream 是一款网络用户界面工具，通过扩散模型为图像生成和处理引入了分层、非破坏性编辑和可移植性等功能。
- 用户可以在现有图像的基础上，在同一图像上进行多项实验，并保存和共享他们的工作流程。
- Opendream 支持扩展功能，并提供用户友好的方法来编写和使用新的扩散功能。

### [反应](https://news.ycombinator.com/item?id=37136898)

- 讨论围绕人工智能技术在图像编辑、艺术和动画中的应用展开，突出了人工智能在这些领域的潜在优势和挑战。
- 涉及的主题包括基于图层的用户界面的开发、人工智能生成的有争议内容的潜在危害、对侵犯版权的担忧以及人工智能训练和使用的道德和法律影响。
- 辩论还探讨了服务提供商的责任、监管的必要性以及对艺术家生计的影响，提出了关于人工智能在这些行业中的未来的重要问题。

## [互联网服务提供商抱怨列出每项收费太难，敦促 FCC 取消新规定](https://arstechnica.com/tech-policy/2023/08/isps-complain-that-listing-every-fee-is-too-hard-urge-fcc-to-scrap-new-rule/)

- 美国宽带行业反对联邦通信委员会披露所有月费的要求，认为这会增加复杂性，加重消费者和供应商的负担。

- 美国联邦通信委员会的宽带标签规则，包括定价和额外收费，目前正在接受美国管理和预算办公室的审查。

- 讨论涉及有关互联网服务提供商的多个话题，如收费透明度、作为公用事业的监管、小费文化、自来水供应、定价透明度以及互联网服务提供商的收费。
- 它还包括对游说、政府腐败、欺骗性定价、新进入者的竞争以及互联网服务提供商的监控和数据收集行为的关注。
- 总的主题是呼吁提高互联网服务提供商行业的透明度、消费者保护和监管。

## [为什么要举办自己的法律硕士课程？](http://marble.onl/posts/why_host_your_own_llm.html)

- 作者提出了支持自托管语言模型（LLM）而不是依赖 OpenAI 的 GPT-4 等公开可用模型的论点。
- 自托管可对模型结构和权重、定制和微调能力以及与人工智能模型的长期关系提供更多控制。
- 自托管可以进行更多的实验、优化，并与不断发展的技术保持同步，而应用程序接口可能无法提供相同程度的访问和理解。

### [反应](https://news.ycombinator.com/item?id=37133504)

- 文章探讨了自托管语言模型 (LLM) 而不是依赖 OpenAI 的 API 的原因和挑战。
- 讨论了托管 LLM 的其他选择，如 Beam、个人硬件和 George Hotz 的 Tiny Box。
- 探讨了自托管 LLM 的可行性和好处，包括成本效益和隐私问题。
