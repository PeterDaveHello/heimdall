## [开源 Chrome 浏览器扩展开发者的诱惑（2021 年）](https://github.com/extesy/hoverzoom/discussions/670)

- Hover Zoom+ Chrome 浏览器扩展的所有者已收到多个商业提案，希望对该扩展进行货币化或购买。
- 出于对信任的担忧和延期开发商的压力，业主拒绝了这些提议。
- 这些建议包括通过合作关系增加收入、用户数据货币化以及整合商务 SDK。
- 业主优先考虑财务安全，因此选择不接受这些报价。

### [反应](https://news.ycombinator.com/item?id=37066680)

- 与会者讨论了开源 Chrome 浏览器扩展货币化的挑战和道德问题，包括用户隐私、数据收集和定向广告等问题。
- 人们对扩展功能监听对话、跟踪离线广告以及通过广告进行心理操纵的潜力存在争议。
- 此外，还讨论了浏览器扩展程序的审核过程，以及货币化、透明度和用户数据控制的可行性和伦理问题。

## [想让员工回到办公室？那就给每个人一间办公室](https://www.washingtonpost.com/opinions/2023/08/09/remote-work-offices-floorplan-privacy/)

- 开放式办公室可能既嘈杂又缺乏私密性，导致工作效率和工作满意度下降。
- 为工人提供单独的办公室可以鼓励他们返回办公室，提高工作效率。
- 由于远程工作的趋势，办公空间的成本不断下降，这可能会抵消提供办公室的成本。

### [反应](https://news.ycombinator.com/item?id=37062650)

- 对于在家工作（WFH）或返回办公室工作，员工有不同的偏好。
- 有些人喜欢全职工作的灵活性和成本节约，有些人则看重社交互动和办公室环境的变化。
- 讨论还涉及阶级和等级问题，包括高管和其他员工在灵活性方面的差异。

## [由 Radically Open Security 完成的基础设施审计](https://mullvad.net/en/blog/2023/8/9/infrastructure-audit-completed-by-radically-open-security/)

- Mullvad VPN 已经接受了 Radically Open Security 的第三次安全审计，重点是其 VPN 基础设施。
- 审计没有发现信息泄漏或客户数据记录，但发现了各种问题，如测试服务器上生产用户流量的可见性。
- Mullvad VPN 计划通过实施审计未授权登录和取消 SSH 支持等措施来解决这些问题。

### [反应](https://news.ycombinator.com/item?id=37060654)

- 文章和评论的重点是 Mullvad 和 ProtonVPN 等 VPN 提供商及其隐私保护措施，其中特别提到 Mullvad 取消了端口转发功能。
- 讨论的主题包括互联网隐私、政府监控、在线审查、美国制裁的影响以及 VPN 的局限性和脆弱性。
- 提出的关切包括数据收集、VPN 提供商的可信度以及网络隐私的复杂性。

## [Arpchat - 仅使用 ARP 给同一网络中的朋友发短信](https://github.com/kognise/arpchat)

- arpchat "项目是一个开源聊天应用程序，它使用地址解析协议（ARP）在网络设备之间传输信息。
- 它支持长信息的传输，包括存在发现和心跳系统以及压缩功能。
- 该项目由 Rust 编写，在 GitHub 上获得了 1.8k stars 的人气。安装说明适用于 Windows 和 Linux，鼓励用户报告任何问题。

### [反应](https://news.ycombinator.com/item?id=37063151)

- ArpChat 是一款消息应用程序，它使用地址解析协议（ARP）在同一网络的用户之间进行通信。
- 导致 ArpChat 延迟的错误已得到修复，但更新版本尚未发布。
- 由于权限缺失，Linux 用户在使用 ArpChat 时可能会遇到一些问题，但我们可以提供解决方案。
- 文章解释了 ARP 在网络中的用途及其在连接 MAC 地址和 IP 地址方面的作用。
- ARP 常用于以太网和 WiFi 网络，本文将探讨它在这些网络中的用法。
- 讨论了无线路由器、交换机和调制解调器使用的辐条-集线器模式，解释了即使在本地网络中，数据通常也要通过路由器。
- 研究了在网络上创建广播信息的潜在漏洞和对 ARP 的操纵。
- 文章还涉及创建网络地图的批处理脚本、无互联网通信方法（点对点聊天、无线网络）以及使用类似网络协议的用户体验等主题。
- 还讨论了将 DNS 用于聊天系统的可能性。

## [SQLedge：在边缘将 Postgres 复制到 SQLite](https://github.com/zknill/sqledge)

- SQLedge 是一款开源工具，它利用 Postgres 逻辑复制功能，将 Postgres 数据库中的更改传输到边缘的本地 SQLite 数据库中。
- 它允许从 SQLite 数据库读取数据并写入上游 Postgres 服务器，确保数据的一致性。
- 该工具包括一个 SQL 生成器、SQL 解析器和 Postgres 线代理，所有配置均从环境变量中获取。

### [反应](https://news.ycombinator.com/item?id=37063238)

- SQLite 数据库应用广泛，有多种工具和项目可用于增强其功能。
- SQLedge 和 SQLitePostgres 等工具可提供数据复制、版本控制、多主复制和查询解析等功能。
- 这些工具为本地数据缓存和查询提供了轻量级选项，并具有与 SQLite 数据库处理相关的各种功能。不过，由于 SQLite 和其他数据库（如 PostgreSQL）的 SQL 方言不同，因此存在一些限制。

## [使 AMD GPU 在 LLM 推理方面具有竞争力](https://blog.mlc.ai/2023/08/09/Making-AMD-GPUs-competitive-for-LLM-inference)

- 文章讨论了 AMD GPU（特别是 Radeon RX 7900 XTX）在 LLM（大型语言模型）推理方面的性能。
- 研究结果表明，7900 XTX 在 Llama2-7B/13B 机型上的运行速度是英伟达 GeForce RTX 4090 的 80%，是 RTX 3090 Ti 的 94%。
- 机器学习编译（MLC）和 ROCm 堆栈的使用优化了 AMD GPU 的性能，并可能支持 Vulkan 和 WebGPU。这项研究强调，需要进一步开发和优化针对 AMD GPU 和其他平台的 MLC 解决方案。

### [反应](https://news.ycombinator.com/item?id=37066522)

- 经过优化的软件堆栈可以提高 AMD GPU 在大型语言模型推理方面的性能。
- 讨论了 ML 编译技术和 TVM Unity 软件栈在英伟达和 AMD GPU 上实现高性能的方法。
- 文章探讨了 AMD 对 ROCm 和 Vulkan 的支持，但对支持的持久性表示担忧。
- 讨论了 AMD ROCm 平台的各种使用案例，包括人工智能、密码破解、GPU 挖矿和视觉特效渲染。
- 对 AMD 和 Nvidia GPU 进行了比较，重点是机器学习算法中内存容量和带宽的限制。
- 微软在其项目中提出了瓶颈问题的解决方案，而 Geohot 的贡献则是直接改善 ROCm。
- 文章建议使用多张 RX 6600 显卡而不是一张 RX 7900 显卡，但对于在没有 AMD NVLink 的情况下如何连接这些显卡仍有疑问。
- 这些事态发展引起了人们对投资 AMD 的思考。

## [西雅图成为美国首个保护临时工免遭突然 "停工 "的城市](https://kuow.org/stories/seattle-becomes-first-in-u-s-to-protect-gig-workers-from-sudden-deactivation)

- 西雅图市议会批准了一项新法律，以保障临时工在 Instacart 和 DoorDash 等平台上不会被突然停用，这在美国尚属首次。
- 该条例实施了更严格的停用准则，并规定公司在终止服务前必须提前 14 天发出通知。
- 这一进展是西雅图旨在保护未加入工会的 "零工 "的更大举措的一部分，包括最近提高最低工资、给予应用程序工人病假时间和福利的法律。预计西雅图市长布鲁斯-哈雷尔（Bruce Harrell）将签署该法律。

### [反应](https://news.ycombinator.com/item?id=37070398)

- 关于 "零工 "的讨论涉及工人保护、分类、福利、挑战以及与出租车等传统服务的比较等多个话题。
- 辩论的重点包括 "零工 "的灵活性、对 "零工经济 "平台剥削的担忧、监管的必要性以及对企业和工人的影响。
- 这些讨论凸显了围绕 "零工 "的复杂性和不同观点，以及正在进行的关于 "零工经济 "中工人权利的辩论。
