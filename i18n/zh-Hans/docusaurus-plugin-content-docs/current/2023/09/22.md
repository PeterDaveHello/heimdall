- 本论述探讨了铅中毒的各种影响，包括铅中毒与学龄前儿童犯罪率的相关性，以及铅在汽车污染和各种产品中的存在。
- 重点包括管道配件中铅的危险和含铅燃气的使用，同时还强调了石棉、水壶中的矿物质沉积以及电热水器的挑战等问题。
- 讨论强调了监管和认识对于解决这些与铅中毒相关的环境和健康问题的重要性。

## [加快 JavaScript 生态系统的运行速度--流氓式的 Polyfills](https://marvinh.dev/blog/speeding-up-javascript-ecosystem-part-6/)

- 目前的问题是，npm 软件包中不必要地包含了 polyfills，导致 node_modules 文件夹的大小超出了要求。
- 包括 "eslint-plugin-react "在内的许多软件包都在导入并直接调用 polyfill 函数，这种做法效率很低，因为 JavaScript 已经广泛支持这些工作。
- 文章认为，JavaScript 生态系统需要重新评估依赖关系的使用，以提高效率并减少磁盘空间的浪费。通过消除不必要的 polyfills，项目中的依赖关系数量可以显著减少。

### [反应](https://news.ycombinator.com/item?id=37602923)

- 文章深入探讨了 JavaScript 中的自动多填充问题，以及在 swc 和 Babel 等工具的催化下，不必要的多填充是如何导致库规模增大的。
- 它讨论了与 IE6 等旧版浏览器保持向后兼容性的困难，提出了分叉库的可能性，以消除非必要的向后兼容性。
- 文章强调了 JavaScript 生态系统中存在的挑战，如高度依赖性、缺乏经验的开发人员和过时的浏览器支持。文章提出了一些解决方法，如减少依赖性、采用依赖性较低的库，以及可能利用 WebAssembly (WASM) 来解决依赖性问题。

## [用较少的训练数据和较小的模型就能超越较大的语言模型](https://blog.research.google/2023/09/distilling-step-by-step-outperforming.html)

- 谷歌的云计算人工智能团队推出了一种名为 "逐步提炼 "的技术，有助于使用较少的数据训练紧凑的特定任务模型。
- 这项技术的工作原理是从大型语言模型（LLM）中提取被称为 "理由 "的信息部分，以监督小型模型的训练。
- 研究表明，这种方法在使用更小的模型和更少的训练数据时，性能比 LLM 更好，为应对大型模型带来的部署挑战提供了更节省资源的解决方案。

### [反应](https://news.ycombinator.com/item?id=37606352)

- 辩论的焦点集中在谷歌的人工智能能力上，提请人们注意研究的成功未必能保证产品的主导地位。
- 我们强调了特定领域模型、多模态训练和精通处理器密集型大型语言模型（LLM）（如 OpenAI 的 GPT-3+）的管理器模型的重要性，以及对批处理框架的需求。
- 未来的可能性包括苹果和英特尔进入 LLM 市场，以及增加硬件的内存容量以更好地适应 LLM。

## [美国证券交易委员会打击绿色营销](https://www.semafor.com/article/09/21/2023/the-sec-cracks-down-on-greenwashing)

- 为了反对绿色营销，欧盟（EU）还禁止在 2026 年之前宣称 "气候中立 "或任何广泛的环保主张。

### [反应](https://news.ycombinator.com/item?id=37598329)

- 美国证券交易委员会（SEC）正在对碳市场中的 "洗绿 "行为实施更严格的监管，因为非营利组织涉嫌在环境可持续性方面进行虚假宣传。
- 关于碳信用额度的可信度和有效性以及进一步规范碳抵消市场的必要性的争论日益激烈。人们对毁林信用额度和骗局的担忧也在增加。
- 文章和评论讨论了潜在的解决方案，如碳税，但也承认相关的挑战，包括可靠地量化碳排放、黑市出现的风险以及核国家带来的复杂性。
