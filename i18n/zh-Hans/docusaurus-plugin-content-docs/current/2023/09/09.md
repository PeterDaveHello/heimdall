# 2023-09-09

## [包子 v1.0.0](https://bun.sh/blog/bun-v1.0)

- Bun 1.0 是一个工具包，旨在简化 JavaScript 和 TypeScript 开发，取代多种工具，并与现有 Node.js 应用程序兼容。
- 它支持 TypeScript、JSX 和内置 Web API，是一款功能强大的 JavaScript 运行时，据称每秒可处理的请求比 Node.js 更多。

- 讨论以 Bun 为中心，Bun 是 Node.js 生态系统中同时支持 CommonJS 和 ES 模块的工具，用户强调了其优点和缺点。

## [也许 Rust 并不是大规模并发用户空间软件的好工具](https://bitbashing.io/async-rust.html)

- 作者阐明了 Async Rust 这种注重现代并发性的编程语言的目的，区分了并行性和并发性，并概述了代码拆分的局限性。
- 由于存在竞赛条件和死锁的风险，基于互斥的并发被认为是有害的；因此，建议使用通道或队列进行线程间通信。然而，对于高级并发问题，线程可能并不足够，这就促使 Rust 提出了 "async/await "模型。
- 尽管有这些优势，Async Rust 也面临着一些挑战，如确保对象的编译器安全性和 Arc（原子引用计数）的普遍使用。Rust 大规模并发用户空间软件应用的适用性仍然值得怀疑。

### [反应](https://news.ycombinator.com/item?id=37435515)

- 讨论内容涉及 Rust 编程的各种主题，如内存安全、并发性、图形开发、生产率和内存管理。
- 与会者分享的经验阐明了使用 Rust 的优势和挑战，以及 Rust 在各种用例中的适用性。
- 互动还提供了与 Python 和 Go 等其他语言的比较，并讨论了继承和抽象等多个编程概念，让读者了解 Rust 的优势、劣势和不断改进的努力。

## [联邦贸易委员会法官裁定 Intuit 的 "免费 "TurboTax 广告误导消费者](https://www.theverge.com/2023/9/8/23864538/turbotax-intuit-ftc-deceptive-practices-free-tax-filing)

- 美国联邦贸易委员会（FTC）裁定，TurboTax 的母公司 Intuit 将其报税服务宣传为 "免费"，从而误导了消费者。
- 在联邦贸易委员会提起诉讼后，Intuit 被要求明确披露任何限制其免费产品的条款，并支付 1.41 亿美元。
- 鉴于在试图找到 Intuit 的实际免费报税服务 Free File 时遇到的困难，国税局正在开发一种政府支持的免费报税服务，将于 2024 年报税季试行。

### [反应](https://news.ycombinator.com/item?id=37437311)

- 讨论围绕用户对 Intuit 公司 TurboTax 产品的不满展开，因为该产品存在未披露费用、欺骗性促销和缺乏透明度等问题。
- 建议将 FreeTaxUSA 和 Cash App Taxes 等其他报税方式作为更透明、更具成本效益的解决方案。
- 游说的影响、税务系统中的腐败、误导性广告以及要求系统改革等话题突出了这些用户投诉的大背景，并暗示了对报税系统进行重大改革的必要性。

## [空气污染导致蜜蜂难以找到花朵](https://www.birmingham.ac.uk/news/2023/bees-struggle-to-find-flowers-because-of-air-pollution)

- 英国生态学与水文学中心和多所大学的一项新研究显示，空气污染会降低花朵的香味，从而阻碍蜜蜂寻找花朵的能力。
- 研究发现，蜜蜂的气味识别能力降低了多达 90%，这表明蜜蜂在对粮食安全至关重要的授粉过程中发挥着关键作用，可能会对野花的丰产和作物产量产生不利影响。
- 研究还表明，臭氧可能会影响昆虫受气味影响的其他行为，如吸引配偶，因此有必要对空气污染采取行动，以保护粮食生产和生物多样性。

### [反应](https://news.ycombinator.com/item?id=37434381)

- 据称，空气污染导致蜜蜂活动减少，难以找到花朵，造成昆虫数量下降。其他因素可能包括交通和噪音污染，以及蜜蜂资源的减少。
- 一些与会者指出，蜜蜂活动和野生动物有所增加，这可能是由于条件有利或努力营造昆虫友好型环境所致。
- 讨论指出了污染对蜜蜂吸引特定植物的可能影响、导致蜜蜂适应污染的进化压力暗示，以及通过催化转换器解决车辆污染的创新方案。

## [解密GNU Coreutils（2018）](https://maizure.org/projects/decoded-gnu-coreutils/)

- 报告介绍了 MaiZure 专为初学者设计的 GNU coreutils 8.3 版本解密项目。
- 这篇文章的主要内容是概述已完成的阶段，深入探讨实用程序的设计和功能，包括有关每个实用程序所采用的解码框架和流行概念的信息。
- 此外，它还介绍了公用设施的维护、更新以及为项目做贡献的流程。

### [反应](https://news.ycombinator.com/item?id=37439535)

- 讨论主要围绕 GNU Coreutils 中的 "yes "命令展开，突出了程序效率与简洁性之间的权衡。
- 它强调了使用 C 语言编写系统实用程序的优势，因为这种选择可以提高性能。
- 一个值得注意的讽刺现象是，有时在设计用于执行无意义任务的程序中也能发现冗长的代码。
