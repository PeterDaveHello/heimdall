# 2023-09-05

## [VSCodium - VSCode 的开源二进制文件](https://vscodium.com)

- VSCodium 是微软 Visual Studio Code（VS Code）的免费开源替代品，由社区驱动，提供 VS Code 的二进制文件，没有专有许可证和遥测/跟踪功能。
- 由于有了 VSCodium，访问 MIT 许可的 VS Code 变得更加方便，可选择直接下载或通过软件包管理器在 Windows、Mac OS 和 Linux 等多个平台上安装。
- VSCodium 还为从 Visual Studio Code 过渡而来的用户提供了 Flatpak 应用程序安装选项和详细文档。

### [反应](https://news.ycombinator.com/item?id=37381335)

- 对话讨论了不同的话题，包括软件信任、开源项目漏洞和微软的做法，以及其他软件的注意事项。
- 与会者强调了验证、供应链安全、数据隐私、数字版权管理（DRM）、许可和扩展兼容性等关键方面。
- 讨论强调了专有实践与 Visual Studio Code 等应用程序中的开源精神之间的压力。

## [用 500 行 Python 语言编写 C 语言编译器](https://vgel.me/posts/c500/)

- 文中讨论了一个用 500 行 Python 编写的针对 WebAssembly 的 C 编译器，尽管有一些限制，但它能成功编译和运行简单的 C 程序。
- 它让我们深入了解该编译器使用的不同模块和类，控制流程的机制，以及如何处理全局声明。
- 编译器采用 WASM 指令和结构化控制流来有效处理循环和条件。

### [反应](https://news.ycombinator.com/item?id=37383913)

- 文章深入探讨了使用 Python 创建 C 语言编译器的复杂性，强调了 C 语言的显著特点、标量数据类型的理解以及使用 infix 符号管理语言。
- 它以现代架构为目标，消除了对非主流平台的需求，同时还讨论了执行编程相关任务的消极和积极方面。
- 文章还探讨了大学教育、创建独特编程语言和特定领域语言、提供制作动态语言的资源等方面的优点，并讨论了编译器的实际定义。

## [索尼向电视博物馆发送版权通知，内容涉及 40 至 60 年前的节目](https://torrentfreak.com/tv-museum-will-die-in-48-hours-unless-sony-retracts-youtube-copyright-strikes-230904/)

- 芝加哥经典电视博物馆是一家保存历史电视内容的非营利性机构，由于索尼公司的版权打击，该博物馆可能在 48 小时内关闭。
- 索尼公司就 40 至 60 年前的电视节目提出的这些版权索赔，危及了该博物馆自 2007 年以来在 YouTube 上的地位。
- 博物馆创始人里克-克莱因正在紧急试图解决版权问题，以防止项目终止。

### [反应](https://news.ycombinator.com/item?id=37379272)

- 讨论涉及多个主题，如版权侵权、归档平台、版权法对内容创作者的影响、数据囤积、自动删除以及平台管理内容的自主权。
- 讨论探讨了事实与法律之间的区别、司法权在忽视某些法律方面的作用以及政治家对互联网技术的理解。
- 报告重点讨论了实体拒绝托管内容的权利、水印的使用、过时作品的保存以及版权删除过程中人的参与等问题。

## [买个有线调制解调器，去坐牢（1999 年）](http://telecom.csail.mit.edu/judy-sammel.html)

- 作者讲述了自己被 Comcast@home 错误指控为有线电视欺诈，并因此受到刑事传唤的亲身经历。
- 尽管面临康卡斯特公司和法院系统的障碍，指控还是被驳回了，之后提交人建议改进康卡斯特公司的程序，以避免类似情况的发生。
- 提交人呼吁明确客户支持的职责，并得到了 Comcast 的道歉；提交人的犯罪记录也因此被消除。

### [反应](https://news.ycombinator.com/item?id=37386397)

- 论坛内容与宽带技术人员的个人轶事有关，涉及枪支安全和立法、特色国家的枪支所有权、电缆盗窃和侵犯版权等主题。
- 讨论内容包括与服务提供商的合作经验、自由市场理论以及与重罪辩护相关的费用。
- 与会者就这些问题发表了不同的意见，强调了这些议题的复杂性和不同观点。

## [潮汐能不是可再生能源](https://cs.stanford.edu/people/zjl/tide.html)

- 该清单包含多篇讨论不同主题的科学论文，包括低温和高温超导理论、超流体和电子隧道。
- 此外，还研究了 LK-99 的重要性和局限性、布朗运动背后的解释以及克鲁克斯辐射计的驱动过程。
- 这份清单挑战了热力学中的误解，探讨了地球磁场是超导体的可能性，讨论了潮汐能不可再生的原因，并介绍了理解相对论和推导 E=mc2 的方法。

### [反应](https://news.ycombinator.com/item?id=37383283)

- 讨论探讨了潮汐能的可再生性、能源消耗的指数级增长以及未来能源项目（如戴森球或蜂群）的可行性。
- 主要议题包括可持续发展的障碍、人口增长、生育率、满足未来能源需求的战略以及能源消耗对环境稳定和经济发展的影响。
- 其他领域包括不平等现象的持续存在、能源开采对地球自转和气候的影响、风力模式对能源生产的影响以及房地产和石油生产行业的动态。

## [傳送門 64 - 任天堂 64 傳送門的翻版](https://github.com/lambertjamesd/portal64)

- GitHub 存储库 "portal64 "由 lambertjamesd 创建，是专为任天堂 64 游戏机定制的游戏《传送门》demake，包含代码、资产和相关文档。
- 摘要提供了使用 Modern SDK、Blender 和其他各种工具构建项目的指南，还概述了未来的任务、功能和当前的 bug。
- 该版本库已积累了大量的星级、观察者和分叉，彰显了它在用户中的受欢迎程度，该项目采用 MIT 许可。

### [反应](https://news.ycombinator.com/item?id=37381407)

- Portal 64 是一个旨在为任天堂 64 游戏机重新制作游戏《传送门》的项目，目前正在开发中。根据用户反馈，其 GitHub 仓库目前缺乏可视化更新。
- 一场争论浮出水面，焦点是在制造和处理旧硬件时，能源效率与生态影响的对比。一些人主张采用节能技术，而另一些人则出于生态原因支持利用现有硬件。
- 相关主题扩展到数字保存、向后兼容性和游戏开发的复杂性。创作者因成功使用 N64 SDK 制作 N64 demake 而闻名，并以与原始硬件兼容而著称。

## [我在几分钟内就为儿子制作了一个飞机观测器](https://louison.substack.com/p/i-built-a-plane-spotter-for-my-son)

- 作者利用 ChatGPT 这一强大技术迅速创建了一个飞机发现工具，并将其改编成一个网络应用程序，根据用户位置提供附近航班的实时通知。
- 作者强调了技术将想法快速转化为现实的潜力，并提出了一个注重创造性见解和有效机器交流的未来。
- 在对未来的思考中，作者假设机器有可能变得高度智能，促使人类冥想并产生创新性的疑问。

### [反应](https://news.ycombinator.com/item?id=37379801)

- 对话围绕使用人工智能语言模型（如 ChatGPT）生成代码以及相关的伦理问题（如知识产权和归属）展开。
- 与会者对 ChatGPT 的实用性表达了截然不同的观点，有的认为它可以快速生成代码片段和原型，有的则对其原创性、准确性和潜在的滥用表示担忧。
