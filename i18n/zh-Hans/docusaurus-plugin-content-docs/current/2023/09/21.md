## [对香草冰淇淋过敏的汽车 (2000)](http://www.cs.cmu.edu/~wkw/humour/carproblems.txt)

- 一位客户报告说，他的庞蒂亚克汽车只有在购买香草冰淇淋时才无法启动，但在购买其他口味的冰淇淋时却能正常启动。
- 工程师调查后发现，由于香草冰淇淋在店门口很容易买到，所以很快就被买走了，导致发动机过热，汽车出现了汽锁问题。
- 这种情况突出表明，即使是看似荒谬的技术问题，也可以有合乎逻辑的真实解释。

### [反应](https://news.ycombinator.com/item?id=37584399)

- Hacker News 的这篇讨论揭示了环境条件导致电子设备故障的各种情况，包括阳光直射影响鼠标、黑暗导致 iPhone 进入启动循环以及炎热天气引发网络中断。
- 对话中还探讨了其他问题，如湿气导致铜缆出现问题、阳光影响传感器、过时技术的故障排除难题，以及断电导致小城市服务器出现问题。
- 其他值得关注的话题包括汽车电池耗尽、斯巴鲁赔偿不足、因律师之间的混淆而撤销指控，以及揭穿雪佛兰 Nova 在西班牙语市场失败的神话。

## [面向本地优先应用程序的 ElectricSQL、Postgres 到 SQLite 主动-主动同步](https://electric-sql.com/)

- ElectricSQL 是一个开源同步层，使开发人员能够使用 Postgres 和 SQLite 创建实时、离线和响应式应用程序。
- 该同步层由 Elixir 构建，即使在编辑冲突或长时间离线使用的情况下，也能保证数据的一致性。
- ElectricSQL 团队包括无冲突复制数据类型 (CRDT) 的发明者和本地优先软件开发专家。

### [反应](https://news.ycombinator.com/item?id=37584049)

- ElectricSQL 是一个开源的本地优先同步层，在 Postgres 和 SQLite 之间提供双向主动-主动同步，支持实时、离线应用程序，并实现模式更改和数据同步控制，即使在长时间离线的情况下也是如此。
- 它采用基于 CRDT 的同步技术来处理冲突编辑，通过为所有客户端使用一个简单通用的服务器与远程服务器进行同步，采用本地优先的方法、与基于 React 的低代码工具的兼容性以及离线优先的架构，因而备受赞誉。
- 虽然 ElectricSQL 还在开发中，但它积极响应用户反馈，支持 IndexedDB 用于浏览器中的虚拟文件系统，并计划在未来支持其他驱动程序，使其与任何 JavaScript 或 TypeScript 框架兼容。

## [使用 GZIP 的 MNIST 准确率为 78%，代码行数不到 10 行](https://jakobs.dev/solving-mnist-with-gzip/)

- 作者分享了一项利用压缩技术对 MNIST 数据集进行分类的实验，该实验使用 GZIP 和 NCD（归一化压缩距离）作为相似性度量，并使用 k-NN（k-最近邻）对数据进行分类。
- 实现的算法计算测试样本和训练样本之间的 NCD，并根据最近邻居的多数类别预测标签，准确率达到约 78%。
- 这篇帖子还提到了之前一个类似但不太成功的实验，并展示了一个不那么模糊的算法版本，以便更好地理解。

### [反应](https://news.ycombinator.com/item?id=37583593)

- 文章探讨了 GZIP 压缩算法在 MNIST 数字分类中的应用，并将其与其他距离测量方法进行了比较。
- 研究得出结论认为，逻辑回归等更简单的模型可能同样有效，复杂性的增加并不能保证结果的改善。
- 文章还深入探讨了压缩算法在语言和图像分类任务中的应用，并强调了对有损压缩的担忧和明确缩略语定义的必要性。

- 文章讨论了 Windows 和 Linux 的向后兼容性对比、智能手机内核支持的维护问题以及廉价手机缺乏软件支持的问题。
