## [美国缉毒局提议对大麻重新分类](https://apnews.com/article/marijuana-biden-dea-criminal-justice-pot-f833a8dae6ceb31a8658a5d65832a3b8)

- 美国缉毒署正在考虑将大麻重新归类为危险性较低的毒品，这有可能重塑美国的毒品政策。
- 拜登总统支持对联邦大麻法进行审查，并对持有大麻的人给予宽大处理。
- 拟议的变革可能会使联邦毒品政策与各州的合法化运动保持一致，并吸引年轻选民，从而可能有利于总统拜登的公众形象。

### [反应](https://news.ycombinator.com/item?id=40213591)

- 讨论深入探讨了美国大麻合法化和监管的各个方面，如重新分类、银行法规的影响、药房安全、商业挑战以及加密货币在交易中的使用。
- 它还探讨了联邦法律的影响、大麻零售市场以及对毒品使用和合法化的不同观点。
- 讨论中重点关注的问题包括错误信息、入门药物和大公司对市场的影响。

## [利用 Pyinfra 简化基础设施自动化](https://pyinfra.com/)

- pyinfra 是一款用于基础架构自动化的 Python 工具，具有速度快、可扩展性强、用途广泛等特点，适用于临时命令、服务部署和配置管理。
- 它提供快速执行、即时调试、幂等操作以及与连接器的集成，使其成为基础设施自动化的全面解决方案。
- 用户可以通过 pip 安装 pyinfra，通过 SSH 运行命令，使用操作定义状态，使用 Python 代码部署任务，还可以使用 Python 软件包生态系统灵活地对其进行扩展。

### [反应](https://news.ycombinator.com/item?id=40211655)

- 比较 Pyinfra、Ansible、Puppet 和 Terraform 等基础设施自动化工具的声明式和命令式方法、复杂性、优势和局限性。
- 用户称赞基于 Python 的 Pyinfra 可用于自动化任务，对 Ansible 和 Puppet 的局限性表示担忧，并强调了状态管理在基础设施自动化中的重要性。
- 讨论内容包括高效管理大型服务器群所面临的挑战、声明式编程的好处，以及 DRY 编程原则与配置管理简易性之间的平衡。

## [Borgo：编译为 Go 的静态类型语言](https://github.com/borgo-lang/borgo)

- Borgo 是一种移植到 Go 的新编程语言，结合了 Go 的简洁性和 Rust 的表现力。
- 它包括代数数据类型、模式匹配、使用 ? 运算符进行错误处理，并与现有的 Go 软件包兼容。
- Borgo 语法与 Rust 相似，带有可选的分号，并提供了一些功能，如用于处理 nil 的 Option 和用于多个返回值的 Result，并通过一个猜谜游戏示例进行了演示。

### [反应](https://news.ycombinator.com/item?id=40211891)

- 讨论深入探讨了 Borgo（一种转译为 Go 的语言），它解决了 Go 代码中常见的问题，并提供了枚举和可选类型等附加功能。
- 用户对 Borgo 的功能很感兴趣，指出了 Go 等有主见语言的优点和缺点，讨论了通过 try/catch 处理错误与 Go 处理错误的比较，并强调了惯用命名规则的重要性。
- 对话还涉及编译器与转译器的区别、向后兼容性的挑战以及编译成其他语言的语言实例，重点是语言设计、错误管理和跨语言兼容性。

## [Extension.js：简化的网络扩展创建工具](https://github.com/cezaraugusto/extension.js)

- 它消除了对复杂配置和设置说明的要求，使开发人员能够专注于编码。

- 社区反馈涉及 Extension.js 的改进建议和即将推出的浏览器支持计划。

## [文件档案系统使用 Unix 工具处理现代数据](https://mgree.github.io/ffs/)

- ffs 是一个文件文件系统，用户可以将 JSON 等半结构化数据挂载到 Unix 文件系统上，并使用标准 shell 工具进行操作。
- 它支持 JSON、YAML 和 TOML 格式，支持就地文件编辑，在 Linux 上需要 FUSE，在 macOS 上需要 macFUSE，采用 GPLv3 许可。
- ffs 由 mgree 开发，与 jq 和 gron 等工具相比，它提供了一种不同的命令行数据处理方法。

### [反应](https://news.ycombinator.com/item?id=40213731)

- 争论的焦点是如何在 macOS 等操作系统上实现用户可访问的文件系统功能，由于苹果公司的 API 限制和对第三方工具（如 osxfuse）的依赖，这些功能面临着挑战。
- 用户对苹果对外部开发者的限制感到不满，担心可靠性和数据完整性，因此提出了利用 NFS 等建议。
- 讨论内容还包括将 Git 提交挂载为文件系统、将缓冲区视为文件、利用 XPath 技术将 XML 映射到文件系统等创新想法，并探讨了技术细节和技术社区内潜在的利弊。
