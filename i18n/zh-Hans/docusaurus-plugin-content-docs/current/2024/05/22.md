# 2024-05-22

## [计算机工程师先驱戈登-贝尔逝世，享年 89 岁](https://arstechnica.com/gadgets/2024/05/gordon-bell-an-architect-of-our-digital-age-dies-at-age-89/)

- 戈登-贝尔（Gordon Bell）是计算机工程师的先驱，也是开发数字设备公司（DEC）极具影响力的微型计算机的关键人物，享年 89 岁。
- 贝尔在创建 VAX 微型计算机生产线方面发挥了重要作用，他还与他人共同创办了第一个大型计算机博物馆，该博物馆后来发展成为硅谷的计算机历史博物馆。
- 他为微软研究院做出了贡献，是 MyLifeBits 生命记录项目的研究对象，并获得了包括美国国家技术奖章和电气和电子工程师协会约翰-冯-诺依曼奖章在内的众多荣誉。

### [反应](https://news.ycombinator.com/item?id=40432688)

- 戈登-贝尔（Gordon Bell）是计算机工程师和投资者的先驱，他的去世给科技行业和他指导过的人留下了重大影响。
- 贝尔因其 "生命记录 "项目 MyLifeBits 而闻名，他对计算机的贡献还包括研发 PDP-8 和创建计算机历史博物馆。
- 尽管贝尔与 DEC 和麻省理工学院有联系，但 ASCII Bell 字符（CTRL-G）的出现却早于贝尔，而且也不是以他的名字命名的；IBM 的鲍勃-贝默（Bob Bemer）在开发 ASCII 的过程中发挥了关键作用。

## [山姆-奥特曼因 OpenAI 未经授权使用斯嘉丽-约翰逊的声音而面临质疑](https://slate.com/technology/2024/05/scarlett-johansson-ai-voice-sam-altman-openai.html)

- 在斯嘉丽-约翰逊（Scarlett Johansson）指控 OpenAI 在其 GPT-4o 模型中使用了与她非常相似的声音而未征得她的同意后，由山姆-奥特曼（Sam Altman）领导的 OpenAI 正受到严格审查。
- 约翰逊拒绝了为该系统配音的邀请，并聘请了律师，导致 OpenAI 暂停使用该语音。
- 这一事件，加上数据使用方面的内部动荡和法律挑战，使人们对奥特曼的透明度和领导力产生了质疑，这与他对负责任的人工智能开发所持的公开立场形成了鲜明对比。

### [反应](https://news.ycombinator.com/item?id=40434800)

- 文章批评萨姆-奥特曼（Sam Altman）涉嫌未经同意使用酷似斯嘉丽-约翰逊（Scarlett Johansson）的配音演员，引起了人们对富人权力集中的道德和法律问题的关注，以及加强监管的必要性。
- 讨论深入探讨了声音克隆的道德影响，引用了 Midler 诉 Ford 和 Waits 诉 Frito Lay 等法律案例，并就未经许可使用名人肖像的合法性和道德性展开了辩论。
- 报告还谈到了人工智能的快速发展、对配音工作的影响以及人工智能技术的广泛影响，强调了法律限制和道德考虑之间的矛盾。

## [数据存储的革命：单一纯文本文件法》，作者：Breck Yunits](https://breckyunits.com/scrollsets.html)

- Breck Yunits 介绍了一种使用空格和换行将表格知识存储在单个纯文本文件中的方法，这种方法比二进制格式更具优势。
- 该系统名为 ScrollSets，可由人类读取，用 git 进行编辑，与传统软件和人工智能兼容，为开源网站 PLDB.io 提供动力。
- 主要功能包括度量、概念、度量和注释，以及将数据分割成多个文件、使用语法分析语言、添加 sortIndex 和类型等属性等增强功能。

### [反应](https://news.ycombinator.com/item?id=40432834)

- 布雷克的文章讨论了将知识存储在单个长纯文本文件中的问题，并对嵌套解析器等概念和语义数据概念的重塑提出了批评。
- 其中提到了 Edna 和 Heynote 等笔记工具，强调了快速访问快捷方式、本地存储等功能，以及图像支持和 OCR（光学字符识别）等潜在增强功能。
- 讨论强调了参考先前工作以避免冗余的重要性、以文件形式组织数据的实用性，以及对纯文本文件在数据存储方面的简便性的反思，尤其是在 COVID-19 大流行期间。

## [为什么要在网站上添加 /Now 页面以及如何添加](https://sive.rs/now2)

- 2015 年，德里克-西弗斯（Derek Sivers）在他的网站上引入了 "现在"（/now）页面的概念，以分享他当前的活动，其灵感来自于一位朋友的网站上没有此类信息。
- 这个想法获得了广泛的支持，促使西弗斯建立了 nownownow.com，这是一个在全球拥有 2300 多个 /now 网页的目录。
- Sivers 鼓励其他人创建自己的/now 网页，并提供了各种平台的使用说明，还表示愿意应要求将这些网页添加到目录中。

### [反应](https://news.ycombinator.com/item?id=40438960)

- 由德里克-西弗斯（Derek Sivers）推广的"/now "网页概念是指个人在个人网站上分享他们当前的活动。
- 一些人认为"/now "页面很有用，但另一些人则指出这些页面很快就会过时，建议使用/about页面、提醒事项和更简便的内容管理系统（CMS）等替代方法。
- 讨论强调了"/now "网页振兴个人网页和博客的潜力，为互联网上多样化的原创内容做出了贡献。

## [流畅位漏洞引发 C 语言内存安全隐患，急需补丁程序](https://xeiaso.net/shitposts/no-way-to-prevent-this/CVE-2024-4323/)

- Fluent Bit 项目中的一个漏洞（CVE-2024-4323）导致网站可靠性工程师和系统管理员不得不紧急修补漏洞。
- 该漏洞源于 C 语言中的 HTTP 解析代码，可通过特定 HTTP GET 请求导致堆损坏和任意代码执行。
- 这一事件凸显了 C 语言内存安全漏洞问题的顽固性，在过去 50 年中，90% 的此类问题都是由 C 语言造成的，尽管强大的编码实践非常重要。

### [反应](https://news.ycombinator.com/item?id=40437379)

- 讨论批评了美国的校园枪击事件和无效的应对措施，采用讽刺的形式讨论了 C 语言程序中缓冲区溢出的问题，并将美国的枪支法规与其他国家进行了比较，强调了全国枪支协会的影响力和公众对普遍背景调查的支持。
- 它对比了美国和瑞士的枪支法，批评了 C/C++ 而支持 Rust 和 Golang 等现代语言，并讨论了编程范例，强调了 Rust 的内存安全和社区动态。
- 对话批评了科技行业对工具的重视，而不是对熟练开发人员和良好管理的重视，强调了对清晰、可维护代码的需求，以及从 C 语言过渡到 Rust 等更安全语言的挑战。

## [Alacritty：使用 OpenGL 的高性能跨平台终端模拟器](https://github.com/alacritty/alacritty)

- Alacritty 是一款使用 OpenGL 的高性能跨平台终端模拟器，支持 BSD、Linux、macOS 和 Windows。
- 它目前还处于测试阶段，存在一些功能缺失和错误，并强调与其他应用程序的集成，而不是重新实现它们的功能。
- Alacritty 根据 Apache License 2.0 开源，预编译的二进制文件可在 GitHub 上获取，并可通过各种软件包管理器进行安装。

### [反应](https://news.ycombinator.com/item?id=40437535)

- 讨论对终端模拟器进行了比较，重点是功能、性能和用户偏好。
- Alacritty 因速度和 GPU 加速而备受赞誉，但缺乏滚动条和 sixel 支持等功能；WezTerm 因 Lua 配置和文档而备受关注；Kitty 很现代，但缺乏本地 Windows 支持。
- 用户在讨论发行版软件包与开发人员二进制文件的可靠性时，主要关注安全、性能、渲染速度、配置灵活性以及与 tmux 和 NeoVim 等工具的兼容性。
