## [优化 Rust 的调用约定以提高性能](https://mcyoung.xyz/2024/04/17/calling-convention/)

- Miguel Young de la Sota 批评了 Rust 的调用约定，提出了一种更有效的替代方案，与 Linux C ABI 标准保持一致，以获得更好的性能。
- 他建议利用 Go 的寄存器 ABI，通过直接指定 LLVM 的新标志 -Zcallconv 来优化 Rust 函数的调用约定，从而提高性能。
- Miguel 的目标是改进 Rust 的代码生成、优化寄存器使用、增强应用程序二进制接口以提高性能并减少内存流量，同时展示 LLVM 的专业知识。

### [反应](https://news.ycombinator.com/item?id=40081314)

- 讨论强调了在优化 Rust 调用约定时衡量性能的重要性，重点是速度而不是外观。
- 涉及的主题包括 Rust 调用约定的稳定性、在 x64 处理器上优化布尔运算、混合托管和非托管代码的挑战，以及 C 和 C++ 中函数返回值的注意事项。
- 对话还涉及语言之间的互操作、LLVM 调用约定在 x86 架构中的使用，以及调用约定对性能和开发人员控制的影响，旨在提高代码执行效率。

## [隐士可重现沙箱中的确定性程序执行](https://github.com/facebookexperimental/hermit)

- Hermit 是一个可重现的容器，通过将程序与时间和随机数生成等非确定性来源隔离，确保程序的执行具有确定性。
- 它拦截系统调用以实现确定性，适用于压力测试、记录/重放、可重现的构建和并发错误调试。
- 虽然它不能将程序与文件系统变化或外部网络隔离开来，但它已不再被积极开发，而是可以通过 Rust 的 cargo 工具运行，该工具根据 BSD-3 授权，支持 x86_64 Linux，并持续支持 Aarch64。

### [反应](https://news.ycombinator.com/item?id=40076848)

- Hermit 是一款沙箱工具，旨在通过拦截系统调用，以密封和可重现的方式运行程序，与 rr 类似，但缺乏对某些系统调用的支持，也没有定期更新。
- 尽管以确定性为目标，但由于 CPU 指令和系统调用拦截等因素，Hermit 可能无法实现完全的确定性，只能将程序限制在单个 CPU 上运行，以便进行分析和调试，而非一般的沙箱。
