# 2024-01-21

## [Spotube：集成了 YouTube 的开源 Spotify 客户端](https://github.com/KRTirtho/spotube)

- Spotube 是一款开源 Spotify 客户端，无需订阅 Spotify Premium 即可使用。
- 它利用 Spotify 的数据 API，并提供 YouTube、Piped.video 或 JioSaavn 等替代音源。
- 其特点包括无广告、可下载曲目、跨平台支持、体积小、数据用量低、匿名登录、时间同步歌词和原生性能。它不收集用户数据，也不支持 Spotify Shows & Podcasts。

### [反应](https://news.ycombinator.com/item?id=39066136)

- 讨论涉及音乐流媒体平台的多个方面，如 API 密钥、Spotify 的播客限制、DRM 问题、音频质量、Spotify 与 YouTube 的比较、随机播放的挫折、对艺术家收入的影响、Bandcamp 等替代平台，以及在经济上支持音乐家的建议。
- 对话强调了提高艺术家报酬和探索其他流媒体模式的重要性。
- 这一全面讨论揭示了音乐流媒体行业当前面临的挑战和潜在的解决方案。

## [动画师的困境：250 亿美元动漫产业中的低工资和超负荷工作](https://hitregbroke.com/anime/)

- 价值 250 亿美元的动漫产业面临着动画师工资低的重大问题。
- 文章讨论了这一问题背后的原因，以及对这一问题缺乏关注的情况，并重点介绍了 A1 Pictures 公司的一例员工自杀事件。
- 日本工作文化中的超负荷工作被认为是解决这一问题的一个挑战。
- 探讨了手冢治虫在该行业中的作用以及经济因素对工作条件的影响。
- 讨论了漫画家对动画改编缺乏控制以及为动画师争取退休福利的努力未获成功的问题。
- 研究了生产委员会的主导地位和可能的解决方案。
- 文章强调，需要解决动画师工资低和生活成本高之间的差距。
- 质量控制和彻底检查在动漫制作中的重要性得到了强调。
- 公众意识和支持被认为是必要的，并呼吁业界采取行动和进行变革。

### [反应](https://news.ycombinator.com/item?id=39074062)

- 日本的动漫和娱乐业正面临着薪酬低、工作条件差的问题。
- 关于公平工资的重要性及其对该行业的潜在影响存在争议。
- 讨论的主题包括人工智能的影响、工会在协商改善就业条件中的作用、动漫中的翻译和本地化，以及人工智能对音乐产业的影响。

## [介绍 Bun Shell：用 JavaScript 简化 Shell 脚本](https://bun.sh/blog/the-bun-shell)

- 由于不同平台的语法和命令存在差异，在 JavaScript 中运行 shell 脚本可能具有挑战性。
- Bun Shell 是 Bun 中的嵌入式语言和解释器，可使用 JavaScript 和 TypeScript 编写跨平台 shell 脚本。
- Bun Shell 旨在为在 JavaScript 中运行 shell 脚本提供一个精简高效的解决方案，解决现有 shell 的复杂性和不一致性。

### [反应](https://news.ycombinator.com/item?id=39071688)

- 用户正在讨论名为 Bun Shell 的新 shell，对兼容性、替换现有命令和臃肿的 coreutils 表示担忧。
- 该工具因其简单、创新和在脚本任务中的实用性而备受赞誉。
- 对话还涉及不同外壳的可用性，并推荐了替代选择。
- Bun 的开发人员对问题和反馈做出了回应，解决了人们对导入名称和稳定性的担忧。
- 讨论范围扩大到编程语言 Zig，用户分享了经验并讨论了其稳定性。
- 其他主题包括 JavaScript 脚本、Node.js 中 shell 命令的性能问题、标记模板、Lit Element 和 rimraf 库。
- 人们对使用 Bun Shell 的安全风险和局限性表示担忧。
- 讨论还探讨了 ZSH、模板语言、JSON 和 YAML 配置文件以及跨平台兼容性。
- 关于同步执行与异步执行以及托管服务提供商的负面经验，出现了争论。

- 讨论还涉及更广泛的话题，如先进支付技术的采用以及现金或数字支付的偏好。
- 还涉及企业接受现金支付的法律义务。

## [富士通 Lifebook AH532 BIOS 消失：调查解决方案](https://blog.timschumi.net/2024/01/20/ah532-bios-investigation.html)

- 作者调查了富士通笔记本电脑 BIOS 设置菜单无法访问的问题。
- 他们尝试不同的方法来解决问题，包括重置 BIOS 设置和刷新 BIOS。
- 他们发现一个特定命令会清除启动菜单并导致 BIOS 设置菜单无法使用，于是提供了一个临时解决方案，并计划稍后永久性地解决这个问题。

### [反应](https://news.ycombinator.com/item?id=39070871)

- 讨论的重点是访问和修改 UEFI 变量所面临的挑战，尤其是在 Linux 操作系统中。
- 对可能的解决方案提出了建议，例如开发一种可在 Linux 和 Windows 上运行的工具。
- 一些用户遇到过闪存问题，建议使用特定的硬件组件。有人提到预装的 Windows 版本有可能解决硬件/固件问题，从而降低了进一步调查的积极性。强调了修改 UEFI 启动变量的综合性多平台解决方案的重要性。
