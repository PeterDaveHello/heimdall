## [“Jelly Star – 最小的安卓13智能手机”](https://www.unihertz.com/en-de/products/jelly-star)

- “Unihertz发布了Jelly Star，被称为世界上最小的Android 13智能手机，售价为209.99美元(原价229.99美元)。”
- “主要特点包括3英寸显示屏、带LED灯的透明背面设计、八核MediaTek Helio G99处理器、8GB内存、256GB存储空间和48MP后置摄像头。”
- “该设备支持全球LTE、NFC、双Nano SIM卡，并包括指纹解锁、USB OTG、FM收音机和GPS等附加功能。”

### [反应](https://news.ycombinator.com/item?id=40956979)

- “Unihertz发布了Jelly Star，被誉为最小的Android 13智能手机，由于其独特的尺寸和功能引起了广泛关注。”
- “用户反应不一，有人称赞其紧凑的设计和滑动输入功能，但也有人批评该品牌的软件更新少和客户服务差。”
- “讨论强调了软件更新的重要性、与LineageOS等自定义ROM的兼容性以及智能手机中可更换电池的需求。”

## [“在最初拒绝之后，苹果已批准首个适用于iOS的PC模拟器。”](https://www.theverge.com/2024/7/13/24198015/apple-utm-se-pc-os-emulator-for-ios)

- “苹果已批准UTM SE，这是首个适用于iOS的PC模拟器，允许用户在iPhone上运行Windows、Mac OS 9和Linux的经典软件和老式游戏。”
- “UTM SE 基于 QEMU 构建，支持 VGA 和终端模式，并模拟 x86、PPC 和 RISC-V 架构，但不包含任何操作系统。”
- “该应用在最初被拒后获得批准，这得益于AltStore团队和另一位开发者的QEMU TCTI实现，现在可以在iOS、iPadOS和visionOS上免费使用。”

### [反应](https://news.ycombinator.com/item?id=40958465)

- “苹果公司在最初拒绝后，已批准首个用于iOS的PC模拟器UTM，这引起了用户的兴奋。”
- “该批准归因于欧盟法规，突显了关于消费者权益、可修复性和现代电子产品所有权的持续讨论。”
- “由于苹果的限制，UTM没有即时编译(JIT)，导致性能较慢，但仍能在未越狱的iPhone上运行各种操作系统，包括64位的Linux。”

## [“对个人的谷歌位置数据没有合理的隐私预期”](https://fourthamendment.com/?p=58338)

- “第四巡回法院裁定，用户在其谷歌位置数据中没有合理的隐私期望(REP)，因为用户自愿与谷歌分享这些数据。”
- “法院的裁决强调，地理围栏(一种执法部门用来收集位置信息的方法)不构成第四修正案下的搜查。”
- “这一裁决意义重大，因为它涉及自2016年以来执法部门日益增加的地理围栏令的使用以及谷歌保护用户隐私的程序。”

### [反应](https://news.ycombinator.com/item?id=40958458)

- “谷歌正在改变其位置数据的处理方式，将数据推送到设备并从服务器中清除，这意味着它将不再响应执法部门的地理围栏搜查令。”
- “这一变化引发了不同的反应，一些用户对隐私的增加表示赞赏，而另一些用户则担心失去对其时间线数据的访问。”
- “加密备份位置数据的功能是可用的，但默认情况下是禁用的，需要手动启用，这让一些用户觉得不直观。”

## [“Talos：用于运行 Kubernetes 的安全、不可变和精简的 Linux 操作系统”](https://www.talos.dev)

- “Talos Linux 是一个安全、不可变且极简的 Linux 操作系统，专为 Kubernetes 设计，支持云平台、裸机和虚拟化平台，系统管理通过 API 完成。”
- “它通过最小化、强化和不可变性来强调安全性、可预测性和可演化性，所有 API 访问都通过双向 TLS (mTLS) 认证进行保护。”
- “Talos Linux 已经可以投入生产使用，支持大型 Kubernetes 集群，并且可以在笔记本电脑上通过 Docker 在仅仅 3 分钟内启动，使其对开发人员来说非常易于访问和高效。”

### [反应](https://news.ycombinator.com/item?id=40956546)

- “Talos 是一个安全、不可变且极简的 Linux 操作系统，专为运行 Kubernetes 而设计，通过基于 API 的配置、升级和调试简化系统维护。”
- “Talos 需要一个完整的磁盘进行安装，这对于存储有限的服务器来说可能是一个挑战，但用户已经探索了诸如 PXE、虚拟化和外部存储等解决方案。”
- “Talos 因其易于设置和维护而备受赞誉，支持 Raspberry Pi 4b，并在其 YouTube 频道上提供教程。”

## [“构建和扩展 Notion 的数据湖”](https://www.notion.so/de-de/blog/building-and-scaling-notions-data-lake)

- “Notion的数据在三年内增长了10倍，这需要开发和扩展他们的数据湖，以管理产品和分析需求，包括Notion AI功能。”
- “主要挑战包括管理480个Fivetran连接器、向Snowflake进行缓慢且昂贵的数据摄取，以及复杂的数据转换逻辑超出标准SQL的能力。”
- “使用S3进行存储、Spark进行处理、以及Kafka Debezium CDC连接器与Apache Hudi进行数据摄取来实现数据湖，显著节省了成本并减少了数据摄取时间，从而成功推出了Notion AI功能。”

### [反应](https://news.ycombinator.com/item?id=40959787)

- “Notion转向数据湖以减少与Fivetran和Snowflake相关的高成本，在2022年节省了超过一百万美元。”
- “这次迁移涉及转移大型Postgres数据集，并利用Iceberg和Delta Lake等工具来支持分析和AI功能。”
- “用户对数据隐私和潜在的数据挖掘表示担忧，但很可能 Notion 会在内部使用这些数据进行分析和人工智能训练。”
