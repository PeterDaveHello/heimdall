## [美国环保局建议将 "永久化学品 "列为有害物质](https://www.npr.org/2024/02/02/1228540721/forever-chemicals-pfas-epa-hazardous)

- 美国环保局建议将九种全氟辛烷磺酸化学品（又称 "永久化学品"）归类为对人类健康有害的物质。
- 全氟辛烷磺酸与癌症和生殖问题等有害影响有关，它们存在于不粘锅和消防泡沫等日常用品中。
- 一项研究发现，美国近一半的自来水中含有 PFAS。
- 环保局的建议公开征求公众意见，以便公众提出建议和反馈。

### [反应](https://news.ycombinator.com/item?id=39230263)

- 讨论涉及多个主题，如环保局对 "永久化学品 "的分类和监管。
- 围绕全氟辛烷磺酸化学品的使用和处置问题存在争论，引发了对潜在健康风险的担忧。
- 对话强调了化学品和塑料生产和使用的复杂性，敦促人们需要更清晰的语言、更深入的研究以及对风险和益处的考虑。

## [SeaweedFS：用于分布式存储的可扩展高效开源文件系统](https://github.com/seaweedfs/seaweedfs)

- SeaweedFS 是一个开源分布式文件系统，设计用于存储和服务大量文件。
- 它支持目录、POSIX 属性、卸载到分布式键值存储，并可与云存储集成。
- SeaweedFS 提供快速的访问速度、可扩展的容量以及数据量管理、缓存、复制、加密、定制等功能。

### [反应](https://news.ycombinator.com/item?id=39235593)

- SeaweedFS 是一种快速分布式存储系统，它结合了数据库和文件系统的特点，可高效处理数据并从断电和数据损坏中恢复。
- 它是用 Go 编程语言编写的，虽然令人惊讶，但其性能却令人赞赏。
- 讨论的主题包括 Go 中的错误处理、每个对象存储一个文件的利弊、对其他存储解决方案的建议，以及微软之前对类似系统的尝试。
- Seaweed Filer、Cassandra 和 Redis 等临时文件存储的有效性以及与 ZFS 的比较也是对话的一部分。
- 总之，与传统存储系统相比，SeaweedFS 提供了一种非等级分布式键值存储。

## [AWS 对 IPv4 使用的收费预计每年将产生 10 亿美元收入，推动 IPv6 的采用](https://www.tomshardware.com/networking/amazons-aws-new-charge-for-using-ipv4-is-expected-to-rake-in-up-to-dollar1b-per-year-change-should-speed-ipv6-adoption)

- AWS 正在对使用公共 IPv4 地址的客户实施每 IP 每小时 0.005 美元的新收费，以促进 IPv6 的采用。

- AWS 因对 IPv4 使用收费而受到批评，导致用户不满。
- 互联网服务提供商提供 IPv6 支持的能力有限，阻碍了 IPv6 的采用。
- 仅使用 IPv4 的流量比例引起了争议，这凸显了 IPv4 的持续盛行。
- 与会者讨论 IPv6 的实施，包括 IP 地址子网的分配以及对定价和采用的影响。
- 讨论了 IPv6 的优点和缺点，以及替代解决方案和 IPv4 的未来。
- 人们普遍呼吁增加对 IPv6 的采用和支持，以应对 IPv4 的挑战和限制。

## [欧洲监管机构在《网络复原力法》和《产品责任指令》中解决开源问题](https://blog.opensource.org/the-european-regulators-listened-to-the-open-source-communities/)

- 开放源码倡议（OSI）和其他开放源码社区一直在与欧洲监管机构讨论 2023 年的《网络复原力法案》（CRA）和《产品责任指令》（PLD）。
- 最初，人们对《计算机用户保护法》对开源开发的潜在影响表示担忧，但经过广泛参与，最终文本解决了个人开发者和开源基金会面临的大部分风险。
- 目前仍有一些挑战需要解决，例如明确《计算机权利法》和《公共资源法》下的责任，开放源码协会建议欧盟委员会向开放源码社区寻求专家建议。
- 呼吁今后的立法工作应包括第四部门，并将在 2024 年第四部门论坛期间举办一系列研讨会，以进一步开展讨论。

### [反应](https://news.ycombinator.com/item?id=39227184)

- 欧盟的《网络复原力法案》（CRA）引起了使用开源软件的小型企业对合规性和成本的担忧。
- 一些人认为，立法可能会阻碍开源软件的采用，阻碍创新。
- 关于开源软件的责任和义务，以及小型项目在获得认证方面所面临的挑战，存在着争论。
- 这些规定可能会影响欧盟公司与非欧盟公司的非认证应用程序竞争。
- 为解决这些问题，欧盟委员会有必要作出改进和提供指导。
