## [Swift 的静态 Linux SDK：为任何 Linux 发行版构建完全静态链接的可执行文件](https://www.swift.org/documentation/articles/static-linux-getting-started.html)

- 静态 Linux SDK 允许在不需要目标系统上的 Swift 运行时和依赖项的情况下，为 Linux 构建 Swift 程序，从而创建完全静态链接的可执行文件。
- 它支持在macOS上进行开发并部署到Linux，消除了运行时开销和版本控制问题，但会导致可执行文件变大且无法共享代码。
- SDK使用Musl C库进行静态链接，这需要对使用C库的软件包进行调整，并且可以为x86-64和ARM64机器构建二进制文件。

### [反应](https://news.ycombinator.com/item?id=40651054)

- Swift 的新静态 SDK 支持用户自定义平台，包括嵌入式系统和 WebAssembly (WASM)，展示了其在 Apple 生态系统之外的扩展。
- Swift迁移到非苹果的GitHub组织，并在AI操作系统中用于安全验证，突显了其在各个技术领域中日益增长的多功能性和应用。
- 开发进展使得在 Alpine 容器中运行 Swift 二进制文件、交叉编译以及对原生 Debian 的支持成为可能，这让那些喜欢在开发虚拟机(VM)中使用 Debian 的开发者感到兴奋。

- 住房成本在过去一个世纪里稳步上升，这是由于人口增长超过了理想土地供应的速度。

- 文本讨论了影响房价的各种因素，包括双收入家庭、城市化以及技术对分布式生活的影响。

## [AES-GCM 安全风险：重用随机数的危险](https://frereit.de/aes_gcm/)

- 文章强调了在AES-GCM加密中使用唯一随机数的重要性，以防止严重的安全故障。
- 它解释了重复使用一次性随机数如何使攻击者在不知道AES密钥的情况下解密密文并验证数据，从而破坏整个加密方案。
- 这篇文章深入探讨了AES-GCM的数学基础，包括伽罗瓦域算术和多项式因式分解，并展示了在重用随机数时如何利用这些原理破解加密。

### [反应](https://news.ycombinator.com/item?id=40653125)

- 文本强调了在AES-GCM加密中重用随机数的风险，特别是在长期密钥使用场景中，如VPN、分组通信和全盘加密。
- 重用随机数可能导致严重的安全漏洞，包括在攻击者知道明文和密文的情况下可能发生的攻击。
- 替代AES-GCM的算法，如AES-GCM-SIV、XSalsa20、XChaCha20以及AEGIS系列算法，因其在性能和安全性方面的优势，尤其是在重用随机数(nonce)可能成为问题的环境中，推荐使用。
