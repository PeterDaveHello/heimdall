## [“韩国电信公司用恶意软件攻击种子用户”](https://www.tomshardware.com/tech-industry/cyber-security/south-korean-telecom-company-attacks-torrent-users-with-malware-over-600000-people-report-missing-files-strange-folders-and-disabled-pcs)

- “韩国主要电信公司KT Corporation自2020年5月起，故意感染超过60万用户的设备，以恶意软件干扰Webhard的基于BitTorrent的网格服务。”
- “恶意软件导致了文件丢失、奇怪的文件夹和电脑无法使用等问题，并被追溯到KT的数据中心。”

- “该事件突显了韩国互联网服务提供商(ISP)持续存在的问题，包括高额费用和潜在的权力滥用。”
- “这种情况引发了关于韩国互联网自由和基础设施状况的讨论。韩国曾以其快速的互联网速度而闻名，但现在因过时的做法和政府控制而受到批评。”

## [“Google Sheets 将其计算工作器从 JavaScript 移植到 WasmGC”](https://web.dev/case-studies/google-sheets-wasmgc)

- “Google Sheets 已从 JavaScript 过渡到 WasmGC(WebAssembly 垃圾回收)以提升性能，特别是针对大型电子表格。”
- “WasmGC 支持像 Java 这样的垃圾回收语言，在网页上提供接近原生的速度，而 Google Sheets 是首批实现这一功能的应用之一。”
- “WasmGC的初始性能比JavaScript慢，但随后的优化显著提高了其速度，展示了其在革新网页应用开发方面的潜力。”

### [反应](https://news.ycombinator.com/item?id=40808820)

- “Google Sheets 将其计算引擎从 JavaScript 转换为 WasmGC，最初性能较慢，但后来进行了优化，使其速度是 JS 版本的两倍。”
- “这次迁移涉及大量的重新实现工作，包括去虚拟化和利用浏览器API，旨在提高性能并利用原生的V8垃圾回收器。”
- “尽管有所改进，一些用户仍然觉得 Google Sheets 比本地应用程序如 Excel 更慢，这突显了网络技术的持续演变和优化。”

## [“科学家们现在认为他们知道大脑产生的废物去了哪里。”](https://www.npr.org/sections/shots-health-news/2024/06/26/g-s1-6177/brain-waste-removal-system-amyloid-alzheimer-toxins)

- “科学家们发现了关于大脑废物清除系统的新细节，这对于维持大脑健康和预防疾病至关重要。”
- “研究发表在《自然》杂志上，揭示了睡眠期间的缓慢电波有助于将充满废物的液体从大脑深处推向表面，在那里它被吸收到血液中并由肝脏和肾脏排出。”
- “这个类淋巴系统对于清除与阿尔茨海默病相关的淀粉样物质至关重要，废物清除受损可能会导致阿尔茨海默病、帕金森病、头痛和抑郁症等各种脑部疾病。”

### [反应](https://news.ycombinator.com/item?id=40805251)

- “科学家们已经确定，深度睡眠在清除大脑废物(包括与阿尔茨海默症和痴呆症相关的淀粉样蛋白)方面起着至关重要的作用。”
- “研究表明，通过光和声音刺激在大脑中引发缓慢的电波可能会增强这种废物清除功能，尽管目前大多数研究都是在小鼠身上进行的。”
- “诸如咖啡消费、锻炼和睡眠模式等因素也在讨论中，因为它们可能对大脑健康产生影响。”

## [“响应时间为500毫秒的语音机器人”](https://fastvoiceagent.cerebrium.ai/)

- “在GPT-4发布后，开始了对语音界面和大型语言模型(LLM)的实验，突显了语音交互与生成式AI的吸引力和新挑战。”
- “实现500毫秒的语音到语音响应时间具有挑战性，但通过将转录、LLM推理和语音生成一起托管、优化数据路由以及确保良好的WiFi和VRAM缓存，是有可能实现的。”
- “一个支持500毫秒响应的语音机器人演示和一个用于在A10/A100/H100 GPU上部署的容器现已可用，目前的指标显示总响应周期为759毫秒，表明了AI的快速发展和未来简化的潜力。”

### [反应](https://news.ycombinator.com/item?id=40805010)

- “通过优化转录、LLM推理和语音生成过程，以及高效的数据路由和缓存，语音机器人实现500毫秒响应时间是可能的。”
- “语音到语音响应的指标总计为759毫秒，但未来具有原生音频功能的大型语言模型可能会进一步减少这一时间。”
- “用户强调了在人工智能互动中低延迟的重要性，特别是在客户服务方面，并讨论了潜在的技术改进，如浏览器支持和设备上的处理。”
