# 2024-06-18

## [“必须立即停止聊天控制”](https://threema.ch/en/blog/posts/stop-chat-control)

- “欧盟委员会的‘聊天控制’提案旨在实施大规模监控，可能会危及公民的隐私和数据安全。”
- “如果通过，它将要求服务提供商扫描消息中的儿童性虐待材料(CSAM)，但批评者认为这对犯罪分子无效，并且对民主有害。”
- “Threema，一项安全通信服务，反对该提案，并可能为了避免遵守规定而离开欧盟，这突显了潜在的滥用问题以及隐私倡导者的反对。”

### [反应](https://news.ycombinator.com/item?id=40715449)

- “实施一个全球系统来监管互联网隐私将面临隐私倡导者和科技公司的巨大阻力。”
- “由于各国对隐私和互联网自由的承诺程度不同，在全球范围内实施这样的系统几乎是不可能的。”

## [“聊天控制：与基本权利不兼容(2022)”](https://freiheitsrechte.org/en/themen/digitale-grundrechte/chatkontrolle)

- “欧盟委员会的《聊天控制条例》草案旨在打击儿童性暴力，但引发了对基本权利的重大担忧。”
- “关键问题包括隐私侵犯、对自由表达的寒蝉效应、容易出错的过滤义务、网站封锁和强制年龄验证。”
- “GFF认为这些措施违反了《欧盟基本权利宪章》，并呼吁重新考虑该草案。”

### [反应](https://news.ycombinator.com/item?id=40715695)

- “欧洲议会正在辩论一项可能侵犯基本权利的‘聊天控制’立法，该立法要求用户选择加入才能发送图片和视频。”
- “批评者认为，该提案与欧盟的《通用数据保护条例》原则相矛盾，可能导致被迫同意，从而引发对隐私和政府过度干预的担忧。”
- “该立法可能很快会被欧洲理事会通过，引发对大规模监控的担忧，并质疑欧盟对保护个人权利的承诺。”

## [“欧盟明天将批准聊天控制”](https://www.patrick-breyer.de/en/council-to-greenlight-chat-control-take-action-now/)

- “欧盟理事会定于2024年6月20日就涉及大规模搜索私人通信的聊天控制进行投票。”
- “投票的时间安排在欧洲选举之后不久，被视为试图避免公众审查。”
- “敦促公民社会立即采取行动，通过联系政府、在网上提高意识和组织抗议活动，因为当前的草案被认为是不可接受的。”

### [反应](https://news.ycombinator.com/item?id=40710993)

- “欧盟准备批准‘聊天控制’法规，该法规要求在Reddit、Twitter、Discord和Steam等平台上扫描所有直接消息，以查找儿童性虐待材料(CSAM)。”
- “批评者认为，这一措施是前所未有的，可能无效，因为违法者可能会转向私人服务，而且这引发了重大的隐私和过度干预问题。”
- “Signal基金会宣布，如果该法规被强制执行，它将退出欧盟，这突显了该提案的争议性。”

## [“已发布 Htmx 2.0.0”](https://htmx.org/posts/2024-06-17-htmx-2-0-0-is-released/)

- “htmx 2.0.0 已经发布，结束了对 Internet Explorer 的支持，并在不改变核心功能或 API 的情况下收紧了一些默认设置。”
- “主要变化包括将扩展移至新仓库、移除已弃用的属性以及修改HTTP DELETE请求的处理。”

### [反应](https://news.ycombinator.com/item?id=40709769)

- “已发布 Htmx 2.0.0，主要特点是清理和取消对 Internet Explorer (IE) 的支持，而不是重大新功能。”
- “开发人员称赞 htmx 简化了网页开发，一位用户用几个 htmx 属性替换了 500 行 JavaScript (JS) 代码，提高了效率和乐趣。”
- “此次发布引发了关于潜在改进和与其他工具比较的讨论，突显了htmx在减少对复杂JS框架依赖方面的作用。”

## [“对Fandom的‘注意力攻击’”](https://j3s.sh/thought/stop-using-fandom.html)

- “Fandom，一个受欢迎的维基网站，因其侵入性广告(包括自动播放视频和不断的打断)而受到批评，优先考虑利润而非用户体验。”
- “2023年，Fandom有争议地用麦当劳的Grimace Shake广告取代了用户内容，导致大量维基迁移到独立域名，如Runescape、Minecraft和Hollow Knight。”
- “鼓励用户通过使用Indie Wiki Buddy等工具、使用广告拦截器以及将他们的维基迁移出Fandom来支持独立维基。”

### [反应](https://news.ycombinator.com/item?id=40711086)

- “社区正在将他们的维基从Fandom迁移到自托管或其他平台，因为Fandom上有侵入性广告和过时内容。”
- “值得注意的例子包括Runescape和Minecraft的维基，它们已经成功地从Fandom转移。”
- “像Indie Wiki Buddy和LibRedirect这样的工具通过将用户重定向到更友好的资源，帮助用户避免使用Fandom，强调了风险投资对用户驱动内容平台的不利影响。”

## [“人类在60万年前开始迅速积累技术知识”](https://news.asu.edu/20240617-science-and-technology-asu-study-points-origin-cumulative-culture-human-evolution)

- “亚利桑那州立大学的研究人员建议，人类大约在60万年前通过社会学习开始迅速积累技术知识，这标志着累积文化的起源。”
- “这项研究发表在《美国国家科学院院刊》上，分析了长达330万年的石器制造技术，指出在大约60万年前复杂性显著增加。”
- “这一时期，可能在中更新世时期，还出现了诸如控制使用火和建造木结构等进步，这表明累积文化早于尼安德特人与现代人类的分化。”

### [反应](https://news.ycombinator.com/item?id=40711284)

- “人类大约在60万年前开始积累技术知识，多个智人种可能共享和交流技术。”
- “‘人类’一词可以指现代人类和整个智人属，但‘古人类’更为精确；关于尼安德特人和丹尼索瓦人是否被视为人类，存在争议。”
- “知识的快速积累与通信的进步有关，可能包括早期形式的语言，这突显了语言在技术转移中的作用。”

## [“400多个LLM的代币价格计算器”](https://github.com/AgentOps-AI/tokencost)

- “Tokencost 是一个实用程序库，旨在通过计算提示和完成中的标记并应用特定模型的定价来估算与大型语言模型 (LLM) 相关的成本。”
- “它解决了在各种模型和定价方案中跟踪成本的挑战，通过提供实时成本估算，帮助用户避免意外账单。”

### [反应](https://news.ycombinator.com/item?id=40710154)

- “Tokencost 是一个实用程序库，旨在通过计算提示和完成中的标记数量并乘以模型成本来估算超过 400 个大型语言模型 (LLM) 的成本。”
- “由AgentOps开发并开源，它通过一个简单的成本字典和实用函数帮助开发者跟踪支出并避免意外账单。”
- “用户建议改进，例如增加对Rust的支持、规范化成本、包括图像和函数调用成本，尽管对于没有公开分词器的模型，准确性仍然存在担忧。”

## [“从LLMs获取结构化输出的每种方法”](https://www.boundaryml.com/blog/structured-output-from-llms)

- “这篇文章讨论了从大型语言模型(LLM)中获取结构化输出(如JSON)的挑战，因为这些模型通常以自然语言返回响应。”
- “它提供了对各种框架的详细比较，这些框架旨在将LLM输出转换为结构化格式，并根据语言支持、JSON处理、提示控制和支持的模型提供商等标准进行评估。”
- “比较的框架包括BAML、Instructor、TypeChat、Marvin、Outlines、Guidance、LMQL、JSONformer、Firebase Genkit、SGLang和lm-format-enforcer，每个框架在处理结构化数据提取方面都有独特的功能和能力。”

### [反应](https://news.ycombinator.com/item?id=40713952)

- “BAML的文章探讨了从大型语言模型(LLM)中获取结构化输出的方法，强调了BAML在处理格式错误的JSON时独特的解析方法。”
- “BAML 提供开源和付费功能，付费选项主要集中在监控和增强 AI 管道。”
- “文章比较了各种框架，并讨论了在实施结构化输出时的挑战和权衡，指出一些用户更喜欢像Pydantic这样的简单方法来进行JSON验证。”

## [“关于基本复杂性的一点说明”](https://olano.dev/blog/a-note-on-essential-complexity)

- “软件工程师有多个重叠且有时相互冲突的目标，例如编写代码、管理复杂性和满足客户需求。”
- “本质复杂性是问题固有的，而偶然复杂性则源于性能问题或次优工具；减少这两者都至关重要。”
- “资深工程师可以通过挑战假设和与利益相关者协商来重新定义问题，从而可能简化需求并减少复杂性。”

- “这篇文章幽默地提到了Stroustrup C++讽刺作品，以突出编程语言中的故意复杂性。”
- “它认为，最小化复杂性对于良好的工程至关重要，平衡短期和长期决策，并确保一致性以避免不必要的复杂化。”
