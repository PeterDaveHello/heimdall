## [基因疗法恢复遗传性耳聋儿童的听力](https://cosmosmagazine.com/science/biology/gene-therapy-restores-hearing-to-children-with-inherited-deafness/)

- 一项临床试验利用基因疗法，特别是通过腺相关病毒（AAV）传递 OTOF 基因的功能拷贝，成功地恢复了五名遗传性耳聋患儿的听力。
- 据观察，两名儿童在听力、语言感知和声音定位方面都有明显改善，并开始欣赏音乐。
- 该研究发表在《自然医学》上，强调了这种方法的潜力，并主张进行更大规模的国际试验，同时指出，尽管存在轻微的不良反应，但这种疗法是安全有效的。

### [反应](https://news.ycombinator.com/item?id=40620056)

- 基因疗法将携带功能性 OTOF 基因的腺相关病毒注入遗传性耳聋儿童的内耳，确保基因的长期表达，从而恢复了他们的听力。
- 讨论探讨了 CRISPR 等基因编辑技术对人体进行彻底改造的潜力，以及将美容基因工程置于医疗应用之上的伦理意义。
- 提出了利用基因疗法根除儿童遗传性耳聋的伦理问题，重点是聋人社区对人工耳蜗植入的同意和争议。

## [Piku：简化多语言个人服务器的 Git 推送部署](https://github.com/piku/piku)

- Piku 受 Dokku 的启发，可将 git 推送部署到个人服务器，支持类似 Heroku 的工作流程，适用于各种编程语言（Python、Node、Go、Java、Clojure、Ruby）。
- 它可以在 ARM 和英特尔架构上部署、管理和扩展多个应用程序，使用 Python、nginx 和 uwsgi 兼容任何云提供商或裸机设置。
- Piku 运行稳定，维护积极，专为低端设备、业余爱好者和教育使用而设计，强调最小化依赖性、简单性和兼容性。

### [反应](https://news.ycombinator.com/item?id=40622704)

- 讨论重点介绍了 Piku，这是一款用于将 Git 推送部署到个人服务器的简约工具，并将其与 Heroku、Dokku 和 Kubernetes 的简单性和易用性进行了比较。
- Piku 的教程因其清晰度而备受赞誉，现已成为 Piku 官方 GitHub 组织的一部分，并计划进一步改进。
- 用户讨论了各种部署设置，包括用于沙箱的 systemd nspawn 和用于提高稳定性的 NixOS，并讨论了对依赖性管理、安全性和 Docker 开销的担忧，他们更倾向于轻量级、可定制的 PaaS 解决方案。

## [伟哥有望改善脑血流以预防痴呆症](https://www.ox.ac.uk/news/2024-06-07-sildenafil-viagra-improves-brain-blood-flow-and-could-help-prevent-dementia)

- 牛津大学发表在《循环研究》（Circulation Research）上的一项研究表明，西地那非（万艾可）可改善血管性痴呆高危患者的脑血流量和脑功能。
- OxHARP 试验是一项双盲、安慰剂对照研究，有 75 名参与者参加，结果发现与西洛他唑相比，西地那非能增强脑血管功能，且副作用较少。
- 这些研究结果表明，西地那非具有预防血管性痴呆症的潜力，因此需要进行更大规模的试验。

### [反应](https://news.ycombinator.com/item?id=40620739)

- 最近的一项研究表明，最初用于治疗心血管问题和勃起功能障碍（ED）的伟哥可能会改善脑血流量，有助于预防痴呆症。
- 使用者报告了各种副作用，如头痛、恶心和鼻塞，但这些副作用通常会随着时间的推移而减轻。
- 讨论涉及更广泛的健康益处、药物研究的不可预测性，以及需要进行严格的研究来确认对阿尔茨海默氏症和血管性痴呆症等疾病的益处。

## [语言模型的革命性变革：无 MatMul 方法将内存使用量减少 10 倍以上](https://arxiv.org/abs/2406.02528)

- 朱瑞杰等人撰写的论文 "Scalable MatMul-free Language Modeling "研究了去除大型语言模型（LLM）中的矩阵乘法（MatMul）操作以降低计算成本的问题。
- 作者的研究表明，他们的不含 MatMul 的模型在十亿参数规模下仍能保持强劲的性能，可与最先进的 Transformers 相媲美，同时大大减少了推理和训练过程中的内存使用量。
- 他们提供了一种 GPU 高效实施方案，可将内存消耗减少 61%；一种优化内核，可将内存消耗减少 10 倍以上；一种定制 FPGA 硬件解决方案，可提高效率，实现类似大脑的处理能力。

### [反应](https://news.ycombinator.com/item?id=40620955)

- 论文 "可扩展的无 MatMul 语言建模 "介绍了一种利用三元权重和量化感知训练（QAT）来提高语言模型效率的新方法。
- 该方法采用了经过优化并与 Triton 融合的内核，减少了过度拟合和内存使用，并结合了线性变换器，最大限度地减少了注意力机制中的乘法运算。
- 该实施方案节省了大量内存，与 Huggingface 变压器兼容，并包含一个使用定制汇编器编程的 FPGA 内核，这表明未来 ASIC 开发具有潜力。

## [瑞典历史性地改用右侧驾驶：Dagen H 的故事](https://en.wikipedia.org/wiki/Dagen_H)

- Dagen H，即 "Högertrafikomläggningen"，是指 1967 年 9 月 3 日，瑞典将左侧驾驶改为右侧驾驶，以减少正面碰撞。
- 这一过渡需要广泛的公众教育、新的道路标识和更换 35 万个交通标志，并在切换期间禁止非必要的交通。
- 虽然这一变化最初减少了事故，但两年内事故率又恢复了正常；有轨电车大部分被公共汽车取代，只有少数有轨电车线路得以保留。

### [反应](https://news.ycombinator.com/item?id=40622912)

- 讨论强调了瑞典不顾公众的强烈反对，于 1967 年改用右侧通行的做法及其最终带来的好处。
- 用户将过去做出此类决定所需的政治勇气与当今在媒体驱动的环境下实施不受欢迎的政策所面临的挑战进行了比较。
- 对话还探讨了全球驾驶规范、《日内瓦道路交通公约》以及右侧驾驶和将英语作为第二官方语言等标准化做法的广泛影响。

## [LLM 代理团队在利用 HPTSA 系统开发零日漏洞方面表现出色](https://arxiv.org/abs/2406.01637)

- Richard Fang 等人撰写的论文 "LLM 代理团队可利用零日漏洞 "研究了大型语言模型（LLM）代理在网络安全中的应用，重点关注零日漏洞。
- 作者介绍了 HPTSA，这是一个由规划代理协调子代理的系统，以克服单个 LLM 代理在处理未知漏洞和长期规划方面的局限性。
- 研究以 15 个真实世界的漏洞为基准，结果表明 HPTSA 基于团队的方法明显优于以前的方法，性能最多可提高 4.5 倍。

### [反应](https://news.ycombinator.com/item?id=40624563)

- 讨论探讨了 Claude-opus 和 ChatGPT 等大型语言模型 (LLM) 在识别代码中的零日漏洞方面的优缺点。
- 虽然 LLM 可以准确定位技术上正确的问题，但其更广泛的相关性还存在争议，对于复杂的漏洞，专业工具和专家知识仍然必不可少。
- 伦理和法律问题、高昂的成本以及实用性问题（尤其是对于开源项目而言），凸显了人们对 LLM 相对于现有网络安全技术的附加值持怀疑态度。
