## [呼吁到 2026 年恢复汽车物理控制](https://arstechnica.com/cars/2024/03/carmakers-must-bring-back-buttons-to-get-good-safety-scores-in-europe/)

- 欧洲新车安全评估协会（Euro NCAP）倡导汽车制造商在 2026 年前重新采用物理控制来控制汽车的基本功能，以消除因过度使用触摸屏而导致的驾驶分心。

### [反应](https://news.ycombinator.com/item?id=39598189)

- 欧洲 NCAP 正在敦促汽车制造商恢复对转向灯和雨刷等关键功能的物理控制，以提高安全性。
- 讨论强调了与触摸屏相关的风险，尤其是对重要控制装置而言，并强调了在车辆中使用方便、易于获取的控制装置的重要性。
- 用户提出的关切包括触摸屏导致的分心和事故，强调了在驾驶过程中在数字和物理控制之间取得平衡的必要性。

## [英伟达限制其芯片之外的 CUDA 软件中的翻译层](https://www.tomshardware.com/pc-components/gpus/nvidia-bans-using-translation-layers-for-cuda-software-to-run-on-other-chips-new-restriction-apparently-targets-zluda-and-some-chinese-gpu-makers)

- Nvidia 更新了许可条款，禁止在非 Nvidia 硬件上运行基于 CUDA 的软件时使用翻译层，这是对 ZLUDA 和中国 GPU 制造商努力的回应。
- 此举旨在维护 Nvidia 在加速计算领域，尤其是人工智能应用领域的领先地位，允许对 CUDA 程序进行合法的重新编译，但鼓励移植 AMD 和英特尔等竞争对手的工具。
- 这一调整可能会促使 Nvidia 在 GPGPU 领域的优势地位发生变化，因为竞争对手会增强硬件，而软件开发商则会探索替代平台。

### [反应](https://news.ycombinator.com/item?id=39592689)

- 英伟达更新了其 CUDA 许可条款，禁止在非英伟达硬件上使用翻译层运行 CUDA 软件，这引发了不同地区关于互操作性和法律后果的争论。
- 讨论围绕 Nvidia 对 AI/ML 开发的影响、供应商锁定的挑战、竞争以及 WebGPU 等替代产品的出现展开，并引用了谷歌对甲骨文等法律案例。
- 人们对 Nvidia 的 CUDA 独占性、供应商锁定的担忧、与奴隶制的类比以及对 AMD 和英特尔提供竞争对手解决方案的猜测众说纷纭，讨论了从 CUDA 到 WebGPU 的潜在转变，并对 Nvidia 的做法进行了审查。
