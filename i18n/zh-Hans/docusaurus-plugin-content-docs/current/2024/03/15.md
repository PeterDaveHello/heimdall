## [成功扩展：Figma 的 Postgres 分片革命](https://www.figma.com/blog/how-figmas-databases-team-lived-to-tell-the-scale/)

- Figma 的数据库团队在其 Postgres 堆栈中成功实施了水平分片，实现了近乎无限的可扩展性，由于专业知识的原因，他们选择了 RDS Postgres 而不是 NoSQL 数据库，从而提高了可扩展性和可靠性，并将对产品逻辑的影响降至最低。
- 分片密钥和散列函数是高效路由数据的关键，通过创建新的 golang 服务 DBProxy 和未分片数据库的分片视图来简化操作，解决模式更新和跨分片事务等难题。
- 该小组计划对其当前的解决方案进行评估，并探索未来改进系统的替代方法。

### [反应](https://news.ycombinator.com/item?id=39706968)

- Figma 选择了水平分片，而不是迁移到 YugabyteDB 来扩展他们的表，并将成本和迁移风险作为坚持使用 RDS 的理由。
- 讨论涉及各种数据库管理策略，包括多租户、每个客户一个数据库以及使用 Citus 进行扩展，重点介绍了每种方法的复杂性和优势。
- 对话中还仔细探讨了依赖分布式数据库的初创企业的考虑因素以及供应商锁定的风险，如谷歌云Spanner。

## [FCC 将最低宽带速度提高到 100Mbps](https://www.pcmag.com/news/fcc-officially-raises-minimum-broadband-metric-from-25mbps-to-100mbps)

- 联邦通信委员会将最低宽带下载速度从 25Mbps 提高到 100Mbps，上传速度从 20Mbps 提高到 100Mbps，以更好地确定服务不足的地区并分配政府资金。
- 未来的计划包括将宽带指标提高到下载 1,000Mbps 和上传 500Mbps。
- 共和党委员提出异议，认为速度基准中应承认像 SpaceX 的 Starlink 这样提供高速连接的卫星互联网服务。

### [反应](https://news.ycombinator.com/item?id=39708957)

- 联邦通信委员会已将最低宽带速度要求提高到 100Mbps，从而引发了关于更高上载速度必要性和互联网技术发展的争论。
- 用户注意到供应商的改进，强调对称连接的重要性以及联邦通信委员会法规对宽带速度的影响。
- 讨论强调了在担心潜在成本增加和接入挑战的情况下，加快速度、建设可靠的基础设施以及为所有用户提供公平的互联网接入的重要性。

## [利用 Skyvern 的 LLM 和计算机视觉实现浏览器任务自动化](https://github.com/Skyvern-AI/skyvern)

- Skyvern 是一款开源工具，利用 LLM 和计算机视觉技术，通过自然语言 API 自动执行基于浏览器的工作流程，从而完成后台任务。
- 它通过利用 LLM 来分析网站布局，并在没有 XPaths 的情况下进行调整，从而应对工作流程可扩展性的挑战，目前支持采购和政府互动等任务。
- 该工具计划进军医疗电子病历数据录入自动化和客户研究领域，目前仍处于早期开发阶段，正在征求用户意见。

### [反应](https://news.ycombinator.com/item?id=39706004)

- Skyvern 是一款开源工具，利用大型语言模型 (LLM) 和计算机视觉技术实现基于浏览器的工作流程自动化，为采购和工作申请等后台任务提供自然语言应用程序接口。
- 讨论围绕利用 LLM 进行网络搜刮、自动化挑战（如成本和防止垃圾邮件）以及验证码服务的潜在漏洞展开。
- 未来的计划包括扩展到医疗数据录入，引入 LLM 路由器为用户提供帮助，以及创建自助式用户界面以改善客户数据库的可访问性。
