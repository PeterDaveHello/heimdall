## [探索《破碎像素地牢》：传统的 Roguelike](https://github.com/00-Evan/shattered-pixel-dungeon)

- Shattered Pixel Dungeon》是一款基于《像素地牢》（Pixel Dungeon）的传统 roguelike 地牢爬行游戏，开源并提供随机关卡、敌人和大量可收集物品。
- 游戏可在安卓、iOS 和桌面平台上运行，官方发布在游戏网站上，ShatteredPixel.com 的博客和 Transifex 上的翻译项目对其进行了补充。

### [反应](https://news.ycombinator.com/item?id=39773641)

- 讨论的重点是热门游戏《破碎像素地牢》，该游戏以其引人入胜的玩法和功能而著称，用户对其上瘾性和开源性赞不绝口。
- 用户对某些缺乏测试或深度的游戏提出了警告，并与其他游戏如《Nethack》和《Cogmind》分享了经验。
- 该游戏拥有一个专门的社区，允许通过 mod 进行定制，并可跨平台访问，一些用户选择使用 F-Droid 以增强安全性。

## [巴黎投资数十亿美元，通过公共住房促进经济多元化](https://www.nytimes.com/2024/03/17/realestate/paris-france-housing-costs.html)

## [通过平板游戏了解鹦鹉](https://news.northeastern.edu/2024/03/20/parrots-playing-tablet-games/)

- 东北大学的研究人员为鹦鹉制作了一款定制的平板电脑游戏，以研究它们与触摸屏的互动，发现它们在游戏过程中更喜欢使用舌头。
- 根据研究结果，我们引入了 "多点阈值 "来提高可用性，这表明触摸屏具有改善鹦鹉生活的潜力，并为宠物科技提供了宝贵的见解。
- 这项研究表明，量身定制的触摸屏技术不仅能使鹦鹉受益，还能使其他动物受益，为宠物科技的未来发展提供了广阔的前景。

### [反应](https://news.ycombinator.com/item?id=39768604)

- 讨论深入探讨了鹦鹉的智力和行为，以及为不同目的利用动物的伦理问题。
- 它质疑人类对动物智力的看法，强调物种间交流和理解的复杂性。
- 对话还涉及平板电脑等技术在研究鹦鹉和与鹦鹉互动方面的潜在用途，包括将鹦鹉作为宠物饲养所面临的挑战，以及为鹦鹉的利益优化 YouTube 等平台。

## [使用 lcl.host 的简易 HTTPS 设置确保本地开发安全](https://anchor.dev/blog/introducing-lcl-host)

- lcl.host 是一款专为网络开发人员设计的免费开发工具，可方便地在本地设置 HTTPS，以提供与生产设置相同的安全开发环境。

- 重点介绍了 Anchor.dev 对 Caddy 项目的支持、SSL 证书生成方法以及 Let's Encrypt 证书在个性化域名方面的优势。

## [GritQL：用 Rust CLI 彻底改变源代码转换](https://github.com/getgrit/gritql)

- GritQL 是一个开源的 Rust CLI，用于搜索和转换源代码，旨在将 grep 等工具的简单性与 codemod 程序的复杂性结合起来。
- 它允许轻松探索代码库，通过附带条件增加复杂性，并迅速迭代重要的代码迁移，以满足 Rust 的最高性能要求。
- 该项目正在寻求与开源社区的合作，并已被许多资源库用于复杂的迁移。

### [反应](https://news.ycombinator.com/item?id=39770908)

- GritQL 是一款 Rust CLI 工具，用于搜索和转换源代码，帮助进行复杂的迁移和重构，具有探索性分析、增量复杂性和命名模式重用等功能。
- GritQL 定位于 grep 和 codemod 工具之间，为大型代码库提供了性能和灵活性，可在多个软件源中使用，并作为开源工具鼓励社区协作。
- 虽然 GritQL 与 ast-grep 类似，但它有自己独特的功能，比如不要求输出代码语法正确，也不支持动态加载自定义语法，而是专注于简化转译过程。

## [错误修复：Btrfs 文件系统损坏已解决](https://tavianator.com/2024/btrfs_bug.html)

- Tavian Barnes 面临的 Linux 内核文件系统损坏错误与 Btrfs 有关，原因是并发线程访问同一范围缓冲区。
- 通过使用 git bisect 进行调试并在虚拟机中复制该问题，开发出了一种使用双重检查锁定来避免数据竞赛的解决方案。
- 问题源于 Btrfs 系统中未初始化的内存，导致块所有权和数据结构不一致，通过修复漏洞，消除了系统错误和数据损坏。

### [反应](https://news.ycombinator.com/item?id=39765715)

- 讨论将深入探讨 Btrfs 和 ZFS 文件系统的使用，包括错误查找、系统迁移、可靠性以及各种选项之间的比较。
- 用户就文件系统的性能、稳定性、部署和风险，特别是在 PostgreSQL 数据库等生产环境中的性能、稳定性、部署和风险，交流经验、关注点和偏好。
- 对文件系统的法律影响、兼容性、数据恢复和未来的潜在发展进行了讨论，强调了文件系统的挑战和优势。
