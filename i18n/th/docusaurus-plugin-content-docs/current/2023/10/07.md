- มีความต้องการที่ระบุสําหรับการสนับสนุนที่ดีขึ้นทรัพยากรที่ได้รับการปรับปรุงและประสิทธิภาพที่ดีขึ้นจาก AMD ผู้เข้าร่วมยังหารือเกี่ยวกับผลกระทบที่อาจเกิดขึ้นจากการแข่งขันด้านราคาและศักยภาพของผู้เข้าร่วมรายใหม่เช่น Intel ในตลาด GPU

## [การอัปเดตบุรุษไปรษณีย์จะลบเนื้อหาทั้งหมดของคุณหากคุณปฏิเสธที่จะสร้างบัญชี](https://news.ycombinator.com/item?id=37792690)

- ผู้ใช้รายงานว่าการใช้ Postman แบบออฟไลน์โดยไม่มีบัญชีส่งผลให้คอลเลกชันและการตั้งค่าทั้งหมดถูกลบเมื่อปฏิเสธที่จะสร้างบัญชีเมื่อได้รับแจ้ง
- แม้จะสร้างบัญชีในภายหลัง แต่ข้อมูลที่ถูกลบก็ไม่ได้กู้คืนแม้ว่าผู้ใช้จะสามารถกู้คืนได้ด้วยตนเองจากการสํารองข้อมูลในตําแหน่งเฉพาะ
- เหตุการณ์นี้นําไปสู่การสูญเสียความไว้วางใจใน Postman จากผู้ใช้ซึ่งตอนนี้กําลังมองหาเครื่องมือทางเลือกในการเปลี่ยนไปสู่

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37792690)

- การอัปเดตล่าสุดของ Postman ซึ่งเป็นเครื่องมือไคลเอ็นต์ API ได้จุดประกายความไม่พอใจของผู้ใช้เนื่องจากตอนนี้ต้องการสร้างบัญชีทําให้เกิดความกังวลเกี่ยวกับการสูญหายของข้อมูล
- ผู้ใช้กําลังสํารวจทางเลือกอื่นสําหรับ Postman ซึ่งรวมถึงเครื่องมือเช่นส่วนขยาย Insomnia, VSCode, HTTPie, Bruno, Kreya, Paw / Rapid API, Hopscotch และ IntelliJ HTTP คําขอ
- การแยกเกิดขึ้นในหมู่ผู้ใช้เหล่านี้ด้วยทางเลือกที่ชอบขดหรือบรรทัดคําสั่งในขณะที่คนอื่น ๆ โต้แย้งเพื่อความสะดวกของเครื่องมือ GUI เช่น Postman แม้จะมีความกังวลเกี่ยวกับราคาและความเข้ากันได้

## [23andMe กล่าวว่าข้อมูลผู้ใช้ถูกขโมยในการโจมตีการบรรจุข้อมูลประจําตัว](https://www.bleepingcomputer.com/news/security/genetics-firm-23andme-says-user-data-stolen-in-credential-stuffing-attack/)

- ข้อมูลที่ถูกบุกรุกประกอบด้วยชื่อที่สมบูรณ์ชื่อผู้ใช้ภาพโปรไฟล์ผลลัพธ์บรรพบุรุษทางพันธุกรรมและตําแหน่งทางภูมิศาสตร์ของผู้ใช้
- เพื่อป้องกันการโจมตีดังกล่าว บริษัท แนะนําให้ผู้ใช้ใช้การตรวจสอบสิทธิ์แบบสองปัจจัยและสร้างรหัสผ่านที่รัดกุมและไม่ซ้ําใครสําหรับบัญชีออนไลน์ของพวกเขา

## [เธรดต่อคอร์/แกน](https://without.boats/blog/thread-per-core/)

- ชุมชน Rust กําลังถกเถียงการใช้งานเริ่มต้นของผู้บริหารแบบมัลติเธรดในการเขียนโปรแกรมแบบอะซิงโครนัส โดยบางคนสนับสนุนสถาปัตยกรรม "เธรดต่อคอร์"
- บทความนี้เจาะลึกแนวคิดสถาปัตยกรรม "การขโมยงาน" และ "แบ่งปันอะไรเลย" โดยสํารวจผลกระทบต่อประสิทธิภาพของระบบ มันอ้างอิงถึงกระดาษของ Pekka Enberg ที่แสดงให้เห็นถึงข้อดีของสถาปัตยกรรมแบบแบ่งปัน
- ผู้เขียนตั้งข้อสงสัยเกี่ยวกับความเป็นไปได้และความเรียบง่ายของการใช้สถาปัตยกรรมของ Enberg เทียบกับวิธีการแบบดั้งเดิมโดยสรุปว่าการขโมยงานอาจเป็นประโยชน์ต่อระบบที่มีสถานะร่วมกัน

- การละเมิดข้อมูลเกิดจากการโจมตีการบรรจุข้อมูลประจําตัวซึ่งเป็นการโจมตีทางไซเบอร์ประเภทหนึ่งที่ใช้ข้อมูลประจําตัวของบัญชีที่ถูกขโมยเพื่อเข้าถึงบัญชีผู้ใช้โดยไม่ได้รับอนุญาต
- รายละเอียดเพิ่มเติมและการอภิปรายเกี่ยวกับเหตุการณ์มีอยู่ในแพลตฟอร์ม Hacker News
