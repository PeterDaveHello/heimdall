## [การนํา HTTP / 3 มาใช้กําลังเติบโตอย่างรวดเร็ว](https://blog.apnic.net/2023/09/25/why-http-3-is-eating-the-world/)

- HTTP/3 ซึ่งเป็นเวอร์ชันใหม่ของ Hypertext Transfer Protocol (HTTP) ได้ถูกรวมเข้ากับเว็บสาธารณะอย่างรวดเร็วหลังจากได้รับมาตรฐานโดย Internet Engineering Task Force (IETF)
- HTTP / 3 พร้อมกับโปรโตคอล QUIC ที่เกี่ยวข้องถูกใช้อย่างกว้างขวางโดย บริษัท ใหญ่ ๆ เช่น Google และ Meta และแทนที่ Transport Control Protocol (TCP) ด้วย QUIC ซึ่งนําเสนอการเข้ารหัสที่ดีขึ้นและการปรับปรุงประสิทธิภาพ
- โปรโตคอลใหม่ให้ความเร็วที่ดีขึ้นสําหรับการโหลดหน้าเว็บและการสตรีมวิดีโอซึ่งเอื้อต่อการยอมรับอย่างรวดเร็วในอุตสาหกรรม

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37777050)

- การอภิปรายหมุนรอบหลายหัวข้อเช่นการใช้โปรโตคอล QUIC เพื่อนําทางข้อ จํากัด ทางอินเทอร์เน็ตในประเทศจีนประโยชน์และอุปสรรคของโปรโตคอล QUIC และการใช้ QoS ใน QUIC
- ความไม่พอใจขององค์กรกับบริการรักษาความปลอดภัยเครือข่าย Zscaler การใช้การสกัดกั้น SSL ภายในเครือข่ายขององค์กรและกลยุทธ์การบรรจุวิดีโอและการเข้ารหัสแต่ละเป็นส่วนหนึ่งของการสนทนา
- ในที่สุดการอภิปรายไตร่ตรองถึงความจําเป็นสําหรับ QUIC API มาตรฐานและประโยชน์ของการใช้ mmap () มากกว่า read() สําหรับการจัดการชุดข้อมูลขนาดใหญ่

## [Bitmagnet: ตัวจัดทําดัชนี BitTorrent ที่โฮสต์ด้วยตนเองโปรแกรมรวบรวมข้อมูล DHT และการค้นหาฝนตกหนัก](https://bitmagnet.io/)

- Bitmagnet เป็นตัวจัดทําดัชนี BitTorrent ที่โฮสต์เองพร้อมคุณสมบัติที่เป็นเอกลักษณ์เช่นโปรแกรมรวบรวมข้อมูล Distributed Hash Table (DHT) ตัวจําแนกเนื้อหาและเครื่องมือค้นหาฝนตกหนัก สิ่งเหล่านี้ช่วยให้ผู้ใช้สามารถค้นหา torrents ได้โดยไม่ต้องพึ่งพาเอนทิตีภายนอก
- โครงการยังอยู่ในช่วงอัลฟ่า แต่มีคุณสมบัติที่มีแนวโน้มหลายอย่างเช่นตัวทําดัชนี BitTorrent ทั่วไปตัวจําแนกเนื้อหาเครื่องมือค้นหาฝนตกหนักและ GraphQL API ซึ่งเป็นเทคโนโลยีที่ทําให้ง่ายต่อการรับข้อมูลจากเซิร์ฟเวอร์ไปยังไคลเอนต์
- ทีมงานกําลังทํางานเกี่ยวกับคุณสมบัติที่มีลําดับความสําคัญสูงที่ยังไม่ได้ดําเนินการเช่นตัวจําแนกสําหรับประเภทเนื้อหาอื่น ๆ การเรียงลําดับผลการค้นหาการเพิ่มประสิทธิภาพการค้นหาและคุณสมบัติที่มีประโยชน์อื่น ๆ เช่นการรับรองความถูกต้องการค้นหาที่บันทึกไว้และการสนับสนุนโปรโตคอล BitTorrent v2 เปิดให้รองรับ GitHub

## [ท่อ Linux เร็วแค่ไหน? (2022)](https://mazzo.li/posts/fast-pipes.html)

- บทความนี้อธิบายเกี่ยวกับการทํางานของท่อ Unix ใน Linux โดยเน้นที่การเพิ่มประสิทธิภาพโปรแกรมทดสอบสําหรับการเขียนข้อมูลและการอ่านผ่านท่อ
- มันเจาะลึกการใช้เทคนิคเช่น vmsplice และประกบกันเพื่อเพิ่มปริมาณงานโดยการลดการคัดลอกข้อมูลและการว่าจ้างเครื่องมือ perf สําหรับการวิเคราะห์ประสิทธิภาพ
- นอกจากนี้ยังกล่าวถึงบทบาทของเพจหน่วยความจําเสมือนในการถ่ายโอนข้อมูลการแปลเสมือนไปยังที่อยู่ทางกายภาพและการประยุกต์ใช้หน้าขนาดใหญ่เพื่อลด TLB (Translation Lookaside Buffer) พลาด

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37782493)

- บทความนี้ให้รายละเอียดเกี่ยวกับการใช้งานและประสิทธิภาพของท่อ Linux โดยเน้นถึงประโยชน์ที่อาจเกิดขึ้นของ vmsplice ซึ่งเป็นกลไกหน่วยความจําที่ใช้ร่วมกันแม้จะมีความท้าทายในการเพิ่มประสิทธิภาพเวอร์ชันต่างๆ
- มันเจาะลึกเข้าไปในไลบรารี API สําหรับการจัดการข้อมูลและการเพิ่มประสิทธิภาพพฤติกรรมที่คาดเดาไม่ได้ของท่อ Linux และกล่าวถึงผลกระทบของท่อแตก
- วิธีการถ่ายโอนข้อมูลทางเลือกเช่นหน่วยความจําที่ใช้ร่วมกันหรือคิวข้อความและเทคนิคการเพิ่มประสิทธิภาพเช่น io_uring และ mmap จะถูกสํารวจ นอกจากนี้ยังกล่าวถึงการประยุกต์ใช้ท่อในการเขียนสคริปต์เชลล์สําหรับงานที่มีประสิทธิภาพสูงเช่นเดียวกับการแลกเปลี่ยนและผลลัพธ์ประสิทธิภาพเมื่อเลือกระหว่างเธรดและท่อ

## [NIST Elliptic Curves เมล็ด Bounty](https://words.filippo.io/dispatches/seeds-bounty/)
