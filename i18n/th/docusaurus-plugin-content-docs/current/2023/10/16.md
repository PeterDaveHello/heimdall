# 2023-10-16

## [มาสเตอร์การ์ดควรหยุดขายข้อมูลของเรา](https://www.eff.org/deeplinks/2023/10/mastercard-should-stop-selling-our-data)

- มูลนิธิ Electronic Frontier Foundation (EFF) ได้ร่วมมือกับกลุ่มวิจัยผลประโยชน์สาธารณะของสหรัฐอเมริกา (U.S. PIRG) ในการรณรงค์เรียกร้องให้มาสเตอร์การ์ดจํากัดการรวบรวมข้อมูลและยุติการขายข้อมูลผู้ถือบัตร
- มีรายงานว่ากลยุทธ์การสร้างรายได้ในปัจจุบันของมาสเตอร์การ์ดซึ่งเกี่ยวข้องกับการวิเคราะห์ข้อมูลธุรกรรมเพื่อจัดหมวดหมู่ผู้ถือบัตรเป็นการละเมิดความเป็นส่วนตัวและความไว้วางใจของผู้บริโภค
- EFF กําลังกดดันให้มาสเตอร์การ์ดเปลี่ยนแปลงแนวทางปฏิบัติด้านข้อมูลเหล่านี้เพื่อรักษาความไว้วางใจของผู้ถือบัตรและเคารพความเป็นส่วนตัวของพวกเขา

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37892684)

- มีการชั่งน้ําหนักข้อดีและข้อเสียของการใช้บัตรเครดิตรวมถึงบัตรเฉพาะเช่น Amex Platinum Card โดยเน้นที่ความวิตกด้านความเป็นส่วนตัวที่เกี่ยวข้องกับการรวบรวมและปรับใช้ข้อมูลโดย บริษัท ต่างๆ

## [ฟินแลนด์จะลงคะแนนเสียงต่อต้านคําสั่งห้ามการเฝ้าระวังและการเข้ารหัสจํานวนมากของสหภาพยุโรป](https://dawn.fi/uutiset/2023/10/14/eu-csam-suomi-eduskunta-kanta)

- พวกเขาย้ําถึงการสนับสนุนการคุ้มครองเด็กทางออนไลน์ แต่โต้แย้งกับการประนีประนอมใด ๆ เกี่ยวกับการเข้ารหัสและการเฝ้าระวังจํานวนมากโดยสนับสนุนให้มุ่งเน้นไปที่บุคคลที่ระบุอย่างชัดเจนแทน
- การตัดสินใจในประเด็นที่ถกเถียงกันนี้จากประเทศสมาชิกของสหภาพยุโรปคาดว่าจะเกิดขึ้นในเร็ว ๆ นี้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37891886)

- การอภิปรายทําให้เกิดความกังวลเกี่ยวกับการบังคับใช้คําสั่งห้ามการสอดแนมประชาชนทั่วไปที่อาจเกิดขึ้นและอภิปรายถึงผลกระทบของการใช้ชีวิตในโลกที่โปร่งใสอย่างสมบูรณ์
- มีวาทกรรมเกี่ยวกับบทบาทที่สําคัญของการเข้ารหัสในการรักษาความเป็นส่วนตัวลักษณะที่ขัดต่อรัฐธรรมนูญที่อาจเกิดขึ้นของกฎหมายและความคิดเห็นที่แตกต่างกันจากประเทศต่างๆ นอกจากนี้ยังให้ความสนใจกับการขาดความตระหนักที่เป็นที่นิยมเกี่ยวกับโครงสร้างและกระบวนการตัดสินใจของสหภาพยุโรป

## [Lotus 1-2-3 สําหรับ Linux](https://lock.cmpxchg8b.com/linux123.html)

- ผู้เขียนจัดการพอร์ต Lotus 1-2-3 ซึ่งเป็นซอฟต์แวร์วินเทจไปยัง Linux โดยการกู้คืนไฟล์ที่สูญหายและใช้แฮ็กต่างๆ
- โฟกัสให้กับข้อกังวลด้านความเข้ากันได้ซึ่งเกี่ยวข้องกับการเรียกระบบและการแปลงไฟล์ออบเจ็กต์ควบคู่ไปกับการลบสัญลักษณ์ออกจากไฟล์ออบเจ็กต์และการเขียน wrappers สําหรับฟังก์ชันที่เข้ากันไม่ได้
- มีการกล่าวถึงการต่อสู้บางอย่างเกี่ยวกับ termios และการออกใบอนุญาต แต่โดยรวมแล้วผู้เขียนถือว่าการเปลี่ยนไปใช้ Linux ประสบความสําเร็จ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37889666)

- บทความนี้ประกาศความพร้อมใช้งานของโปรแกรมสเปรดชีต Lotus 1-2-3 สําหรับ Linux โดยเน้นย้ําถึงความสําคัญในการรักษาข้อมูล
- การสนทนายังหมุนรอบความคิดถึงสําหรับฉาก BBS (Bulletin Board System) ที่ผ่านมาโดยเน้นการฟื้นคืนชีพของจริยธรรมและวัฒนธรรมย้อนยุค 80/90
- ปัญหาเกี่ยวกับการละเมิดลิขสิทธิ์ซอฟต์แวร์การเก็บถาวรของคู่มือซินธิไซเซอร์เกร็ดเล็กเกร็ดน้อยส่วนตัวการโฆษณาที่ตรงเป้าหมายและการจัดการที่อาจเกิดขึ้นโดย บริษัท เทคโนโลยียังกล่าวถึงในบทความ

## [Signtime.apple: ล่ามภาษามือตัวต่อตัวโดย Apple](https://www.signtime.apple/applecare/us-EN/asl)

- 'SignTime' ซึ่งเป็นบริการล่ามภาษามือแบบตัวต่อตัวสําหรับการช็อปปิ้งและการสนับสนุนเซสชันไม่สามารถใช้งานได้ในขณะนี้
- ขอแนะนําให้ผู้ใช้ลองเข้าถึงบริการอีกครั้งในภายหลัง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37890176)

- Apple ได้เปิดตัวบริการใหม่ที่มีชื่อว่า Signtime.apple ซึ่งออกแบบมาเพื่อนําเสนอล่ามภาษามือส่วนบุคคลสําหรับการสนับสนุนลูกค้า โดยเฉพาะสําหรับลูกค้าที่ใช้ภาษามืออเมริกัน (ASL)
- บริการนี้ช่วยให้ลูกค้าที่หูหนวกหรือมีปัญหาทางการได้ยินสามารถโต้ตอบกับตัวแทนของ Apple โดยใช้ ASL ผ่านวิดีโอแชท
- มีความกังวลเกี่ยวกับการโทรสแปมที่อาจเกิดขึ้นเส้นทางการตรวจสอบและการละเมิดพร้อมกับความสับสนเกี่ยวกับชื่อโดเมนของ Apple และผลกระทบด้านความปลอดภัย ถึงกระนั้นความมุ่งมั่นของ Apple ในด้านคุณภาพและประสบการณ์ของลูกค้าก็ได้รับการยอมรับในเชิงบวก

## [การตรวจสอบ SSH: การตรวจสอบความปลอดภัยของเซิร์ฟเวอร์ SSH และไคลเอ็นต์](https://github.com/jtesta/ssh-audit)

- พื้นที่เก็บข้อมูล jtesta/ssh-audit ได้รับการอัปเดตด้วยการปรับปรุงสําหรับการตรวจสอบเซิร์ฟเวอร์ SSH และการกําหนดค่าไคลเอ็นต์ ซึ่งขณะนี้รองรับ Python 3.7-3.11
- การอัปเดตรวมถึงคําแนะนําข้อมูลความปลอดภัยแพ็คเกจที่สร้างไว้ล่วงหน้าส่วนหน้าของเว็บนโยบายใหม่การแก้ไขข้อบกพร่องและความเข้ากันได้ที่เพิ่มขึ้นใน Ubuntu 22.04 LTS
- นอกจากนี้ยังมีคีย์โฮสต์แบบขยายการแลกเปลี่ยนคีย์การเข้ารหัส MACs การตรวจสอบซอฟต์แวร์ไคลเอนต์และตัวเลือกเอาต์พุต JSON เพื่อความคล่องตัวมากขึ้น

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37892028)

- การสนทนามุ่งเน้นไปที่ความสําคัญของการรักษาความปลอดภัยเซิร์ฟเวอร์ SSH และไคลเอนต์โดยเน้นปัญหาเกี่ยวกับความปลอดภัยของการตั้งค่าเริ่มต้น
- การใช้อัลกอริธึมที่แตกต่างกันและความน่าเชื่อถือของคีย์ Ed25519 ถูกนํามาอภิปราย
- มีความเห็นที่แตกต่างกันเกี่ยวกับความปลอดภัยของ Trust on First Use (TOFU) ในโปรโตคอลการเข้ารหัสและจําเป็นต้องมีขั้นตอนการตรวจสอบเพิ่มเติมหรือไม่

## [Omnivore - ฟรีโอเพ่นซอร์สอ่านมันในภายหลัง App](https://omnivore.app/)

- Login Omnivore เป็นแอปโอเพ่นซอร์สแบบอ่านมันในภายหลังที่มีไว้สําหรับผู้อ่านที่กว้างขวางซึ่งช่วยให้สามารถบันทึกเนื้อหาประเภทต่างๆสําหรับการอ่านในภายหลังโดยปราศจากสิ่งรบกวน
- ผู้ใช้ไม่เพียง แต่สามารถเพิ่มบันทึกย่อและไฮไลต์ แต่ยังจัดระเบียบและซิงค์รายการอ่านของพวกเขาในทุกอุปกรณ์
- แอพนี้มีฟังก์ชันการแปลงข้อความเป็นคําพูดและการรวมเข้ากับระบบการจัดการความรู้ส่วนบุคคล (PKM) ยอดนิยมทําให้ผู้ใช้สามารถควบคุมไลบรารีการอ่านได้อย่างสมบูรณ์

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37890742)

- Omnivore เป็นแอปอ่านมันในภายหลังแบบโอเพ่นซอร์สฟรีที่เชื่อมโยงกับ RSS และจดหมายข่าวทางอีเมล แต่ผู้ใช้ได้แสดงความกังวลเกี่ยวกับกลยุทธ์การสร้างรายได้ในอนาคตและการขาดคุณสมบัติบางอย่าง
- ปัจจุบันแอปพลิเคชันรองรับเฉพาะ iOS เท่านั้น แต่นักพัฒนามีแผนที่จะเปิดตัวเวอร์ชัน Windows คุณสมบัติที่เป็นเอกลักษณ์ ได้แก่ ความสามารถในการส่งออกไฮไลท์ใช้ปลั๊กอินสําหรับ Obsidian และ Notion และเซิร์ฟเวอร์โฮสต์ด้วยตนเอง
- Omnivore แข่งขันกับแอพเช่น Pocket, Raindrop.io, Instapaper และ Readwise แม้ว่าผู้ใช้จะพบว่าคําแนะนําการโฮสต์ด้วยตนเองมีความซับซ้อนเมื่อเทียบกับคู่แข่งและรูปแบบการระดมทุนยังคงคลุมเครือ

## [ทําไมต้อง Async Rust?](https://without.boats/blog/why-async-rust/)

- บทความนี้ตรวจสอบการประยุกต์ใช้และวิวัฒนาการของกลไกการทํางานพร้อมกันในภาษาการเขียนโปรแกรม Rust โดยเฉพาะอย่างยิ่งการมุ่งเน้นไปที่ไวยากรณ์ async / await

- ด้วยคําอธิบายของการใช้ Rust ที่ประสบความสําเร็จในบริการเครือข่ายผู้เขียนยังแสดงความกังวลเกี่ยวกับระบบนิเวศของ Rust ที่พึ่งพา async / waiting มากเกินไปและเสนอการปรับปรุงที่อาจเกิดขึ้น

- บทความนี้เน้นถึงภัยคุกคามในการยุติการเข้าถึงอินเทอร์เน็ตเนื่องจากการอ้างสิทธิ์การละเมิดลิขสิทธิ์และความจําเป็นในการเป็นกลางทางเน็ตเพื่อรักษาสภาพแวดล้อมออนไลน์ที่เป็นธรรม
- มันนําความสนใจไปที่บทบาทที่ถกเถียงกันของผู้ให้บริการอินเทอร์เน็ต (ISP) ในการตรวจสอบการถ่ายโอนข้อมูลที่เกี่ยวข้องกับกฎหมายลิขสิทธิ์และทรัพย์สินทางปัญญา
- งานชิ้นนี้เรียกร้องให้มีการประเมินนโยบายลิขสิทธิ์ที่มีอยู่ใหม่ซึ่งเหมาะสมกับยุคดิจิทัล พร้อมกับการอภิปรายเกี่ยวกับจริยธรรมของการละเมิดลิขสิทธิ์และผลกระทบต่อผู้สร้างเนื้อหา
