- ผู้เขียนได้สร้างทางแยกของ Emacs ของตัวเองที่เรียกว่าสาขา "หลัก" ซึ่งพวกเขาได้ย้อนกลับการเปลี่ยนแปลงที่มีปัญหาและทําการปรับปรุงของตนเอง พวกเขากําลังสนับสนุนให้ผู้อื่นเข้าร่วมและมีส่วนร่วมในสาขาของพวกเขา

- ผู้ใช้กําลังหารือว่าควรเปลี่ยนลักษณะการทํางานเริ่มต้นหรือหากลักษณะการทํางานใหม่ควรเป็นทางเลือกสําหรับผู้ใช้ที่จะเลือก
- การอภิปรายเจาะลึกกระบวนการพัฒนาคุณค่าของความคิดเห็นของผู้ใช้และพฤติกรรมของบุคคลบางคนพร้อมข้อเสนอแนะรวมถึงการปลอมซอฟต์แวร์หรือหาทางประนีประนอมเพื่อจัดการกับข้อกังวล

## [AST-grep: เครื่องมือ CLI สําหรับการค้นหาโครงสร้างโค้ด ซับใน และการเขียนใหม่](https://github.com/ast-grep/ast-grep)

- AST-grep เป็นเครื่องมือบรรทัดคําสั่งที่ช่วยให้สามารถค้นหาโครงสร้างโค้ดซับและเขียนใหม่โดยใช้ประโยชน์จากแผนผังไวยากรณ์นามธรรม (AST)
- มันทํางานคล้ายกับ grep แต่แทนที่จะจับคู่ข้อความจะตรงกับโหนด AST
- ผู้ใช้สามารถเขียนรูปแบบเหมือนโค้ดเพื่อจับคู่โค้ดที่มีโครงสร้างทางไวยากรณ์เดียวกัน
- AST-grep ให้การจัดการโค้ดขั้นสูงการจับคู่รูปแบบที่ใช้งานง่ายและ API ที่เหมือน jQuery สําหรับการเดินทาง AST
- สามารถติดตั้งผ่านตัวจัดการแพ็คเกจหรือสร้างขึ้นจากแหล่งที่มา
- เครื่องมือนี้มีจุดมุ่งหมายเพื่อลดความซับซ้อนของการเขียนโปรแกรม AST และมีประโยชน์อย่างยิ่งสําหรับผู้เขียนไลบรารีโอเพ่นซอร์สผู้นําด้านเทคโนโลยีและนักวิจัยด้านความปลอดภัย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38590984)

- Ast-grep, grep-ast และผู้ค้นหาไวยากรณ์ถูกเน้นเป็นเครื่องมือที่มีประสิทธิภาพสําหรับการค้นหาโครงสร้างโค้ดการซับและการเขียนใหม่โดยนําเสนอฟังก์ชันในการค้นหาและแสดงรหัสที่ตรงกัน
- มีความกังวลเกี่ยวกับเอกสารและความซับซ้อนของเครื่องมือเหล่านี้ซึ่งบ่งบอกถึงพื้นที่ที่สามารถปรับปรุงได้
- เครื่องมืออื่น ๆ เช่น semgrep, Conscat และ OpenRewrite ถูกกล่าวถึงเป็นตัวเลือกสําหรับการปรับโครงสร้างโค้ดและระบุความคล้ายคลึงกันทางความหมายขยายช่วงของเครื่องมือที่มีอยู่สําหรับนักพัฒนา
- ข้อ จํากัด ของการสร้างแบบจําลองภาษาและความท้าทายในการกําหนดความเท่าเทียมกันทางความหมายได้รับการยอมรับโดยเน้นความซับซ้อนของงานเหล่านี้
- มีการหารือเกี่ยวกับการปรับปรุงและการปรับปรุงที่เป็นไปได้สําหรับเครื่องมือเหล่านี้ซึ่งแสดงให้เห็นถึงความมุ่งมั่นในการพัฒนาและนวัตกรรมอย่างต่อเนื่องในด้านการวิเคราะห์และปรับแต่งโค้ด

- Krasue เป็นโทรจันการเข้าถึงระยะไกลที่ช่วยให้ผู้โจมตีสามารถควบคุมเครือข่ายเป้าหมายจากระยะไกล
- Krasue ใช้รูทคิทต่างๆ เพื่อซ่อนกิจกรรมของมัน ปลอมตัว "alive pings" เป็นข้อความ RTSP และมีรูทคิทที่สกัดกั้น kill() syscall เพื่อเอาชีวิตรอดจากความพยายามในการปิดระบบ เวกเตอร์การติดตั้งที่เป็นไปได้รวมถึงการใช้ประโยชน์จากช่องโหว่การโจรกรรมข้อมูลประจําตัวหรือการอัปเดตซอฟต์แวร์โทรจัน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38590827)

- การสนทนากล่าวถึงหัวข้อต่าง ๆ ที่เกี่ยวข้องกับความปลอดภัยของ Linux รวมถึงการค้นพบรูทคิท Krasue และความกังวลเกี่ยวกับช่องโหว่ของระบบ Linux
- มันสํารวจประสิทธิภาพของซอฟต์แวร์ป้องกันไวรัสและความจําเป็นในการเพิ่มมาตรการรักษาความปลอดภัยบนเดสก์ท็อป Linux
- การสนทนายังตรวจสอบการใช้งานระบบสิทธิ์รันไทม์ที่อาจเกิดขึ้นบน Linux และเปรียบเทียบคุณสมบัติด้านความปลอดภัยของ Linux กับระบบปฏิบัติการอื่น ๆ

## [ปรับปรุงการเข้าถึงเว็บ: ส่งเสริมการรวม RSS สําหรับเว็บไซต์](https://rknight.me/please-expose-your-rss/)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38595855)

- ขาดการสนับสนุนแบบเนทีฟสําหรับการโฆษณาในฟีด RSS ทําให้เป็นตัวเลือกที่ต้องการสําหรับผู้ที่ชอบประสบการณ์ที่ไม่มีโฆษณา
- ความไม่สอดคล้องกันของเนื้อหาในฟีด RSS ซึ่งอาจแตกต่างกันไปในรูปแบบและคุณภาพ
- ข้อดีของการใช้โปรแกรมอ่าน RSS เพื่อความสะดวกและการรวมเนื้อหาส่วนบุคคล
- การอภิปรายเกี่ยวกับการขูดเนื้อหาโดยทางโปรแกรมการขายข้อมูลจากฟีด RSS และการเปิดเผยจํานวนสมาชิก
- คําแนะนําในการขยาย RSS เพื่อรวมโพสต์ในฟอรัมและรวมไซต์แบบคงที่เข้ากับฟีด fediverse
- การอภิปรายเกี่ยวกับการออกแบบเว็บไซต์การตั้งค่าและการบํารุงรักษาฟีด RSS และการเปรียบเทียบระหว่างผู้อ่าน RSS และผู้รวบรวมทางสังคม
- การปรากฏตัวของโฆษณาในฟีด RSS เป็นหัวข้อของการอภิปราย

## [การปรับปรุงประสิทธิภาพอินเทอร์เน็ต: ขอแนะนําสถาปัตยกรรม L4S สําหรับเวลาแฝงต่ํา การสูญเสียต่ํา และปริมาณงานที่ปรับขนาดได้](https://datatracker.ietf.org/doc/rfc9330/)

- RFC 9330 เปิดตัวสถาปัตยกรรม L4S ซึ่งมีจุดมุ่งหมายเพื่อปรับปรุงแอปพลิเคชันอินเทอร์เน็ตโดยลดความล่าช้าในการเข้าคิวและเปิดใช้งานเวลาแฝงต่ําและปริมาณงานที่ปรับขนาดได้
- สถาปัตยกรรม L4S เสนอกลไกการควบคุมความแออัดใหม่ที่ทํางานร่วมกับการควบคุมที่มีอยู่และใช้ Explicit Congestion Notification (ECN) เวอร์ชันแก้ไข
- เป้าหมายคือการให้บริการอินเทอร์เน็ตคุณภาพสูงพร้อมประสิทธิภาพที่ดีขึ้นสําหรับแอปพลิเคชันแบบโต้ตอบเช่นการเล่นเกมการสตรีมและการประชุมทางวิดีโอ
- สถาปัตยกรรมสามารถปรับใช้ได้ทีละน้อยและเข้ากันได้กับเครือข่ายประเภทต่างๆ
- ข้อความประกอบด้วยการอภิปรายเกี่ยวกับข้อควรพิจารณาในการปรับใช้การควบคุมการจราจรและข้อควรพิจารณาด้านความปลอดภัย
- มันเน้นความสําคัญของข้อเสนอแนะ ECN ที่ถูกต้องและความสามารถในการปรับขนาดในอัลกอริธึมการควบคุมความแออัด
- เอกสารนี้ให้การอ้างอิงถึงทรัพยากรเพิ่มเติมเกี่ยวกับการควบคุมความแออัดโปรโตคอลเครือข่ายและบริการที่แตกต่าง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38597744)

- บริการอินเทอร์เน็ต Low Latency, Low Loss และ Scalable Throughput (L4S) มีจุดมุ่งหมายเพื่อลดเวลาแฝงและปรับปรุงประสิทธิภาพสําหรับแอปพลิเคชันเช่นแฮงเอาท์วิดีโอ
- L4S ช่วยลดอาการบวมของบัฟเฟอร์และปรับปรุงการส่งสัญญาณความแออัดเพื่อบรรเทาความล่าช้าและการหยุดชะงักในการส่งข้อมูล
- L4S สามารถใช้งานได้โดยโปรโตคอลการขนส่งต่างๆ รวมถึง TCP และ QUIC และมีศักยภาพในการปรับปรุงประสิทธิภาพของเครือข่ายเมื่อนํามาใช้โดย ISP

## [ส่วนผสมเฉื่อยในสารกําจัดศัตรูพืชอาจเป็นภัยคุกคามต่อผึ้งมากขึ้น](https://theconversation.com/inert-ingredients-in-pesticides-may-be-more-toxic-to-bees-than-scientists-thought-218005)

- ส่วนผสมเฉื่อยในสารกําจัดศัตรูพืชซึ่งก่อนหน้านี้ถือว่าไม่เป็นพิษอาจเป็นอันตรายต่อผึ้งมากกว่าที่คิดตามการวิจัยใหม่
- การศึกษาทําให้ผึ้งสัมผัสกับยาฆ่าเชื้อราที่มีส่วนผสมทั้งออกฤทธิ์และเฉื่อยและพบว่าสูตรทั้งหมดทําให้ความจําของผึ้งลดลงในขณะที่ส่วนผสมที่ออกฤทธิ์เพียงอย่างเดียวไม่ได้
- ส่วนผสมเฉื่อยมีหน้าที่แตกต่างกันและถูกควบคุมแตกต่างจากสารออกฤทธิ์ทําให้ยากที่จะตรวจสอบความเป็นพิษ จําเป็นต้องมีการทดสอบสูตรสารกําจัดศัตรูพืชอย่างครอบคลุมเพื่อทําความเข้าใจผลกระทบต่อสุขภาพของผึ้ง การเปลี่ยนแปลงกฎระเบียบของสารกําจัดศัตรูพืชสามารถช่วยปกป้องผึ้งโดยการระบุส่วนผสมเฉื่อยบนฉลากและต้องมีการทดสอบเพิ่มเติมเกี่ยวกับผลกระทบระยะยาว

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38593008)

- ส่วนผสม "เฉื่อย" ในสารกําจัดศัตรูพืชซึ่งโดยทั่วไปถือว่าไม่เป็นพิษอาจเป็นอันตรายต่อผึ้งมากกว่าที่เชื่อกันก่อนหน้านี้
- การศึกษาพบว่าสูตรที่สมบูรณ์ของยาฆ่าเชื้อราทําให้ความจําของผึ้งลดลงในขณะที่สารออกฤทธิ์เพียงอย่างเดียวไม่ได้
- การใช้สารกําจัดศัตรูพืชมีผลเสียโดยตรงต่อผึ้งและส่งผลทางอ้อมต่อแหล่งอาหารของพวกเขาทําให้มีความเสี่ยงต่อปรสิตและโรค กฎระเบียบและความโปร่งใสเกี่ยวกับส่วนผสมเฉื่อยในสารกําจัดศัตรูพืชทําให้เกิดความกังวลเกี่ยวกับอันตรายที่อาจเกิดขึ้น
