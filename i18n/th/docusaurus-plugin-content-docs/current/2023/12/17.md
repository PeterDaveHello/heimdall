## [ขอแนะนํา SSH3: ประสบการณ์เชลล์ที่รวดเร็วและปลอดภัยยิ่งขึ้นด้วย HTTP/3 และ QUIC](https://github.com/francoismichel/ssh3)

- SSH3 เป็นโปรโตคอล SSH เวอร์ชันแก้ไขที่ใช้กลไก HTTP เพื่อมอบประสบการณ์เชลล์ที่รวดเร็วและปลอดภัยยิ่งขึ้น
- ใช้ QUIC + TLS1.3 สําหรับการสร้างช่องทางที่ปลอดภัยและการอนุญาต HTTP สําหรับการรับรองความถูกต้องของผู้ใช้
- SSH3 ให้การสร้างเซสชันที่เร็วขึ้นวิธีการรับรองความถูกต้องที่ได้รับการปรับปรุง (OAuth2.0, OpenID Connect) ความทนทานต่อการโจมตีการสแกนพอร์ตการส่งต่อพอร์ต UDP และคุณสมบัติของโปรโตคอล QUIC ที่ทันสมัย
- อนุญาตให้ซ่อนเซิร์ฟเวอร์ SSH และรองรับใบรับรอง X.509 สําหรับการรับรองความถูกต้อง
- SSH3 กําลังทดลองและสามารถติดตั้งได้โดยการดาวน์โหลดไบนารีหรือรวบรวมโค้ดจากแหล่งที่มา
- เซิร์ฟเวอร์ SSH3 จําเป็นต้องปรับใช้เพื่อใช้ SSH3 และไคลเอ็นต์ SSH3 รองรับวิธีการรับรองความถูกต้องต่างๆ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38664729)

- เธรดความคิดเห็นครอบคลุมหัวข้อต่างๆ ที่เกี่ยวข้องกับ SSH เช่น ความเข้ากันได้กับโปรโตคอล เช่น HTTP/3 และ QUIC การใช้ใบรับรองสําหรับการรับรองความถูกต้อง และศักยภาพสําหรับ SSH เวอร์ชันใหม่
- นอกจากนี้ยังมีการกล่าวถึงโปรโตคอลทางเลือกเช่น Mosh
- การอภิปรายประกอบด้วยมุมมองและข้อกังวลที่หลากหลายเกี่ยวกับความปลอดภัยประสิทธิภาพการใช้งานการเซ็นเซอร์เครือข่ายการซ่อนโปรโตคอลและการจัดการอุปกรณ์

## [พบช่องโหว่ด้านความปลอดภัยหลายอย่างใน MongoDB; แนะนําการอัปเดต](https://www.mongodb.com/alerts)

- มีการระบุช่องโหว่ด้านความปลอดภัยหลายรายการใน MongoDB เวอร์ชันต่างๆ และส่วนประกอบที่เกี่ยวข้อง
- ช่องโหว่เหล่านี้รวมถึงการเปิดเผยข้อมูลการโจมตีแบบปฏิเสธการให้บริการและการยกระดับสิทธิ์
- ขอแนะนําให้อัปเดตเป็นเวอร์ชันล่าสุดเพื่อลดความเสี่ยงที่เกี่ยวข้องกับช่องโหว่เหล่านี้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38667596)

- MongoDB Atlas เป็นหัวข้อของการอภิปรายและการอภิปรายโดยมีการวิพากษ์วิจารณ์เกี่ยวกับช่องโหว่ด้านความปลอดภัยในการตรวจสอบสิทธิ์แบบสองปัจจัยทาง SMS และการตอบสนองของ MongoDB ต่อการละเมิด
- มีความกังวลเกี่ยวกับการสูญหายของข้อมูลและการวิพากษ์วิจารณ์เกี่ยวกับข้อดีและข้อเสียของการใช้ MongoDB เป็นฐานข้อมูล
- ความเข้ากันได้ของ MongoDB กับ Realm สําหรับแอปพลิเคชันมือถือสถานะปัจจุบันของ Jepsen และการวิเคราะห์ฐานข้อมูลและปัญหาการออกใบอนุญาตกับ MongoDB ก็ได้รับการกล่าวถึงเช่นกัน

## [การศึกษาเผยให้เห็นประสิทธิภาพที่แตกต่างกันและการรับรู้ของผู้ใช้ CAPTCHA สมัยใหม่](https://arxiv.org/abs/2307.12108)
