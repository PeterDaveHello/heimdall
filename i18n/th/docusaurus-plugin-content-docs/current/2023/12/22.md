# 2023-12-22

## [ArXiv ปรับปรุงการเข้าถึงด้วยเวอร์ชัน HTML สําหรับเอกสารการวิจัย](https://blog.arxiv.org/2023/12/21/accessibility-update-arxiv-now-offers-papers-in-html-format/)

- ขณะนี้ ArXiv กําลังสร้างเอกสารการวิจัยเวอร์ชัน HTML ที่ส่งใน TeX/LaTeX เพื่อปรับปรุงการเข้าถึงสําหรับผู้ใช้ที่มีความบกพร่องทางร่างกาย
- รูปแบบ HTML เข้ากันได้กับโปรแกรมอ่านหน้าจอและเทคโนโลยีช่วยเหลืออื่น ๆ ที่ใช้โดยนักวิจัยที่มีความบกพร่องทางการอ่าน
- การแปลง HTML ยังอยู่ในขั้นตอนการทดลองและอาจไม่สามารถแปลงเอกสารทุกฉบับได้อย่างถูกต้อง แต่ผู้เขียนควรตรวจสอบและปรับปรุงการแสดงผล

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38724665)

- ArXiv ได้แนะนํารูปแบบ HTML สําหรับเอกสารทางวิทยาศาสตร์ทําให้สามารถเข้าถึงได้มากขึ้นบนอุปกรณ์มือถือและได้รับการตอบรับเชิงบวกสําหรับโหมดมืดและการจัดรูปแบบที่น่าพอใจ
- มีการถกเถียงกันว่าผู้เขียนควรได้รับอนุญาตให้อัปโหลดเวอร์ชัน HTML ของตนเองหรือไม่เนื่องจากความกังวลเกี่ยวกับความปลอดภัยและความสอดคล้อง
- มีการอภิปรายเกี่ยวกับความจําเป็นในการปรับปรุงคุณสมบัติการช่วยสําหรับการเข้าถึง เช่น ลิงก์การอ้างอิงโดยตรงและกระดานสนทนา ตลอดจนโซลูชันสําหรับบุคคลตาบอดและการเรียงพิมพ์อักษรเบรลล์ทางคณิตศาสตร์

## [ฮีโร่สนับสนุน: ทักทายผู้ที่ทํางานตลอดเวลาเพื่อให้ระบบทํางานต่อไป](https://news.ycombinator.com/item?id=38725015)

- สิ่งนี้เน้นย้ําถึงความทุ่มเทและความมุ่งมั่นที่จําเป็นเพื่อให้แน่ใจว่าการทํางานของระบบที่สําคัญเป็นไปอย่างราบรื่นแม้ในสถานการณ์ที่ท้าทาย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38725015)

- การอภิปรายครอบคลุมหัวข้อที่หลากหลายเช่นความสําคัญของการเป็นเจ้าของความผิดพลาดการเปลี่ยนแปลงทางวัฒนธรรมในการแสดงความขอโทษและความท้าทายของการสนับสนุนด้านเทคนิคและการบริการลูกค้า
- นอกจากนี้ยังมีการกล่าวถึงความยากลําบากที่พบในช่วงเทศกาลวันหยุดและผลกระทบของการอัปเดตซอฟต์แวร์และการเปิดตัว
- ประเด็นอื่น ๆ ได้แก่ ประสบการณ์ในการถูกเรียกให้ทํางานและความสําคัญของความมั่นคงความกตัญญูและการแก้ปัญหาในอุตสาหกรรมต่างๆ

## [HotspotHelper API ของ Apple เปิดใช้งานการติดตามตําแหน่งโดยไม่ได้รับความยินยอมจากผู้ใช้เป็นเวลาเกือบทศวรรษ](https://wingu.se/2023/11/30/only-apple-can-do-allow-apps-tracking-users-location-without-consensus.html)

- Apple อนุญาตให้แอพติดตามตําแหน่งของผู้ใช้โดยไม่ได้รับความยินยอมหรือความรู้มาเกือบทศวรรษผ่าน HotspotHelper API ซึ่งทําให้เกิดข้อกังวลด้านความเป็นส่วนตัว
- แอปหลัก ๆ เช่น WeChat และ Alipay ได้ใช้ความสามารถนี้แล้วซึ่งเพิ่มผลกระทบของการติดตามตําแหน่งโดยไม่ได้รับความยินยอม
- ขอแนะนําให้ Apple ให้ตัวเลือกแก่ผู้ใช้ในการปิดใช้งานคุณสมบัตินี้และต้องได้รับอนุญาตอย่างชัดเจนสําหรับการติดตามตําแหน่งในขณะที่คําถามยังคงอยู่ว่า Apple สามารถรักษาความเป็นส่วนตัวและจัดการกับข้อกังวลเหล่านี้ได้หรือไม่

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38720656)

- การอภิปรายกล่าวถึงหัวข้อต่างๆรวมถึงสิทธิ์ของแอพการใช้งานมือถือในเอเชียและความกังวลเกี่ยวกับความเป็นส่วนตัวและการควบคุมใน superapps แบบรวมศูนย์
- ผู้ใช้แสดงความกังวลเกี่ยวกับการอนุญาตการติดตามตําแหน่งในแอป iOS เช่น WeChat และ Alipay
- การสนทนาสํารวจความชุกของซูเปอร์แอพมือถือในตลาดจีนโดยมีความคิดเห็นที่แตกต่างกันเกี่ยวกับการเชื่อมต่อและผลกระทบด้านความเป็นส่วนตัว

## [Beeper ท้าทายการแทรกแซงของ Apple กับแอพของพวกเขา](https://blog.beeper.com/p/beeper-moving-forward)

- มีรายงานว่า Apple กําลังแทรกแซงคุณสมบัติ Beeper Mini ของ Beeper ทําให้ Beeper สร้างซอฟต์แวร์เชื่อมต่อ iMessage แบบโอเพ่นซอร์สและมุ่งเน้นไปที่การสร้างแอพแชทที่ดีที่สุด
- Beeper อ้างว่าแอปมีความปลอดภัยและช่วยเพิ่มความปลอดภัยในการแชทระหว่างผู้ใช้ iPhone และ Android โดยใช้การเข้ารหัสแบบ end-to-end
- Beeper วิพากษ์วิจารณ์คํากล่าวอ้างของ Apple และแนะนําว่า Apple กําลังบล็อก Beeper Mini เพื่อสร้างความเสียหายให้กับลูกค้าของตนเองในขณะที่ตั้งคําถามถึงแผนการของพวกเขาที่จะสนับสนุน RCS และแสดงความกังวลเกี่ยวกับการควบคุม iMessage ของ Apple เนื่องจากตําแหน่งทางการตลาดที่โดดเด่น

- ข้อมูลเฉพาะของวิธีการเสียไม่ได้กล่าวถึงในข้อความ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38720544)

## [ศาลเยอรมันรับรองสิทธิ์ของผู้ใช้ในการใช้ Adblockers](https://fsfe.org/news/2023/news-20231220-01.sl.html)

- คดีในศาลสองคดีล่าสุดในเยอรมนีมุ่งเน้นไปที่การต่อสู้เพื่อควบคุมผู้ใช้ผ่านเทคโนโลยีโดยเฉพาะที่เกี่ยวข้องกับ adblockers
- Axel Springer SE ซึ่งเป็น บริษัท สิ่งพิมพ์รายใหญ่ฟ้อง Eyeo GmbH ผู้สร้าง Adblock Plus สําหรับการบล็อกโฆษณาของพวกเขา
- ศาลตัดสินในความโปรดปรานของ Eyeo โดยยืนยันว่าผู้ใช้มีสิทธิ์ที่จะตัดสินใจว่าจะใช้ adblockers หรือไม่และไม่ละเมิดกฎหมายลิขสิทธิ์ อย่างไรก็ตาม Axel Springer ได้รับอนุญาตให้ยกเว้นผู้ใช้ที่มี adblockers จากการเข้าถึงเนื้อหาของพวกเขาและอาจใช้รูปแบบการชําระเงิน
- กรณีเหล่านี้สร้างแบบอย่างทางกฎหมายที่สําคัญเกี่ยวกับเสรีภาพของผู้ใช้และสิทธิในการเลือกบุคคลในขณะที่ใช้อินเทอร์เน็ต

- ผู้ให้บริการเนื้อหาโต้แย้งสิทธิ์ในการควบคุมการเข้าถึงเนื้อหาของตนและกําหนดวิธีการแสดง
- การสนทนายังพิจารณาถึงความเป็นไปได้ของอินเทอร์เน็ตที่ไม่มีโฆษณาผลกระทบต่อเว็บไซต์และผู้สร้างเนื้อหาและรูปแบบธุรกิจทางเลือกเช่นการสมัครสมาชิก ความเป็นส่วนตัวมัลแวร์และพลังของยักษ์ใหญ่อินเทอร์เน็ตเป็นข้อกังวลเพิ่มเติม

- นอกจากนี้ยังมีการหารือเกี่ยวกับหัวข้อที่กว้างขึ้นเช่นความเป็นส่วนตัวกฎหมายและค่าตอบแทนของลูกค้าพร้อมกับประเด็นที่เกี่ยวข้องกับการ์ดแสดงผลคู่บนแล็ปท็อปและอิทธิพลของนโยบายของ บริษัท ทุนนิยมที่มีต่อพนักงาน

## [การลักลอบนําเข้า SMTP: การส่งอีเมลปลอมทั่วโลก](https://sec-consult.com/blog/detail/smtp-smuggling-spoofing-e-mails-worldwide/)

- นักวิจัยได้ค้นพบเทคนิคการโจมตีที่เรียกว่าการลักลอบนําเข้า SMTP ที่ช่วยให้ผู้คุกคามสามารถส่งอีเมลปลอมจากที่อยู่อีเมลใดก็ได้โดยใช้เซิร์ฟเวอร์ SMTP ที่มีช่องโหว่
- มีการแก้ไขช่องโหว่ใน Microsoft และ GMX แล้ว แต่ขอแนะนําให้บริษัทที่ใช้ Cisco Secure Email อัปเดตการกําหนดค่าเริ่มต้นด้วยตนเอง
- ช่องโหว่นี้ส่งผลกระทบต่อผู้ให้บริการอีเมลหลายรายทั่วโลก และนักวิจัยได้สร้างเครื่องมือวิเคราะห์เพื่อตรวจสอบช่องโหว่

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38727987)

- การลักลอบนําเข้า SMTP เป็นวิธีที่ผู้โจมตีใช้เพื่อจัดการกับเซิร์ฟเวอร์ SMTP และอีเมลปลอม
- บทความนี้วิพากษ์วิจารณ์การเน้นการโจมตีที่ประสบความสําเร็จแทนที่จะรักษาความปลอดภัยซอฟต์แวร์โอเพ่นซอร์สโดยเน้นถึงความเสี่ยงต่อความปลอดภัยของระบบอีเมล
- มีข้อกังวลเกี่ยวกับการตั้งค่าเริ่มต้นใน Cisco และบทความแนะนําให้ใช้นโยบาย DMARC ที่เข้มงวดและ SPF เพื่อลดความเสี่ยง
- มีการกล่าวถึงช่องโหว่ในเซิร์ฟเวอร์อีเมล Postfix และการโต้เถียงเกี่ยวกับการเปิดเผยการโจมตีการปลอมแปลงอีเมล
- CERT/CC ได้อนุญาตให้เผยแพร่ข้อมูลเกี่ยวกับการลักลอบนําเข้า SMTP ต่อสาธารณะโดยพิจารณาว่าเป็นคุณสมบัติมากกว่าข้อบกพร่อง
