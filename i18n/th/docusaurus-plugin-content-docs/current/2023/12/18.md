## [DeWave: แปลคลื่นสมองเป็นคําบนหน้าจอ](https://www.iflscience.com/new-mind-reading-braingpt-turns-thoughts-into-text-on-screen-72054)

- นักวิจัยจากมหาวิทยาลัยเทคโนโลยีซิดนีย์ได้พัฒนาเทคโนโลยีที่เรียกว่า DeWave ที่สามารถแปลคลื่นสมองเป็นคําบนหน้าจอได้
- DeWave ใช้การบันทึก EEG จากหมวกที่ผู้เข้าร่วมสวมใส่เพื่อฉายคําที่พวกเขากําลังอ่านบนหน้าจออย่างเงียบ ๆ
- ซึ่งแตกต่างจากวิธีการก่อนหน้านี้ DeWave ไม่จําเป็นต้องฝังสมองหรือเข้าถึงเครื่อง MRI ทําให้ใช้งานได้จริงและสะดวกยิ่งขึ้น

- ผู้เข้าร่วมแสดงความกระตือรือร้นเพื่อประโยชน์ที่อาจเกิดขึ้นรวมถึงการสื่อสารที่ดีขึ้นและการใช้งานทางการแพทย์
- ความกังวลเกี่ยวกับความเป็นส่วนตัวการเฝ้าระวังและการใช้เทคโนโลยีในทางที่ผิดก็เพิ่มขึ้นเช่นกัน

## [สถาปัตยกรรมการประมวลผล CDNA 3 ของ AMD: ท้าทาย Nvidia ด้วยคุณสมบัติขั้นสูง](https://chipsandcheese.com/2023/12/17/amds-cdna-3-compute-architecture/)

- AMD ได้เปิดตัว CDNA 3 Compute Architecture เพื่อพยายามแข่งขันกับ Nvidia ในตลาดการประมวลผล GPU
- สถาปัตยกรรมมีเทคโนโลยีบรรจุภัณฑ์ขั้นสูงการตั้งค่าหน่วยความจําที่มีประสิทธิภาพและการปรับปรุงที่สําคัญเพื่อเพิ่มปริมาณงานและการใช้งาน
- GPU MI300X ใช้ชิปเล็ตที่มีการประมวลผลแบบแยกส่วนและแคชที่ใช้ร่วมกันเอาชนะข้อ จํากัด ของรุ่นก่อนหน้าและรวมคุณสมบัติต่างๆเช่น "Infinity Cache" เพื่อปรับปรุงปริมาณการใช้หน่วยความจํา สถาปัตยกรรมจัดลําดับความสําคัญของการเพิ่มปริมาณการประมวลผลการคูณเมทริกซ์และความจุแคชคําสั่ง MI300X ยังมีแบนด์วิดธ์แบบ die-to-die ที่เพิ่มขึ้นและมีส่วนประกอบ APU ทําให้ CPU และ GPU สามารถใช้พื้นที่ที่อยู่หน่วยความจําเดียวกันได้ ผู้เขียนกําลังทดสอบซีรีส์ MI300 อย่างกระตือรือร้นและรอคอยผลิตภัณฑ์ในอนาคตจาก AMD

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38675258)

- การอภิปรายเกี่ยวกับสถาปัตยกรรม GPU ของ AMD สัมผัสกับหัวข้อต่างๆรวมถึงความท้าทายในการใช้ภาษา GPU และแบ็กเอนด์การประมวลผลการเปรียบเทียบระหว่างกราฟิกการ์ด AMD และ Nvidia และการขาดการสนับสนุนอย่างเป็นทางการสําหรับ AMD GPU
- นอกจากนี้ยังมีการกล่าวถึงการใช้โปรเซสเซอร์ VLIW และกลยุทธ์การแคชเฉพาะใน GPU ซึ่งให้ความกระจ่างเกี่ยวกับความซับซ้อนและข้อควรพิจารณาที่เกี่ยวข้องกับการเขียนโปรแกรม GPU และการออกแบบสถาปัตยกรรม
- การอภิปรายเหล่านี้ให้ข้อมูลเชิงลึกที่มีค่าเกี่ยวกับโลกของเทคโนโลยี GPU และนําเสนอความเข้าใจที่ลึกซึ้งยิ่งขึ้นเกี่ยวกับความซับซ้อนของสถาปัตยกรรม GPU ของ AMD

## [ถอดรหัสภาษาของ VC: ข้อมูลเชิงลึกและเคล็ดลับสําหรับสตาร์ทอัพ](https://jacobbartlett.substack.com/p/yes-actually-means-no-the-curious)

- บทความนี้สํารวจรูปแบบการสื่อสารของผู้ร่วมทุน (VCs) เมื่อมีส่วนร่วมกับสตาร์ทอัพและผู้ประกอบการ
- ผู้เขียนแบ่งปันประสบการณ์ส่วนตัวในการรับการปฏิเสธจาก VCs และให้ข้อมูลเชิงลึกที่มีค่าเกี่ยวกับความหมายที่ซ่อนอยู่เบื้องหลังคําพูดของพวกเขา
- เคล็ดลับการเอาชีวิตรอดมีให้กับผู้ก่อตั้งที่กําลังมองหาเงินทุนช่วยให้พวกเขาสํารวจภูมิทัศน์ VC ได้อย่างมีประสิทธิภาพมากขึ้น

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38677251)

- ผู้ร่วมทุน (VCs) ในระบบนิเวศสตาร์ทอัพมักจะให้คําตอบที่คลุมเครือแทนที่จะพูดว่า "ไม่" โดยตรงทําให้เกิดการถกเถียงกันเกี่ยวกับภาษาและพฤติกรรมของพวกเขา
- ความสําคัญของการแยกความแตกต่างระหว่าง VCs ที่มีประสบการณ์และผู้เล่นรายย่อยถูกเน้นในการอภิปราย
- ครอบคลุมหัวข้อต่างๆรวมถึงข้อดีข้อเสียของการระดมทุน VC ความท้าทายที่สตาร์ทอัพต้องเผชิญอคติในการรับรู้แนวคิดการเริ่มต้นการได้มาซึ่งลูกค้าแหล่งเงินทุนทางเลือกความท้าทายและความเสี่ยงที่ VCs ต้องเผชิญชื่อเสียงและการเชื่อมต่อในการรักษาความปลอดภัยเงินทุนและปัญหาการเลือกปฏิบัติและอคติในอุตสาหกรรม

## [การปรับปรุงโมเดลพื้นฐานด้วย Promptbase: Medprompt Methodology และ Performance Boost](https://github.com/microsoft/promptbase)

- Promptbase เป็นชุดทรัพยากรและสคริปต์ตัวอย่างที่ครอบคลุมเพื่อเพิ่มประสิทธิภาพของโมเดลพื้นฐานเช่น GPT-4
- วิธีการ Medprompt ซึ่งรวมอยู่ใน Promptbase ใช้กลยุทธ์สามประการ (ช็อตไม่กี่ช็อตแบบไดนามิกห่วงโซ่ความคิดที่สร้างขึ้นเองและคะแนนเสียงข้างมาก) เพื่อให้ได้ประสิทธิภาพระดับผู้เชี่ยวชาญ
- Promptbase ได้ขยายกรอบ Medprompt เพื่อผลลัพธ์ที่ดีขึ้นในเกณฑ์มาตรฐาน MMLU และมีการวางแผนกรณีศึกษาและการสัมภาษณ์เพิ่มเติม แพลตฟอร์มนี้มีรายละเอียดการใช้งานและคําแนะนําสําหรับการเรียกใช้สคริปต์

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38673954)

- การอภิปรายข่าวแฮ็กเกอร์หมุนรอบวิศวกรรมที่รวดเร็วในรูปแบบภาษาโดยผู้ใช้จะพูดถึงประสิทธิภาพและความสําคัญในการเพิ่มประสิทธิภาพ
- ผู้เข้าร่วมบางคนโต้แย้งถึงความจําเป็นของวิศวกรรมที่รวดเร็วในขณะที่คนอื่นตั้งคําถามถึงความสําคัญของมัน
- มีการเปรียบเทียบระหว่างวิศวกรรมที่รวดเร็วและวิศวกรรมสังคมและผู้ใช้รายหนึ่งแสดงความไม่พอใจกับการขาดความสามารถทางวิศวกรรมที่รวดเร็วในโมเดล NLP เช่น ChatGPT

## [ห้องปฏิบัติการนิวเคลียร์ฟิวชั่นของสหรัฐอเมริกาบรรลุ 'การจุดระเบิด' หลายครั้ง](https://www.nature.com/articles/d41586-023-04045-8)
