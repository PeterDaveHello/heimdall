# 2023-08-16

## [เราลดต้นทุนในการสร้าง Mastodon ในระดับ Twitter ลง 100x](https://blog.redplanetlabs.com/2023/08/15/how-we-reduced-the-cost-of-building-twitter-at-twitter-scale-by-100x/)

- Red Planet Labs ประสบความสําเร็จในการลดต้นทุน 100 เท่าสําหรับการสร้างอินสแตนซ์ Mastodon ขนาด Twitter โดยใช้แพลตฟอร์ม Rama
- อินสแตนซ์นี้สร้างขึ้นด้วยโค้ดเพียง 10,000 บรรทัด เมื่อเทียบกับ 1 ล้านบรรทัดของ Twitter
- พระรามนําเสนอความสามารถในการปรับขนาดประสิทธิภาพและความทนทานต่อความผิดพลาดทําให้การพัฒนาแบ็กเอนด์ง่ายขึ้น ทีมงานวางแผนที่จะปล่อย Rama สําหรับการดาวน์โหลดสาธารณะและการใช้งาน Mastodon แบบโอเพนซอร์ส PState ใน Mastodon ใช้เพื่อติดตามข้อมูลและปรับปรุงประสิทธิภาพ มีการกล่าวถึงเทคนิคการเพิ่มประสิทธิภาพและมีการกล่าวถึงการเผยแพร่เอกสารพระรามและ API

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37137110)

- บทความนี้มุ่งเน้นไปที่ Rama ซึ่งเป็นเฟรมเวิร์กแบ็กเอนด์ใหม่ที่มีจุดมุ่งหมายเพื่อลดความซับซ้อนและลดต้นทุนในการสร้างแอปพลิเคชันที่ปรับขนาดได้
- ผู้แสดงความคิดเห็นพูดถึงการเปรียบเทียบที่ทํากับ Twitter และตั้งคําถามถึงความสามารถในการปรับขนาดของพระราม
- การอภิปรายยังเจาะลึกถึงความท้าทายของแพลตฟอร์มแบบกระจายอํานาจข้อดีข้อเสียของการใช้ Entity Component Systems ในการพัฒนาเกมและความสําคัญของความปลอดภัยและการกลั่นกรอง

## [ในที่สุด Firefox ก็มีประสิทธิภาพเหนือกว่า Google Chrome ใน SunSpider](https://www.phoronix.com/news/Firefox-Faster-SunSpider)

- Firefox แซงหน้า Google Chrome ในเกณฑ์มาตรฐาน SunSpider JavaScript ซึ่งนับเป็นความสําเร็จที่สําคัญสําหรับนักพัฒนา Mozilla
- แม้จะประสบความสําเร็จนี้ Chrome ยังคงมีประสิทธิภาพเหนือกว่า Firefox ในเกณฑ์มาตรฐาน JetStream 2.0 ที่มีความต้องการมากขึ้น
- Firefox มุ่งเน้นไปที่การปรับปรุงความเร็วในการอัปโหลดและแนะนําการปรับปรุงอื่น ๆ เพื่อเพิ่มประสิทธิภาพโดยรวม

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37134092)

- เธรดการสนทนามุ่งเน้นไปที่หัวข้อต่างๆที่เกี่ยวข้องกับเว็บเบราว์เซอร์เช่น Firefox, Chrome และ Safari
- ผู้ใช้มีส่วนร่วมในการอภิปรายเกี่ยวกับเกณฑ์มาตรฐานประสิทธิภาพตัวเลือกการปรับแต่งการใช้หน่วยความจําและส่วนต่อประสานผู้ใช้
- มีความกังวลเกี่ยวกับความปลอดภัยผู้จัดการรหัสผ่านและการครอบงําของ Chrome และการอภิปรายล้อมรอบการลดลงของ Firefox และผลกระทบที่อาจเกิดขึ้นจากการควบคุมเว็บของ Google

## [แถลงการณ์ OpenTF](https://opentf.org/)

- แถลงการณ์ OpenTF ทําให้เกิดความกังวลเกี่ยวกับการเปลี่ยนแปลงใบอนุญาตใน Terraform ซึ่งเป็นแพลตฟอร์มโอเพ่นซอร์สและสนับสนุนการย้อนกลับการเปลี่ยนแปลงหรือสร้างรากฐานเพื่อรักษาเวอร์ชันโอเพ่นซอร์ส
- บริษัท และบุคคลต่างๆมีส่วนร่วมอย่างแข็งขันในการริเริ่มชุมชนโอเพ่นซอร์สเพื่อปรับปรุงโครงการและส่งเสริมการทํางานร่วมกัน
- เมื่อวันที่ 15 สิงหาคม พ.ศ. 2023 กลุ่มบุคคลมีส่วนร่วมในการพัฒนารายบุคคลและความพยายามของชุมชนโอเพ่นซอร์สเช่นการทดสอบเอกสารการให้คําปรึกษาและการใช้เครื่องมือโอเพ่นซอร์ส

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37133054)

- แถลงการณ์ OpenTF เป็นการอภิปรายโดยนักพัฒนาที่สนับสนุนการแยก Terraform เป็นมูลนิธิที่ไม่แสวงหาผลกําไรเนื่องจากความกังวลเกี่ยวกับการขาดการบํารุงรักษาและการสนับสนุนจาก Hashicorp เจ้าของปัจจุบัน
- นักพัฒนาบางคนโต้แย้งส้อมเพื่อปรับปรุงภาษาและจัดลําดับความสําคัญของความต้องการของชุมชนในขณะที่คนอื่น ๆ หวังว่าจะเปลี่ยนใจจาก Hashicorp
- การอภิปรายยังกล่าวถึงผลิตภัณฑ์การจัดการลับข้อ จํากัด ของการเขียนโปรแกรมแบบประกาศผลกระทบต่อชุมชนโอเพ่นซอร์สระบบนิเวศของผู้ให้บริการการสร้างรายได้โดย Hashicorp ตัวเลือกการออกใบอนุญาตทางเลือกการเปลี่ยนชื่อและรีแบรนด์โครงการการขาดกรอบการทดสอบใน Terraform ปัญหาความเป็นเจ้าของและความกังวลเกี่ยวกับใบอนุญาตใหม่
- ผู้ใช้แสดงการสนับสนุน Terraform แบบโอเพนซอร์สและสํารวจโซลูชันทางเลือก
- นอกจากนี้ยังมีการกล่าวถึงความกังวลเกี่ยวกับการเปลี่ยนใบอนุญาตผลกระทบต่อการเติบโตของรายได้ข้อ จํากัด ความเข้ากันได้และอิทธิพลของผู้ขายระบบคลาวด์ที่มีต่อเครื่องมือโอเพนซอร์ส

## [LLaMa.cpp เป็นไปได้อย่างไร?](https://finbarr.ca/how-is-llama-cpp-possible/)

- ผู้เขียนตรวจสอบความเป็นไปได้ในการเรียกใช้รหัสการอนุมาน LLaMa ใน C ++ ดิบบนฮาร์ดแวร์ต่างๆรวมถึงอุปกรณ์ที่มี RAM จํากัด เช่น Raspberry Pi
- มีการกล่าวถึงความสําคัญของแบนด์วิดท์หน่วยความจําในการเรียนรู้เชิงลึกพร้อมกับประโยชน์ของการใช้โมเดลที่มีความแม่นยําต่ําเพื่อลดการใช้หน่วยความจํา
- การคํานวณและตัวอย่างมีให้สําหรับการเรียกใช้ LLaMa บนฮาร์ดแวร์ต่างๆโดยเน้นที่ความสําคัญของแบนด์วิดท์หน่วยความจําเมื่อสุ่มตัวอย่างจากหม้อแปลง นอกจากนี้ผู้เขียนยังเน้นย้ําถึงความสําคัญของการกลั่นและการฝึกอบรมเพิ่มเติมสําหรับโมเดลขนาดเล็กเพื่อปรับปรุงประสิทธิภาพ อย่างไรก็ตามพวกเขารับทราบถึงความเป็นไปได้ของข้อผิดพลาดในการคํานวณและสนับสนุนข้อเสนอแนะ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37140013)

- การอภิปรายหมุนรอบแบบจําลองเชิงปริมาณความสามารถของแบบจําลองภาษาลามะการอนุมานที่ใช้ CPU ข้อ จํากัด แบนด์วิดท์หน่วยความจําศักยภาพของ AI ในการจับคู่ประสิทธิภาพของมนุษย์ข้อ จํากัด และศักยภาพของแบบจําลองภาษาสําหรับ AGI และความท้าทายในการรับข้อความที่มนุษย์สร้างขึ้น
- ผู้เข้าร่วมครอบคลุมแพลตฟอร์มวิธีการและการกําหนดค่าฮาร์ดแวร์ที่เฉพาะเจาะจง
- มีการแสดงความคิดเห็นที่แตกต่างกันเกี่ยวกับความเป็นไปได้และความก้าวหน้าในอนาคตของโมเดล AI

## [ผู้ให้บริการอินเทอร์เน็ตบ่นว่าการแสดงรายการค่าธรรมเนียมทั้งหมดยากเกินไปกระตุ้นให้ FCC ยกเลิกกฎใหม่](https://arstechnica.com/tech-policy/2023/08/isps-complain-that-listing-every-fee-is-too-hard-urge-fcc-to-scrap-new-rule/)

- อุตสาหกรรมบรอดแบนด์ของสหรัฐอเมริกากําลังคัดค้านข้อกําหนดของ FCC ในการเปิดเผยค่าธรรมเนียมรายเดือนทั้งหมด โดยให้เหตุผลว่าจะเพิ่มความซับซ้อนและภาระให้กับทั้งผู้บริโภคและผู้ให้บริการ
- กลุ่มล็อบบี้แนะนําให้รวมข้อความอธิบายเกี่ยวกับค่าธรรมเนียมคล้ายกับภาษีที่รัฐบาลกําหนดแทนที่จะแสดงรายการค่าธรรมเนียมทั้งหมด
- ผู้สนับสนุนผู้บริโภควิพากษ์วิจารณ์ข้อเสนอนี้โดยระบุว่าอนุญาตให้ผู้ให้บริการอินเทอร์เน็ตซ่อนต้นทุนการบริการที่แท้จริง
- กฎการติดฉลากบรอดแบนด์ของ FCC รวมถึงการกําหนดราคาและค่าใช้จ่ายเพิ่มเติมกําลังได้รับการตรวจสอบโดยสํานักงานบริหารและงบประมาณของสหรัฐอเมริกา

- นอกจากนี้ยังรวมถึงความกังวลเกี่ยวกับการล็อบบี้การทุจริตของรัฐบาลการกําหนดราคาที่หลอกลวงการแข่งขันจากผู้เข้าร่วมใหม่และการเฝ้าระวังและการรวบรวมข้อมูลของ ISP
- ธีมโดยรวมคือการเรียกร้องให้มีความโปร่งใสการคุ้มครองผู้บริโภคและกฎระเบียบในอุตสาหกรรม ISP มากขึ้น

## [ตัวเลือก CSS: คู่มือภาพ](https://fffuel.co/css-selectors/)

- คู่มือภาพนี้ให้คําอธิบายเกี่ยวกับตัวเลือก CSS ต่างๆ รวมถึงคลาสหลอกและองค์ประกอบหลอก พร้อมตัวอย่างการใช้งาน
- คู่มือนี้ครอบคลุมตัวเลือกสําหรับองค์ประกอบ คลาส รหัส แอตทริบิวต์ และสถานะต่างๆ ขององค์ประกอบอินพุต
- นอกจากนี้ยังกล่าวถึงคลาสหลอก CSS และองค์ประกอบหลอกเพิ่มเติมสําหรับการกําหนดเป้าหมายและจัดแต่งทรงผมองค์ประกอบเฉพาะภายในหน้าเว็บและให้ทรัพยากรเช่นเครื่องกําเนิด SVG เครื่องมือและทรัพยากรการออกแบบ โปรดทราบว่ามีลิงค์พันธมิตรสําหรับบริการที่เกี่ยวข้อง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37132754)

- มีการกล่าวถึงคุณสมบัติขั้นสูงเช่น nth-child และตัวเลือกที่มีค่าเฉพาะที่จุดเริ่มต้นหรือจุดสิ้นสุด
- บทความนี้ยังสํารวจเครื่องมือสําหรับการกําหนดตัวเลือก CSS ทั่วไปและเน้นถึงประโยชน์ของไวยากรณ์นิพจน์ปกติใน CSS

## [ฉันควรอ่านสัญกรณ์ระบบประเภทอย่างไร](https://langdev.stackexchange.com/questions/2692/how-should-i-read-type-system-notation)

- Programming Language Design and Implementation Stack Exchange เป็นแพลตฟอร์มสําหรับนักออกแบบภาษาการเขียนโปรแกรมและผู้ดําเนินการเพื่อแลกเปลี่ยนคําถามและคําตอบ
- คําถามล่าสุดบนเว็บไซต์ขอคําแนะนําเกี่ยวกับสัญกรณ์ระบบการอ่านประเภท
- ไซต์นี้ให้คําอธิบายพื้นฐานรวมถึงไวยากรณ์ไวยากรณ์ความสัมพันธ์การตัดสินสัจพจน์และกฎการอนุมาน นอกจากนี้ยังกล่าวถึงบทบาทของบริบทหรือสภาพแวดล้อมประเภทในการกําหนดประเภทตัวแปรและข้อควรพิจารณาอื่น ๆ ในการระบุระบบประเภท

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37138807)

- การทําความเข้าใจและการใช้สัญกรณ์ระบบประเภทเป็นสิ่งสําคัญในวิทยาการคอมพิวเตอร์และภาษาโปรแกรม
- บทความนี้สํารวจการพัฒนาทางประวัติศาสตร์ของสัญกรณ์ระบบประเภทและการใช้ตรรกะ
- มีการกล่าวถึงมุมมองและประสบการณ์ที่แตกต่างกันเกี่ยวกับการจัดวางและการประชุมของคําอธิบายประกอบประเภทในภาษาการเขียนโปรแกรม

## [ทําไมต้องเป็นเจ้าภาพ LLM ของคุณเอง?](http://marble.onl/posts/why_host_your_own_llm.html)

- ผู้เขียนนําเสนอข้อโต้แย้งเพื่อสนับสนุนโมเดลภาษาที่โฮสต์ด้วยตนเอง (LLMs) แทนที่จะพึ่งพาโมเดลที่เปิดเผยต่อสาธารณะเช่น GPT-4 ของ OpenAI
- การโฮสต์ด้วยตนเองให้การควบคุมสถาปัตยกรรมโมเดลและน้ําหนักความสามารถในการปรับแต่งและการปรับแต่งที่ดีขึ้นและความสัมพันธ์ระยะยาวกับโมเดล AI
- การโฮสต์ด้วยตนเองช่วยให้สามารถทดลองเพิ่มประสิทธิภาพและติดตามเทคโนโลยีที่พัฒนาแล้วได้มากขึ้นในขณะที่ API อาจไม่ได้ให้การเข้าถึงและความเข้าใจในระดับเดียวกัน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37133504)

- บทความนี้สํารวจเหตุผลและความท้าทายของ Language Model Models (LLMs) ที่โฮสต์ด้วยตนเองแทนที่จะพึ่งพา API ของ OpenAI
- มีการกล่าวถึงตัวเลือกอื่นสําหรับการโฮสต์ LLM เช่น Beam ฮาร์ดแวร์ส่วนบุคคลและ Tiny Box ของ George Hotz
- มีการสํารวจความเป็นไปได้และประโยชน์ของ LLM ที่โฮสต์ด้วยตนเองรวมถึงความคุ้มค่าและข้อกังวลด้านความเป็นส่วนตัว

## [Modern CSV เวอร์ชัน 2](https://www.moderncsv.com/modern-csv-2-is-now-available/)

- เวอร์ชันใหม่นี้นําเสนอการปรับปรุงหลายอย่าง รวมถึง UI/UX ที่ปรับปรุงแล้ว ประสิทธิภาพที่ดีขึ้น คุณสมบัติเพิ่มเติม เอกสารที่อัปเดต และการสนับสนุนดั้งเดิมสําหรับ Apple Silicon
- ขอแนะนําให้ผู้ใช้ลองใช้เวอร์ชันใหม่และให้ข้อเสนอแนะแก่นักพัฒนาซอฟต์แวร์
