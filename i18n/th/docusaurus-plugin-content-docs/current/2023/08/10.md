## [การตรวจสอบโครงสร้างพื้นฐานเสร็จสมบูรณ์โดย Radically Open Security](https://mullvad.net/en/blog/2023/8/9/infrastructure-audit-completed-by-radically-open-security/)

- Mullvad VPN ผ่านการตรวจสอบความปลอดภัยครั้งที่สามโดย Radically Open Security โดยมุ่งเน้นไปที่โครงสร้างพื้นฐาน VPN
- การตรวจสอบไม่พบการรั่วไหลของข้อมูลหรือการบันทึกข้อมูลลูกค้า แต่ระบุปัญหาต่าง ๆ เช่นการมองเห็นปริมาณการใช้งานของผู้ใช้ที่ใช้งานจริงบนเซิร์ฟเวอร์ทดสอบ
- Mullvad VPN วางแผนที่จะแก้ไขปัญหาเหล่านี้โดยใช้มาตรการต่างๆ เช่น การตรวจสอบการเข้าสู่ระบบโดยไม่ได้รับอนุญาตและการลบการสนับสนุน SSH

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37060654)

- บทความและความคิดเห็นมุ่งเน้นไปที่ผู้ให้บริการ VPN รวมถึง Mullvad และ ProtonVPN และแนวทางปฏิบัติด้านความเป็นส่วนตัวของพวกเขาโดยมีการกล่าวถึง Mullvad โดยเฉพาะในการลบคุณสมบัติการส่งต่อพอร์ต
- หัวข้อที่กล่าวถึง ได้แก่ ความเป็นส่วนตัวทางอินเทอร์เน็ตการเฝ้าระวังของรัฐบาลการเซ็นเซอร์ออนไลน์ผลกระทบของการคว่ําบาตรของสหรัฐอเมริกาและข้อ จํากัด และช่องโหว่ของ VPN
- ความกังวลที่เกิดขึ้นรวมถึงการรวบรวมข้อมูลความน่าเชื่อถือของผู้ให้บริการ VPN และความซับซ้อนของความเป็นส่วนตัวออนไลน์

## [Arpchat – ส่งข้อความถึงเพื่อนของคุณในเครือข่ายเดียวกันโดยใช้เพียง ARP](https://github.com/kognise/arpchat)

- โครงการ "arpchat" เป็นแอปพลิเคชันแชทโอเพ่นซอร์สที่ใช้ Address Resolution Protocol (ARP) สําหรับการส่งข้อความระหว่างอุปกรณ์เครือข่าย
- รองรับการส่งข้อความยาวและรวมถึงการค้นพบสถานะและระบบการเต้นของหัวใจรวมถึงฟังก์ชั่นการบีบอัด
- โครงการนี้เขียนด้วย Rust และได้รับความนิยมบน GitHub ด้วยดาว 1.8k คําแนะนําในการติดตั้งมีให้สําหรับ Windows และ Linux และขอแนะนําให้ผู้ใช้รายงานปัญหาใด ๆ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37063151)

- ArpChat เป็นแอปส่งข้อความที่ใช้ Address Resolution Protocol (ARP) สําหรับการสื่อสารระหว่างผู้ใช้ในเครือข่ายเดียวกัน
- ข้อผิดพลาดที่ทําให้เกิดเวลาแฝงใน ArpChat ได้รับการแก้ไขแล้ว แต่เวอร์ชันที่อัปเดตยังไม่ได้รับการเผยแพร่
- ผู้ใช้ Linux อาจประสบปัญหากับ ArpChat เนื่องจากสิทธิ์ที่ขาดหายไป แต่มีวิธีแก้ไข
- บทความนี้อธิบายถึงประโยชน์ของ ARP ในระบบเครือข่ายและบทบาทในการเชื่อมต่อที่อยู่ MAC กับที่อยู่ IP
- ARP มักใช้ในเครือข่ายอีเธอร์เน็ตและ WiFi และบทความจะสํารวจการใช้งานในเครือข่ายเหล่านี้
- มีการกล่าวถึงโมเดล spoke-and-hub ที่ใช้โดยเราเตอร์ไร้สายสวิตช์และโมเด็มโดยอธิบายว่าข้อมูลโดยทั่วไปจะผ่านเราเตอร์แม้ในเครือข่ายท้องถิ่นอย่างไร
- มีการตรวจสอบช่องโหว่ที่อาจเกิดขึ้นและการจัดการ ARP สําหรับการสร้างข้อความที่ออกอากาศบนเครือข่าย
- บทความนี้ยังครอบคลุมหัวข้อต่างๆเช่นสคริปต์แบทช์สําหรับการสร้างแผนที่เครือข่ายวิธีการสื่อสารโดยไม่ใช้อินเทอร์เน็ต (การแชทแบบเพียร์ทูเพียร์เครือข่ายไร้สาย) และประสบการณ์ของผู้ใช้ด้วยโปรโตคอลเครือข่ายที่คล้ายกัน
- นอกจากนี้ยังมีการกล่าวถึงความเป็นไปได้ของการใช้ DNS สําหรับระบบแชท

## [SQLedge: ทําซ้ํา Postgres ไปยัง SQLite บน Edge](https://github.com/zknill/sqledge)

- SQLedge เป็นเครื่องมือโอเพ่นซอร์สที่ใช้การจําลองแบบลอจิคัล Postgres เพื่อถ่ายโอนการเปลี่ยนแปลงจากฐานข้อมูล Postgres ไปยังฐานข้อมูล SQLite ภายในเครื่องบนขอบ
- ช่วยให้สามารถอ่านจากฐานข้อมูล SQLite และเขียนไปยังเซิร์ฟเวอร์ Postgres ต้นน้ําเพื่อให้มั่นใจถึงความสอดคล้องของข้อมูล
- เครื่องมือนี้ประกอบด้วยตัวสร้าง SQL, ตัวแยกวิเคราะห์ SQL และพร็อกซีสาย Postgres และการกําหนดค่าทั้งหมดจะถูกดึงมาจากตัวแปรสภาพแวดล้อม

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37063238)

- ฐานข้อมูล SQLite ใช้กันอย่างแพร่หลายและมีเครื่องมือและโครงการหลายอย่างที่พร้อมใช้งานเพื่อปรับปรุงฟังก์ชันการทํางาน
- เครื่องมือเช่น SQLedge และ SQLitePostgres มีคุณสมบัติเช่นการจําลองข้อมูลประโยชน์การควบคุมเวอร์ชันการจําลองแบบหลายต้นแบบและการแยกวิเคราะห์แบบสอบถาม
- เครื่องมือเหล่านี้มีตัวเลือกที่มีน้ําหนักเบาสําหรับการแคชและการสืบค้นข้อมูลในเครื่องด้วยฟังก์ชันต่างๆที่เกี่ยวข้องกับการจัดการฐานข้อมูล SQLite อย่างไรก็ตามมีข้อ จํากัด เนื่องจากความแตกต่างในภาษาถิ่น SQL ระหว่าง SQLite และฐานข้อมูลอื่น ๆ เช่น PostgreSQL

- มันเน้นการปิดเครื่องมือ Prosecraft เนื่องจากความกังวลของผู้เขียนและการวิพากษ์วิจารณ์เครื่องมือ Shaxpir
- บทสรุปยังครอบคลุมถึงการอภิปรายเกี่ยวกับความถูกต้องตามกฎหมายและการใช้เนื้อหาที่สร้างโดย AI การอภิปรายเกี่ยวกับลิขสิทธิ์และการใช้งานที่เหมาะสมผลกระทบของ AI ต่อการเรียนรู้และความคิดสร้างสรรค์ความกังวลเกี่ยวกับความเป็นส่วนตัวของข้อมูลและผลกระทบของบริการสตรีมมิ่งต่อความพร้อมใช้งานของเนื้อหา

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37060974)

- การอภิปรายหมุนรอบการใช้ prosecraft.io ซึ่งเป็นเครื่องมือที่สร้างสถิติและเมตริกเกี่ยวกับหนังสือ
- หัวข้อการอภิปรายรวมถึงการละเมิดลิขสิทธิ์การใช้งานที่เหมาะสมและความจําเป็นในการได้รับความยินยอมจากผู้เขียน
- นอกจากนี้ยังมีการกล่าวถึงผลกระทบของเทคโนโลยี AI ต่ออุตสาหกรรมสร้างสรรค์การสูญเสียงานจริยธรรมความถูกต้องและความคิดสร้างสรรค์ของมนุษย์

## [ทําให้ AMD GPU สามารถแข่งขันได้สําหรับการอนุมาน LLM](https://blog.mlc.ai/2023/08/09/Making-AMD-GPUs-competitive-for-LLM-inference)

- บทความนี้กล่าวถึงประสิทธิภาพของ AMD GPU โดยเฉพาะ Radeon RX 7900 XTX สําหรับการอนุมาน LLM (Large Language Model)
- การศึกษาแสดงให้เห็นว่า 7900 XTX ทํางานที่ 80% ของความเร็วของ GeForce RTX 4090 ของ NVIDIA และ 94% ของความเร็วของ RTX 3090 Ti สําหรับรุ่น Llama2-7B / 13B
- การใช้การรวบรวมแมชชีนเลิร์นนิ่ง (MLC) และสแต็ก ROCm ช่วยเพิ่มประสิทธิภาพบน AMD GPU โดยอาจรองรับ Vulkan และ WebGPU การศึกษาเน้นย้ําถึงความจําเป็นในการพัฒนาและเพิ่มประสิทธิภาพของโซลูชัน MLC สําหรับ AMD GPU และแพลตฟอร์มอื่น ๆ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37066522)

- กองซอฟต์แวร์ที่ปรับให้เหมาะสมสามารถปรับปรุงประสิทธิภาพของ AMD GPU สําหรับการอนุมานโมเดลภาษาขนาดใหญ่
- เทคนิคการรวบรวม ML และสแต็คซอฟต์แวร์ TVM Unity ถูกกล่าวถึงว่าเป็นวิธีการเพื่อให้ได้ประสิทธิภาพสูงทั้ง NVIDIA และ AMD GPU
- บทความนี้สํารวจการสนับสนุนของ AMD สําหรับ ROCm และ Vulkan แต่ทําให้เกิดความกังวลเกี่ยวกับอายุการใช้งานที่ยาวนานของการสนับสนุน
- มีการกล่าวถึงกรณีการใช้งานต่างๆ ของแพลตฟอร์ม ROCm ของ AMD รวมถึง AI, การถอดรหัสรหัสผ่าน, การขุด GPU และการเรนเดอร์ VFX
- มีการเปรียบเทียบระหว่าง AMD และ Nvidia GPU โดยเน้นที่ข้อ จํากัด ของความจุ RAM และแบนด์วิดท์ในอัลกอริธึมการเรียนรู้ของเครื่อง
- Microsoft กําลังเสนอวิธีแก้ปัญหาคอขวดในโครงการของพวกเขาในขณะที่การมีส่วนร่วมของ Geohot มุ่งเน้นไปที่การปรับปรุง ROCm โดยตรง
- บทความนี้แนะนําให้ใช้การ์ด RX 6600 หลายใบแทนที่จะเป็น RX 7900 หนึ่งใบ แต่ยังคงมีคําถามเกี่ยวกับวิธีเชื่อมโยงหากไม่มี NVLink ที่เทียบเท่ากับ AMD
- การพัฒนาเหล่านี้ทําให้เกิดข้อควรพิจารณาเกี่ยวกับการลงทุนใน AMD

## [ซีแอตเทิลกลายเป็นแห่งแรกในสหรัฐอเมริกาที่ปกป้องคนงานกิ๊กจากการ 'ปิดใช้งาน' อย่างกะทันหัน](https://kuow.org/stories/seattle-becomes-first-in-u-s-to-protect-gig-workers-from-sudden-deactivation)

- สภาเมืองซีแอตเทิลได้อนุมัติกฎหมายใหม่เพื่อปกป้องคนงานกิ๊กจากการปิดใช้งานอย่างกะทันหันบนแพลตฟอร์มเช่น Instacart และ DoorDash ซึ่งถือเป็นกฎหมายดังกล่าวฉบับแรกในสหรัฐอเมริกา
- ข้อบัญญัติใช้แนวทางที่เข้มงวดมากขึ้นสําหรับการปิดใช้งานและกําหนดให้ บริษัท ต่างๆแจ้งให้ทราบล่วงหน้า 14 วันก่อนการยกเลิก
- การพัฒนานี้เป็นส่วนหนึ่งของความคิดริเริ่มขนาดใหญ่ในซีแอตเทิลที่มุ่งปกป้องคนงานกิ๊กที่ไม่ได้เป็นส่วนหนึ่งของสหภาพแรงงานรวมถึงกฎหมายล่าสุดที่เพิ่มค่าแรงขั้นต่ําและให้เวลาป่วยและผลประโยชน์แก่คนงานที่ใช้แอพ กฎหมายนี้คาดว่าจะลงนามโดยบรูซฮาร์เรลล์นายกเทศมนตรีเมืองซีแอตเทิล

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37070398)

- การอภิปรายเกี่ยวกับงานกิ๊กครอบคลุมหัวข้อต่างๆเช่นการคุ้มครองคนงานการจําแนกประเภทผลประโยชน์ความท้าทายและการเปรียบเทียบกับบริการแบบดั้งเดิมเช่นรถแท็กซี่
