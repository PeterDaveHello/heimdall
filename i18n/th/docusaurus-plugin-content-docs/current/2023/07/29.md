# 2023-07-29

## [Cap'n Proto 1.0](https://capnproto.org/news/2023-07-28-capnproto-1.0.html)

- Cap'n Proto ซึ่งเป็นโปรโตคอลการส่งข้อความได้เปิดตัวเวอร์ชัน 1.0 ของการใช้งานอ้างอิง C ++
- โปรโตคอลนี้ถูกใช้ในการผลิตเป็นเวลาหลายปีซึ่งบ่งบอกถึงความน่าเชื่อถือ
- การเปิดตัวรวมถึงการเพิ่มประสิทธิภาพเพื่อประสิทธิภาพที่ดีขึ้น
- มีการเปลี่ยนแปลงเพื่อให้สามารถยกเลิก RPC ได้
- แผนสําหรับเวอร์ชัน 2.0 รวมถึงการแก้ไข C ++ API และไลบรารีชุดเครื่องมือ KJ C ++ เพื่อปรับปรุงประสบการณ์ของนักพัฒนา
- การเปลี่ยนแปลงเหล่านี้จะต้องใช้คอมไพเลอร์ที่รองรับ C ++ 20 และอาจทําให้ KJ เป็นมิตรกับ coroutine มากขึ้น
- การเปลี่ยนแปลงที่เข้ากันไม่ได้ย้อนหลังจะไม่เกิดขึ้นกับรูปแบบการออกหมายเลขกํากับหรือโปรโตคอล RPC
- การเปิดตัว 2.0 อย่างเป็นทางการไม่คาดว่าจะเกิดขึ้นสองสามปี

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=36908309)

- Ian Denhardt ผู้สนับสนุนหลักในการดําเนินการ Go ของ Cap'n Proto ได้เสียชีวิตแล้ว ทําให้ชุมชน Cap'n Proto และ Sandstorm อยู่ในความโศกเศร้า
- Cap'n Proto เป็นรูปแบบการออกหมายเลขกํากับข้อมูลประสิทธิภาพสูงที่ใช้ในแพลตฟอร์ม Workers ของ Cloudflare
- มีการกล่าวถึงประโยชน์ของ Cap'n Proto รวมถึงการออกหมายเลขกํากับและ deserialization ที่มีประสิทธิภาพ
- การอภิปรายยังรวมถึงการสนับสนุนภาษาสําหรับ Cap'n Proto พร้อมตัวอย่างของการใช้งานในภาษาต่างๆ
- มีการสํารวจศักยภาพในการพัฒนาและปรับปรุง Cap'n Proto ต่อไป
- การสนทนาสัมผัสกับหัวข้อต่างๆเช่นฟิลด์ที่จําเป็นและไม่จําเป็นในการพัฒนาซอฟต์แวร์การพิมพ์แบบคงที่และความท้าทายของการออกแบบโปรโตคอล
- รูปแบบการออกหมายเลขกํากับอื่น ๆ เช่น Avro, protobuf และ flatbuffers จะถูกเปรียบเทียบกับ Cap'n Proto
- คําแนะนําสําหรับไลบรารีและโปรโตคอลการออกหมายเลขกํากับไบนารีอื่น ๆ จะกล่าวถึงควบคู่ไปกับ Cap'n Proto

- ผู้ใช้แบ่งปันประสบการณ์และความทรงจําในการดูทีวีในอดีต
- มีการหารือเกี่ยวกับช่องโฆษณาและรายการเฉพาะจากยุค 90
- ปัญหาลิขสิทธิ์และใบอนุญาตถูกกล่าวถึงในบริบทของการดูรายการทีวีรุ่นเก่า

## [ผู้ก่อตั้ง Sci-Hub ได้รับรางวัล EFF สําหรับการให้การเข้าถึงความรู้ทางวิทยาศาสตร์](https://torrentfreak.com/sci-hubs-alexandra-elbakyan-receives-eff-award-for-providing-access-to-scientific-knowledge-230728/)

- Alexandra Elbakyan ผู้ก่อตั้ง Sci-Hub ได้รับการยอมรับจาก Electronic Frontier Foundation (EFF) สําหรับความพยายามของเธอในการให้การเข้าถึงความรู้ทางวิทยาศาสตร์ฟรี
- Sci-Hub เป็นทรัพยากรที่มีค่าสําหรับนักเรียนและนักวิจัยซึ่งได้รับการยกย่องสําหรับบทบาทในการช่วยชีวิต
- Elbakyan ยังคงปกป้องภารกิจของ Sci-Hub แม้จะมีความท้าทายทางกฎหมายและความเสียหายนับล้านที่เป็นหนี้กับสํานักพิมพ์รายใหญ่
- EFF ยอมรับงานของ Elbakyan ในการท้าทายระบบการเผยแพร่ทางวิชาการในปัจจุบันและช่วยให้สามารถเข้าถึงข้อมูลทางวิทยาศาสตร์ได้หลายล้านคน
- Elbakyan จะได้รับรางวัลโดยตรงในพิธีในเดือนกันยายน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=36906435)

- Alexandra Elbakyan ผู้ก่อตั้ง Sci-Hub ได้รับรางวัลผู้บุกเบิก Electronic Frontier Foundation (EFF) จากความพยายามของเธอในการทําให้ความรู้ทางวิทยาศาสตร์สามารถเข้าถึงได้
- Elbakyan แสดงความไม่พอใจในการยอมรับรางวัลในนามของ Sci-Hub มากกว่าเป็นการส่วนตัวโดยเปรียบเทียบกับ Linus Torvalds และ Linux
- การอภิปรายเน้นย้ําถึงความสําคัญของ Sci-Hub ในการทําให้ความรู้เป็นประชาธิปไตยและความพร้อมของแพลตฟอร์มทางเลือกสําหรับการเข้าถึงเอกสารทางวิทยาศาสตร์
- อย่างไรก็ตามมีความกังวลว่ารางวัลนี้อาจถูกใช้เพื่อบ่อนทําลาย EFF ในคดีทางกฎหมายที่เกี่ยวข้องกับการละเมิดลิขสิทธิ์

## [Worldcoin ไม่ได้แย่อย่างที่คิด: มันแย่กว่านั้น](https://blockworks.co/news/worldcoin-privacy-concerns)

- Worldcoin เป็นระบบการเงินใหม่ที่รวบรวมข้อมูลไบโอเมตริกซ์ที่ละเอียดอ่อนและสัญญาว่าจะให้ความเท่าเทียมกันและความเป็นธรรม
- บทความนี้ทําให้เกิดความกังวลเกี่ยวกับวิธีการรวบรวมข้อมูลที่น่าสงสัยที่ใช้โดย Worldcoin และการสร้างตลาดมืดที่มีศักยภาพสําหรับข้อมูลประจําตัวที่ได้รับการยืนยัน
- ประสิทธิภาพของมาตรการความเป็นส่วนตัวของ Worldcoin และผลที่ตามมาของการเชื่อมโยงลักษณะไบโอเมตริกซ์ที่ไม่เปลี่ยนแปลงกับเงินก็ถูกตั้งคําถามเช่นกัน
- แผนการจัดจําหน่ายของ Worldcoin ซึ่งเกี่ยวข้องกับการจัดสรรโทเค็นจํานวนมากให้กับคนวงในถูกวิพากษ์วิจารณ์
- บทความชี้ให้เห็นว่า Worldcoin อาจไม่ปฏิวัติหรือเท่าเทียมอย่างที่กล่าวอ้าง

## [รัฐบาลสหราชอาณาจักรใกล้จะกัดกร่อนการเข้ารหัสทั่วโลก](https://www.eff.org/deeplinks/2023/07/uk-government-very-close-eroding-encryption-worldwide)

- การยืนยันของรัฐบาลว่าสามารถสแกนข้อความเพื่อหาเนื้อหาที่ผิดกฎหมายในขณะที่ปกป้องความเป็นส่วนตัวนั้นขัดแย้งกัน
- ผู้สนับสนุนกําลังเรียกร้องให้ฝ่ายนิติบัญญัติของสหราชอาณาจักรปกป้องการเข้ารหัสและความเป็นส่วนตัวโดยการรวมการแก้ไขร่างกฎหมาย

## [เหตุใด DNS จึงยังเรียนรู้ได้ยาก](https://jvns.ca/blog/2023/07/28/why-is-dns-still-hard-to-learn/)

- การเรียนรู้และการแก้ไขปัญหา DNS อาจเป็นเรื่องที่ท้าทายเนื่องจากลักษณะที่ซ่อนอยู่ของระบบ
- แคชของตัวแก้ไขและการสนทนาระหว่างตัวแก้ไขและเนมเซิร์ฟเวอร์ที่เชื่อถือได้มักจะถูกซ่อนจากผู้ใช้
- ผู้เขียนแนะนําให้สอนผู้คนเกี่ยวกับระบบที่ซ่อนอยู่เหล่านี้และให้การมองเห็นและการดีบักข้อมูลใน DNS มากขึ้น
- เครื่องมือที่สับสนและผลลัพธ์ของพวกเขาก่อให้เกิดความท้าทายในการแก้ไขปัญหา DNS และผู้เขียนเสนอการปรับปรุงเอาต์พุตเครื่องมือและสร้างเครื่องมือบรรทัดคําสั่งที่ใช้งานง่ายยิ่งขึ้น
- ความยากลําบากในการจัดทําเอกสารปัญหา DNS ทั่วไปเกิดขึ้นจากการเปลี่ยนแปลงของประสบการณ์ของแต่ละบุคคล
- การสัมผัสไม่บ่อยนักและความกลัวในการทดลองกับ DNS มีส่วนทําให้เกิดความยากลําบากในการเรียนรู้เรื่องนี้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=36909427)

- การเรียนรู้ DNS (Domain Name System) ก่อให้เกิดความท้าทายและความซับซ้อน
- การดีบักปัญหา DNS เป็นปัญหาทั่วไป
- อุปสรรคในการทําความเข้าใจ DNS จําเป็นต้องได้รับการแก้ไข
- จําเป็นต้องมีเอกสารและเครื่องมือที่ดีขึ้นเพื่อปรับปรุงการเรียนรู้ DNS
- มีการกล่าวถึงการปรับปรุงอินเทอร์เฟซผู้ใช้ของเครื่องมือหลัก FOSS (ซอฟต์แวร์ฟรีและโอเพ่นซอร์ส)
- การใช้เครื่องหมายอัฒภาคในการสืบค้น DNS เป็นหัวข้อถกเถียงกัน
- ความง่ายในการเรียนรู้ DNS ถูกตั้งคําถาม
- การทําความเข้าใจโครงสร้างพื้นฐานถือว่ามีความสําคัญในการเรียนรู้ DNS
- ทักษะทางเทคนิคที่ลดลงในหมู่นักพัฒนาที่อายุน้อยกว่าเป็นเรื่องที่น่ากังวล
- มีการกล่าวถึงโปรโตคอล DNS และไฟล์โซน
- มีการกล่าวถึงตัวเลือกซอฟต์แวร์ DNS ทางเลือก
- ประสบการณ์เฉพาะเกี่ยวกับการแก้ไขปัญหา DNS จะถูกแชร์
- DNS ใช้เพื่อวัตถุประสงค์ต่างๆ

## [โหมด No-GIL กําลังมาสําหรับ Python](https://lwn.net/Articles/939568/)

- สภากํากับดูแล Python กําลังพิจารณาข้อเสนอเพื่อให้ Global Interpreter Lock (GIL) เป็นตัวเลือกใน CPython
- สิ่งนี้จะเปิดใช้งานโหมด GIL-less ใน Python ซึ่งอาจปรับปรุงกระบวนการทํางานพร้อมกัน
- สภากําลังใช้ความระมัดระวังเพื่อให้แน่ใจว่ามีความเข้ากันได้แบบย้อนหลังและหลีกเลี่ยงสถานการณ์แยก Python 2 และ 3
- พวกเขากําลังหารือเกี่ยวกับข้อกําหนดสําหรับความเข้ากันได้ของ ABI และจัดการกับผลกระทบต่อความเข้ากันได้แบบย้อนหลัง
- ผู้วิจารณ์บางคนได้แสดงความกังวลเกี่ยวกับปัญหาความเข้ากันได้ที่อาจเกิดขึ้นและแนะนําแนวทางทางเลือกในการปรับปรุงกระบวนการทํางานพร้อมกันใน Python

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=36914401)

- Python กําลังแนะนําโหมด No-GIL (Global Interpreter Lock) ตามที่ยืนยันโดยโพสต์บน lwn.net
- Global Interpreter Lock เป็นกลไกใน Python ที่อนุญาตให้รัน Python bytecode ได้ครั้งละหนึ่งเธรดเท่านั้น
- โหมด No-GIL มีจุดมุ่งหมายเพื่อลบข้อ จํากัด นี้และอนุญาตให้หลายเธรดรัน Python bytecode พร้อมกัน
