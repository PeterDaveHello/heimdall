## [ข้อผิดพลาดใน macOS 14 Sonoma ป้องกันไม่ให้แอพของเราทํางาน](https://mullvad.net/en/blog/2023/9/13/bug-in-macos-14-sonoma-prevents-our-app-from-working/)

- ข้อผิดพลาดนี้อาจนําไปสู่การรั่วไหลภายใต้การตั้งค่าเฉพาะซึ่งก่อให้เกิดความเสี่ยงด้านความปลอดภัยที่อาจเกิดขึ้นสําหรับผู้ใช้
- ในขณะที่ปัญหาได้รับการรายงานไปยัง Apple ยังไม่มีวิธีแก้ไข ดังนั้นขอแนะนําให้ผู้ใช้อยู่บน macOS 13 Ventura จนกว่าจะมีการแก้ไข

## [โทรศัพท์ของผู้ร่วมก่อตั้ง Meduza ที่ติดเชื้อ Pegasus](https://meduza.io/en/feature/2023/09/13/the-million-dollar-reporter)

- ฝ่ายบริหารของ Biden ขึ้นบัญชีดํา NSO Group ในปี 2021 โดยห้ามไม่ให้เข้าถึงเทคโนโลยีของอเมริกาเนื่องจากความกังวลเกี่ยวกับการใช้ในทางที่ผิดและการกํากับดูแลด้านกฎระเบียบ
- Meduza, Access Now และ Citizen Lab ร่วมกันตรวจสอบเหตุการณ์โดยเปิดเผยการใช้เพกาซัสในทางที่ผิดอย่างกว้างขวางและตอกย้ําความกังวลเกี่ยวกับการเฝ้าระวังของรัฐบาลและการกําหนดเป้าหมายของนักข่าวฝ่ายตรงข้ามและนักเคลื่อนไหว

## [สเปกโตรมิเตอร์ใกล้ IR DIY](https://caoyuan.scripts.mit.edu/ir_spec.html)

- สเปกโตรมิเตอร์ทํางานโดยใช้ตะแกรงเลี้ยวเบนและโฟโตไดโอดท่ามกลางส่วนประกอบอื่น ๆ มันวัดความละเอียดสเปกตรัมและความยาวคลื่นของแสง
- นอกเหนือจากการหารือเกี่ยวกับความท้าทายที่เกี่ยวข้องกับการลดสัญญาณรบกวนและการรบกวนแล้ว Cao ยังพูดถึงการสอบเทียบช่วงไดนามิกของสเปกโตรมิเตอร์และการวัดสเปกตรัมการส่งผ่าน พวกเขาอาจเปิดเผยสเปกตรัมเพิ่มเติมในอนาคต

## [เมื่อ MFA ไม่ใช่ MFA หรือวิธีที่เราถูกฟิชชิ่ง](https://retool.com/blog/mfa-isnt-mfa/)

- Retool ซึ่งเป็นแพลตฟอร์มสําหรับการสร้างเครื่องมือภายในต้องเผชิญกับการโจมตีแบบสเปียร์ฟิชชิงที่นําไปสู่การเข้าถึงบัญชีลูกค้าบนคลาวด์ 27 บัญชีโดยไม่ได้รับอนุญาตเผยให้เห็นช่องโหว่ของระบบต่อการโจมตีทางวิศวกรรมสังคม
- การโจมตีเกิดขึ้นเมื่อพนักงานยอมจํานนต่อการโจมตีแบบฟิชชิงทาง SMS และให้รหัสการตรวจสอบสิทธิ์แบบหลายปัจจัย (MFA) ซึ่งต่อมาทําให้ VPN และระบบผู้ดูแลระบบภายในของบริษัทเสียหาย
- Retool แนะนําให้ปรับปรุงความปลอดภัย เช่น หลีกเลี่ยงการจัดเก็บรหัส MFA ในระบบคลาวด์ ส่งเสริมการฝึกอบรมพนักงานและการใช้คีย์ความปลอดภัยฮาร์ดแวร์สําหรับ MFA และการใช้กลยุทธ์การป้องกันเชิงลึก ขณะนี้บริษัทกําลังร่วมมือกับหน่วยงานบังคับใช้กฎหมายและบริษัทนิติวิทยาศาสตร์เพื่อสอบสวนเหตุการณ์ที่เกิดขึ้น

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37500895)

- การอภิปราย Hacker News ครอบคลุมแง่มุมต่างๆเช่นการระมัดระวังการโทร / ข้อความที่ขอข้อมูลส่วนบุคคลความจําเป็นในการตรวจสอบข้อมูลประจําตัวของผู้โทรและความต้องการระบบการตรวจสอบย้อนกลับมาตรฐาน
- ผู้เข้าร่วมจะหารือเกี่ยวกับจุดแข็งและขีดจํากัดของมาตรการรักษาความปลอดภัยต่างๆ รวมถึงการเข้ารหัสรหัสผ่าน การตรวจสอบสิทธิ์แบบหลายปัจจัย และไบโอเมตริกซ์ และแบ่งปันกรณีปัญหาการฉ้อโกงที่ผิดพลาด
- ปัญหาต่างๆเช่นการใช้เทคโนโลยี deepfake ในการละเมิดความปลอดภัยการปกป้องสินทรัพย์ดิจิทัลและข้อตกลงที่ครอบคลุมเกี่ยวกับข้อกําหนดของโซลูชันความปลอดภัยที่ใช้งานง่ายและมีประสิทธิภาพก็มีรายละเอียดเช่นกัน

## [อย่าใช้ Discord เป็นฟอรัมถาม &ตอบของคุณ](https://kraktoos.com/posts/dont-use-discord-as-forum/)

- ผู้เขียนกีดกันการใช้ Discord เป็นฟอรัมถาม & ตอบ (คําถามและคําตอบ) สําหรับโครงการโดยอ้างถึงลักษณะที่วุ่นวายคุณสมบัติการค้นหาและการค้นพบที่ต่ํากว่ามาตรฐานและความเสี่ยงในการไม่ต่อเนื่องที่อาจเกิดขึ้น
- ข้อความแนะนําทางเลือกที่ดีกว่าเช่นฟอรัมชุมชนเฉพาะเว็บไซต์ที่มีชื่อเสียงเช่น Stack Overflow หรือ Reddit และระบบติดตามปัญหาที่ใช้ Git
- ผู้เขียนเน้นย้ําถึงความสําคัญของการเลือกแพลตฟอร์มอย่างพิถีพิถันเพื่อวัตถุประสงค์ในการถามตอบโดยตระหนักถึงผลกระทบต่อความคืบหน้าของโครงการและการมีปฏิสัมพันธ์กับชุมชน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37502258)

- จุดสนใจหลักของการอภิปรายคือการใช้ Discord เป็นฟอรัมถาม & ตอบโดยมีมุมมองที่แตกต่างกันเกี่ยวกับประสิทธิภาพเมื่อเทียบกับแพลตฟอร์มเช่นปัญหา GitHub ฟอรัมเฉพาะและรายชื่อผู้รับจดหมาย
- ความไม่เป็นทางการและความสะดวกในการใช้งานของ Discord ได้รับการยกย่อง แต่ข้อ จํากัด ในการค้นหาและการจัดระเบียบจะถูกเน้น ทางเลือกของแพลตฟอร์มขึ้นอยู่กับความต้องการของแต่ละบุคคลเช่นการควบคุมต้นทุนความสะดวกในการติดตั้งและแนวโน้มทางประชากรศาสตร์
- พูดคุยเกี่ยวกับความล้าสมัยที่อาจเกิดขึ้นของฟอรัมและรายชื่อผู้รับจดหมายแบบดั้งเดิมและความเป็นไปได้ของแพลตฟอร์มที่ครอบคลุมมากขึ้นด้วยเครื่องมือค้นหาที่จัดทําดัชนีการตอบกลับที่ซ้อนกันและฟังก์ชันเสียง

## [ข้าม API จัดส่งฐานข้อมูลของคุณ](https://fly.io/blog/skip-the-api/)

- LiteFS เป็นฐานข้อมูล SQLite แบบกระจายทั่วโลกซึ่งเดิมออกแบบมาเพื่อกระจายข้อมูลทางภูมิศาสตร์เพื่อให้มั่นใจถึงเวลาตอบสนองที่รวดเร็ว
- มีการค้นพบการใช้ LiteFS แบบใหม่ทําให้สามารถแทนที่เลเยอร์ API ระหว่างบริการได้ สิ่งนี้ทําได้โดยการกระจายฐานข้อมูลที่สมบูรณ์ไปยังฝั่งไคลเอ็นต์ทําให้สามารถสืบค้น SQL และเข้าร่วมข้อมูลตามความต้องการของลูกค้าได้
- แบบจําลองเป็นแบบอ่านอย่างเดียวซึ่งจําเป็นต้องมี API สําหรับการอัปเดตข้อมูล แม้ว่าจะมีข้อ จํากัด บางประการ แต่เทคนิคนี้จะเป็นประโยชน์ในสถานการณ์ที่ API ทั่วไปไม่เพียงพอ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37497345)

## [การสนับสนุน WASI ใน Go](https://go.dev/blog/wasi)

- ทิศทางในอนาคตสําหรับ Go รวมถึงการส่งออกฟังก์ชัน Go ไปยัง WebAssembly และความเข้ากันได้กับ WASI API ที่กําลังพัฒนาและนักพัฒนามีช่องทางเช่น Gophers Slack และตัวติดตามปัญหา Go เพื่อมีส่วนร่วม

- ด้วยการลงคะแนนเสียงแบบสหภาพแรงงานในขณะนี้คนงาน VFX จะเข้าสู่การเจรจาต่อรองร่วมกันกับ Marvel Studios แม้ว่าการเจรจาอย่างต่อเนื่องกับกลุ่มแรงงานอื่น ๆ อาจทําให้การหารือเหล่านี้ล่าช้า
