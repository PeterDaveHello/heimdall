# 2023-09-11

## [Microsoft ยังไม่หยุดบังคับ Edge สําหรับผู้ใช้ Windows 11](https://www.ctrl.blog/entry/windows-system-components-default-edge.html)

- การตั้งค่าเว็บเบราว์เซอร์เริ่มต้นใน Windows 11 ยังคงไม่เปลี่ยนแปลงแม้จะมีการอ้างสิทธิ์ในบล็อกโพสต์ว่า Microsoft จะพิจารณาการตั้งค่าเบราว์เซอร์ของผู้ใช้
- ความเข้าใจผิดเกิดจากบันทึกการเปลี่ยนแปลงบิลด์ตัวอย่าง Windows 11 Insider ล่าสุด ทําให้หลายคนเชื่อว่า Microsoft ยอมรับแรงกดดันด้านกฎระเบียบ
- การทดสอบเชิงลึกเผยให้เห็นว่า Windows 11 เสนอ Microsoft Edge เป็นเบราว์เซอร์เริ่มต้นอย่างต่อเนื่องโดยแทนที่เบราว์เซอร์ที่ผู้ใช้เลือก รายงานเบื้องต้นเกี่ยวกับการเปลี่ยนแปลงถูกตีความผิดและเผยแพร่อย่างกว้างขวางในสื่อโดยไม่มีการยืนยันอย่างเป็นทางการจาก Microsoft

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37461449)

- การอภิปรายหลักหมุนรอบความไม่พอใจของผู้ใช้กับความพยายามล่วงล้ําของ Microsoft Edge ที่จะเป็นเบราว์เซอร์เริ่มต้นและข้อ จํากัด กับระบบปฏิบัติการ Windows
- มีความปรารถนาที่โดดเด่นสําหรับความหลากหลายของเบราว์เซอร์มากขึ้นโดยผู้ใช้แสดงความกังวลเกี่ยวกับการครอบงําของ Google และบางคนชอบเบราว์เซอร์ทางเลือกเช่น Chromium หรือ Brave
- ผู้ใช้กล่าวถึงระบบปฏิบัติการทางเลือกเช่น Ubuntu, Arch Linux และการกระจาย Windows ที่กําหนดเองในขณะเดียวกันพูดคุยเกี่ยวกับเครื่องมือต่างๆเช่น Emacs และ VSCode

## [Knightmare: A DevOps Cautionary Tale (ค.ศ. 2014)](https://dougseven.com/2014/04/17/knightmare-a-devops-cautionary-tale/)

- ผลงานชิ้นนี้เล่าถึงการล่มสลายของ Knight Capital Group ซึ่งเป็น บริษัท ผู้ให้บริการทางการเงินที่ดําเนินการในเวลาไม่ถึงหนึ่งชั่วโมงเนื่องจากการปรับใช้ระบบการซื้อขายอัตโนมัติไม่สําเร็จ
- มันเน้นถึงลักษณะที่จําเป็นของการปรับใช้อัตโนมัติและทําซ้ําได้ในอาณาจักร DevOps
- ความสําคัญของการจัดการการกําหนดค่าที่ถูกต้องและการทดสอบในการปรับใช้ซอฟต์แวร์ก็เน้นเช่นกัน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37459495)

- การสนทนามุ่งเน้นไปที่เหตุการณ์ Knightmare ปี 2014 ที่ความล้มเหลวของ DevOps ทําให้เกิดความสูญเสียทางการเงินอย่างมากสําหรับ Knight Capital เนื่องจากการเขียนโค้ดที่เข้ากันไม่ได้และขาด kill switch
- ผู้เข้าร่วมเน้นย้ําถึงความจําเป็นของระบบอัตโนมัติ kill switch และการบริหารความเสี่ยงเพื่อป้องกันปัญหาดังกล่าว ซึ่งบ่งบอกถึงความสําคัญของการทํางานร่วมกันระหว่างภาคซอฟต์แวร์และการเงิน
- บทสนทนายังกล่าวถึงความเสี่ยงที่เกี่ยวข้องกับการซื้อขายความถี่สูงผลพวงของข้อผิดพลาดในการปรับใช้และความสําคัญของกรอบการทํางานที่ถูกต้องระบบอัตโนมัติและความรับผิดชอบในอุตสาหกรรมการเงิน

## [โลกร้อนที่สุด 3 เดือนเป็นประวัติการณ์ อุณหภูมิทะเลที่ไม่เคยมีมาก่อนและสภาพอากาศที่รุนแรง](https://public.wmo.int/en/media/press-release/earth-had-hottest-three-month-period-record-unprecedented-sea-surface)

- ในช่วงสามเดือนที่ผ่านมาเป็นช่วงที่ร้อนแรงที่สุดเป็นประวัติการณ์สําหรับโลกโดยจําลองรูปแบบการเปลี่ยนแปลงสภาพภูมิอากาศที่ทําให้เกิดเหตุการณ์สภาพอากาศที่รุนแรงและอุณหภูมิพื้นผิวทะเลที่ไม่เคยมีมาก่อน
- สิงหาคม 2023 กลายเป็นเดือนสิงหาคมที่อบอุ่นที่สุดเท่าที่เคยมีมา ทําให้ปีนี้กลายเป็นปีที่อบอุ่นที่สุดเป็นอันดับสองในประวัติการณ์ ตามหลังเพียง 2016
- นักวิทยาศาสตร์ระบุว่าสภาพภูมิอากาศที่ทําลายสถิติเหล่านี้ส่งผลต่อการเปลี่ยนแปลงสภาพภูมิอากาศและเรียกร้องให้ดําเนินการอย่างเร่งด่วนเพื่อป้องกันผลกระทบต่อสิ่งแวดล้อมที่เลวร้าย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37455534)

- การอภิปรายครอบคลุมถึงความเร่งด่วนและความจําเป็นในการดําเนินการกับการเปลี่ยนแปลงสภาพภูมิอากาศหารือเกี่ยวกับแนวทางแก้ไขที่เป็นไปได้และผลกระทบต่าง ๆ รวมถึงเหตุการณ์สภาพอากาศที่รุนแรงภาวะโลกร้อนที่เกี่ยวข้องกับมนุษย์และผลกระทบที่อาจเกิดขึ้นจากการเปลี่ยนแปลงสภาพภูมิอากาศ
- หัวข้อที่สําคัญ ได้แก่ วิศวกรรมธรณีผลกระทบทางการเมืองผลกระทบต่อการเกษตรและการผลิตอาหารและความรับผิดชอบของประเทศต่างๆในการจัดการการเปลี่ยนแปลงสภาพภูมิอากาศ
- มีการถกเถียงกันเกี่ยวกับการปฏิเสธการเปลี่ยนแปลงสภาพภูมิอากาศความพยายามในการลดการปล่อยมลพิษด้วยความรู้สึกโดยรวมของความเร่งด่วนกังวลเกี่ยวกับผลกระทบในอนาคตของการเปลี่ยนแปลงสภาพภูมิอากาศแนะนําความคิดเห็นและมุมมองที่หลากหลาย

## [เครื่องยนต์ Peredvizhnikov: เอ็นจิ้นเกมล็อคฟรีเขียนใน C ++ 20](https://github.com/eduard-permyakov/peredvizhnikov-engine)

- เครื่องยนต์ Peredvizhnikov เป็นเอ็นจิ้นเกมที่ปราศจากการล็อคที่เขียนด้วย C ++ 20 ซึ่งออกแบบในรูปแบบนักแสดงของการคํานวณพร้อมกันทําให้สามารถพัฒนาตรรกะแบบขนานและการแยกการซิงโครไนซ์ระหว่างเธรด
- เครื่องยนต์มีความทนทานต่อความผิดพลาดรับประกันว่าจะคงอยู่แม้ว่าเธรดของคนงานจะถูกฆ่ารวมถึงอัลกอริธึมที่ปราศจากการล็อคและได้รับใบอนุญาต GPLV3 แต่สามารถได้รับใบอนุญาตแตกต่างกันหากได้รับอนุญาตจากผู้สร้าง
- ปัจจุบันเอ็นจิ้นรองรับเฉพาะ Linux และต้องใช้ Clang ++ 16 สําหรับการสร้าง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37456471)

## [Linux NAT a ping อย่างไร](https://devnonsense.com/posts/how-does-linux-nat-a-ping/)

- ผู้เขียนได้ทําการทดลองเพื่อทําความเข้าใจว่า Network Address Translation (NAT) ทํางานอย่างไรเกี่ยวกับแพ็กเก็ต ICMP ในสภาพแวดล้อม Linux
- พวกเขาได้เจาะลึกโครงสร้างและกลไกของข้อความตอบกลับ ICMP echo และ echo พร้อมกับการใช้คําสั่ง 'ping' และการสุ่มของการเลือกตัวระบุ ICMP
- การใช้เครื่องมือที่เรียกว่า 'bpftrace' สําหรับการติดตามฟังก์ชั่นเคอร์เนลผู้เขียนสังเกตและยืนยันการปรับเปลี่ยนในที่อยู่ IP ต้นทางและปลายทางในแพ็กเก็ต ICMP เนื่องจากกระบวนการ NAT

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37455621)

- บทความสํารวจหัวข้อต่างๆ ที่เกี่ยวข้องกับ Network Address Translation (NAT), IPv6 และการใช้ ping สําหรับเครือข่ายเพียร์ทูเพียร์ รวมถึงวิธีที่ Linux จัดการกับ NAT และ ping
- มีการกล่าวถึงข้อดี ความท้าทาย และข้อจํากัดของการกําหนดค่า NAT และ IPv6 ที่มีอยู่ รวมถึงโซลูชันที่เป็นไปได้สําหรับการปรับปรุงการสื่อสารแบบเพียร์ทูเพียร์
- ในขณะที่ดูเหมือนว่าจะมีมุมมองและข้อกังวลที่หลากหลายเกี่ยวกับเทคโนโลยีเหล่านี้และการนําไปใช้บทความมีจุดมุ่งหมายเพื่อให้มุมมองที่ครอบคลุมของหัวข้อ

## [ฉันสาย GPS ของจักรยานของฉันเพื่อสั่งพิซซ่าระหว่างการแข่งขันกรวด](https://steele.blue/geofence-pizza-ordering/)

## [Google ไม่มีการจดทะเบียนโดเมนใหม่อีกต่อไป](https://domains.google/)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37455574)

- Google ได้โอนบริการจดทะเบียนโดเมน Google Domains ไปยัง Squarespace การพัฒนานี้จุดประกายการอภิปรายและทําให้เกิดความกังวลเกี่ยวกับกลยุทธ์ของ Google และการจัดสรรทรัพยากรในอนาคต
- ขอแนะนําให้ใช้ตัวเลือกผู้รับจดทะเบียนโดเมนอื่น ๆ เช่น Porkbun, Namecheap, Dynadot, Cloudflare, Nearlyfreespeech และ Gandi มีความกังวลเกี่ยวกับการจัดการใหม่ของ Gandi ในขณะที่ Cloud DNS ซึ่งเป็นบริการจัดการโดเมนของ Google Cloud ยังคงไม่ได้รับผลกระทบ ในทางกลับกันความกังวลเกี่ยวกับอนาคตของ Google Cloud Platform (GCP) จะถูกบันทึกไว้
- การเปลี่ยนแปลงนี้เน้นประเด็นเรื่องความไว้วางใจความน่าเชื่อถือและความเสี่ยงในการพึ่งพา บริษัท เดียวสําหรับบริการต่างๆ ความสนใจถูกดึงไปที่ความจําเป็นในการพิจารณาผู้ให้บริการทางเลือกและการจัดการโดเมนเปรียบเทียบของ Google

## [ระบบการออกแบบสําหรับรัฐบาลกลาง](https://designsystem.digital.gov/)

- US Web Design System (USWDS) เป็นเครื่องมือสําหรับรัฐบาลกลางที่ช่วยในการสร้างเว็บไซต์ของรัฐบาลที่เข้าถึงได้และเป็นมิตรกับมือถือ
- มีส่วนประกอบรูปแบบโทเค็นการออกแบบและยูทิลิตี้สําหรับการออกแบบและใช้งานประสบการณ์ผู้ใช้
- USWDS เป็นชุมชนโอเพ่นซอร์สที่ให้การสนับสนุนหน่วยงานและเว็บไซต์ของรัฐบาลต่างๆ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37459341)

- รัฐบาลสหรัฐฯ ได้ประกาศใช้ US Web Design System (USWDS) ในความพยายามที่จะปรับปรุงเว็บไซต์ของรัฐบาลให้ทันสมัยและปรับปรุงบริการดิจิทัล ซึ่งสะท้อนให้เห็นถึงความคิดริเริ่มที่คล้ายคลึงกันในประเทศอื่นๆ
- การอภิปรายยังคงแบ่งตามความสําเร็จของโครงการเหล่านี้โดยมีประเด็นหลักของความขัดแย้งที่เกี่ยวข้องกับการใช้งานการออกแบบที่เรียบง่ายและคุณค่าของระบบมาตรฐานและไลบรารีไอคอน
- การถกเถียงกันเกี่ยวกับการใช้สาธารณสมบัติและใบอนุญาต GPL ในการดําเนินงานของรัฐบาลซึ่งความกังวลและการเก็งกําไรเกิดขึ้นเกี่ยวกับการใช้ระบบไอคอนที่มีอยู่และการบํารุงรักษาเทคโนโลยีที่คาดหวัง
