## [โซลูชันการคํานวณสําหรับ Othello Uncovered: เกมสามารถนําไปสู่การเสมอด้วยการเล่นที่สมบูรณ์แบบนักวิทยาศาสตร์ Hiroki Takizawa กล่าว](https://arxiv.org/abs/2310.19387)

- Hiroki Takizawa ได้ออกกระดาษประกาศว่าเกมที่ซับซ้อนของ Othello ได้รับการแก้ไขด้วยการคํานวณแล้ว
- Othello มีสถานะความท้าทายที่สําคัญในด้านวิทยาการคอมพิวเตอร์เนื่องจากมีบันทึกเกมที่มีศักยภาพประมาณสิบ octodecillion และตําแหน่งเกมที่มีศักยภาพสิบ octillion
- ความสําเร็จนี้บอกเป็นนัยว่าตอนนี้ได้พิสูจน์แล้วว่าการเล่นที่สมบูรณ์แบบโดยผู้เล่นทั้งสองใน Othello สามารถนําไปสู่การเสมอกันได้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38141366)

- เอกสารล่าสุดโดย Hiroki Takizawa เปิดเผยว่าเกมของ Othello ซึ่งถือเป็นหนึ่งในเกมที่ซับซ้อนที่สุดในโลกเนื่องจากบันทึกและตําแหน่งเกมที่เป็นไปได้มากมายได้รับการแก้ไขด้วยการคํานวณ
- Othello เป็นความท้าทายที่สําคัญสําหรับวิทยาการคอมพิวเตอร์เนื่องจากการประมาณการคร่าวๆของบันทึกเกมที่เป็นไปได้สิบ octodecillion และตําแหน่งเกมที่เป็นไปได้สิบ octillion
- ความก้าวหน้านี้หมายความว่าการเล่นที่สมบูรณ์แบบโดยผู้เข้าร่วมทั้งสองสามารถแสดงให้เห็นด้วยการคํานวณเพื่อให้เกิดผลเสมอ

## [NewPipe: แอป YouTube แบบโอเพ่นซอร์สที่เป็นมิตรกับความเป็นส่วนตัวสําหรับผู้ใช้ Android ที่หลากหลายรองรับหลายแพลตฟอร์มแล้ว](https://newpipe.net/)

- NewPipe เป็นแอปพลิเคชั่น YouTube ที่เน้นความเป็นส่วนตัวที่มีน้ําหนักเบาสําหรับอุปกรณ์ Android ที่มุ่งมอบประสบการณ์ผู้ใช้ที่ไม่มีโฆษณาบนสมาร์ทโฟน
- ในฐานะที่เป็นแอปโอเพ่นซอร์สที่มีอยู่ใน GitHub NewPipe มีคุณสมบัติมากมายรวมถึงการใช้งานแบบออฟไลน์การสมัครสมาชิกประวัติวิดีโอและความเข้ากันได้กับแพลตฟอร์มอื่น ๆ เช่น PeerTube, SoundCloud และ Bandcamp
- แอปที่ขับเคลื่อนโดยชุมชนที่กําลังเติบโตสนับสนุนการมีส่วนร่วมของผู้ใช้และการแปลโดยมีข้อกําหนดสําหรับการอัปเดตที่รวดเร็วยิ่งขึ้นผ่านการเพิ่มที่เก็บ F-Droid ให้กับลูกค้า

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38144400)

- NewPipe เป็นแอป YouTube แบบโอเพ่นซอร์สที่เน้นความเป็นส่วนตัวสําหรับอุปกรณ์ Android ซึ่งออกแบบมาเพื่อมอบประสบการณ์ YouTube ที่ใช้งานง่ายและไม่มีโฆษณาโดยไม่มีสิทธิ์ล่วงล้ํา
- นอกจาก YouTube แล้วแอปยังรองรับแพลตฟอร์มต่างๆรวมถึง PeerTube, SoundCloud และ Bandcamp คุณสมบัติของมันรวมถึงเครื่องเล่นพื้นหลังเครื่องเล่นป๊อปอัปรองรับการใช้งานออฟไลน์การสมัครสมาชิกบุ๊กมาร์กและประวัติวิดีโอ
- การอัปเดตของแอปสามารถรับได้เร็วขึ้นโดยการเพิ่มที่เก็บ F-Droid ให้กับลูกค้าและยินดีต้อนรับการมีส่วนร่วมและการแปลจากผู้ใช้ ในฐานะที่เป็นแอปโอเพ่นซอร์สมีให้บริการฟรีบน GitHub

## [ล่าสุด Bevy 0.12 Game Engine Release Features Deferred Rendering, Improved Shadow Quality และ Optimized Renderer](https://bevyengine.org/news/bevy-0-12/)

- เอ็นจิ้นเกม Bevy 0.12 เปิดตัวโดยมีการอัปเดตหลายอย่างเช่น Deferred Rendering, Bevy Asset V2, PCF Shadow Filtering และ Renderer Optimizations
- เวอร์ชันนี้แนะนําการนําเข้า Rusty Shader, Suspend & Resume บน Android และ Automatic Batching และ Instancing of Draw Commands รวมถึงการปรับปรุงอื่น ๆ
- แม้จะมีการอัปเดตที่สําคัญสําหรับเกม 2D และ 3D แต่แนะนําให้ใช้โหมดไปข้างหน้าเนื่องจากความเข้ากันได้ของอุปกรณ์ที่กว้างขึ้น Bevy Asset V2 ในระยะแรกรองรับฟังก์ชันต่างๆ เช่น การนําเข้าสินทรัพย์และเหตุการณ์โหลดการพึ่งพาสินทรัพย์ซ้ํา

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38144417)

- Bevy 0.12 เอ็นจิ้นเกม Bevy เวอร์ชันใหม่ที่มีการอัปเดตเช่น Deferred Rendering, Bevy Asset V2, PCF Shadow Filtering, StandardMaterial Light Transmission และ Renderer Optimizations

- ในขณะที่การอัปเดตรองรับทั้งเกม 2D และ 3D ขอแนะนําให้ใช้โหมดไปข้างหน้าสําหรับความเข้ากันได้ของอุปกรณ์ที่แพร่หลาย Bevy Asset V2 แม้ว่าจะอยู่ในช่วงเริ่มต้น แต่ก็รองรับการนําเข้าสินทรัพย์และเหตุการณ์โหลดการพึ่งพาสินทรัพย์ซ้ํา

## [การปรับปรุงการควบคุมผู้ป่วย: ข้อดีของซอฟต์แวร์ฟรีในอุปกรณ์การแพทย์มากกว่าเวอร์ชันที่เป็นกรรมสิทธิ์](https://www.fsf.org/blogs/community/software-that-supports-your-body-should-always-respect-your-freedom)

- บทความวิพากษ์วิจารณ์ข้อเสียของซอฟต์แวร์ที่เป็นกรรมสิทธิ์ในอุปกรณ์การแพทย์ซึ่งบ่งชี้ว่าผู้ใช้มีการควบคุมน้อยที่สุดซึ่งนําไปสู่ความผิดปกติของอุปกรณ์ที่อาจเกิดขึ้นหลังจากการอัปเดต
- เสนอว่าหากอุปกรณ์เหล่านี้ทํางานบนซอฟต์แวร์ฟรีผู้ใช้สามารถทําการเปลี่ยนแปลงที่จําเป็นและพึ่งพาชุมชนนักพัฒนาสําหรับการแก้ปัญหาข้อบกพร่องในเวลาที่เหมาะสม
- ชิ้นนี้วิงวอนให้ผู้ป่วยตั้งคําถามถึงประเภทของซอฟต์แวร์ที่ใช้ในความช่วยเหลือทางการแพทย์ของพวกเขาและเพื่อสนับสนุนซอฟต์แวร์ฟรีมากกว่ารูปแบบที่เป็นกรรมสิทธิ์เพื่อเพิ่มการควบคุมและความน่าเชื่อถือ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38143180)

- บทความนี้เน้นข้อบกพร่องของซอฟต์แวร์ที่เป็นกรรมสิทธิ์ในอุปกรณ์การแพทย์ที่เน้นช่องโหว่เนื่องจากการอัปเดตซอฟต์แวร์ที่ไม่สามารถควบคุมได้ซึ่งอาจทําให้เกิดการทํางานผิดปกติ
- มันชี้ให้เห็นว่าซอฟต์แวร์ฟรีจะช่วยให้ผู้ใช้สามารถแก้ไขและพึ่งพาชุมชนนักพัฒนาสําหรับการแก้ไขข้อบกพร่องสร้างระบบที่เชื่อถือได้และควบคุมได้มากขึ้น
- โซลูชันที่นําเสนอสนับสนุนให้ผู้ป่วยตรวจสอบซอฟต์แวร์ที่ใช้ในอุปกรณ์การแพทย์และเลือกใช้ซอฟต์แวร์ฟรีแทนซอฟต์แวร์ที่เป็นกรรมสิทธิ์เพื่อการควบคุมและความรับผิดชอบที่ดีขึ้น

- พบช่องโหว่ซีโร่เดย์สี่รายการใน Microsoft Exchange ที่สามารถใช้ประโยชน์จากระยะไกลเพื่อเรียกใช้รหัสที่กําหนดเองหรือเปิดเผยข้อมูลที่ละเอียดอ่อน
- Microsoft ได้รับแจ้งถึงช่องโหว่เหล่านี้โดย Zero Day Initiative (ZDI) ของ Trend Micro แต่ตามคําตัดสินถือว่าไม่รุนแรงพอที่จะสั่งให้ดําเนินการทันที
- ZDI ได้แนะนําให้ จํากัด การโต้ตอบกับแอปพลิเคชัน Exchange การใช้การรับรองความถูกต้องแบบหลายปัจจัยและกล่าวว่าช่องโหว่บางอย่างจําเป็นต้องเข้าถึงข้อมูลประจําตัวอีเมลก่อน Microsoft ระบุว่าได้ใช้มาตรการป้องกันและจะแก้ไขปัญหาเหล่านี้ในการอัปเดตและผลิตภัณฑ์ในอนาคต

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38145461)

- พบช่องโหว่ซีโร่เดย์สี่จุด รวมถึงข้อบกพร่องของการเรียกใช้โค้ดจากระยะไกล (RCE) และปัญหาการตรวจสอบความถูกต้องของข้อมูลใน Microsoft Exchange ซึ่งอาจนําไปสู่การเรียกใช้โค้ดโดยไม่ได้รับอนุญาตหรือการเปิดเผยข้อมูลที่ละเอียดอ่อน
- แม้จะได้รับแจ้งจาก Zero Day Initiative (ZDI) ของ Trend Micro แต่ Microsoft ก็ไม่ได้พิจารณาว่าปัญหาเหล่านี้รุนแรงพอสําหรับการดําเนินการทันที ช่องโหว่ที่ระบุทั้งหมดจําเป็นต้องมีการรับรองความถูกต้องสําหรับการแสวงหาประโยชน์
- ZDI แนะนําให้จํากัดการโต้ตอบกับแอปพลิเคชัน Exchange และใช้การรับรองความถูกต้องแบบหลายปัจจัยเป็นกลยุทธ์การบรรเทาผลกระทบ ในขณะเดียวกัน Microsoft อ้างว่าได้ใช้มาตรการป้องกันและวางแผนที่จะแก้ไขช่องโหว่เหล่านี้ในการอัปเดตและผลิตภัณฑ์ในอนาคต ปัญหาบางอย่างได้รับการแก้ไขแล้วหรือต้องการการเข้าถึงอีเมลก่อน
