# 2023-11-03

## [Sam Bankman-Fried ถูกตัดสินว่ามีความผิด](https://www.nytimes.com/live/2023/11/02/business/sam-bankman-fried-trial)

- Sam Bankman-Fried ผู้ก่อตั้งการแลกเปลี่ยน crypto FTX ถูกตัดสินว่ามีความผิดฐานฉ้อโกงและการสมรู้ร่วมคิดเจ็ดครั้งโดยกล่าวหาว่าเขายักยอกเงินประมาณ 10 พันล้านดอลลาร์จากผู้ใช้ของเขา
- การล่มสลายของ FTX และค่าใช้จ่ายที่ตามมากําลังถูกใช้เป็นตัวอย่างของอันตรายและส่วนเกินในอุตสาหกรรมสกุลเงินดิจิทัล
- พยานรวมถึงอดีตแฟนสาวและอดีตพนักงานของเขาเป็นพยานต่อ Bankman-Fried ระหว่างการพิจารณาคดีโดยกล่าวหาว่าเขายักยอกเงินของลูกค้าสําหรับค่าใช้จ่ายส่วนตัวและการบริจาคทางการเมือง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38122061)

- การอภิปรายออนไลน์ที่เชื่อมโยงกับโพสต์ของ New York Times กล่าวถึงความเชื่อมั่นของ Sam Bankman-Fried ซีอีโอของ FTX (SBF) เนื่องจากถูกกล่าวหาว่าประพฤติมิชอบในพื้นที่ crypto รวมถึงการยักยอกเงินของลูกค้า
- ผู้เข้าร่วมการอภิปรายตั้งคําถามถึงความประพฤติของศาลของ SBF และการปฏิเสธที่จะให้คําตอบที่ชัดเจนโดยมีข้อโต้แย้งเกี่ยวกับตัวละครการเลี้ยงดูและผลกระทบที่อาจสร้างความเสียหายต่อกรณีนี้อาจมีต่อการรับรู้และกฎระเบียบของอุตสาหกรรมคริปโต
- การสนทนาของผู้ใช้ยังรวมถึงขั้นตอนทางกฎหมายการตั้งคําถามและความเป็นไปได้ที่ออทิสติกอาจส่งผลกระทบต่อการตอบสนองของศาลนอกเหนือจากคําเตือนเกี่ยวกับธงแดงสําหรับการดําเนินธุรกิจที่ฉ้อโกงโดยเน้นย้ําถึงความจําเป็นในการโปร่งใสและการควบคุมภายในที่แข็งแกร่ง

- แพลตฟอร์ม Meta ที่ Facebook เป็นเจ้าของกําลังดําเนินการเพื่อยึดมั่นและพิจารณารูปแบบการสมัครสมาชิกเพื่อให้เป็นไปตามกฎระเบียบใหม่ ซึ่งอาจส่งผลกระทบต่อผู้ใช้ประมาณ 250 ล้านคนทั่วยุโรป

## [คําศัพท์ Git ที่สับสน](https://jvns.ca/blog/2023/11/01/confusing-git-terminology/)

- Julia Evans ลดความซับซ้อนของคําศัพท์ Git โดยเน้นว่าความเข้าใจอย่างถ่องแท้เกี่ยวกับข้อกําหนดเหล่านี้มีความสําคัญต่อการใช้ Git อย่างมีประสิทธิภาพซึ่งเป็นระบบโอเพ่นซอร์สสําหรับการติดตามการเปลี่ยนแปลงในซอฟต์แวร์
- เธอเจาะลึกคําศัพท์ที่ซับซ้อนของ Git เช่น "HEAD and heads", "ours and theirs during merging or rebasing," "detached HEAD state," "ahead of 'origin/main' by 1 commit" เป็นต้น ซึ่งอธิบายความแตกต่างระหว่างพวกเขา
- บทความนี้เน้นถึงความต้องการความชัดเจนเพิ่มเติมในคําศัพท์ของ Git เนื่องจากลักษณะและความซับซ้อนหลายแง่มุมให้ความกระจ่างเกี่ยวกับแนวคิดเช่น "รีเซ็ต", "ย้อนกลับ", "กู้คืน", คําว่า "ติดตาม" ในบริบทที่แตกต่างกันและความแตกต่างระหว่าง "ผสาน", "rebase" และ "cherry-pick"

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38112951)

- เธรด Hacker News เน้นถึงความยากลําบากของผู้ใช้ในการทําความเข้าใจคําศัพท์ที่ซับซ้อนของ Git ซึ่งเป็นระบบควบคุมเวอร์ชันที่แพร่หลายในการพัฒนาซอฟต์แวร์และกราฟ acyclic (DAG) ที่กํากับโดยพื้นฐาน
- ผู้ใช้ประสบปัญหาในการทําความเข้าใจคําสั่ง Git อาร์กิวเมนต์เริ่มต้นและความซับซ้อนของการรวมในขณะที่ประเมินเอกสารที่ชัดเจนและคําสั่ง 'git reflog'
- แม้จะมีปัญหาเหล่านี้ แต่ผู้ใช้ยกย่องบทบาทสําคัญของ Git ในการพัฒนาซอฟต์แวร์และความเหนือกว่าระบบควบคุมเวอร์ชันทางเลือกเช่น Fossil, SVN, Perforce และ Mercurial

## [ในปี พ.ศ. 1886 สหรัฐอเมริกาได้มอบหมายให้วาดภาพสีน้ําของผลไม้ที่รู้จักทุกชิ้น (2019)](https://www.openculture.com/2019/06/the-us-government-commissioned-7500-watercolor-paintings.html)

- ในปี พ.ศ. 2429 รัฐบาลสหรัฐได้มอบหมายภาพสีน้ํา 7,500 ภาพของผลไม้ทุกชนิดที่รู้จักซึ่งรู้จักกันในชื่อ USDA Pomological Watercolor Collection
- เผยแพร่สู่สาธารณะในปี 2015 โดย Parker Higgins นักเคลื่อนไหว EFF คอลเล็กชันนี้มีบอท Twitter ที่ส่งภาพแบบสุ่มจากที่เก็บถาวรและได้รับคําวิจารณ์ที่หลากหลาย
- ข้อความนี้ยังกล่าวถึงแหล่งข้อมูลการศึกษาฟรีประเภทต่างๆรวมถึง Open Culture, LLC ซึ่งเป็น บริษัท ที่ให้บริการเช่นหนังสือเสียงหลักสูตรออนไลน์ภาษาและตําราเรียน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38121250)

- สหรัฐอเมริกาเคยมอบหมายให้วาดภาพผลไม้ที่รู้จักทุกชิ้นคล้ายกับภาพวาดนกของ John James Audubon ซึ่งทั้งสองเป็นของสะสมที่มีค่าในปัจจุบัน
- มีการถกเถียงกันอย่างต่อเนื่องเกี่ยวกับบุคคลในประวัติศาสตร์เช่น Audubon โดยเฉพาะอย่างยิ่งการเชื่อมโยงไปยังประเด็นที่ถกเถียงกันเช่นการเป็นทาสและควรอนุรักษ์หรือรื้อถอนเกียรติยศและอนุสาวรีย์ที่เชื่อมโยงกับพวกเขาหรือไม่
- การแปลงงานประวัติศาสตร์เป็นดิจิทัลแม้จะมีค่าใช้จ่ายสูง แต่ก็ถือว่ามีประโยชน์เนื่องจากช่วยให้สามารถอนุรักษ์เข้าถึงได้กว้างขึ้นและมีศักยภาพในการทํากําไรผ่านการขายการเข้าถึงความละเอียดสูง

## [เจ้าของบ้านรายใหญ่ใช้ซอฟต์แวร์เพื่อสมรู้ร่วมคิดกับราคาค่าเช่าคดี DC กล่าวว่า](https://arstechnica.com/tech-policy/2023/11/14-big-landlords-used-software-to-collude-on-rent-prices-dc-lawsuit-says/)

## [ทดลองใช้ Android WebView Media Integrity API ต้นปีหน้า](https://android-developers.googleblog.com/2023/11/increasing-trust-for-embedded-media.html)

- ทีม Android เป็นผู้บุกเบิก Android WebView Media Integrity API รุ่นทดลอง โดยมีเป้าหมายเพื่อเพิ่มมาตรการรักษาความปลอดภัยและความเป็นส่วนตัวสําหรับสื่อฝังตัวในแอปพลิเคชัน Android
- API นี้ซึ่งมีกําหนดการทดสอบกับผู้ให้บริการสื่อฝังตัวที่เลือกในปีหน้าพยายามเปิดใช้งานการแสดงสื่อฝังตัวบนหน้าเว็บภายในแอปในขณะที่รักษาสภาพแวดล้อมที่ปลอดภัยและป้องกันการฉ้อโกงและการละเมิดที่อาจเกิดขึ้น
- API นี้นําเสนอคําตัดสินความสมบูรณ์ของอุปกรณ์และแอปให้กับผู้ให้บริการสื่อ API นี้ช่วยให้มั่นใจได้ว่าสตรีมสื่อจะทํางานภายในสภาพแวดล้อมที่เชื่อถือได้และปลอดภัยทั้งหมดนี้โดยไม่เปิดเผยตัวระบุผู้ใช้หรืออุปกรณ์ นอกจากนี้ ผู้ให้บริการเนื้อหาสื่อยังได้รับเชิญให้เข้าร่วมโปรแกรมเล่นระหว่างพัฒนาในปีหน้า

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38118627)

- Google ตั้งใจที่จะเปิดตัว Android WebView Media Integrity API ทดลองในปีหน้าซึ่งนําไปสู่ความกังวลในหมู่นักวิจารณ์เกี่ยวกับผลกระทบที่อาจเกิดขึ้นกับ adblockers และการเปิดกว้างการไม่เปิดเผยตัวตนและการกระจายอํานาจของเว็บ
- การถกเถียงแพร่กระจายไปทั่วหัวข้อต่างๆ เช่น ความสมดุลระหว่างความปลอดภัยและเสรีภาพ การยับยั้ง Google จากการควบคุมกระบวนการตัดสินใจ และผลกระทบของข้อเสนอ Web Environment Integrity (WEI) ที่เลื่อนออกไปของ Google สําหรับ Chrome
- ยักษ์ใหญ่ด้านเทคโนโลยีแม้จะระงับข้อเสนอ WEI สําหรับ Chrome แต่ก็มีแผนที่จะนําไปใช้กับ Android WebView API ซึ่งนําไปสู่การอภิปรายเกี่ยวกับช่องโหว่ของเว็บวิวและมาตรการรับมือที่อาจเกิดขึ้น

## [Google ละทิ้ง API ความสมบูรณ์ของสภาพแวดล้อมเว็บ](https://www.theregister.com/2023/11/02/google_abandons_web_environment_integrity/)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38121994)

- ผู้ใช้กล่าวถึงการบังคับใช้ Trusted Platform Modules (TPMs) โดย Chromium การต่อสู้กับบอทและการเปลี่ยนแปลงมาตรฐาน DRM โดยเฉพาะอย่างยิ่งการเปลี่ยนแปลงไปสู่การริเริ่มเว็บแบบปิดมากขึ้น
- การตรวจสอบชุมชนขยายไปถึงการจัดการชื่อเสียงของแบรนด์ของ Google และพวกเขากําลังเรียกร้องให้มีโซลูชันที่จัดลําดับความสําคัญของความต้องการของผู้ใช้ปลายทางโดยไม่ขัดขวางนวัตกรรม

- Cloudflare ซึ่งเป็น บริษัท ด้านประสิทธิภาพเว็บและความปลอดภัยต้องเผชิญกับปัญหาการบริการอย่างกว้างขวางที่ส่งผลกระทบต่อฟังก์ชันต่างๆเช่น Logpush, Cloudflare Dash, Stream API และ Radar เนื่องจากไฟฟ้าดับที่ศูนย์ข้อมูล

- ปัญหาเริ่มต้นเมื่อวันที่ 2 พฤศจิกายน 2023 แต่เครือข่ายการจัดส่งเนื้อหา (CDN) และคุณลักษณะด้านความปลอดภัยของ บริษัท ไม่ได้รับผลกระทบโดยเน้นถึงความยืดหยุ่นของแง่มุมเหล่านี้เมื่อเผชิญกับปัญหาดังกล่าว

## [แถลงการณ์ร่วมว่าด้วยความปลอดภัยและการเปิดกว้างของ AI](https://open.mozilla.org/letter/)

- เมื่อวันที่ 31 ตุลาคม พ.ศ. 2566 Mozilla ได้เผยแพร่แถลงการณ์ร่วมที่สนับสนุนความโปร่งใส การเปิดกว้าง และการเข้าถึงเทคโนโลยี AI อย่างกว้างขวางเพื่อป้องกันอันตรายที่อาจเกิดขึ้น
- แถลงการณ์ดังกล่าววิพากษ์วิจารณ์การครอบงําของเอกชนเหนือโมเดล AI พื้นฐาน และเตือนถึงกฎระเบียบที่เร่งรีบซึ่งอาจขัดขวางการแข่งขันและนวัตกรรม
- ผู้ลงนามผู้เชี่ยวชาญจากโดเมนที่หลากหลายสนับสนุนโอเพ่นซอร์สและวิทยาศาสตร์แบบเปิดเพื่อทําความเข้าใจความเสี่ยงของ AI สร้างความมั่นใจในความรับผิดชอบของสาธารณชนและลดอุปสรรคสําหรับผู้มาใหม่ในตลาด

## [แซลลี่เพิกเฉยต่อคําแนะนําก่อนหน้านี้](https://www.haihai.ai/pen15/)

- Ben Stein ผู้ร่วมก่อตั้ง QuitCarbon กล่าวถึงปัญหาการฉีดทันทีใน Language Learning Models (LLMs) ที่ค้นพบระหว่างการใช้เกมผจญภัยข้อความ AI โดยระบุว่าการบุกรุกดังกล่าวสามารถใช้โดยผู้ประสงค์ร้ายเพื่อถามคําร้องที่สร้างความเสียหายหรือไม่เหมาะสม
- สไตน์พบว่าการขอให้ LLM ฆ่าเชื้ออินพุตก่อนประมวลผลทําให้สามารถระบุและต่อต้านความพยายามในการจัดการได้อย่างมีประสิทธิภาพ โมเดลนี้แสดงให้เห็นถึงประสิทธิภาพที่เชื่อถือได้ในการตรวจจับคําขอที่ไม่เหมาะสม
- แม้จะมีผลลัพธ์เบื้องต้นที่มีแนวโน้ม แต่ Stein ยอมรับว่าจําเป็นต้องมีการทดสอบเพิ่มเติมและโต้แย้งว่าโซลูชันในอนาคตอาจต้องการการป้องกันในตัวในแอปพลิเคชันหรือมาตรการที่ Edge คล้ายกับไฟร์วอลล์เว็บแอปพลิเคชัน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38120097)

- Hacker News ดําเนินการอภิปรายเกี่ยวกับความท้าทายในการจัดการเนื้อหาที่ไม่เหมาะสมในข้อความของแฟน ๆ ในเกม NBA และความกังวลในวงกว้างเกี่ยวกับความปลอดภัยการเซ็นเซอร์และภาษาในเทคโนโลยีโดยมุ่งเน้นไปที่ Large Language Models (LLMs) และข้อความแจ้งภาษาที่สมบูรณ์ของ Turing
- มีการแนะนํากลยุทธ์ต่าง ๆ รวมถึงการ จํากัด การแจ้งเตือนบางอย่างทําให้การจัดการยากขึ้นและใช้ผู้ช่วยพิสูจน์ แต่กระบวนการแยกแยะการควบคุมจากข้อมูลยังคงซับซ้อน
- ความจําเป็นในการเข้ารหัสที่มีประสิทธิภาพช่องโหว่ที่เป็นไปได้และความเสี่ยงที่อาจเกิดขึ้นจากคําสั่งที่ไม่ถูกต้องหรือไม่ปลอดภัยก็ถูกถกเถียงกันแม้ว่าจะไม่มีทางออกที่ชัดเจนหรือฉันทามติในการอภิปราย
