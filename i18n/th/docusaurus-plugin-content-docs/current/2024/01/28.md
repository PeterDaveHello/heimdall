## [การสร้างโมเดลภาษาที่เหมือน ChatGPT ของคุณเอง: คําแนะนําทีละขั้นตอน](https://github.com/rasbt/LLMs-from-scratch)

- "สร้างแบบจําลองภาษาขนาดใหญ่ (ตั้งแต่เริ่มต้น)" เป็นหนังสือที่ให้คําแนะนําทีละขั้นตอนสําหรับการสร้างแบบจําลองภาษาของคุณเอง
- หนังสือเล่มนี้มีคําอธิบาย ไดอะแกรม และตัวอย่างที่ชัดเจนเพื่อช่วยให้ผู้อ่านเข้าใจแต่ละขั้นตอนของกระบวนการ
- เป็นไปตามแนวทางเดียวกันกับการสร้างโมเดลเช่น ChatGPT
- รหัสประกอบและเอกสารเพิ่มเติมสามารถเข้าถึงได้บนที่เก็บ GitHub
- หัวข้อที่ครอบคลุมในหนังสือเล่มนี้ ได้แก่ การทําความเข้าใจโมเดลภาษาขนาดใหญ่การทํางานกับข้อมูลข้อความกลไกความสนใจในการเข้ารหัสการใช้โมเดล GPT การฝึกอบรมล่วงหน้าเกี่ยวกับข้อมูลที่ไม่มีป้ายกํากับการปรับแต่งสําหรับการจัดประเภทข้อความการปรับแต่งความคิดเห็นของมนุษย์การใช้โมเดลภาษาขนาดใหญ่ในทางปฏิบัติและการแนะนํา PyTorch

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39156778)

- ผู้เขียนกําลังเขียนคู่มือและหนังสือที่ครอบคลุมเกี่ยวกับการสร้างโมเดลภาษาที่เหมือน ChatGPT (LLM) ตั้งแต่เริ่มต้น โดยมีเป้าหมายเพื่อให้ความรู้และทําให้ LLM กระจ่างขึ้นในขณะที่ให้ความช่วยเหลือในทางปฏิบัติในการใช้โมเดลที่กําหนดเอง
- แม้ว่าการฝึกอบรม LLM ตั้งแต่เริ่มต้นอาจไม่สามารถทําได้ทางการเงินสําหรับคนส่วนใหญ่ แต่ผู้เขียนแนะนําให้ใช้เฟรมเวิร์ก เช่น หม้อแปลง HF หรือ axolotl อย่างไรก็ตาม บริษัท ที่ได้รับทุนสนับสนุนอย่างดีได้แสดงให้เห็นว่าการฝึกอบรมตั้งแต่เริ่มต้นเป็นไปได้
- หนังสือเล่มนี้ครอบคลุมถึงการฝึกล่วงหน้า การปรับแต่งอย่างละเอียด และการบรรทุกตุ้มน้ําหนักที่ฝึกไว้ล่วงหน้า และผู้อ่านสนใจคําแนะนําการเรียนรู้แบบเสริมแรง การอภิปรายยังรวมถึงคําแนะนําสําหรับ RL โครงข่ายประสาทเทียม และทรัพยากรหม้อแปลงไฟฟ้า
- ผู้เขียนปกป้องค่าใช้จ่ายของหนังสือกล่าวถึงคูปองที่มีอยู่และยินดีรับข้อเสนอแนะผ่าน Manning Forum หรือ GitHub
- หนังสือเล่มนี้คาดว่าจะมีความยาว 400-500 หน้า และมีวิดีโอสอนสําหรับการแนะนําที่สั้นกว่า มีการถกเถียงกันถึงความเหมาะสมของวิดีโอสอนสําหรับผู้เริ่มต้น แต่ผู้สร้างรับรองว่ากําลังดําเนินการบทเพิ่มเติม

## [เครื่องบินอับปางไม่ได้สร้างการซื้อขายโดยใช้ข้อมูลภายใน: ผู้ใช้ Law Stack Exchange หารือเกี่ยวกับโบอิ้งวาง](https://law.stackexchange.com/questions/98706/is-it-insider-trading-if-i-bought-boeing-puts-while-i-am-inside-the-wrecked-airp)

- ผู้ใช้บนแพลตฟอร์ม Law Stack Exchange พูดคุยกันว่าการซื้อ Boeing put options ระหว่างเครื่องบินตกจะถือเป็นการซื้อขายโดยใช้ข้อมูลภายในหรือไม่
- การสนทนาจะตรวจสอบคําจํากัดความของการซื้อขายหลักทรัพย์โดยใช้ข้อมูลภายในและการมีส่วนร่วมที่อาจเกิดขึ้นของ ก.ล.ต.
- ในที่สุดก็สรุปได้ว่าการอยู่ในเครื่องบินที่ตกไม่เข้าข่ายเป็นข้อมูลวงในดังนั้นกฎหมายการซื้อขายหลักทรัพย์โดยใช้ข้อมูลภายในจะไม่มีผลบังคับใช้ การขอคําแนะนําทางกฎหมายและด้านการศึกษาของแพลตฟอร์มจะถูกเน้น

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39157797)

- การอภิปรายมุ่งเน้นไปที่คําจํากัดความและความถูกต้องตามกฎหมายของการซื้อขายหลักทรัพย์โดยใช้ข้อมูลภายใน โดยเฉพาะเกี่ยวกับการซื้อขายหุ้นโดยใช้ข้อมูลที่ไม่เปิดเผยต่อสาธารณะ
- ผู้เข้าร่วมถกเถียงกันว่าการกระทําบางอย่าง เช่น การสังเกตปัญหาของผลิตภัณฑ์หรือประสบกับความล้มเหลวของซอฟต์แวร์ ควรถือเป็นการซื้อขายโดยใช้ข้อมูลภายในหรือไม่
- การสนทนายังครอบคลุมถึงภาระหน้าที่ในการรักษาความลับ ความสัมพันธ์ที่ได้รับความไว้วางใจ และกฎระเบียบต่างๆ ในประเทศต่างๆ โดยเน้นถึงความซับซ้อนและลักษณะส่วนตัวของการกําหนดการซื้อขายหลักทรัพย์โดยใช้ข้อมูลภายใน

## [จุดสิ้นสุดของช่องทําเครื่องหมาย Square: วิวัฒนาการและอนาคตของการควบคุมผู้ใช้](https://tonsky.me/blog/checkbox/)

- ด้วยเว็บและ CSS มีแนวโน้มของการคิดค้นและออกแบบการควบคุมเหล่านี้ใหม่ซึ่งนําไปสู่การใช้ช่องทําเครื่องหมายกลมและช่องทําเครื่องหมายที่มีเครื่องหมายถูก
- Apple เพิ่งเปิดตัวช่องทําเครื่องหมายทรงกลมใน visionOS ใหม่ซึ่งเบี่ยงเบนไปจากการประชุมสี่เหลี่ยมจัตุรัสแบบดั้งเดิม ผู้เขียนคาดการณ์ว่าช่องทําเครื่องหมายสี่เหลี่ยมอาจถูกแทนที่ด้วยการสลับในปี 2024

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39161339)

- โดยเน้นย้ําถึงอิทธิพลของตัวเลือกการออกแบบของ Apple และความท้าทายที่ผู้สูงอายุต้องเผชิญในการนําทางการออกแบบ UI ที่ทันสมัย
- มีการอภิปรายระหว่างปุ่มตัวเลือกและช่องทําเครื่องหมาย พร้อมกับศักยภาพในการสร้างสรรค์นวัตกรรมในการออกแบบ UI ในขณะที่ยังคงยึดมั่นในอนุสัญญาที่กําหนดไว้

## [Sxmo: สภาพแวดล้อม Linux ที่เรียบง่ายสําหรับอุปกรณ์มือถือที่รองรับระบบสัมผัสและ SSH](https://sxmo.org/)

- Sxmo เป็นสภาพแวดล้อมที่เรียบง่ายสําหรับอุปกรณ์มือถือ Linux ด้วยอินเทอร์เฟซที่ขับเคลื่อนด้วยเมนูที่ใช้งานง่ายและรองรับการควบคุมแบบสัมผัสและท่าทาง
- ผู้ใช้สามารถ SSH ลงในโทรศัพท์และใช้งานเหมือนกับเครื่อง Alpine Linux อื่น ๆ ทําให้สะดวกสําหรับการใช้งานระยะไกล
- แพลตฟอร์มนี้สามารถปรับแต่งได้สูงและเขียนสคริปต์ได้ทําให้ผู้ใช้สามารถเปิดสคริปต์สําหรับการกระทําต่างๆและผูกไว้กับท่าทางสัมผัสหรือปุ่มทางกายภาพ มันทํางานบนลีนุกซ์แบบดั้งเดิมและรองรับแอพ FOSS

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39155103)

- ระบบปฏิบัติการ SXMO กําลังถูกกล่าวถึงโดยเน้นที่การออกแบบฟังก์ชันการทํางานและความเข้ากันได้กับ PinePhone
- ผู้ใช้ได้แบ่งปันประสบการณ์และความคิดเห็นของพวกเขา, แสดงมุมมองทั้งบวกและลบเกี่ยวกับ SXMO.
- การสนทนาครอบคลุมหัวข้อต่างๆเช่นข้อบกพร่องในการออกแบบข้อบกพร่องคุณสมบัติตัวเลือกการปรับแต่ง ROM ทางเลือกวิธีแก้ปัญหาแอปธนาคารประโยชน์ของตัวจัดการหน้าต่างการเรียงต่อกันบนโทรศัพท์กิจกรรมที่จะเกิดขึ้นการพัฒนา Python และ Qt และไม่มีตัวเลือก PIN / รหัสผ่านสําหรับการปลดล็อกอุปกรณ์

## [อินสแตนซ์ GitLab ที่มีช่องโหว่ทําให้เซิร์ฟเวอร์ 5.3K ถูกครอบครองบัญชี](https://www.scmagazine.com/news/gitlab-password-reset-bug-leaves-more-than-5-3k-servers-up-for-grabs)

- เซิร์ฟเวอร์มากกว่า 5,300 เครื่องไม่ได้ใช้แพตช์ที่สําคัญใน GitLab ทําให้บัญชีของนักพัฒนาเสี่ยงต่อการถูกยึดครองจากระยะไกล
- ข้อบกพร่องใน GitLab ช่วยให้ผู้โจมตีสามารถรีเซ็ตรหัสผ่านและรับลิงก์รีเซ็ตในอีเมลของตนเองโดยไม่ต้องโต้ตอบกับผู้ใช้
- มีการเผยแพร่การอัปเดตความปลอดภัยแล้ว แต่อินสแตนซ์ GitLab จํานวนมากยังคงไวต่อช่องโหว่นี้ ขอแนะนําให้ผู้ใช้ตรวจสอบสัญญาณของการใช้ประโยชน์ในบันทึกและเปิดใช้งานการรับรองความถูกต้องด้วยสองปัจจัย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39159002)

- ข้อผิดพลาดในการรีเซ็ตรหัสผ่านใน GitLab ทําให้เซิร์ฟเวอร์มากกว่า 5.3K มีช่องโหว่ทําให้เกิดความกังวลเกี่ยวกับความปลอดภัยของบัญชีและการเข้าถึงโดยไม่ได้รับอนุญาต
- มีการหารือเกี่ยวกับการเลือกภาษาโปรแกรมและแนวทางปฏิบัติที่ดีที่สุดเพื่อลดจุดบกพร่องและช่องโหว่ในโครงการในอนาคต
- มีการวิพากษ์วิจารณ์เกี่ยวกับทีมรักษาความปลอดภัยและการจัดการคุณลักษณะด้านความปลอดภัยของบัญชี ตลอดจนการอภิปรายเกี่ยวกับวัฒนธรรมทางวิศวกรรมและการใช้พารามิเตอร์ที่แข็งแกร่งใน Ruby on Rails

## [นักวิจัยของ MIT พัฒนาการพิมพ์โลหะเหลวอย่างรวดเร็วสําหรับชิ้นส่วนอลูมิเนียมขนาดใหญ่](https://news.mit.edu/2024/researchers-demonstrate-rapid-3d-printing-liquid-metal-0125)
