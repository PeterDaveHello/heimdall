# 2024-06-03

## [ยานโวเอเจอร์ 1: โลกได้รับ 1500 โฟตอนต่อบิตที่ 8.3 GHz](https://physics.stackexchange.com/questions/816698/how-many-photons-are-received-per-bit-transmitted-from-voyager-1)

- การอภิปรายเกี่ยวกับ Physics Stack Exchange ประมาณการว่ายานโวเอเจอร์ 1 ส่งโฟตอนประมาณ 2.6 x 10^22 โฟตอนต่อบิต โดยโลกได้รับโฟตอนประมาณ 1500 โฟตอนต่อบิตที่ 8.3 GHz
- การคํานวณจะพิจารณาปัจจัยต่างๆ เช่น ความถี่ในการส่งสัญญาณ ขนาดจานรับ และระยะห่างจากโลก โดยเน้นย้ําถึงความท้าทายของการรับสัญญาณเนื่องจากสัญญาณรบกวน
- การสนทนายังอ้างอิงถึงขีดจํากัดของแชนนอน ซึ่งเป็นแนวคิดพื้นฐานในทฤษฎีข้อมูลที่กําหนดอัตราสูงสุดของการส่งข้อมูลที่ปราศจากข้อผิดพลาดผ่านช่องทางการสื่อสาร

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40561872)

- การอภิปราย Physics Stack Exchange สํารวจจํานวนโฟตอนที่ได้รับต่อบิตที่ส่งจากยานโวเอเจอร์ 1 ซึ่งริเริ่มโดยนักวิจัยการแก้ไขข้อผิดพลาดควอนตัม
- การสนทนาเปรียบเทียบวิธีการแก้ไขข้อผิดพลาดแบบคลาสสิก เช่น วิธีการใน DRAM และสายเคเบิลใต้ทะเล กับการแก้ไขข้อผิดพลาดควอนตัม โดยเน้นถึงความซับซ้อนของระบบควอนตัม
- หัวข้อต่างๆ ได้แก่ ขีดจํากัดแชนนอน การนับโฟตอน และความท้าทายของการสื่อสารในอวกาศทางไกล ตลอดจนเทคนิคขั้นสูง เช่น เสาอากาศหลายตัวและการประมวลผลภายหลังแบบดิจิทัลเพื่อปรับปรุงการรับสัญญาณ

## [คู่มือสําหรับผู้เริ่มต้นในการพัฒนาอุปกรณ์ USB ด้วย NUCLEO-F103RB](https://popovicu.com/posts/making-usb-devices/)

- บทความนี้ทําหน้าที่เป็นคู่มือสําหรับผู้เริ่มต้นในการพัฒนาอุปกรณ์ USB โดยเน้นที่โครงการแบบ end-to-end ง่ายๆ โดยใช้บอร์ดพัฒนา NUCLEO-F103RB
- อธิบายพื้นฐานของ USB รวมถึงการเชื่อมต่อทางกายภาพและคู่ดิฟเฟอเรนเชียลสําหรับการถ่ายโอนข้อมูล และให้คําแนะนําที่เป็นประโยชน์สําหรับการกําหนดเส้นทาง PCB
- ผู้เขียนวิจารณ์ความซับซ้อนของการตั้งค่า STM32 และแนะนํา Linux เป็นทางเลือกที่คล่องตัวยิ่งขึ้นสําหรับการพัฒนาอุปกรณ์ USB โดยสนับสนุนเฟรมเวิร์กที่ยืดหยุ่นมากขึ้นในอนาคต

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40560300)

- การอภิปรายเน้นย้ําถึงความท้าทายในการสร้างอุปกรณ์ USB โดยเน้นที่การจัดการกระแสไหลเข้า ความซับซ้อนของรหัส USB แบบเปลือยโลหะ และการใช้เครื่องมือต่างๆ เช่น libusb
- ผู้ใช้แนะนําบอร์ดสไตล์ Arduino ระบบนิเวศ ESP32 และคอนโทรลเลอร์ขนาดเล็กที่มี VUSB ในขณะเดียวกันก็หารือเกี่ยวกับข้อจํากัดในการถ่ายโอนข้อมูล USB และ ID ผู้ขาย USB ที่มีราคาสูง
- โซลูชันที่ใช้งานได้จริง ได้แก่ การใช้ Raspberry Pi Zeroes เพื่อสร้างต้นแบบและใช้ประโยชน์จากทรัพยากรโอเพ่นซอร์ส พร้อมการอภิปรายเพิ่มเติมเกี่ยวกับปัญหาทางกฎหมายเกี่ยวกับ VID และ PID และการแปลงพอร์ต USB-A เป็น USB-C
