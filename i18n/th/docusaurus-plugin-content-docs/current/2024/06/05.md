# 2024-06-05

## [เหตุใดคีย์สังเคราะห์จึงเหนือกว่าคีย์ธรรมชาติในการออกแบบฐานข้อมูล](https://blog.ploeh.dk/2024/06/03/youll-regret-using-natural-keys/)

- บล็อกโพสต์ของ Mark Seemann โต้แย้งการใช้คีย์สังเคราะห์เหนือคีย์ธรรมชาติในการออกแบบฐานข้อมูล โดยอ้างถึงความน่าเชื่อถือและความสมบูรณ์ของข้อมูล
- เขาใช้เกร็ดเล็กเกร็ดน้อยส่วนตัวเกี่ยวกับข้อผิดพลาดของหมายเลขตัวถังรถยนต์เพื่อเน้นปัญหาเกี่ยวกับกุญแจธรรมชาติ เช่น ข้อผิดพลาดในการป้อนข้อมูลและการรับรองความเป็นเอกลักษณ์
- ความคิดเห็นของผู้อ่านให้มุมมองเพิ่มเติม โดยกล่าวถึงบทบาทของคีย์ธรรมชาติในการสร้างแบบจําลองข้อมูลเชิงตรรกะและความท้าทายด้วยไลบรารี ORM (Object-Relational Mapping)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40580549)

- การอภิปรายวิพากษ์คีย์ธรรมชาติในฐานข้อมูล โดยแนะนํา ID ที่ไม่ซ้ําใครและมนุษย์สามารถอ่านได้ซึ่งสร้างโดยฟังก์ชัน JavaScript/TypeScript พร้อมองค์ประกอบเวลาเพื่อลดการกระจายตัว
- แนะนําให้ใช้คีย์หลักที่เพิ่มขึ้นโดยอัตโนมัติ 64 บิตสําหรับการดําเนินการภายในเพื่อเพิ่มประสิทธิภาพและลดการขยายตัวของดัชนีในขณะที่รหัสสตริงแบบสุ่มช่วยเพิ่มความปลอดภัยสําหรับข้อมูลสาธารณะ
- การสนทนาเน้นย้ําถึงการแลกเปลี่ยนระหว่างคีย์ธรรมชาติและคีย์สังเคราะห์ โดยสนับสนุนคีย์ตัวแทนเพื่อความเสถียร การอ้างอิงที่สอดคล้องกัน และความสมบูรณ์ของข้อมูลที่เพิ่มขึ้น

## [คู่มือเคลื่อนไหวสําหรับซีรีส์ฟูริเยร์: จากแวดวงสู่ Epicycles](https://www.andreinc.net/2024/04/24/from-the-circle-to-epicycles)

- บทความ "From the Circle to Epicycles (Part 1)" แนะนําฟูริเยร์ซีรีส์ โดยอธิบายแนวคิดพื้นฐาน เช่น ฟังก์ชันตรีโกณมิติ อัตลักษณ์ของออยเลอร์ และไซนัสอยด์โดยใช้แอนิเมชั่นเพื่อความเข้าใจที่ดีขึ้น
- ความสําคัญของ π และคุณสมบัติของไซนัสอยด์และเอพิไซเคิล ซึ่งนําไปสู่การแนะนําฟูริเยร์ซีรีส์และการใช้งานในการประมาณรูปคลื่นที่ซับซ้อน
- บทความนี้ยังกล่าวถึงการสลายตัวของฟูริเยร์เซตของรูปคลื่นต่างๆ รวมถึงคลื่นสี่เหลี่ยม สามเหลี่ยม และฟันเลื่อยย้อนกลับ และแนะนําเครื่องมือสร้างภาพที่เรียกว่า "Fourier Series Machinery" เพื่อแสดงแนวคิดเหล่านี้

- มีการถกเถียงกันเกี่ยวกับการใช้องศากับเรเดียน โดยให้ความสําคัญกับเรเดียนในบริบททางทฤษฎี โดยเน้นย้ําถึงข้อจํากัดของการศึกษาแบบดั้งเดิมและคุณค่าของเนื้อหาออนไลน์ทางเลือก

## [นักข่าวแบ่งปันข้อความที่ตลกขบขันและแปลกประหลาดจากขโมย iPhone](https://gothamist.com/news/things-the-guys-who-stole-my-phone-have-texted-me-to-try-to-get-me-to-unlock-it)

- Veronica de Souza นักข่าวถูกขโมย iPhone ของเธอและต่อมาได้รับข้อความหลอกลวงจากโจรที่พยายามปลดล็อกโทรศัพท์
- ข้อความหลอกลวงรวมถึงการแจ้งเตือน Apple Pay ปลอมและการข่มขู่ว่าจะประมูลข้อมูลส่วนบุคคลของเธอในตลาดมืด
- De Souza แบ่งปันประสบการณ์ของเธอเพื่อเน้นย้ําถึงลักษณะที่แปลกประหลาดและมักจะตลกขบขันของการหลอกลวงเหล่านี้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40578460)

- ผู้ใช้ Gothamist ได้รับข้อความจากโจรที่พยายามปลดล็อกโทรศัพท์ที่ถูกขโมยซึ่งนําไปสู่การอภิปรายเกี่ยวกับการใช้ข้อความเกี่ยวกับการสังหารหมู่ที่จัตุรัสเทียนอันเหมินเพื่อกระตุ้นการเซ็นเซอร์ของจีนเพื่อยับยั้ง
- การสนทนาดังกล่าวทําให้เกิดความกังวลเกี่ยวกับความเสี่ยงต่อครอบครัวของผู้รับในประเทศจีนบทบาททางสังคมของ WeChat และนโยบายความปลอดภัยและการซ่อมแซมอุปกรณ์ของ Apple รวมถึงการรวมชิ้นส่วนเข้ากับเมนบอร์ดเพื่อป้องกันการโจรกรรม
- แม้จะมีมาตรการป้องกันการโจรกรรมของ Apple แต่การโจรกรรม iPhone ยังคงแพร่หลายโดยโทรศัพท์ที่ถูกขโมยมักจะถูกถอดชิ้นส่วนทําให้เกิดการถกเถียงกันเกี่ยวกับความสามารถในการซ่อมแซมผลกระทบต่อสิ่งแวดล้อมและประสิทธิภาพของคุณสมบัติด้านความปลอดภัยเช่น "Find My iPhone" นอกจากนี้ยังมีการหารือถึงความเฉยเมยของผู้บังคับใช้กฎหมายและความจําเป็นในการใช้มาตรการทางกฎหมายที่ดีขึ้นและการศึกษาของผู้ใช้

## [อิสราเอลใช้บัญชีปลอมเพื่อโน้มน้าวฝ่ายนิติบัญญัติของสหรัฐฯ เกี่ยวกับสงครามฉนวนกาซา](https://www.haaretz.com/israel-news/security-aviation/2024-06-05/ty-article-magazine/.premium/israel-secretly-targeted-american-lawmakers-with-gaza-war-influence-campaign/0000018f-e7c8-d11f-a5cf-e7cb62af0000)

- กระทรวงกิจการพลัดถิ่นของอิสราเอลดําเนินการรณรงค์อิทธิพลแอบแฝงโดยมุ่งเป้าไปที่ฝ่ายนิติบัญญัติผิวดําและหัวก้าวหน้ารุ่นเยาว์ในสหรัฐอเมริกาและแคนาดาเพื่อโน้มน้าวความคิดเห็นของสาธารณชนเกี่ยวกับสงครามฉนวนกาซา
- การรณรงค์นี้ดําเนินการโดยบริษัทการเมืองแห่งหนึ่งใช้บัญชีและเว็บไซต์ปลอมเพื่อเผยแพร่เนื้อหาที่สนับสนุนอิสราเอลและอิสลาม
- ปฏิบัติการนี้ได้รับการรายงานครั้งแรกโดย Haaretz ในเดือนมีนาคม

- การหยุดแคมเปญอิทธิพลแอบแฝงของ OpenAI ด้วย ChatGPT จุดประกายการถกเถียงเกี่ยวกับบทบาทของ AI ในการหลอกลวงและสงครามทางอินเทอร์เน็ต โดยเน้นย้ําถึงบริบททางประวัติศาสตร์ของการโฆษณาชวนเชื่อและวิวัฒนาการของการจารกรรมผ่านโซเชียลมีเดีย
- การสนทนาวิพากษ์วิจารณ์การจัดการสื่อ โดยเฉพาะอย่างยิ่งในความขัดแย้งระหว่างอิสราเอลกับปาเลสไตน์ และอิทธิพลของกลุ่มล็อบบี้เช่น AIPAC ในขณะที่กล่าวถึงผลกระทบที่กว้างขึ้นสําหรับเสรีภาพในการพูดและวาทกรรมทางการเมือง

## [ไดรเวอร์ Vulkan 1.3 ตัวแรกสําหรับ M1 ของ Apple บรรลุความสอดคล้อง 98.3%](https://rosenzweig.io/blog/vk13-on-the-m1-in-1-month.html)

- ไดรเวอร์ "Honeykrisp" เป็นการใช้งานที่สอดคล้องกับ Vulkan 1.3 ตัวแรกสําหรับฮาร์ดแวร์ M1 ของ Apple ซึ่งพัฒนาขึ้นโดยไม่มีการยกเว้นการพกพาและใช้ไดรเวอร์ NVK ของ Faith Ekstrand สําหรับ NVIDIA GPU
- โครงการประสบความสําเร็จครั้งสําคัญ รวมถึงอัตราการผ่าน 99.6% สําหรับ Vulkan 1.1 และ 98.3% สําหรับ Vulkan 1.3 และตัวเรนเดอร์ Vulkan ในตัวสําหรับ SuperTuxKart และ Zink
- เป้าหมายในอนาคตรวมถึงการสนับสนุน Direct3D ผ่าน DXVK และ vkd3d-proton สําหรับเกม Windows บน Asahi Linux ในขณะที่ปัจจุบันนําเสนอไดรเวอร์ OpenGL 4.6 ที่สอดคล้องกันสําหรับเกม Linux

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40585842)

- Alyssa Rosenzweig พัฒนาไดรเวอร์ Vulkan 1.3 สําหรับชิป M1 ของ Apple ในเวลาเพียงหนึ่งเดือน ทําให้เกิดการอภิปรายเกี่ยวกับ Hacker News เกี่ยวกับการเล่นเกมบนสถาปัตยกรรม ARM และ macOS
- การอภิปรายวิพากษ์วิจารณ์ความชอบของ Apple ที่มีต่อ Metal มากกว่า Vulkan ซึ่งจํากัดความพร้อมใช้งานของเกมดั้งเดิมบน macOS เมื่อเทียบกับ Windows และ Linux และเน้นย้ําถึงความซับซ้อนของการรองรับ API กราฟิกหลายตัว
- ผู้ใช้แสดงความไม่พอใจกับ Game Porting Toolkit ของ Apple และความต้องการพื้นที่เก็บข้อมูลสูงสําหรับการอัปเดตเกม โดยเน้นย้ําถึงความจําเป็นในการรองรับการเล่นเกมที่ดีขึ้นและความเข้ากันได้ของ API ที่กว้างขึ้นเพื่อดึงดูดเกม AAA มาสู่ macOS

## [CoPilot+ และข้อกังวลด้านความเป็นส่วนตัวของ Microsoft จุดประกายฟันเฟืองของผู้ใช้และการตรวจสอบด้านกฎระเบียบ](https://www.antipope.org/charlie/blog-static/2024/06/is-microsoft-trying-to-commit-.html)

- Charles Stross วิจารณ์การเปิดตัว CoPilot+ ของ Microsoft ซึ่งเป็นส่วนเสริมที่ใช้ AI สําหรับ Windows โดยเปรียบเทียบกับ Clippy ที่ไม่เป็นที่นิยม แต่มี AI ขั้นสูงและมีข้อบกพร่องมากกว่า
- Stross ทําให้เกิดความกังวลด้านความเป็นส่วนตัวเกี่ยวกับคุณลักษณะ "Recall" ใหม่ของ Microsoft ใน Windows 11 ซึ่งจัดเก็บกิจกรรมของผู้ใช้ในฐานข้อมูลที่ไม่ได้เข้ารหัส ก่อให้เกิดความเสี่ยงต่อข้อมูลที่ละเอียดอ่อนและจุดประกายการโต้เถียงและการตรวจสอบด้านกฎระเบียบ
- บล็อกนี้กล่าวถึงผลกระทบที่กว้างขึ้นสําหรับความเป็นส่วนตัว ความปลอดภัย และการผลักดันของอุตสาหกรรมเทคโนโลยีสําหรับการอัปเกรดฮาร์ดแวร์และระบบปฏิบัติการ โดยผู้ใช้บางคนกําลังพิจารณาทางเลือกอื่น เช่น Linux

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40585212)

- คุณลักษณะ "Recall" ใหม่ของ Microsoft ซึ่งใช้ OCR บนอุปกรณ์ (การรู้จําอักขระด้วยแสง) และฐานข้อมูล SQLite ที่ค้นหาได้จุดประกายการถกเถียงกันว่ามีคุณสมบัติเป็น AI หรือไม่ และทําให้เกิดข้อกังวลด้านความเป็นส่วนตัวและความปลอดภัย
- การอภิปรายวิพากษ์วิจารณ์กลยุทธ์ที่กว้างขึ้นของ Microsoft รวมถึงการมุ่งเน้นไปที่บริการคลาวด์เช่น Azure ความเกี่ยวข้องที่ลดลงของ Windows และการจัดการความเป็นส่วนตัวและความปลอดภัยของข้อมูลของบริษัท

- โดยเน้นถึงประโยชน์ของ systemd-tmpfiles สําหรับการจัดการไฟล์ชั่วคราวและความซับซ้อนของเค้าโครงระบบไฟล์ รวมถึงการอภิปรายเกี่ยวกับการรวม '/var/tmp' และ '/var/cache'
- การสนทนายังครอบคลุมถึงเครื่องมือคอนเทนเนอร์ เช่น Snapcraft และ Flatpak ผลกระทบของ tmpfs ต่อประสิทธิภาพของระบบ การสึกหรอของ SSD และการจัดการหน่วยความจํา โดยมีความคิดเห็นที่หลากหลายเกี่ยวกับประสิทธิภาพของ swap และ zswap

## [การศึกษาพบว่าโครงการซอฟต์แวร์ Agile มีอัตราความล้มเหลวสูงกว่า Non-Agile ถึง 268%](https://www.theregister.com/2024/06/05/agile_failure_rates/)

- การศึกษาโดยที่ปรึกษา Engprax พบว่าโครงการซอฟต์แวร์ Agile มีแนวโน้มที่จะล้มเหลวมากกว่า 268% เมื่อเทียบกับโครงการที่ไม่ใช่ Agile
- การวิจัยที่เกี่ยวข้องกับวิศวกรซอฟต์แวร์ 600 คนจากสหราชอาณาจักรและสหรัฐอเมริกาเน้นว่าโครงการที่มีข้อกําหนดที่ชัดเจนและเป็นเอกสารก่อนการพัฒนามีแนวโน้มที่จะประสบความสําเร็จ 97%
- การศึกษาชี้ให้เห็นว่าข้อกําหนดที่แข็งแกร่ง วิศวกรรม และความปลอดภัยทางจิตวิทยาสําหรับการแก้ปัญหามีความสําคัญต่อความสําเร็จของโครงการ โดยสนับสนุนแนวทางที่สมดุลระหว่างวิธีการแบบ Agile และวิธีการแบบดั้งเดิม เช่น Waterfall

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40584901)

- การศึกษาที่อ้างว่าโครงการซอฟต์แวร์ Agile มีอัตราความล้มเหลวสูงขึ้น 268% ถูกวิพากษ์วิจารณ์ว่ามีอคติ ขาดความโปร่งใสของข้อมูล และคําจํากัดความที่มีข้อบกพร่อง
- นักวิจารณ์โต้แย้งว่า Agile มักมีลักษณะผิดและนําไปใช้ในทางที่ผิดซึ่งนําไปสู่ปัญหาต่างๆเช่นหนี้ทางเทคนิคความเหนื่อยหน่ายของผู้เชี่ยวชาญและความล้มเหลวของโครงการโดยเน้นย้ําถึงความสําคัญของการใช้งานที่เหมาะสมความยืดหยุ่นและการปรับตัว
- การอภิปรายเปรียบเทียบ Agile กับวิธีการของ Waterfall โดยสังเกตความเหมาะสมของ Agile สําหรับสภาพแวดล้อมแบบไดนามิกและการพัฒนาแบบวนซ้ํา ในขณะที่ Waterfall ดีกว่าสําหรับโครงการที่มีข้อกําหนดคงที่ โดยเน้นถึงความจําเป็นในการสร้างสมดุลระหว่างหลักการ Agile กับข้อกําหนดในทางปฏิบัติ วิศวกรรมและการออกแบบ
