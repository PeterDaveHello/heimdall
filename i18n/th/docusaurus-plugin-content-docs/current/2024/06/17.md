# 2024-06-17

## [MicroMac, แมคอินทอชในราคาต่ำกว่า £5](https://axio.ms/projects/2024/06/16/MicroMac.html)

- โครงการที่ชื่อว่า Pico-mac ใช้ไมโครคอนโทรลเลอร์ Raspberry Pi RP2040 เพื่อจำลอง Macintosh 128K ทำให้สามารถรันซอฟต์แวร์เก่าของ Macintosh ได้
- หน่วยความจำ RAM ขนาด 264KB และหน่วยความจำแฟลชขนาด 2MB ของ RP2040 เพียงพอที่จะรองรับหน่วยความจำ 128KB ของ Mac, โปรแกรมจำลอง และภาพดิสก์ที่มีระบบปฏิบัติการและซอฟต์แวร์
- โครงการนี้เกี่ยวข้องกับการสร้างอีมูเลเตอร์ Mac 128K, การสร้างวงจรสำหรับการแสดงผลวิดีโอ VGA, และการปรับปรุงประสิทธิภาพผ่านการโอเวอร์คล็อกและการปรับปรุงอินเตอร์พรีเตอร์ ทำให้ได้ประสิทธิภาพประมาณ 1.4 MIPS.

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40699684)

- MicroMac, คอมพิวเตอร์แมคอินทอชที่มีราคาไม่ถึง 5 ปอนด์, ได้รับความสนใจอย่างมาก โดยเน้นวิธีการทำเองที่มีต้นทุนต่ำสำหรับการใช้คอมพิวเตอร์ย้อนยุค
- โครงการนี้เกี่ยวข้องกับการใช้ไมโครคอนโทรลเลอร์ RP2040 เพื่อจำลองเครื่อง Macintosh 68k ซึ่งให้ประสบการณ์ที่สมจริงมากกว่าการใช้โปรแกรมจำลองบน Linux
- การสนทนายังครอบคลุมถึงความท้าทายและวิธีแก้ปัญหาเชิงสร้างสรรค์ในการจัดหาชิ้นส่วนเทคโนโลยีเก่า โดยเน้นถึงความสามารถในการหาทรัพยากรและความสนใจของชุมชนในโครงการคอมพิวเตอร์ย้อนยุค

- ผู้เขียนสนับสนุนให้ผู้อื่นยืนยันสิทธิผู้บริโภคของตนและไม่ยอมรับระยะเวลาการรับประกันที่น้อยเกินไป โดยเน้นความสำคัญของช่องทางกฎหมายเช่น NCAT ในการแก้ไขข้อพิพาทดังกล่าว

## [NumPy 2.0](https://numpy.org/devdocs/release/2.0.0-notes.html)

- NumPy 2.0.0 เป็นการเปิดตัวหลักครั้งแรกตั้งแต่ปี 2006 พัฒนามาเป็นเวลา 11 เดือนโดยผู้ร่วมพัฒนา 212 คน มีคุณสมบัติใหม่ที่สำคัญ การปรับปรุงประสิทธิภาพ และการเปลี่ยนแปลงที่ทำให้ไม่เข้ากันกับ API ของ Python และ C
- การอัปเดตที่สำคัญรวมถึง dtype สตริงที่มีความยาวตัวแปรใหม่, ฟังก์ชันการเรียงลำดับที่เร่งความเร็ว, และการแยกที่ชัดเจนระหว่าง API สาธารณะและส่วนตัว, พร้อมกับ API C สาธารณะใหม่สำหรับการสร้าง dtype ที่กำหนดเอง
- ผู้ใช้ควรทราบถึงการเปลี่ยนแปลงที่อาจทำให้เกิดปัญหาและปัญหาความเข้ากันได้ย้อนหลัง โดยมีคู่มือการย้ายข้อมูลที่จัดเตรียมไว้เพื่อช่วยในการอัปเดตโค้ด

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40699470)

- NumPy 2.0 ได้ถูกปล่อยออกมาแล้ว โดยมีการเปลี่ยนแปลงที่สำคัญ รวมถึงการเลิกใช้บาง APIs สาธารณะและการสนับสนุนใหม่สำหรับการจัดการสตริง
- การเปิดตัวนี้มีเป้าหมายเพื่อแก้ไขจุดอ่อนทางประวัติศาสตร์ในการดำเนินการกับสตริงภายใน NumPy ซึ่งอาจเป็นประโยชน์ต่อกรณีการใช้งาน NLP (การประมวลผลภาษาธรรมชาติ) และ LLM (โมเดลภาษาขนาดใหญ่)
- ความเข้ากันได้กับ NumPy 1.x ยังคงรักษาไว้เป็นส่วนใหญ่ ยกเว้นฟังก์ชัน "byte_bounds" และแนะนำให้ผู้ใช้ตรึงการพึ่งพาเพื่อหลีกเลี่ยงปัญหาระหว่างการเปลี่ยนแปลง
