## [อัลลัน แมคโดนัลด์ปฏิเสธที่จะอนุมัติการปล่อยยานชาเลนเจอร์ เปิดโปงการปกปิด (2021)](https://www.npr.org/2021/03/07/974534021/remembering-allan-mcdonald-he-refused-to-approve-challenger-launch-exposed-cover)

- อัลลัน แมคโดนัลด์ ซึ่งเสียชีวิตในวัย 83 ปี เป็นบุคคลสำคัญในเหตุการณ์ภัยพิบัติของกระสวยอวกาศชาเลนเจอร์ โดยเป็นที่รู้จักจากการปฏิเสธที่จะอนุมัติการปล่อยกระสวยเนื่องจากความกังวลด้านความปลอดภัย
- หลังจากเหตุการณ์ภัยพิบัติ McDonald ได้เปิดโปงการปกปิดของ NASA โดยเปิดเผยว่ามีวิศวกรที่คัดค้านการปล่อยยานเนื่องจากอุณหภูมิที่เย็นจัดส่งผลกระทบต่อ O-rings ซึ่งนำไปสู่การถูกลดตำแหน่งชั่วคราวของเขา
- แมคโดนัลด์ต่อมาได้เป็นผู้นำในการออกแบบใหม่ของจรวดบูสเตอร์ ร่วมเขียนบันทึกที่เป็นที่ยอมรับเกี่ยวกับภัยพิบัติ และกลายเป็นผู้สนับสนุนการตัดสินใจที่มีจริยธรรมในวิศวกรรม

- แม้จะมีแรงกดดันอย่างหนัก แมคโดนัลด์และวิศวกรคนอื่น ๆ ก็ถูกผู้บริหารระดับสูงปฏิเสธ ทำให้เกิดภัยพิบัติของยานชาเลนเจอร์
- แมคโดนัลด์ต่อมาได้สนับสนุนการตัดสินใจที่มีจริยธรรม โดยเน้นถึงความตึงเครียดระหว่างความถูกต้องทางเทคนิคและแรงกดดันจากการบริหารในโครงการที่มีความเสี่ยงสูง

## [เบราว์เซอร์ Ladybird กางปีกออก](https://lwn.net/Articles/976822/)

- Ladybird เป็นโครงการเว็บเบราว์เซอร์โอเพนซอร์สที่แยกออกมาจาก SerenityOS โดย Andreas Kling โดยมีเป้าหมายที่จะเป็นอิสระจาก Chrome และเขียนด้วยภาษา C++ ภายใต้ใบอนุญาต BSD
- แม้จะยังอยู่ในช่วงพัฒนาเริ่มต้นและขาดคุณสมบัติมากมาย แต่ Ladybird แสดงให้เห็นถึงศักยภาพด้วยฟังก์ชันพื้นฐานและการมีส่วนร่วมจากชุมชนอย่างมาก โดยมุ่งเน้นที่ระบบ Linux, macOS และระบบที่คล้าย UNIX พร้อมการสนับสนุน Windows ผ่าน WSL
- ปฏิกิริยาของชุมชนมีความหลากหลาย บางคนเห็นถึงศักยภาพ ขณะที่บางคนแนะนำให้มุ่งเน้นไปที่เบราว์เซอร์ที่มีอยู่แล้ว แต่ผู้สนับสนุนโต้แย้งว่าโครงการใหม่ๆ เช่น Ladybird มีความสำคัญต่อระบบนิเวศของเบราว์เซอร์ที่มีสุขภาพดี

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40746804)

- เบราว์เซอร์ Ladybird ซึ่งเป็นเบราว์เซอร์เว็บที่สร้างขึ้นโดยชุมชนใหม่ กำลังได้รับความสนใจในฐานะตัวเลือกที่สามารถใช้งานได้ทุกวัน โดยแตกต่างจากตัวเลือกหลักอย่าง Chrome และ Firefox
- การอภิปรายเน้นถึงความท้าทายในการสร้างเบราว์เซอร์ที่เรียบง่ายและปลอดภัยยิ่งขึ้นซึ่งรองรับเฉพาะส่วนย่อยของเทคโนโลยีเว็บ โดยต้องสมดุลระหว่างการใช้งานและการยอมรับของผู้ใช้
- โครงการนี้ถูกมองว่าเป็นโอกาสในการเรียนรู้ที่มีค่าสำหรับนักพัฒนารายใหม่ โดยมีขั้นตอนการสร้างที่มีการบันทึกไว้อย่างดีและมีพื้นที่กว้างขวางสำหรับการมีส่วนร่วม
