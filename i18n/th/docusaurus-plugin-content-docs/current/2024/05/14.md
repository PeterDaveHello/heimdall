## [Amazon S3 หยุดการเรียกเก็บเงินสําหรับรหัสข้อผิดพลาด HTTP ที่เลือก](https://aws.amazon.com/about-aws/whats-new/2024/05/amazon-s3-no-charge-http-error-codes/)

- Amazon S3 จะหยุดการเรียกเก็บเงินจากลูกค้าสําหรับรหัสข้อผิดพลาด HTTP ที่เฉพาะเจาะจง เช่น HTTP 403 (Access Denied) ที่ไม่ได้เกิดจากลูกค้า ซึ่งจะส่งผลต่อบัคเก็ต S3 ทั้งหมดในรีเจี้ยน AWS ทั้งหมด
- การเปลี่ยนแปลงนี้ไม่จําเป็นต้องแก้ไขแอปพลิเคชันของลูกค้า ซึ่งช่วยลดค่าใช้จ่ายสําหรับข้อผิดพลาดที่ไม่คาดคิด
- สําหรับรายละเอียดเพิ่มเติม โปรดดูที่หน้าการเรียกเก็บเงินสําหรับการตอบสนองต่อข้อผิดพลาดของ Amazon S3
