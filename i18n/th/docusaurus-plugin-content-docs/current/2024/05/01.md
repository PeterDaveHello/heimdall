## [ปรับปรุงระบบอัตโนมัติของโครงสร้างพื้นฐานด้วย Pyinfra](https://pyinfra.com/)

- pyinfra เป็นเครื่องมือ Python สําหรับระบบอัตโนมัติของโครงสร้างพื้นฐาน ซึ่งมีความเร็ว ความสามารถในการปรับขนาด และความเก่งกาจสําหรับคําสั่งเฉพาะกิจ
- มีการดําเนินการที่รวดเร็วการดีบักทันทีการดําเนินการที่มีศักยภาพและการรวมเข้ากับตัวเชื่อมต่อทําให้เป็นโซลูชันที่ครอบคลุมสําหรับระบบอัตโนมัติของโครงสร้างพื้นฐาน
- ผู้ใช้สามารถติดตั้ง pyinfra ผ่าน pip เรียกใช้คําสั่งผ่าน SSH กําหนดสถานะโดยใช้การดําเนินการ และปรับใช้งานโดยใช้โค้ด Python โดยมีความยืดหยุ่นในการขยายโดยใช้ระบบนิเวศของแพ็คเกจ Python

## [Borgo: ภาษาที่พิมพ์แบบคงที่ที่รวบรวมเพื่อไป](https://github.com/borgo-lang/borgo)

- Borgo เป็นภาษาโปรแกรมใหม่ที่แปลงเป็น Go โดยผสมผสานความเรียบง่ายของ Go เข้ากับการแสดงออกของ Rust
- ประกอบด้วยประเภทข้อมูลพีชคณิตการจับคู่รูปแบบการจัดการข้อผิดพลาดด้วย โอเปอเรเตอร์และเข้ากันได้กับแพ็คเกจ Go ที่มีอยู่
- ไวยากรณ์ Borgo คล้ายกับ Rust โดยมีเครื่องหมายอัฒภาคเสริม และมีคุณสมบัติต่างๆ เช่น ตัวเลือกสําหรับการจัดการศูนย์ ผลลัพธ์สําหรับค่าส่งคืนหลายค่า ซึ่งแสดงให้เห็นผ่านตัวอย่างเกมเดา

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40211891)

## [Extension.js: เครื่องมือสร้างส่วนขยายเว็บที่คล่องตัว](https://github.com/cezaraugusto/extension.js)

- Extension.js เป็นเครื่องมือพัฒนาสําหรับส่วนขยายเบราว์เซอร์ที่ปรับปรุงกระบวนการโดยให้การสนับสนุนในตัวสําหรับ TypeScript, WebAssembly, React และ JavaScript ที่ทันสมัย
- ขจัดข้อกําหนดสําหรับการกําหนดค่าที่ซับซ้อนและคําแนะนําในการตั้งค่า ทําให้นักพัฒนามีสมาธิกับการเขียนโค้ด
- เครื่องมือนี้สามารถรวมเข้ากับสคริปต์ npm ได้อย่างราบรื่นทําให้การสร้างส่วนขยายข้ามเบราว์เซอร์ง่ายขึ้นโดยไม่จําเป็นต้องกําหนดค่าบิลด์ เดิมทีสร้างขึ้นเพื่อให้ความรู้เกี่ยวกับการพัฒนาส่วนขยาย แต่ปัจจุบันเปิดให้สาธารณชนใช้โดยสนับสนุนข้อเสนอแนะ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40210627)

- Extension.js เป็นเครื่องมือ CLI ที่ทําให้การสร้างส่วนขยายเว็บง่ายขึ้น รองรับ TypeScript, WebAssembly, React และ JavaScript สมัยใหม่ เข้ากันได้อย่างสมบูรณ์กับ Chrome และ Edge พร้อมความเข้ากันได้ของ Firefox และ Safari ในอนาคตในวาระการประชุม
- การสนทนาบนแพลตฟอร์มเช่น Github และ HN มุ่งเน้นไปที่คุณสมบัติ Extension.js การเปรียบเทียบกับเฟรมเวิร์กเช่น Plasmo และกลยุทธ์สําหรับการปรับเปลี่ยนหน้าเว็บและ SPA โดยใช้เครื่องมือเช่น Mutation Observers และ CSS
- ความคิดเห็นของชุมชนเกี่ยวข้องกับคําแนะนําในการปรับปรุงและแผนการสนับสนุนเบราว์เซอร์ที่กําลังจะมีขึ้นสําหรับ Extension.js

## [ระบบไฟล์ไฟล์: การจัดการข้อมูลสมัยใหม่ด้วยเครื่องมือ Unix](https://mgree.github.io/ffs/)

- ffs เป็นระบบไฟล์ที่ช่วยให้ผู้ใช้เมานต์ข้อมูลกึ่งโครงสร้าง เช่น JSON บนระบบไฟล์ Unix ซึ่งช่วยให้สามารถจัดการด้วยเครื่องมือเชลล์มาตรฐานได้
- รองรับรูปแบบ JSON, YAML และ TOML ทําให้สามารถแก้ไขไฟล์แบบแทนที่ได้ ซึ่งต้องใช้ FUSE บน Linux และ macFUSE บน macOS และได้รับอนุญาตภายใต้ GPLv3
- พัฒนาโดย mgree ffs นําเสนอแนวทางที่แตกต่างสําหรับการประมวลผลข้อมูลบรรทัดคําสั่งเมื่อเทียบกับเครื่องมือเช่น jq และ gron
