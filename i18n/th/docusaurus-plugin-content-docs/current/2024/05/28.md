## [gh-dash: แดชบอร์ด GitHub CLI ที่ปรับแต่งได้สําหรับการประชาสัมพันธ์และปัญหา](https://github.com/dlvhdr/gh-dash)

- **gh-dash** เป็นส่วนขยาย GitHub CLI ที่นําเสนอแดชบอร์ดที่ปรับแต่งได้สําหรับจัดการคําขอดึง (PR) และปัญหา
- คุณสมบัติที่สําคัญ ได้แก่ ส่วนที่กําหนดค่าได้ตัวเลือกการค้นหาและการปรับแต่งธีมและเลย์เอาต์มุมมองโดยละเอียดการกําหนดค่าหลายรายการและช่วงเวลารีเฟรชอัตโนมัติ
- ขั้นตอนการติดตั้ง: ติดตั้ง GitHub CLI (v2.0.0+) ติดตั้งส่วนขยายด้วย 'gh extension install dlvhdr/gh-dash' และติดตั้งแบบอักษร Nerd สําหรับการแสดงไอคอน

## [Transformers แสดงความสามารถในการให้เหตุผลโดยนัยผ่านการ grokking การศึกษาเผย](https://arxiv.org/abs/2405.15071)

- บทความ "Grokked Transformers are Implicit Reasoners" สํารวจว่าหม้อแปลงสามารถให้เหตุผลโดยปริยายเหนือความรู้แบบพาราเมตริกโดยเน้นที่องค์ประกอบและประเภทการให้เหตุผลเปรียบเทียบหรือไม่
- ผลการวิจัยระบุว่าหม้อแปลงสามารถเรียนรู้การให้เหตุผลโดยนัยผ่านการฝึกอบรมที่กว้างขวางนอกเหนือจากการ overfitting หรือที่เรียกว่า grokking แต่ความสามารถในการสรุปแตกต่างกันไป: พวกเขาต่อสู้กับองค์ประกอบ แต่ประสบความสําเร็จด้วยการเปรียบเทียบในตัวอย่างที่ไม่กระจาย
- การศึกษาชี้ให้เห็นถึงการปรับปรุงการตั้งค่าข้อมูลและการฝึกอบรมการปรับปรุงสถาปัตยกรรมที่เป็นไปได้เช่นการแบ่งปันความรู้ข้ามชั้นและแสดงให้เห็นว่าหม้อแปลงไฟฟ้าที่คลําได้อย่างสมบูรณ์มีประสิทธิภาพดีกว่ารุ่นต่างๆเช่น GPT-4-Turbo และ Gemini-1.5-Pro ในงานการให้เหตุผลที่ซับซ้อน

- ข้อความแสดงข้อผิดพลาดใหม่ช่วยให้โปรแกรมเมอร์ JavaScript แก้ไขการใช้ '===' เป็น '==' และการเผยแพร่รับทราบการมีส่วนร่วมจากนักพัฒนาหลายราย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40498092)

- รุ่น Gleam 1.2.0 ปรับปรุงความทนทานต่อข้อผิดพลาดและประสบการณ์ของนักพัฒนา โดยมีไวยากรณ์ที่ทันสมัยและระบบประเภท Hindley-Milner ที่แข็งแกร่ง
- Gleam รวบรวมทั้ง Beam VM และ JavaScript ของ Erlang ซึ่งให้ประสิทธิภาพที่ดีและเปรียบเทียบกับ Elixir ซึ่งกําลังก้าวไปสู่การพิมพ์แบบค่อยเป็นค่อยไป
- ชุมชนที่ใช้งานอยู่สนับสนุนโครงการต่างๆ เช่น เฟรมเวิร์กส่วนหน้า Lustre และเว็บแอป Nestful แม้ว่าผู้ใช้บางคนจะพบว่าเอกสารขาดในด้านความปลอดภัยและการทํางานพร้อมกันของประเภท

## [Openkoda: แพลตฟอร์มโอเพ่นซอร์สสําหรับการพัฒนาแอปธุรกิจอย่างรวดเร็ว](https://github.com/openkoda/openkoda)

- Openkoda เป็นแพลตฟอร์มการพัฒนาที่มุ่งเร่งการสร้างแอปพลิเคชันทางธุรกิจและเครื่องมือภายในโดยมอบฟังก์ชันที่สร้างไว้ล่วงหน้าและคุณสมบัติที่พร้อมใช้งานทันที
- โมเดลการเช่าหลายแบบ และใช้เทคโนโลยีที่คุ้นเคย เช่น Java, Spring Boot, JavaScript, HTML, Hibernate และ PostgreSQL ทําให้สามารถปรับแต่งและผสานรวมได้อย่างกว้างขวาง
- Openkoda เป็นโอเพ่นซอร์สภายใต้ใบอนุญาต MIT รวมถึงคุณสมบัติต่างๆ เช่น การจัดการผู้ใช้ CMS การจัดตารางงาน และเสนอบริการคลาวด์ที่มีการจัดการเพื่อการปรับใช้และการปรับขนาดที่ง่ายดาย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40502956)

- Openkoda เป็นแพลตฟอร์มโอเพ่นซอร์สสําหรับการสร้างแอปพลิเคชันทางธุรกิจหลักโดยเสนอความเป็นเจ้าของซอร์สโค้ดเต็มรูปแบบและหลีกเลี่ยงข้อ จํากัด ทางการค้าซึ่งแตกต่างจาก Salesforce
- เมื่อเทียบกับ ERP ของ FOSS (ซอฟต์แวร์ฟรีและโอเพ่นซอร์ส) อื่นๆ เช่น Odoo แต่มีความกังวลน้อยกว่าเกี่ยวกับการสร้างรายได้ และถูกมองว่าเป็นทางเลือกที่คุ้มค่าสําหรับ Salesforce
- Openkoda ใช้ Java เพื่อประสิทธิภาพและระบบนิเวศที่สมบูรณ์ แม้ว่าจะมีการถกเถียงกันเกี่ยวกับการใช้คําฟุ่มเฟือย และให้อิมเมจ Docker เพื่อให้การผสานรวมง่ายขึ้นสําหรับองค์กรที่ไม่ใช่ Java
