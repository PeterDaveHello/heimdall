## [Big Tech ต่อต้านกฎหมายตลาดดิจิทัลของสหภาพยุโรป เผชิญกับค่าปรับ](https://www.eff.org/deeplinks/2024/05/big-tech-eu-drop-dead)

- พระราชบัญญัติตลาดดิจิทัล (DMA) ของสหภาพยุโรปมีจุดมุ่งหมายเพื่อเพิ่มขีดความสามารถให้กับผู้ใช้โดยกําหนดให้ผู้เฝ้าประตูด้านเทคโนโลยีอนุญาตร้านแอปของบุคคลที่สามและเกตเวย์ที่ทํางานร่วมกันได้ในขณะที่ห้ามการเลือกตนเองและปรับปรุงกฎระเบียบด้านความเป็นส่วนตัว
- บริษัทเทคโนโลยีขนาดใหญ่ เช่น Apple และ Meta กําลังต่อต้านการปฏิบัติตาม DMA ซึ่งอาจส่งเสริมการแข่งขัน ความรับผิดชอบ และการควบคุมผลกําไรของผู้ใช้และพนักงานในอุตสาหกรรมเทคโนโลยีที่เพิ่มขึ้น
- สหภาพยุโรปกําลังบังคับใช้ DMA ผ่านการสอบสวนและกําหนดค่าปรับสําหรับบริษัทที่ไม่ปฏิบัติตามกฎระเบียบ โดยเน้นย้ําถึงความสําคัญของการปฏิบัติตามข้อกําหนดในการส่งเสริมสภาพแวดล้อมทางเทคโนโลยีที่มีการแข่งขันสูงและมีผู้บริโภคเป็นศูนย์กลางมากขึ้น

## [คอมพิวเตอร์ RISC-V 64 บิตแบบโอเพ่นซอร์สที่น่าเชื่อถือ](https://www.contrib.andrew.cmu.edu/~somlo/BTCP/)

- Gabriel L. Somlo ตั้งเป้าที่จะพัฒนาคอมพิวเตอร์ RISC-V 64 บิตที่เชื่อถือได้ ฟรี เข้ากันได้กับ Linux และโฮสต์ด้วยตนเองโดยใช้เครื่องมือโอเพนซอร์สและคอมไพเลอร์บน FPGA เพื่อลดความเสี่ยงของแบ็คดอร์ของฮาร์ดแวร์
- โครงการมุ่งเน้นไปที่ HDL ที่โปร่งใสและแหล่งที่มาของซอฟต์แวร์เพื่อเพิ่มความน่าเชื่อถือ ซึ่งแตกต่างจากความคิดริเริ่มต่างๆ เช่น lowRISC ที่รวมส่วนประกอบที่เป็นกรรมสิทธิ์
- ผลงานของ Somlo นําเสนอการสร้าง yoloRISC ซึ่งเป็น SoC สาธิตที่ทํางานบนเครื่องมือโอเพ่นซอร์สที่ออกแบบมาสําหรับบอร์ดพัฒนา Lattice ECP5 5G Versa

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40399987)

- บทความนี้เจาะลึกถึงความซับซ้อนของการรวมแบ็คดอร์เข้ากับฮาร์ดแวร์ โดยเน้นที่ FPGA และ RAM ทําให้เกิดความกังวลเกี่ยวกับผลกระทบด้านความมั่นคงของชาติ
- โดยเน้นย้ําถึงความท้าทายในการรวมซีพียูที่ซ่อนอยู่ เสนอโซลูชันที่เป็นไปได้ และแบ่งปันข้อมูลเชิงลึกจากผู้ที่ชื่นชอบการพัฒนาเครื่อง RISC-V ที่โฮสต์ด้วยตนเองด้วยบอร์ด FPGA
- การอภิปรายครอบคลุมถึงความเร็วหลักของ FPGA, การเพิ่มประสิทธิภาพ, การดีบักบนระบบที่ซบเซา, การใช้ FPGA ในการพัฒนา, ค่าใช้จ่ายที่เกี่ยวข้องกับ FPGA และความก้าวหน้าใน toolchains โอเพนซอร์สสําหรับ FPGA
