# 2024-03-02

## [Apple รองรับ Progressive Web Apps ใน EU ใน iOS 17.4](https://appleinsider.com/articles/24/03/01/apple-reverses-course-on-death-of-progressive-web-apps-in-eu)

- Apple ได้ตัดสินใจที่จะสนับสนุน Progressive Web Apps ในสหภาพยุโรปใน iOS 17.4 ตามความคิดเห็นของผู้บริโภคและพระราชบัญญัติตลาดดิจิทัลของสหภาพยุโรป
- การเปลี่ยนแปลงนี้จัดการกับความท้าทายด้านความปลอดภัยและความเป็นส่วนตัวด้วยเว็บแอป และรวมถึงการรับรองแอป iOS ลดค่าธรรมเนียม และเปิดใช้งานการสนับสนุนเอ็นจิ้นเบราว์เซอร์หลายตัว
- บริษัทต่างๆ เช่น Spotify และ Epic Games ได้แสดงความไม่พอใจกับการปรับเปลี่ยนเหล่านี้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39563618)

- ในตอนแรก Apple ปิดใช้งานการรองรับ Progressive Web Apps (PWA) บน Safari ในสหภาพยุโรป แต่กลับรายการเนื่องจากภัยคุกคามทางกฎหมายเกี่ยวกับค่าคอมมิชชั่นของ App Store
- มีข้อสงสัยว่า Apple ชอบแอพที่มาพร้อมเครื่องมากกว่า PWA เพื่อผลกําไรและการควบคุมที่อาจเกิดขึ้นทําให้เกิดคําถามเกี่ยวกับความมุ่งมั่นในการสนับสนุน PWA
- การอภิปรายรวมถึงการปฏิบัติตามกฎระเบียบของสหภาพยุโรปของ Apple ความกังวลเกี่ยวกับการสนับสนุนเอ็นจิ้นเบราว์เซอร์ของบุคคลที่สามสําหรับ PWA บน iOS และผลกระทบของแนวทางสวนที่มีกําแพงล้อมรอบต่อนักพัฒนาและผู้ใช้ ซึ่งจุดประกายการอภิปรายเกี่ยวกับความโปร่งใสและทางเลือกในระบบนิเวศของแอป

## [Jan Marsalek ซีโอโอของ Fugitive Wirecard เปิดเผยว่าเป็นสายลับ GRU ที่ยาวนานกว่าทศวรรษ](https://theins.ru/en/politics/269612)

- Jan Marsalek ซีโอโอของ Wirecard ได้รับการเปิดเผยว่าทํางานเป็นสายลับให้กับ GRU ของรัสเซียเป็นเวลาสิบปี ตามรายงานของ Roman Dobrokhotov, Christo Grozev และ Michael Weiss เมื่อวันที่ 1 มีนาคม 2024

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39561021)

- การอภิปรายกล่าวถึงเรื่องอื้อฉาวของ Wirecard ซึ่งเกี่ยวข้องกับบริษัทตรวจสอบบัญชี เช่น EY และ KPMG พร้อมกับข้อสงสัยเกี่ยวกับความเชื่อมโยงของ Jan Marsalek กับหน่วยข่าวกรองรัสเซีย
- หัวข้อครอบคลุมตั้งแต่การเมืองรัสเซียคริสตจักรออร์โธดอกซ์รัสเซียไปจนถึงความหมายของบุคคลเช่น Alexei Navalny ในภูมิภาคโดยสัมผัสกับประเด็นที่เชื่อมโยงถึงกันต่างๆเช่นความปลอดภัยนโยบายพลังงานและพลวัตทางภูมิรัฐศาสตร์ในยุโรป
- โดยเน้นย้ําถึงข้อกังวลและหัวข้อที่หลากหลาย โดยเน้นความสัมพันธ์ที่ซับซ้อนระหว่างการฉ้อโกงทางการเงิน การจารกรรม การเมือง และอิทธิพลระดับภูมิภาค

## [บริษัทในสหราชอาณาจักรยอมรับการทํางาน 4 วันต่อสัปดาห์ โดยรายงานผลประโยชน์ที่ยั่งยืน](https://www.npr.org/2024/02/27/1234271434/4-day-workweek-successful-a-year-later-in-uk)

- หลายบริษัทได้นําการทํางานสี่วันต่อสัปดาห์มาใช้ ซึ่งนําไปสู่พนักงานที่มีความสุขมากขึ้น ลดการลาออก และปรับปรุงประสิทธิภาพ ควบคู่ไปกับผลประโยชน์ที่ยั่งยืนด้านสุขภาพและความสมดุลระหว่างชีวิตและการทํางาน
- ความสําเร็จแตกต่างกันไปขึ้นอยู่กับข้อกําหนดเฉพาะของบริษัทและประสิทธิภาพของแผนการดําเนินงาน โดยบางบริษัททํานโยบายถาวรในขณะที่บางบริษัทประเมินผลกระทบ
- ความท้าทายรวมถึงการสร้างสมดุลระหว่างความคาดหวังของลูกค้าการรักษาความสามารถในการปรับตัวในสถานการณ์ที่ท้าทายและรับประกันความยืดหยุ่นในพนักงาน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39562760)

- การใช้สัปดาห์ทํางาน 4 วันใน บริษัท ในสหราชอาณาจักร 61 แห่งส่งผลให้มีการรักษาพนักงานเพิ่มขึ้นสุขภาพจิตที่ดีขึ้นและคุณภาพการทํางานที่ดีขึ้น แต่ผลกระทบต่อประสิทธิภาพและประสิทธิผลยังคงไม่แน่นอน
- ความท้าทายที่กล่าวถึง ได้แก่ ปัญหาการทํางานระยะไกลการสนับสนุนด้านไอทีปัญหาขององค์กรและแรงจูงใจของพนักงานในขณะที่มีการถกเถียงกันถึงข้อโต้แย้งสําหรับและต่อต้านสัปดาห์การทํางานที่สั้นลงสําหรับผู้ปฏิบัติงานที่มีความรู้
- การสนทนากล่าวถึงความกังวลเกี่ยวกับสหภาพแรงงานความเป็นไปได้ในการทํางานระยะไกลและการต่อต้านการเปลี่ยนแปลงรูปแบบสัปดาห์การทํางานแบบดั้งเดิมกระตุ้นให้เกิดการถกเถียงกันเกี่ยวกับการสร้างสมดุลระหว่างชั่วโมงการทํางานประสิทธิภาพการทํางานและความเป็นอยู่ที่ดีในภูมิทัศน์การทํางานที่กําลังพัฒนาในปัจจุบัน

## [WhatsApp ได้รับรหัสสปายแวร์เพกาซัสในการชนะทางกฎหมาย](https://arstechnica.com/tech-policy/2024/03/whatsapp-finally-forces-pegasus-spyware-maker-to-share-its-secret-code/)

- WhatsApp สามารถเข้าถึงรหัสสปายแวร์เพกาซัสของ NSO Group หลังจากมีข้อพิพาททางกฎหมาย ทําให้สามารถเจาะลึกฟังก์ชันการทํางานที่สมบูรณ์ของสปายแวร์ได้
- ศาลยกฟ้องความพยายามของ NSO ในการให้รายละเอียดเกี่ยวกับเลเยอร์การติดตั้งเพียงอย่างเดียว โดยอนุญาตให้ WhatsApp เข้าถึงในวงกว้างขึ้นเพื่อสํารวจความสามารถของสปายแวร์
- NSO ไม่จําเป็นต้องเปิดเผยลูกค้าหรือโครงสร้างเซิร์ฟเวอร์ โดยมีกําหนดเริ่มทดลองใช้ในปี 2025

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39566766)

- WhatsApp กําลังต่อสู้ทางกฎหมายกับผู้ผลิตสปายแวร์เพกาซัสเกี่ยวกับการแบ่งปันรหัสลับทําให้เกิดความกังวลเกี่ยวกับความมั่นคงของชาติและการกํากับดูแลในระบบศาล FISA
- การอภิปรายเกี่ยวกับไวยากรณ์อิทธิพลของสหรัฐฯในการธนาคารทั่วโลกสกุลเงินดิจิทัลที่หลีกเลี่ยงการควบคุมของรัฐบาลและเขตอํานาจศาลของระบบกฎหมายอเมริกันเหนือหน่วยงานต่างประเทศ
- การสนทนาเจาะลึกถึงประเด็นขัดแย้งทางจริยธรรมในการรักษาความปลอดภัยทางไซเบอร์และงานที่เป็นที่ถกเถียงกันของ Project Zero ในการระบุช่องโหว่ในระบบต่างๆ

## [CSS-Tricks: การเพิ่มขึ้น การตก และโอกาสที่สูญเสียไป](https://chriscoyier.net/2024/02/28/where-im-at-on-the-whole-css-tricks-thing/)

- ผู้เขียนซึ่งขาย CSS-Tricks ให้กับ DigitalOcean เห็นว่าเนื้อหาและชุมชนของไซต์ถูกรื้อถอนหลังจากจุดเริ่มต้นที่มีความหวัง
- แม้จะพยายามเรียกคืนความเป็นเจ้าของเพื่อฟื้นฟูไซต์ แต่ DigitalOcean ปฏิเสธข้อเสนอของผู้เขียน
- แผนที่ผู้เขียนเสนอให้เปิดไซต์ใหม่ไม่ได้ปฏิบัติตาม

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39560705)

- การเข้าซื้อกิจการ CSS-Tricks ของ DigitalOcean ซึ่งเป็นเว็บไซต์ยอดนิยมสําหรับนักพัฒนาส่วนหน้า ได้รับปฏิกิริยาที่หลากหลายจากชุมชน
- บางคนยกย่อง CSS-Tricks สําหรับทรัพยากรที่มีค่า ในขณะที่คนอื่นๆ วิพากษ์วิจารณ์ DigitalOcean ที่ลบเนื้อหาที่จําเป็นและขาดการสื่อสารกับผู้ก่อตั้ง
- มีความกังวลเกี่ยวกับผลกระทบของการขายบล็อกนักพัฒนาอิสระให้กับบริษัทขนาดใหญ่ โดยมีความกังวลเกี่ยวกับคุณภาพและการสูญเสียเนื้อหาที่อาจเกิดขึ้น พร้อมกับการวิพากษ์วิจารณ์ Tailwind CSS และความกังวลเกี่ยวกับผลกระทบของทางลัดในการพัฒนาเว็บต่อโอกาสระยะยาวของอุตสาหกรรม

## [Atlanta Magnet Man: รักษาถนนให้ปลอดภัย ขี่ทีละครั้ง](https://www.wabe.org/the-atlanta-magnet-man-is-saving-our-car-tires-one-bike-ride-at-a-time/)

## [การเพิ่มความปลอดภัย cURL ด้วยเทคนิคการคลุมเครือขั้นสูง](https://blog.trailofbits.com/2024/03/01/toward-more-effective-curl-fuzzing/)

## [ขอแนะนํา JSR: JavaScript Registry](https://jsr.io)

- JSR เป็นรีจิสทรีของแพ็คเกจโอเพ่นซอร์สที่ปรับให้เหมาะกับ JavaScript และ TypeScript สมัยใหม่ โดยจัดเตรียมเอกสาร API ไฟล์ .d.ts และการแปลงสําหรับความเข้ากันได้ข้ามรันไทม์
- เข้ากันได้กับ Node, Deno และ Cloudflare Workers JSR ทํางานร่วมกับตัวจัดการแพ็คเกจ JavaScript และปรับปรุง npm โดยนําเสนอโมดูลที่พิมพ์อย่างหนักและการสนับสนุนตัวแก้ไข
- JSR ไม่ได้แทนที่ npm แต่ทําหน้าที่เป็นเครื่องมือเสริม เพื่อให้มั่นใจว่านักพัฒนาจะได้รับประสบการณ์ที่ราบรื่นด้วยการพิมพ์ที่มีประสิทธิภาพและความสามารถในการทํางานข้ามรันไทม์

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39561594)

- JavaScript Registry (JSR) แนะนํารีจิสทรีของแพ็คเกจใหม่โดยเน้นที่คุณลักษณะต่างๆ เช่น การสร้างเอกสาร API ไฟล์ .d.ts และความเข้ากันได้ข้ามรันไทม์ผ่านการทรานสไพล์ โดยมีเป้าหมายเพื่อเสริม npm แทนที่จะแทนที่
- แม้จะมีความสงสัยเกี่ยวกับการกระจายตัวในระบบนิเวศ JavaScript (JS) แต่ JSR ก็ให้ประโยชน์ต่างๆ เช่น ความเข้ากันได้กับ TypeScript และ JavaScript การสร้างที่เก็บแพ็คเกจส่วนกลาง และการจัดการขอบเขตและโดเมน โดยแยกความแตกต่างจาก npm, Deno และ Node
- การอภิปรายเกี่ยวกับ JSR เน้นความคิดเห็นที่แตกแยกเกี่ยวกับยูทิลิตี้ โดยบางส่วนเน้นถึงข้อดีสําหรับผู้เขียนแพ็คเกจและผู้บริโภค
