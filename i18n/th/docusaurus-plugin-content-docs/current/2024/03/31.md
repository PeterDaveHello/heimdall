## [Linux Landlock Syscall Check ปรับปรุงที่เก็บ Xz.git](https://git.tukaani.org/?p=xz.git;a=blobdiff;f=CMakeLists.txt;h=d2b1af7ab0ab759b6805ced3dff2555e2a4b3f8e;hp=76700591059711e3a4da5b45cf58474dac4e12a7;hb=328c52da8a2bbb81307644efdb58db2c422d9ba7;hpb=eb8ad59e9bab32a8d655796afd39597ea6dcc64d)

- คอมมิตในที่เก็บ xz.git จะกล่าวถึงการทดสอบคุณลักษณะ Linux Landlock ใน Autotools และ CMake builds โดยรวมการตรวจสอบการคอมไพล์เพื่อตรวจสอบความพร้อมใช้งานของ syscalls ที่จําเป็นสําหรับการใช้งาน Linux Landlock
- เมื่อ syscalls ที่จําเป็นพร้อมใช้งาน SANDBOX_COMPILE_DEFINITION จะถูกกําหนดค่าเป็น "HAVE_LINUX_LANDLOCK" และเปิดใช้งาน SANDBOX_FOUND เพื่อรองรับความเข้ากันได้กับเทคนิคแซนด์บ็อกซ์ Landlock ใน XZ Utils

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39874404)

- ปัญหาอักขระตัวเดียวในโครงการ Xz ปิดใช้งานระบบควบคุมการเข้าถึง Linux landlock ทําให้เกิดการอภิปรายเกี่ยวกับการป้องกันการโจมตีที่คล้ายกันโดยใช้อักขระที่ไม่ได้มาตรฐานในซอร์สโค้ด
- เหตุการณ์ดังกล่าวเน้นย้ําถึงความสําคัญของการจัดการข้อผิดพลาดในตัวอย่างการตรวจจับคุณลักษณะ และเน้นย้ําถึงความท้าทายด้วยช่องว่าง แท็บ และช่องว่างในการเข้ารหัส Python
- การอภิปรายครอบคลุมถึงการดัดแปลงโดยเจตนาในการพัฒนาซอฟต์แวร์ความซับซ้อนของการทดสอบและความปลอดภัยการลบฟังก์ชันการทํางานของ landlock ผลกระทบด้านความปลอดภัยของการตรวจจับคุณลักษณะความท้าทายในการเขียนซอฟต์แวร์แบบพกพาและความก้าวหน้าในการกําหนดเวอร์ชันไลบรารีและการจัดการแพ็คเกจ

- เน้นย้ําถึงความสําคัญของการล้างข้อมูลการควบคุมอายุการใช้งานของวัตถุที่แม่นยําและการจัดการทรัพยากรในการเขียนโปรแกรมโดยเน้นถึงบทบาทสําคัญของการจัดการหน่วยความจําในการคํานวณและการเพิ่มประสิทธิภาพ

## [เครื่องมือ OCR บนเว็บประมวลผล PDF และรูปภาพทันที](https://simonwillison.net/2024/Mar/30/ocr-pdfs-images/)

- Simon Willison กล่าวถึงความท้าทายในการดึงข้อมูลจาก PDF และรูปภาพในการประชุมวารสารศาสตร์ข้อมูล Story Discovery At Scale
- เขาสร้างเว็บแอป tools.simonwillison.net/ocr โดยใช้ประโยชน์จาก Tesseract OCR ในเบราว์เซอร์สําหรับ PDF และรูปภาพ โดยไม่จําเป็นต้องอัปโหลดข้อมูลไปยังเซิร์ฟเวอร์
- ด้วยการใช้โมเดล LLM เขาได้พัฒนาและปรับปรุงเครื่องมืออย่างมีประสิทธิภาพด้วยคุณสมบัติต่างๆ เช่น การลากและวางและ OCR รูปภาพ ทําให้มั่นใจได้ถึงความยั่งยืนด้วยส่วนประกอบ JavaScript และ WebAssembly แบบคงที่

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39877391)

- การสนทนามุ่งเน้นไปที่การใช้เทคโนโลยี OCR (การรู้จําอักขระด้วยแสง) โดยเฉพาะ tesseract.js เพื่อแยกข้อความจาก PDF และรูปภาพโดยตรงในเบราว์เซอร์

- ผู้ใช้พูดคุยเกี่ยวกับการอ่านซ้ํา การเลือกประเภท และคุณค่าของวรรณกรรมที่ซับซ้อน ปกป้องงานของมอร์ริสันท่ามกลางการพิจารณา เช่น ห้องสมุดโรงเรียนและผลกระทบทางการเมือง

## [เปิดเผยความสุ่มด้วยความซับซ้อนของ Kolmogorov](https://smunshi.net/kolmogorov-complexity-and-compression-distance.html)

- อลิซสงสัยว่าเล่นผิดกติกาในเกมโยนเหรียญกับบ็อบเนื่องจากลําดับของหางทั้งหมดนําไปสู่การถกเถียงกันเกี่ยวกับความน่าจะเป็นและการสุ่มของลําดับ
- Kolmogorov Complexity ถูกนํามาใช้เพื่อวัดการสุ่มโดยการค้นหาโปรแกรมที่สั้นที่สุดเพื่อสร้างสตริงจัดการกับการพึ่งพาภาษาด้วยภาษาสากลและเครื่องทัวริง
- Normalized Information Distance และ Normalized Compression Distance คํานวณโดยใช้ Kolmogorov Complexity ซึ่งช่วย Alice ในการชิงไหวชิงพริบ Bob ในเกม

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39877276)

- บทความนี้เจาะลึกถึงความซับซ้อนและระยะการบีบอัดของ Kolmogorov โดยเน้นวิธีการที่ไม่เชื่อเรื่องพระเจ้าโดยใช้แคลคูลัสแลมบ์ดาเพื่อกําหนดความซับซ้อน
- สํารวจความท้าทายในการเปรียบเทียบคําอธิบายที่เหมาะสมที่สุดในภาษาต่างๆ ข้อจํากัดของเครื่องทัวริง และความสําคัญของตัวอักษรไบนารีในวิธีการบีบอัด
- การอภิปรายครอบคลุมการค้นหาเชิงสัญลักษณ์ การสุ่ม การสร้างเมล็ดพันธุ์ การบีบอัด และความสัมพันธ์ระหว่างความซับซ้อนของเอนโทรปีแชนนอนและโคลโมโกรอฟ โดยให้ข้อมูลเชิงลึกเกี่ยวกับการวัดความซับซ้อนนอกเหนือจากวิธีการดั้งเดิม เช่น เอนโทรปีของแชนนอน
