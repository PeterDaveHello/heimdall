# 2024-02-09

## [FCC แบนเสียง AI ใน Robocalls](https://www.fcc.gov/document/fcc-makes-ai-generated-voices-robocalls-illegal)

- FCC ได้ประกาศว่าการใช้เสียงที่สร้างโดย AI ใน robocalls นั้นผิดกฎหมายภายใต้ TCPA
- คําตัดสินนี้ได้รับการรับรองอย่างเป็นเอกฉันท์โดย FCC เพื่อจัดการกับเทคโนโลยีเสียงและวิดีโอปลอมลึกที่ใช้ในการโทรแบบโรโบคอลและข้อความหลอกลวง
- การตัดสินใจนี้มีจุดมุ่งหมายเพื่อต่อสู้กับการใช้เสียงเทียมเพื่อปกป้องผู้บริโภคจากกิจกรรมฉ้อโกง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39304736)

- ความถูกต้องตามกฎหมายของเสียงที่สร้างโดย AI ใน robocalls กําลังเป็นที่ถกเถียงกัน โดยมีข้อโต้แย้งสําหรับและต่อต้านกฎระเบียบ
- หลักการของ Chevron Deference อยู่ภายใต้การตรวจสอบข้อเท็จจริงเกี่ยวกับเสียง AI ในการโทรแบบโรโบคอล
- การอภิปรายในวงกว้างรวมถึงความสมดุลของอํานาจระหว่างหน่วยงานบริหารและสภานิติบัญญัติความท้าทายในการควบคุมเทคโนโลยีใหม่และการขาดฉันทามติในประเด็นที่ถกเถียงกัน

## [Mozilla แต่งตั้ง CEO คนใหม่ท่ามกลางการเปลี่ยนแปลงความเป็นส่วนตัวของข้อมูล](https://fortune.com/2024/02/08/mozilla-firefox-ceo-laura-chambers-mitchell-baker-leadership-transition/)

- Mozilla Corp. ได้ประกาศการเปลี่ยนแปลงความเป็นผู้นําเมื่อ CEO Mitchell Baker ก้าวลงจากตําแหน่งเพื่อมุ่งเน้นไปที่ AI และความปลอดภัยทางอินเทอร์เน็ต
- Laura Chambers สมาชิกคณะกรรมการของ Mozilla ที่มีประสบการณ์ในบริษัทต่างๆ เช่น Airbnb และ PayPal จะดํารงตําแหน่ง CEO ชั่วคราว
- ความเคลื่อนไหวดังกล่าวเกิดขึ้นเนื่องจาก Mozilla มีเป้าหมายที่จะจัดการกับข้อกังวลด้านความเป็นส่วนตัวและความท้าทายที่เกิดจากเทคโนโลยีขนาดใหญ่และ AI ในขณะเดียวกันก็มุ่งเน้นไปที่การพัฒนาผลิตภัณฑ์และนโยบายที่ให้ความสําคัญกับการควบคุมข้อมูลผู้ใช้และเสนอทางเลือกให้กับรูปแบบธุรกิจที่มีอยู่

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39302744)

- Mozilla ได้แต่งตั้งซีอีโอคนใหม่โดยมุ่งเน้นที่ความเป็นส่วนตัวของข้อมูล ซึ่งจุดประกายการอภิปรายเกี่ยวกับความเป็นผู้นําและส่วนแบ่งการตลาดที่หดตัวของบริษัท
- มีการถกเถียงกันเกี่ยวกับเงินเดือนที่หนักหน่วงของอดีตซีอีโอและความท้าทายที่ Mozilla ต้องเผชิญในการแข่งขันกับ Google Chrome
- หัวข้ออื่น ๆ ที่กําลังกล่าวถึง ได้แก่ ค่าตอบแทนผู้บริหารความล้มเหลวของ FirefoxOS และกลยุทธ์ในการดึงดูดผู้ใช้ไปยังเบราว์เซอร์ Firefox

## [Gemini เปิดตัวแอพมือถือและโมเดล AI ขั้นสูงสําหรับการเข้ารหัสและความคิดสร้างสรรค์](https://blog.google/products/gemini/bard-gemini-advanced-app/)

- ประสบการณ์ขั้นสูงของ Gemini มีให้บริการในกว่า 150 ประเทศ โดยจะมีการเพิ่มภาษาและสถานที่เพิ่มเติมในอนาคต เป็นส่วนหนึ่งของแผน Google One AI Premium ราคา $19.99/เดือน และเสนอการสอนพิเศษ ความช่วยเหลือในการเขียนโค้ด และการสร้างเนื้อหาในแบบของคุณ ผู้ใช้สามารถทํางานร่วมกันและขอความช่วยเหลือบนอุปกรณ์ Android และ iOS

- Google ได้รีแบรนด์โมเดลภาษา "Bard" เป็น "Gemini" และเปิดตัวแอพมือถือและ Gemini Advanced
- การตัดสินใจเปลี่ยนชื่อได้จุดประกายปฏิกิริยาที่หลากหลาย โดยผู้ใช้บางคนชอบความสัมพันธ์ของ Bard กับคําพูด และคนอื่นๆ พบว่าราศีเมถุนเหมาะสมกว่าสําหรับการเชื่อมต่อกับการสํารวจอวกาศ
- การอภิปรายยังกล่าวถึงความสําคัญของชื่อผลิตภัณฑ์การรับรู้ของผู้ใช้ความสามารถของราศีเมถุนความกังวลเกี่ยวกับการตีความผิดหรือความสับสนที่อาจเกิดขึ้นตําแหน่งของ Google ในการแข่งขัน AI การรักษาความสามารถและข้อ จํากัด ของโมเดล AI ในการทําความเข้าใจภาษาและการตอบสนองที่ถูกต้อง

## [ตอนนี้ VirtualBox รองรับไฮเปอร์ไวเซอร์ KVM พร้อมการเปิดตัวโอเพ่นซอร์สจาก Cyberus Technology](https://cyberus-technology.de/articles/vbox-kvm-public-release)

- Cyberus Technology ได้พัฒนาแบ็กเอนด์ KVM แบบโอเพ่นซอร์สสําหรับ VirtualBox ทําให้จอภาพเครื่องเสมือนทํางานบนไฮเปอร์ไวเซอร์ Linux KVM แทนที่จะเป็นโมดูลเคอร์เนลที่เป็นกรรมสิทธิ์
- รุ่นนี้ให้ผู้ใช้มีตัวเลือกอื่นในการเรียกใช้ VirtualBox บนไฮเปอร์ไวเซอร์ KVM ซึ่งอาจปรับปรุงประสิทธิภาพและความเข้ากันได้
- ลักษณะโอเพ่นซอร์สของแบ็กเอนด์เปิดโอกาสให้นักพัฒนามีส่วนร่วมปรับปรุงและปรับแต่งการผสานรวมของ VirtualBox กับ KVM

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39300317)

- การสนทนากล่าวถึงการเปิดตัวแบ็กเอนด์ KVM แบบโอเพ่นซอร์สสําหรับ VirtualBox ซึ่งช่วยเพิ่มตัวเลือกซอฟต์แวร์เวอร์ชวลไลเซชัน
- ผู้ใช้เปรียบเทียบ VirtualBox กับตัวเลือกเครื่องเสมือนอื่น ๆ โดยพิจารณาจากปัจจัยต่างๆเช่นประสิทธิภาพความเป็นมิตรต่อผู้ใช้และความเข้ากันได้
- หัวข้อต่างๆ ได้แก่ ความผิดหวังกับตัวเลือก GUI สําหรับการจําลองเสมือน การอภิปรายเกี่ยวกับ GPU passthrough, virt-manager และเทคโนโลยีการจําลองเสมือนต่างๆ เช่น UML และ KVM รวมถึงการกล่าวถึงคุณลักษณะด้านความปลอดภัย ปัญหาใบอนุญาต และการพัฒนาที่อาจเกิดขึ้นในอนาคต

## [Ollama เพิ่มความเข้ากันได้กับ OpenAI Chat Completion API เปิดใช้งานการใช้เครื่องมือเพิ่มเติมในท้องถิ่น](https://ollama.ai/blog/openai-compatibility)

- ขณะนี้แพลตฟอร์ม Ollama เข้ากันได้กับ OpenAI Chat Completion API ซึ่งขยายขีดความสามารถในการทํางานกับเครื่องมือและแอปพลิเคชันในเครื่องมากขึ้น
- ผู้ใช้สามารถตั้งค่า Ollama ได้โดยดาวน์โหลดแพลตฟอร์มและดึงโมเดล จากนั้นเข้าถึง API ผ่าน cURL หรือไลบรารี OpenAI Python หรือ JavaScript
- ตัวอย่างการผสานรวมแสดงให้เห็นว่าสามารถใช้ Ollama กับไลบรารีอื่นๆ ได้อย่างไร รวมถึง Vercel AI SDK และ Autogen และกล่าวถึงการปรับปรุงในอนาคตสําหรับความเข้ากันได้ของ OpenAI

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39307330)

- การสนทนาเกี่ยวกับโมเดลภาษา OpenAI รวมถึงโมเดลต่างๆ เช่น Ollama, Deepseek Coder 33B และ Gemini Pro และ Ultra
- ผู้ใช้แบ่งปันประสบการณ์ ข้อกังวล และความชอบที่เกี่ยวข้องกับประสิทธิภาพ ความเข้ากันได้
- การสนทนายังกล่าวถึงหัวข้อต่างๆ เช่น OpenAI API, ความต้องการข้อกําหนดและเอกสารที่ชัดเจนยิ่งขึ้น, มาตรฐาน API ทั่วไป, เครื่องมือและไลบรารี เช่น Langchain และหม้อแปลง Huggingface และการอภิปรายเกี่ยวกับการกระจายซอฟต์แวร์ ราคา และกรอบการปรับใช้

## [การเพิ่มประสิทธิภาพ VSE ของ Blender โดยไม่ได้ตั้งใจนําไปสู่การปรับปรุงที่สําคัญ](https://aras-p.info/blog/2024/02/06/I-accidentally-Blender-VSE/)

- ผู้เขียนทํางานโดยไม่ได้ตั้งใจกับ Video Sequence Editor (VSE) ของ Blender และทําการปรับปรุงประสิทธิภาพหลายอย่างที่จะรวมอยู่ในรุ่นถัดไป
- พวกเขาปรับปรุงการออกแบบเวกเตอร์สโคปโปรแกรมแก้ไขภาพใช้มัลติเธรดสําหรับการแปลง RGB↔YUV และแก้ไขปัญหาเกี่ยวกับการสุ่มตัวอย่างเสียง
- นอกจากนี้ พวกเขายังเพิ่มตัวเลือกการกรองใหม่สําหรับการปรับขนาดภาพและปรับเอฟเฟกต์ต่างๆ ให้เหมาะสมใน VSE แม้จะไม่คุ้นเคยกับพื้นที่และฐานโค้ด แต่นักพัฒนาก็ประสบความสําเร็จอย่างมากในเวลาเพียงสองเดือนของการทํางานนอกเวลา และพวกเขาตั้งใจที่จะสนับสนุน Blender ต่อไปในอนาคต

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39305994)

- ผู้เขียนแบ่งปันประสบการณ์ของพวกเขาในการมีประสิทธิผลมากขึ้นในโครงการที่ไม่ได้รับค่าจ้างเมื่อเทียบกับโครงการที่ได้รับค่าจ้างโดยตั้งคําถามกับความเชื่อที่ว่าค่าจ้างและกําไรเป็นแรงจูงใจเพียงอย่างเดียวในการทํางาน
- มีการกล่าวถึงแนวคิดของการทํางานและการปลดความเพลิดเพลินออกจากการทํางานหรือการเล่นโดยเน้นย้ําถึงความสําคัญของการค้นหาแรงจูงใจในการทํางาน
- นอกจากนี้ยังมีการกล่าวถึงผลกระทบของการพัฒนาซอฟต์แวร์ ทางเลือกระหว่างงานที่ได้รับค่าจ้างและงานที่มีผลกระทบ และศักยภาพในการช่วยเหลือ AI ในการสร้างแอปเครื่องเล่นเพลงที่ดีขึ้นสําหรับ iOS

## [Microsoft เปิดตัว Sudo สําหรับ Windows ใน Windows 11 Insider Preview Build](https://devblogs.microsoft.com/commandline/introducing-sudo-for-windows/)

- Microsoft ได้เปิดตัว Sudo สําหรับ Windows ใน Windows 11 Insider Preview Build 26052 ทําให้ผู้ใช้สามารถเรียกใช้คําสั่งยกระดับจากเซสชันคอนโซลที่ไม่ได้ยกระดับ
- Sudo สําหรับ Windows เป็นโครงการโอเพ่นซอร์สบน GitHub และมอบโซลูชันตามหลักสรีรศาสตร์สําหรับผู้ใช้ที่ต้องการยกระดับคําสั่งโดยไม่ต้องเปิดคอนโซลยกระดับใหม่
- Sudo เวอร์ชันปัจจุบันสําหรับ Windows รองรับตัวเลือกการกําหนดค่าสามแบบและต้องการการยืนยันจากผู้ใช้ผ่านกล่องโต้ตอบ UAC สําหรับระดับความสูง Microsoft วางแผนที่จะขยายเอกสารประกอบและแบ่งปันเพิ่มเติมเกี่ยวกับผลกระทบด้านความปลอดภัยของโครงการและแผนโอเพนซอร์สในอนาคต

- การอภิปรายจะกล่าวถึงหัวข้อที่เกี่ยวข้องกับ Windows หลายหัวข้อ เช่น การแนะนํา Sudo สําหรับ Windows และการรวมทีมเพื่อโซลูชันที่มีประสิทธิภาพ
- มีความกังวลเกี่ยวกับการใช้ชื่อ "sudo" และอาจทําให้เกิดความสับสน
- นอกจากนี้ยังมีการกล่าวถึงความสามารถของ PowerShell สําหรับการพัฒนา GUI ความผิดหวังกับประสบการณ์ของนักพัฒนา Windows และปัญหาเกี่ยวกับมาตรการรักษาความปลอดภัยและการปรับใช้ซอฟต์แวร์

- มีการถกเถียงกันอย่างต่อเนื่องเกี่ยวกับผลกระทบและจริยธรรมของการละเมิดลิขสิทธิ์ ตลอดจนความกังวลเกี่ยวกับบริษัทที่ผิดสัญญาและโอนภาระผูกพันผ่านธุรกรรมทางธุรกิจ

## [ระบบแบบกระจาย: รายการเรื่องรออ่านที่ครอบคลุมซึ่งครอบคลุมทฤษฎี แบบจําลอง และการตรวจจับข้อผิดพลาด](https://ferd.ca/a-distributed-systems-reading-list.html)

- เอกสารนี้เป็นรายการเรื่องรออ่านของระบบแบบกระจายที่ให้ข้อมูลพื้นฐานและทรัพยากรในหัวข้อ
- แบบจําลองต่างๆ ที่ใช้ในทฤษฎีระบบแบบกระจาย และโหมดความล้มเหลวทางทฤษฎี
- กล่าวถึงความสําคัญของการตรวจจับข้อผิดพลาดในระบบแบบกระจายทฤษฎีบท CAP การส่งผ่านข้อความการสั่งซื้อและแบบจําลองความสอดคล้องที่แตกต่างกัน

- ประสบการณ์จริงได้รับการเน้นย้ําว่ามีความสําคัญ โดยมีคําแนะนํา เช่น การออกแบบโครงการ การใช้ VM หรือฮาร์ดแวร์เก่า และการสํารวจเฟรมเวิร์กและแอปพลิเคชัน
- แนะนําให้ใช้หลักสูตร Distributed Systems ของ MIT และแหล่งข้อมูลอื่นๆ สําหรับการเรียนรู้เพิ่มเติม แต่ไม่มีคําแนะนําเฉพาะสําหรับหนังสือเกี่ยวกับธุรกรรมแบบกระจาย

## [หน่วยข่าวกรองของตํารวจเนเธอร์แลนด์สอดแนมกลุ่มชุมชนอย่างผิดกฎหมาย ละเมิดกฎหมายความเป็นส่วนตัว](https://nltimes.nl/2024/02/08/police-intelligence-services-unlawfully-spied-whole-population-groups)

- การสืบสวนละเมิดกฎหมายของเนเธอร์แลนด์เนื่องจากการบุกรุกความเป็นส่วนตัวที่เกี่ยวข้อง และหน่วยข่าวกรองไม่ได้แสดงให้เห็นถึงความจําเป็นในการสืบสวนในวงกว้างอย่างเหมาะสม

## [รัฐบาลแคนาดาลงทุนในการต่อสู้กับการโจรกรรมรถยนต์ด้วยการแบนอุปกรณ์](https://www.canada.ca/en/public-safety-canada/news/2024/02/federal-action-on-combatting-auto-theft.html)

- รัฐบาลแคนาดากําลังดําเนินการของรัฐบาลกลางเพื่อแก้ไขปัญหาการโจรกรรมรถยนต์และเพิ่มความปลอดภัยชายแดน
- สํานักงานบริการชายแดนแคนาดา (CBSA) จะลงทุน 28 ล้านดอลลาร์เพื่อปรับปรุงความสามารถในการสืบสวนและต่อสู้กับการโจรกรรมรถยนต์
- หน่วยงานรัฐบาลต่างๆ รวมถึง Royal Canadian Mounted Police (RCMP), Department of Justice Canada, Transport Canada และ Innovation, Science and Economic Development Canada (ISED) กําลังร่วมมือกันเพื่อเสริมสร้างกรอบกฎหมาย ปรับปรุงมาตรฐานความปลอดภัยของยานพาหนะให้ทันสมัย และพัฒนาโซลูชันเพื่อป้องกันการโจรกรรมรถยนต์

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39308731)

- การสนทนาครอบคลุมหัวข้อต่างๆ เช่น การห้ามใช้อุปกรณ์ Flipper Zero ในแคนาดา ประสิทธิภาพของประชาธิปไตยทางตรง และข้อบกพร่องด้านความปลอดภัยในการเข้ารหัสกุญแจรถยนต์
- มีความคิดเห็นที่แตกต่างกันเกี่ยวกับการห้ามอุปกรณ์บางอย่างและผลกระทบที่อาจเกิดขึ้นกับการโจรกรรมรถยนต์และอาชญากรรม
- การสนทนายังกล่าวถึงหัวข้อที่ไม่เกี่ยวข้อง เช่น กฎระเบียบของรัฐบาล เศรษฐกิจ และการย้ายถิ่นฐาน โดยมุ่งเน้นโดยรวมที่ประสิทธิภาพของกฎระเบียบ บทบาทของรัฐบาล และผลกระทบต่อบุคคลและสังคม

## [เอฟเฟกต์ทางเข้าประตู: การย้ายระหว่างสถานที่ต่างๆ ทําให้สูญเสียความทรงจําได้อย่างไร](https://en.wikipedia.org/wiki/Doorway_effect)

- "เอฟเฟกต์ทางเข้าประตู" เป็นปรากฏการณ์ทางจิตวิทยาที่การสูญเสียความทรงจําเกิดขึ้นเมื่อเคลื่อนที่ไปมาระหว่างสถานที่หรือผ่านทางเข้าประตู
- เกิดจากสมองสร้างแบบจําลองหน่วยความจําแยกต่างหากสําหรับสภาพแวดล้อมที่แตกต่างกัน
- ประตูทําหน้าที่เป็นขอบเขตของเหตุการณ์และส่งผลต่อการเก็บรักษาความทรงจําซึ่งนําไปสู่ความทรงจําที่ผิดพลาดเพิ่มขึ้น แต่ไม่มีผลต่อการลืม

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39307677)

- The Doorway Effect เป็นปรากฏการณ์ที่ผู้คนมักลืมจุดประสงค์เมื่อเข้าห้องใหม่ ซึ่งนําไปสู่การสนทนาบน Reddit เกี่ยวกับประสบการณ์ส่วนตัวและกลยุทธ์ในการเอาชนะสิ่งรบกวนที่เกิดจากสภาพแวดล้อมที่เปลี่ยนแปลง
- การสนทนาบน Reddit ครอบคลุมหัวข้อที่เกี่ยวข้อง รวมถึงการเรียกคืนหน่วยความจําตามบริบท การแสดงตนเชิงพื้นที่ การเชื่อมโยงหน่วยความจํา และประสบการณ์ของคนตาบอด
- ความคิดเห็นบางส่วนแสดงความสงสัยเกี่ยวกับความถูกต้องของการศึกษาทางจิตวิทยาและเน้นย้ําถึงความสําคัญของการออกแบบการทดลองและการจําลองแบบที่เข้มงวด
