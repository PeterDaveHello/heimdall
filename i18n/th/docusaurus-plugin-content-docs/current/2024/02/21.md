## [เพิ่มความเป็นส่วนตัวบน Signal: ซ่อนหมายเลขโทรศัพท์ของคุณด้วยชื่อผู้ใช้](https://signal.org/blog/phone-number-privacy-usernames/)

- Signal กําลังเพิ่มความเป็นส่วนตัวโดยทําให้ผู้ใช้สามารถปกปิดหมายเลขโทรศัพท์ของตนโดยค่าเริ่มต้นและใช้ชื่อผู้ใช้ที่ไม่ซ้ํากันสําหรับการสื่อสารควบคุมว่าใครสามารถค้นพบได้ด้วยหมายเลขโทรศัพท์
- ชื่อผู้ใช้จะถูกเก็บไว้เป็นส่วนตัวในการแชทและสามารถแลกเปลี่ยนเพื่อเชื่อมต่อโดยไม่ต้องเปิดเผยหมายเลขโทรศัพท์โดยผู้ใช้มีความยืดหยุ่นในการปรับการตั้งค่าและชื่อผู้ใช้ได้ตลอดเวลา
- คุณสมบัติใหม่เหล่านี้เป็นทางเลือกและมีเป้าหมายเพื่อรักษาความเป็นส่วนตัวระหว่างการโต้ตอบบนแพลตฟอร์ม Signal

## [ในการป้องกันสถาปัตยกรรมที่เรียบง่าย: กรณีของ บริษัท $ 1.7B Wave](https://danluu.com/simple-architectures/)

- Wave ซึ่งเป็นบริษัทมูลค่า 1.7 พันล้านดอลลาร์เลือกใช้สถาปัตยกรรมแอป CRUD ที่เรียบง่ายพร้อมเสาหิน Python บน Postgres ซึ่งคล้ายกับ Stackoverflow ซึ่งแสดงให้เห็นถึงความชอบในความเรียบง่ายมากกว่าความซับซ้อนในความสามารถในการปรับขนาด
- บริษัทประมวลผลคําขอรายเดือนหลายพันล้านรายการด้วย Python แบบซิงโครนัสและคิวงาน ซึ่งต้องเผชิญกับอุปสรรคในการผสานรวมโทรคมนาคมและการปฏิบัติตามกฎระเบียบระหว่างการขยายตัวทั่วโลก
- แม้ว่าเทรนด์เทคโนโลยีจะชอบการตั้งค่าที่ซับซ้อน แต่ Wave ก็ใช้ GraphQL, Kubernetes และโปรโตคอลการขนส่งแบบกําหนดเองสําหรับ API โดยเน้นที่ความง่ายในการปฏิบัติงานและความคุ้มค่าด้วยทีมวิศวกรขนาดเล็ก

- การอภิปรายครอบคลุมถึงความสามารถในการปรับขนาด การบํารุงรักษา ความซับซ้อน และการตัดสินใจทางสถาปัตยกรรมที่มีอิทธิพลต่อประสิทธิภาพและความสมบูรณ์ของข้อมูล โดยเฉพาะอย่างยิ่งในบริการทางการเงิน
- เน้นการสร้างสมดุลระหว่างความเรียบง่ายและความซับซ้อนบรรลุวัตถุประสงค์ทางธุรกิจบรรลุผลลัพธ์ที่จับต้องได้และการจัดการการเปลี่ยนผ่านระหว่างบทบาทและเทคโนโลยีในสาขา

## [ความก้าวหน้าของ SSD แซงหน้าข้อเสนอระบบคลาวด์](http://databasearchitects.blogspot.com/2024/02/ssds-have-become-ridiculously-fast.html)

- บล็อกโพสต์เน้นย้ําถึงความก้าวหน้าของเทคโนโลยี SSD ในด้านความเร็วและความจุ ซึ่งตรงกันข้ามกับการปรับปรุง SSD สินค้าโภคภัณฑ์กับประสิทธิภาพ SSD บนคลาวด์ที่ซบเซาจากผู้จําหน่ายรายใหญ่ เช่น AWS และ Azure
- มีการคาดเดาสาเหตุที่เป็นไปได้สําหรับช่องว่างประสิทธิภาพของ SSD บนคลาวด์ เช่น ความกังวลเกี่ยวกับความล้มเหลวของอุปกรณ์
- โพสต์สนับสนุนการเปิดตัว SSD ที่รวดเร็วขึ้นในอนาคตในอินสแตนซ์ระบบคลาวด์ โดยชี้ไปที่แนวโน้มที่มีความหวังสําหรับความสามารถในการจัดเก็บข้อมูลบนคลาวด์ที่ได้รับการปรับปรุง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=39443679)

- การอภิปรายจะเจาะลึกถึงความท้าทายด้านประสิทธิภาพของ SSD ในระบบคลาวด์ โดยจัดการกับข้อจํากัดของโปรโตคอลเครือข่าย การเพิ่มประสิทธิภาพพื้นที่จัดเก็บข้อมูลในแพลตฟอร์มต่างๆ เช่น AWS การกําหนดค่า CPU และผลกระทบทางเศรษฐกิจของการตั้งค่าพื้นที่จัดเก็บข้อมูล
- หัวข้อต่างๆ ได้แก่ ผลกระทบของการกําหนดค่าพื้นที่จัดเก็บข้อมูลต่อประสิทธิภาพของ VM เวลาแฝงของเครือข่าย กลยุทธ์ในการเพิ่มประสิทธิภาพระบบคลาวด์ และการอภิปรายเกี่ยวกับการปรับใช้โปรโตคอลการสื่อสารและการแยกฮาร์ดแวร์ในบริการคลาวด์
- นอกจากนี้ยังเน้นย้ําถึงความสําคัญของ SSD ที่เชื่อมต่อในเครื่อง ปัญหาเวลาแฝงของเครือข่าย ไดรฟ์ระดับองค์กรที่เชื่อถือได้ และความสมดุลระหว่างประสิทธิภาพและความยืดหยุ่นในตัวเลือกการจัดเก็บข้อมูลคลัสเตอร์ฐานข้อมูล
