# 2024-07-05

## [แบตเตอรี่: ราคาจะถูกได้แค่ไหน?](https://aukehoekstra.substack.com/p/batteries-how-cheap-can-they-get)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40877337)

- แบตเตอรี่ลิเธียมไอรอนฟอสเฟต (LiFePO4) สำหรับใช้ในบ้านมีราคาปัจจุบันอยู่ที่ $89 ต่อกิโลวัตต์ชั่วโมงจากผู้จำหน่ายที่มีชื่อเสียงในประเทศจีน โดยจะมีค่าใช้จ่ายเพิ่มเติม $30 ต่อกิโลวัตต์ชั่วโมงหากซื้อในสหรัฐอเมริกา
- แบตเตอรี่โซเดียมมีราคาอยู่ที่ $130 ต่อกิโลวัตต์ชั่วโมง แต่มีประสิทธิภาพน้อยกว่าแบตเตอรี่ LiFePO4 ถึง 26%
- โพสต์นี้เน้นถึงความคุ้มค่าและประสิทธิภาพของแผงโซลาร์เซลล์และแบตเตอรี่ โดยมีผู้ใช้บางรายที่สามารถเปลี่ยนบ้านของตนไปใช้พลังงานแสงอาทิตย์และการเก็บพลังงานในแบตเตอรี่ได้สำเร็จ ทำให้ลดการพึ่งพาบริษัทพลังงานแบบดั้งเดิมลงได้

## [ฉันไม่มีตัวสร้าง และฉันต้องเริ่มต้น](https://consteval.ca/2024/07/03/initialization/)

- ในภาษา C++ หากไม่ได้มีการกำหนดคอนสตรัคเตอร์ไว้ คอมไพเลอร์อาจสร้างคอนสตรัคเตอร์เริ่มต้น ซึ่งอาจทำให้เกิดออบเจ็กต์ที่ไม่ได้รับการกำหนดค่าเริ่มต้น
- การกำหนดค่าเริ่มต้นแบบดีฟอลต์ไม่ทำอะไรสำหรับประเภทที่ไม่ใช่คลาสและไม่ใช่อาร์เรย์ ในขณะที่การกำหนดค่าเริ่มต้นแบบค่าเริ่มต้นจะกำหนดค่าเป็นศูนย์สำหรับประเภทที่ไม่ใช่คลาสและจากนั้นจึงกำหนดค่าเริ่มต้นแบบดีฟอลต์
- การพึ่งพาคอนสตรัคเตอร์ที่สร้างโดยคอมไพเลอร์อาจส่งผลให้เกิดพฤติกรรมที่ไม่สามารถคาดเดาได้; แนะนำให้เขียนคอนสตรัคเตอร์ที่กำหนดเองเพื่อหลีกเลี่ยงปัญหาเหล่านี้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40880932)

- C++ มีความซับซ้อนเนื่องจากลักษณะการรองรับหลายแนวคิดของมัน ซึ่งสามารถนำไปใช้ในแอปพลิเคชันต่างๆ เช่น ระบบฝังตัว วิดีโอเกม เซิร์ฟเวอร์ และ GUI
- ทางเลือกอย่าง C และ Rust ไม่ครอบคลุมทุกกรณีการใช้งานของ C++ ทำให้มันเป็นภาษาที่มีเอกลักษณ์แต่ท้าทาย
- ในขณะที่ความซับซ้อนของ C++ เป็นสิ่งจำเป็นสำหรับประสิทธิภาพและความยืดหยุ่น แต่มันอาจจะเป็นเรื่องที่หนักหนาสำหรับทีมที่มีประสบการณ์น้อยกว่า

## [ญี่ปุ่นเปิดตัวหุ่นยนต์มนุษย์ขนาดใหญ่เพื่อบำรุงรักษาเส้นทางรถไฟ](https://www.theguardian.com/world/article/2024/jul/04/japan-train-robot-maintain-railway-lines)

- หุ่นยนต์ซึ่งสามารถควบคุมจากระยะไกลได้ มีแขนขนาดใหญ่ที่สามารถยกน้ำหนักได้ถึง 40 กิโลกรัมและยืดขึ้นในแนวตั้งได้ถึง 12 เมตร โดยมีเป้าหมายเพื่อแก้ไขปัญหาการขาดแคลนแรงงานและลดอุบัติเหตุ
- บริษัทมีแผนที่จะขยายการใช้งานของหุ่นยนต์สำหรับงานบำรุงรักษาต่างๆ ในอนาคต

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40877648)

- ญี่ปุ่นได้เปิดตัวหุ่นยนต์ฮิวแมนนอยด์ขนาดใหญ่สำหรับการบำรุงรักษาเส้นทางรถไฟ ซึ่งควบคุมจากระยะไกลคล้ายกับหุ่นยนต์ผ่าตัด
- หุ่นยนต์ที่สามารถยืดได้ถึง 12 เมตรบนเครนแบบยืดหดได้ ถูกออกแบบมาเพื่อทำงานเช่น การตัดกิ่งไม้และการตรวจสอบ
- นวัตกรรมนี้มีเป้าหมายเพื่อแก้ไขปัญหากำลังแรงงานสูงวัยและการขาดแคลนแรงงานในญี่ปุ่น แม้ว่าจะมีความสงสัยเกี่ยวกับประสิทธิภาพเมื่อเทียบกับวิธีการแบบดั้งเดิมก็ตาม

## [ใส่โลโก้ DVD ไว้ที่มุม (2023)](https://eieio.games/game-diary/game-6-get-the-dvd-logo-into-the-corner/)

- "เกมที่ 6: วางโลโก้ DVD ไว้ที่มุม" เป็นเกมแอคชั่นอาร์เคดที่สร้างโดยมาร์กาเร็ต ลียงส์ พัฒนาในเวลาเพียง 4 วันโดยใช้แพลตฟอร์ม PICO-8
- เกมนี้ถูกสร้างขึ้นที่ Recurse Center และมีให้เล่นบน itch.io โดยโค้ดของเกมสามารถเข้าถึงได้บน GitHub เป็นเกมสั้น ๆ (45 วินาที) ที่สร้างขึ้นจากมุขตลก
- PICO-8 เป็น "คอนโซลแฟนตาซี" ที่เลียนแบบคอนโซลในยุค 80 โดยมีข้อจำกัดในด้านกราฟิก เสียง และขนาดของโค้ด ทำให้เหมาะสำหรับโครงการเกมขนาดเล็กและเรียบง่าย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40883277)

- โพสต์นี้พูดถึง Recurse Center ซึ่งเป็นโปรแกรมที่พักฟรีสำหรับโปรแกรมเมอร์ที่ส่งเสริมความคิดสร้างสรรค์และการเติบโตส่วนบุคคล โดยผู้เขียนได้แบ่งปันประสบการณ์ของพวกเขาในการสร้างเกม 8 เกมใน 12 สัปดาห์
- มีความสนใจอย่างมากในคอนโซลแฟนตาซีเช่น Pico-8 ซึ่งได้รับการยกย่องในเรื่องข้อจำกัดที่ส่งเสริมการสร้างสรรค์และโครงการเขียนโค้ดที่รวดเร็วและสนุกสนาน
- การสนทนานี้รวมถึงการอ้างอิงถึงการเล่นเกมย้อนยุคและสภาพแวดล้อมการเขียนโปรแกรมต่างๆ ซึ่งเน้นถึงความชื่นชมของชุมชนต่อการเขียนโค้ดเชิงสร้างสรรค์ภายในข้อจำกัดที่มีอยู่

## [Adreno iGPU ของ Snapdragon X Elite](https://chipsandcheese.com/2024/07/04/the-snapdragon-x-elites-adreno-igpu/)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40878222)

- Adreno iGPU ของ Snapdragon X Elite เคยทำผลงานได้ดีกว่า GPU ระดับกลางจาก Apple, AMD, และ NVIDIA ในการทดสอบเบนช์มาร์ค แต่ตอนนี้กลับประสบปัญหาในการรับมือกับโหลดกราฟิกสมัยใหม่ รวมถึงการเล่นเกมบนเดสก์ท็อปด้วย
- Qualcomm เผชิญกับการตัดสินใจเชิงกลยุทธ์ระหว่างการรักษาความเป็นผู้นำในกราฟิกมือถือหรือการมุ่งเน้นไปที่ประสิทธิภาพการเล่นเกมระดับสูง โดยมีปัญหาที่เกิดขึ้นอย่างต่อเนื่องในไดรเวอร์ Adreno ที่ทีม Dolphin ได้สังเกตเห็น

## [การย้ายไปยัง RTOS บน RP2040](https://blog.brixit.nl/moving-to-a-rtos-on-the-rp2040/)

- แม้จะมีความท้าทายเริ่มต้นกับ FreeRTOS เช่น printf ทำให้เกิดการค้างและขาดการแยกฮาร์ดแวร์ แต่ก็ยังคงเป็นตัวเลือกที่เหมาะสมที่สุดเนื่องจากการรวมเข้ากับ pico-sdk

- ตัวเลือก RTOS ที่สำคัญที่กล่าวถึงได้แก่ FreeRTOS, Zephyr, และ NuttX ซึ่งแต่ละตัวมีจุดแข็งและจุดอ่อนของตัวเองในแง่ของการสนับสนุน ความเป็นมิตรต่อผู้ใช้ และความเข้ากันได้กับฮาร์ดแวร์
- การอภิปรายยังครอบคลุมถึงสภาพแวดล้อมการพัฒนาอื่น ๆ เช่น Rust และ MicroPython โดยเน้นถึงประโยชน์ของพวกเขาสำหรับการพัฒนาแบบฝังตัวบน RP2040

## [การซ่อมแซมเนื้อเยื่อกระดูกโดยใช้ปะการังและฟองน้ำทะเล](https://web.stanford.edu/group/mota/education/Physics%2087N%20Final%20Projects/Group%20Gamma/bone.htm)

## [ทำไม Vivaldi ถึงไม่ตามกระแส AI ในปัจจุบัน?](https://vivaldi.com/blog/technology/vivaldi-wont-allow-a-machine-to-lie-to-you/)

- LLMs วิเคราะห์ข้อมูลจำนวนมาก ซึ่งอาจนำไปสู่การเผยแพร่วัสดุที่มีลิขสิทธิ์โดยไม่ตั้งใจและการเปิดเผยข้อมูลส่วนตัว
- Vivaldi มุ่งมั่นที่จะมอบประสบการณ์การท่องเว็บที่มีฟีเจอร์หลากหลายและมีจริยธรรม โดยยังคงเปิดรับความก้าวหน้าของการเรียนรู้ของเครื่องในอนาคตที่ให้ความสำคัญกับความเป็นส่วนตัวและการใช้งานที่ง่ายดาย

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40880322)

- การตัดสินใจของ Vivaldi ที่จะไม่รวม AI โดยเฉพาะอย่างยิ่ง Large Language Models (LLMs) ได้ก่อให้เกิดการถกเถียงอย่างมากในหมู่ผู้ใช้
- ผู้สนับสนุนท่าทีของ Vivaldi เน้นย้ำถึงความกังวลเกี่ยวกับความน่าเชื่อถือและการใช้พลังงานสูงของ LLMs ในขณะที่ผู้วิจารณ์โต้แย้งว่า LLMs มีคุณสมบัติที่มีคุณค่าและสามารถเป็นประโยชน์ต่อผู้ใช้ได้
- บทความที่อธิบายการตัดสินใจของ Vivaldi ถูกวิจารณ์ว่าแสดงความลำเอียงและมองข้ามประโยชน์ที่เป็นไปได้ของ LLMs โดยมีหลายคนสนับสนุนให้การรวม AI เป็นฟีเจอร์ทางเลือกในเบราว์เซอร์

## [การเขียนโปรแกรมใน Unison](https://lwn.net/Articles/978955/)

- Unison เป็นภาษาการเขียนโปรแกรมที่ได้รับอนุญาตจาก MIT ซึ่งเก็บโค้ดในฐานข้อมูลแทนที่จะเป็นไฟล์ข้อความ ทำให้การเขียนโปรแกรมแบบกระจายง่ายขึ้น
- มันใช้ระบบการตั้งชื่อที่ไม่ซ้ำกันโดยอิงจากแฮช ทำให้สามารถเปลี่ยนชื่อและจัดการเวอร์ชันได้อย่างราบรื่น และมีระบบความสามารถในการติดตามการพึ่งพาและการโต้ตอบของโค้ด
- ออกแบบสำหรับการประมวลผลแบบคลาวด์ Unison นำเสนอบริการที่มีการจัดการผ่าน Unison Cloud แต่เผชิญกับความท้าทายเช่นการขาดอินเทอร์เฟซฟังก์ชันต่างประเทศ (FFI) ที่เสถียรและความต้องการเครื่องมือที่กำหนดเอง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=40882133)

- วิธีการของ Unison ในการจัดการโค้ดที่ไม่เปลี่ยนแปลงซึ่งเก็บไว้ในฐานข้อมูลช่วยแก้ปัญหาทั่วไปในภาษาที่ใช้ REPL แบบดั้งเดิม แม้ว่าผู้ใช้บางคนจะกังวลเกี่ยวกับภาษาสมัยใหม่ที่รวมเครื่องมือในตัวมากเกินไป
