## [ความเห็นอกเห็นใจต่อผู้ใช้ที่มีเพศสัมพันธ์กับซอฟต์แวร์ของคุณ](https://docs.buttplug.io/docs/dev-guide/intro/buttplug-ethics/)

- การแบ่งปันโค้ดเทคโนโลยีทางเพศบนแพลตฟอร์มเช่น GitHub สามารถช่วยสร้างแนวปฏิบัติที่ดีที่สุดได้ แต่ผู้พัฒนาควรระมัดระวังกับสื่อที่มีเนื้อหาชัดเจนและพิจารณาการโฮสต์เนื้อหาที่ละเอียดอ่อนด้วยตนเอง
- โครงการ Buttplug เน้นความปลอดภัย ความเป็นอิสระ และสิทธิมนุษยชน โดยส่งเสริมพฤติกรรมที่รับผิดชอบและการศึกษาในแถลงการณ์ภารกิจของตน

- การสนทนานี้รวมถึงการแลกเปลี่ยนที่มีทั้งความขบขันและความจริงจังเกี่ยวกับการใช้งานซอฟต์แวร์ วิธีการทดสอบ และการใช้งานที่เป็นไปได้ในด้านอื่น ๆ เช่น สุขภาพ

## [รายละเอียดเบื้องต้นเกี่ยวกับสาเหตุที่ทำให้ CSAgent.sys ของ CrowdStrike ล่ม](https://twitter.com/patrickwardle/status/1814343502886477857)

- ปัญหาเกิดจากที่อยู่ที่ไม่ได้ถูกแมปจากอาร์เรย์ของพอยน์เตอร์ ทำให้เกิดการแครชเมื่อเข้าถึงหน่วยความจำ
- คำสั่งที่เกิดข้อผิดพลาดเฉพาะคือ "mov r9d, [r8]," โดยที่ R8 ถือที่อยู่ที่ไม่ถูกต้อง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=41021366)

- เหตุการณ์นี้เลวร้ายลงเนื่องจากโค้ดเคอร์เนลที่เขียนไม่ดี การทดสอบภายในที่ขาดหาย การตั้งค่าการอัปเดตที่ถูกละเลย การเปิดตัวทั่วโลกพร้อมกัน และการเสียหายของข้อมูลในขั้นต้น
- สิ่งนี้เน้นย้ำถึงความจำเป็นในการจัดการข้อผิดพลาดที่แข็งแกร่ง การปรับใช้แบบขั้นบันได และการทดสอบอย่างละเอียด รวมถึงการทดสอบแบบ fuzz เพื่อป้องกันความล้มเหลวที่คล้ายกัน

## [ความสำคัญของตัวอักษร "j" ที่ท้ายเลขโรมันคืออะไร? (2013)](https://genealogy.stackexchange.com/questions/3748/what-is-the-significance-of-the-character-j-at-the-end-of-a-roman-numeral)

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=41022653)

- ตัวอักษร "j" ที่ท้ายตัวเลขโรมันถูกใช้ในประวัติศาสตร์เพื่อบ่งบอกถึงจุดสิ้นสุดของตัวเลข เพื่อให้แน่ใจว่ามีความชัดเจนและป้องกันการแก้ไข
- การปฏิบัตินี้มีความสำคัญในเอกสารที่เขียนด้วยมือเพื่อหลีกเลี่ยงการตีความผิด เช่น การแยกแยะ "xviij" (18) จาก "xviii" (17) หรือ "xviiii" (19).
- แนวทางการพิมพ์ที่คล้ายคลึงกันมีอิทธิพลต่อการพัฒนาของไดกราฟ "ij" ในภาษาดัตช์ ซึ่งเน้นให้เห็นถึงผลกระทบของแนวทางเหล่านี้ต่อการพัฒนาภาษา

## [การสื่อสารของขีปนาวุธ Minuteman](https://computer.rip/2024-07-20-minuteman-missile-communications.html)

- ผู้เขียนวางแผนที่จะเปลี่ยนแพลตฟอร์มโฮสติ้ง ซึ่งจะมีค่าใช้จ่ายเพิ่มขึ้น และขอการสนับสนุนจากผู้อ่านผ่านการบริจาค โดยเสนอเนื้อหาพิเศษและการเข้าถึงวิดีโอ YouTube ใหม่ก่อนใครสำหรับผู้สนับสนุน
- สหรัฐอเมริกามีขีปนาวุธข้ามทวีป Minuteman III จำนวน 400 ลูก ซึ่งบริหารจัดการโดยกองบินขีปนาวุธที่ 90, 91 และ 341 ของกองทัพอากาศ โดยแต่ละกลุ่มของสถานีปล่อยขีปนาวุธสิบแห่งจะถูกตรวจสอบโดยสถานีเตือนภัยขีปนาวุธหนึ่งแห่ง
- การสื่อสารของ Minuteman รวมถึงระบบภายนอกเช่นเสาอากาศ LF, HF, UHF, และ SHF และระบบภายในเช่นระบบสายเคเบิลระหว่างไซต์ที่แข็งแกร่ง (HICS) โดยมีการวางแผนการอัปเกรดในอนาคตภายใต้โปรแกรม Sentinel

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=41019604)

- โพสต์นี้กล่าวถึงระบบการสื่อสารของขีปนาวุธ Minuteman โดยเน้นการเปลี่ยนแปลงจากเทคโนโลยีเก่าเช่น DSL ผ่านสาย HICS ไปสู่สายไฟเบอร์ออปติกสมัยใหม่เพื่อเพิ่มความจุและความน่าเชื่อถือ
- การสนทนานี้รวมถึงข้อมูลเชิงลึกเกี่ยวกับความท้าทายของการป้องกัน EMP (Electromagnetic Pulse) และฟ้าผ่าสำหรับระบบการสื่อสารเหล่านี้ โดยเน้นถึงข้อดีของสายเคเบิลใยแก้วนำแสงที่ไม่เป็นสื่อนำไฟฟ้า
- โพสต์นี้ยังมีเรื่องเล่าและคำแนะนำสำหรับการเยี่ยมชมสถานที่จรวดประวัติศาสตร์ เช่น สถานีควบคุมการปล่อยจรวด Minuteman ในเซาท์ดาโคตา และไซโลจรวด Titan II ในแอริโซนา ซึ่งให้บริบททางประวัติศาสตร์สำหรับเทคโนโลยีเหล่านี้

## [คุณคิดว่าคุณรู้เรื่องเงากล่องดีแล้วหรือ?](https://dgerrells.com/blog/how-not-to-use-box-shadows)

- บทความนี้สำรวจการใช้เงากล่องใน CSS อย่างสร้างสรรค์และไม่ธรรมดา นอกเหนือจากการปรับปรุง UI ทั่วไป
- ผู้เขียนได้ทดลองใช้เงากล่องสำหรับศิลปะมินิมอล, แอนิเมชัน, เอฟเฟกต์ 3 มิติ, และแม้กระทั่งการติดตามรังสี เพื่อแสดงให้เห็นถึงความหลากหลายของคุณสมบัติ CSS นี้
- การทดลองเน้นถึงความสามารถในการทำงานของฮาร์ดแวร์สมัยใหม่ เช่น แล็ปท็อป M1 ในการจัดการกับเอฟเฟกต์ภาพที่ซับซ้อนโดยใช้เงากล่อง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=41024664)

- การอภิปรายมุ่งเน้นไปที่วิธีการที่ความโปร่งใสในการเรนเดอร์กราฟิกส่งผลกระทบต่อการจัดกลุ่ม GPU และการวาดทับ ซึ่งเป็นสิ่งสำคัญสำหรับการเพิ่มประสิทธิภาพการทำงาน
- ผู้ใช้แบ่งปันเทคนิคต่างๆ เช่น การใช้ signed distance fields (SDF) เพื่อการเรนเดอร์ที่มีประสิทธิภาพมากขึ้น โดยสังเกตว่าประสิทธิภาพจะแตกต่างกันไปในแต่ละอุปกรณ์และเบราว์เซอร์
- ข้อมูลเชิงลึกประกอบด้วยประสบการณ์เชิงปฏิบัติและลิงก์ไปยังแหล่งข้อมูลที่เกี่ยวข้องและบทความทางประวัติศาสตร์สำหรับการอ่านเพิ่มเติม

## [rr – เครื่องมือดีบักที่สามารถบันทึกและเล่นซ้ำสำหรับภาษา C/C++](https://rr-project.org/)

- rr เป็นเครื่องมือดีบักสำหรับภาษา C/C++ บนระบบปฏิบัติการ Linux ที่ช่วยเพิ่มประสิทธิภาพให้กับ gdb โดยการอนุญาตให้เล่นซ้ำแบบกำหนดได้และการย้อนกลับที่มีประสิทธิภาพ
- มันรองรับการบันทึกและการเล่นซ้ำของแอปพลิเคชันต่างๆ จัดการงานหลายกระบวนการ และผสานรวมกับการเขียนสคริปต์ gdb และ IDEs
- rr มีความโดดเด่นในเรื่องของการใช้ทรัพยากรต่ำ ความสามารถในการใช้งานบนเคอร์เนล Linux มาตรฐาน และความสามารถในการทำให้การดีบักบั๊กที่เกิดขึ้นเป็นครั้งคราวง่ายและรวดเร็วยิ่งขึ้น

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=41023547)

- rr เป็นดีบักเกอร์สำหรับการบันทึกและเล่นซ้ำสำหรับภาษา C/C++ ที่ช่วยในการวิศวกรรมย้อนกลับและการดีบักโค้ดที่ซับซ้อน โดยอนุญาตให้ตั้งจุดหยุด (breakpoints), จุดเฝ้าดู (watchpoints), และการย้อนกลับ (reverse-continuation) เพื่อติดตามการเปลี่ยนแปลงของตัวแปรและระบุข้อบกพร่อง
- มันใช้ gdb (GNU Debugger) และรองรับภาษาที่คอมไพล์แล้วหลายภาษา เช่น Rust, Go, และ Julia บน Linux แต่มีข้อจำกัดกับบั๊กที่เกี่ยวกับการทำงานพร้อมกันและไม่รองรับ GPUs.
- โหมดความวุ่นวายของ rr สามารถช่วยเปิดเผยปัญหาการทำงานพร้อมกันบางอย่าง และเครื่องมือขั้นสูงเช่น Pernosco ซึ่งอิงจาก rr เสนอฐานข้อมูลที่สามารถสืบค้นได้ของการทำงานของโปรแกรม

- แต่ละบริษัทกำลังใช้มาตรฐานเช่น UCIe (Universal Chiplet Interconnect Express) และ BoW (Bridge of Wires) สำหรับการเชื่อมต่อชิปเล็ต โดยที่ Intel ใช้ EMIB (Embedded Multi-die Interconnect Bridge) และ Samsung และ TSMC กำลังสำรวจวิธีการ 2.5D และ 3D-IC (Integrated Circuit)
- โรงหล่อกำลังก้าวหน้าไปสู่ระดับแองสตรอมในกระบวนการผลิตและนวัตกรรมในด้านต่างๆ เช่น การส่งพลังงานด้านหลังและวัสดุพื้นผิวแก้ว โดยมีเป้าหมายในการส่งมอบโซลูชันที่ปรับแต่งได้และมีประสิทธิภาพสูงอย่างมีประสิทธิภาพ

## [การจัดการทรัพยากรของ Classic Mac OS ใน ResEdit](https://eclecticlight.co/2024/07/20/managing-classic-mac-os-resources-in-resedit/)

- ระบบไฟล์ของ Macintosh มีสองส่วน: ส่วนข้อมูล (data fork) และส่วนทรัพยากร (resource fork) โดยส่วนหลังจะเก็บข้อมูลที่มีโครงสร้างเช่น ไอคอน, ไดอะล็อก, และชิ้นส่วนของโค้ด
- ResEdit ของ Apple ซึ่งเป็นโปรแกรมแก้ไขทรัพยากรที่อัปเดตล่าสุดในปี 1994 อนุญาตให้ผู้ใช้ปรับแต่งไฟล์ระบบ, ไดอะล็อก, และรูปแบบแป้นพิมพ์ แต่มีความเสี่ยงด้านความปลอดภัยเนื่องจากมัลแวร์มักจะใช้ประโยชน์จาก resource forks
- ใน Mac OS X, resource forks ได้เปลี่ยนไปเป็น extended attributes และทรัพยากรของแอปพลิเคชันถูกจัดระเบียบเป็น bundles โดยแอปพลิเคชันสมัยใหม่ถูกล็อกด้วยลายเซ็น ทำให้ไม่สามารถปรับแต่งได้อย่างกว้างขวางเหมือนที่เคยทำได้ด้วย ResEdit
