# 2023-11-03

## [סם בנקמן-פריד הורשע](https://www.nytimes.com/live/2023/11/02/business/sam-bankman-fried-trial)

- סם בנקמן-פריד, מייסד בורסת הקריפטו FTX, הורשע בשבעה סעיפים של הונאה וקשירת קשר, בטענה כי מעיל בכ-10 מיליארד דולר ממשתמשיו.
- נפילתה של FTX וההאשמות שבאו בעקבותיה משמשות כדוגמאות לסכנות ולחריגות בתעשיית הקריפטו.
- עדים, כולל חברתו לשעבר ועובדים בעבר, העידו נגד בנקמן-פריד במהלך המשפט, והאשימו אותו במעילה בכספי לקוחות להוצאות אישיות ותרומות פוליטיות.

### [תגובות](https://news.ycombinator.com/item?id=38122061)

- דיונים מקוונים, הקשורים לפוסט בניו יורק טיימס, דנים בהרשעתו של מנכ"ל FTX סם בנקמן-פריד (SBF) בגין התנהגות בלתי הולמת לכאורה במרחב הקריפטו, כולל מעילה בכספי לקוחות.
- משתתפי הדיון מטילים ספק בהתנהלותו של SBF בבית המשפט ובסירובו לספק תשובות ברורות, עם טיעונים הנוגעים לאופיו, חינוכו, וההשפעה המזיקה הפוטנציאלית שעשויה להיות למקרה זה על התפיסה והרגולציה של תעשיית הקריפטו.
- שיחות משתמשים כוללות גם הליכים משפטיים, הטלת ספק בתקינות ואפשרויות שאוטיזם עשוי להשפיע על תגובות בית המשפט, בנוסף לאזהרות על דגלים אדומים לפרקטיקות עסקיות מזויפות, המדגישות את הצורך בשקיפות ובבקרות פנימיות חזקות.

## [טרמינולוגיה מבלבלת של Git](https://jvns.ca/blog/2023/11/01/confusing-git-terminology/)

- ג'וליה אוונס מפשטת טרמינולוגיה מורכבת של Git, ומדגישה כי הבנה מעמיקה של מונחים אלה חיונית לשימוש יעיל ב- Git, מערכת קוד פתוח למעקב אחר שינויים בתוכנה.
- היא צוללת לעומק המונחים המורכבים של גיט כמו "ראש וראש", "שלנו ושלהם במהלך מיזוג או ביסוס מחדש", "מצב ראש מנותק", "לפני 'מקור/עיקרי' בהתחייבות אחת", בין היתר, ומבהירה את ההבדלים ביניהם.
- המאמר מדגיש את הצורך בבהירות נוספת בטרמינולוגיה של Git בשל אופיו ומורכבותו רבת הפנים, ושופך אור על מושגים כגון "איפוס", "החזרה", "שחזור", המונח "מסלול" בהקשרים שונים, וההבחנות בין "מיזוג", "בסיס מחדש" ו"קטיף דובדבנים".

### [תגובות](https://news.ycombinator.com/item?id=38112951)

- שרשור חדשות Hacker הדגיש את קשיי המשתמש בהבנת המינוח המורכב של Git, מערכת בקרת גרסאות נפוצה בפיתוח תוכנה, והגרף האציקלי המכוון הבסיסי שלה (DAG).
- משתמשים התקשו להבין פקודות Git, ארגומנטים של ברירת מחדל ואת המורכבות של מיזוג תוך הערכת תיעוד ברור והפקודה 'git reflog'.
- למרות בעיות אלה, משתמשים שיבחו את התפקיד המכריע של Git בפיתוח תוכנה ואת עליונותה על פני מערכות בקרת גרסאות חלופיות כמו Fossil, SVN, Perforce ו- Mercurial.

- הטקסט מזכיר גם סוגים שונים של משאבים חינוכיים בחינם, כמו גם Open Culture, LLC, חברה המציעה שירותים כמו ספרי שמע, קורסים מקוונים, שפות וספרי לימוד.

### [תגובות](https://news.ycombinator.com/item?id=38121250)

- ארה"ב הזמינה בעבר ציורים של כל פרי ידוע, בדומה לציורי הציפורים של ג'ון ג'יימס אודובון, שניהם פריטי אספנים יקרי ערך כיום.
- יש ויכוח מתמשך על דמויות היסטוריות כמו אודובון, בעיקר על הקשר שלהן לנושאים שנויים במחלוקת כמו עבדות, והאם יש לשמר או לפרק כיבודים ואנדרטאות הקשורים אליהם.
- דיגיטציה של יצירות היסטוריות, למרות היותה יקרה, נחשבת מועילה מכיוון שהיא מאפשרת שימור, גישה רחבה יותר ופוטנציאל לרווח באמצעות מכירת גישה ברזולוציה גבוהה.

## [API ניסיוני של תקינות מדיה של Android WebView בתחילת השנה הבאה](https://android-developers.googleblog.com/2023/11/increasing-trust-for-embedded-media.html)

- צוות Android הוא חלוץ בפיתוח API ניסיוני של Android WebView Media Integrity, שמטרתו להגביר את אמצעי האבטחה והפרטיות עבור מדיה משובצת ביישומי Android.
- API זה, המתוכנן לבדיקה עם ספקי מדיה משובצת נבחרים בשנה הבאה, מבקש לאפשר הצגה של מדיה מוטמעת בדפי אינטרנט בתוך יישומים תוך שמירה על סביבה מאובטחת ומניעת הונאה ושימוש לרעה פוטנציאליים.
- ממשק API זה, המציע לספקי מדיה פסק דין על תקינות המכשיר והאפליקציה, מבטיח שזרמי מדיה יפעלו בסביבה מהימנה ומאובטחת, וכל זאת מבלי לחשוף מזהי משתמש או מכשיר. יתר על כן, ספקי תוכן מדיה מוזמנים לתוכנית גישה מוקדמת בשנה הבאה.

### [תגובות](https://news.ycombinator.com/item?id=38118627)

- גוגל מתכוונת לשחרר API ניסיוני של שלמות המדיה של Android WebView בשנה הבאה, מה שיוביל לחששות בקרב מבקרים לגבי השפעות אפשריות על חוסמי פרסומות והפתיחות, האנונימיות והביזור של הרשת.
- הוויכוחים נסובים סביב נושאים כמו האיזון בין ביטחון לחירות, ריסון Google משליטה בתהליכי קבלת החלטות, וההשלכות של הצעת Google שנדחתה לשלמות סביבת האינטרנט (WEI) עבור Chrome.
- ענקית הטכנולוגיה, למרות שהשעתה את הצעת ה-WEI שלה לכרום, מתכננת להטמיע אותה ב-API של Android WebView, מה שיוביל לדיונים על נקודות תורפה של צפיות ברשת ואמצעי נגד פוטנציאליים.

## [Raspberry Pi מקבל השקעה אסטרטגית מ-Arm](https://newsroom.arm.com/news/raspberry-pi-investment)

- חברת המוליכים למחצה ועיצוב התוכנה Arm רכשה נתח מיעוט ב-Raspberry Pi כדי לחזק את השותפות ארוכת השנים שלהם המשרתת את קהילת מפתחי ה-IoT.
- Arm מספקת ל-Raspberry Pi פתרונות מחשוב עתירי ביצועים מאז 2008 ולאחרונה פרסמה עדכונים שונים, כולל מדריך לכלי רכב מוגדרי תוכנה ומדריך נגן מוזיקה.
- התוכניות העתידיות של החברה כוללות התמקדות ביישומי AI אתיים, ממשל AI וחקירת הפוטנציאל של תוכנות בעלות ביצועים גבוהים בתעשיית הרכב.

### [תגובות](https://news.ycombinator.com/item?id=38113525)

- חברת הטכנולוגיה Arm השקיעה אסטרטגית ב-Raspberry Pi, ככל הנראה כדי להרתיע את האחרונה ממעבר פוטנציאלי לטכנולוגיית RISC-V.
- משתמשים הדגישו את היתרונות הפוטנציאליים של המעבר ל-RISC-V, כולל הגמישות והעלות האפקטיבית שלו במונחים של שטח פרוסות, אך גם הביעו חששות לגבי הבדלי תמיכה בתוכנה והזמן הדרוש ל-RISC-V כדי להתאים ליכולות של ARM.
- יש חוסר שביעות רצון של המשתמש מסדרת Raspberry Pi האחרונה עקב היעדר תמיכה בשדרוג ותכונות חסרות.

## [Google נוטשת API של שלמות סביבת האינטרנט](https://www.theregister.com/2023/11/02/google_abandons_web_environment_integrity/)

- גוגל הפסיקה את הפיתוח של Web Environment Integrity API השנוי במחלוקת, שנועד בתחילה לרסן הונאות מקוונות, בשל ביקורת על כך שהוא דומה מדי לניהול זכויות דיגיטלי (DRM) עבור אתרים, מה שעלול לאיים על החופש המקוון ולהכתיב את העדפת הדפדפן.
- כעת, גוגל מתכננת להשתמש בגרסה מוגבלת יותר של טכנולוגיה זו באופן בלעדי עבור Android WebViews, גרסה של Chrome שתוכננה להיות משולבת ביישומי אנדרואיד.
- שינוי מיקוד זה נוצר לאחר משוב ביקורתי משמעותי מקהילת הטכנולוגיה; צוות אנדרואיד של גוגל אישר שאין תוכניות ליישם פונקציונליות זו מעבר ל- Android WebViews ומדיה משולבת.

### [תגובות](https://news.ycombinator.com/item?id=38121994)

- Google עצרה את הפיתוח של Web Environment Integrity API, והפנתה את המיקוד שלה ל-Android WebView Media Integrity API, מה שהוביל לדיונים מקוונים בנוגע לחששות אבטחה אפשריים ולעלייה פוטנציאלית בפרקטיקות ניהול זכויות דיגיטליות (DRM) לא הוגנות.
- המשתמשים דנו באכיפה של מודולי פלטפורמה מהימנים (TPM) על ידי Chromium, במאבק נגד בוטים ובשינוי תקני DRM, במיוחד המעבר ליוזמות אינטרנט סגורות יותר.
- הביקורת הקהילתית נוגעת לניהול המוניטין של המותג שלה על ידי גוגל, והם קוראים לפתרונות שמתעדפים את צרכי משתמשי הקצה מבלי לפגוע בחדשנות.
