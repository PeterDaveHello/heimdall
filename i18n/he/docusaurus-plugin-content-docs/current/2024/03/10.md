## [ברונו: לקוח API ידידותי ל-Git מהדור הבא](https://www.usebruno.com/)

- Bruno הוא לקוח API מהיר וקוד פתוח מאתגר כלים כמו Postman ו Insomnia.
- הוא משתמש בשפת סימון טקסט כדי לאחסן נתוני בקשות API במערכת הקבצים, ומאפשר שיתוף פעולה באמצעות Git ומערכות דומות.
- ההתמקדות היא בפרטיות נתונים, המציעה גישה לא מקוונת בלבד ללא סנכרון ענן, עם הזמנות מראש פתוחות עבור מהדורת הזהב המוזלת.

### [תגובות](https://news.ycombinator.com/item?id=39653718)

- תסכולי המשתמשים מתמקדים בלקוחות API כמו Postman המחייבים חשבון ענן, בעוד שלאחרים כמו Insomnia יש מגבלות.
- ברונו, לקוח API בקוד פתוח, זוכה לשבחים על מהירות ותאימות Git, בניגוד ללחץ מוניטיזציה מצד חברות.
- הדיונים מכסים את היתרונות והחסרונות של כלי בדיקת API כמו Bruno, Postman ו- Insomnia, ומדגישים את האתגרים בשיתוף פעולה בתיעוד API.

## [חקירת כוחה של שיחה אחר דחיפת ערך](https://thunderseethe.dev/posts/bet-on-cbpv/)

- Call-by-Push-Value (CBPV) היא אסטרטגיית הערכה חדשנית הממזגת תכונות הן מ- Call-by-Value (CBV) והן מ- Call-by-Name/Need (CBN), ומציעה יצירת קוד יעילה יותר ויכולות שפת תכנות מורחבות.
- CBPV מבדיל ערכים מחישובים, משפר את היסק הסוגים, מנהל תופעות לוואי ומייעל פונקציות מסדר גבוה יותר, ובכך מציג פרספקטיבות חדשות לניתוח תוכניות ושיפור שפה.
- ההפרדה המפורשת בין ערכים וחישובים ב- CBPV סוללת את הדרך לשיפור הבנת התוכנית וחדשנות בעיצוב שפה.

### [תגובות](https://news.ycombinator.com/item?id=39653895)

- הפוסט דן בערך קריאה אחר דחיפה (CBPV) בחשבון למדא, ומשווה אותו להערכה עצלה ויישומו בשפות כמו Haskell ו- PureScript.
- הוא מכסה נושאים כגון חבטות, שרשור פונקציות ואריות פונקציות, תוך הדגשת היתרונות והאתגרים הקשורים למושגים אלה בשפות תכנות.
- המחקר מספק תובנות לגבי יישום CBPV ומשמעותו בפרדיגמות תכנות פונקציונליות.
