## [Amazon S3 מפסיק לחייב עבור קודי שגיאה נבחרים של HTTP](https://aws.amazon.com/about-aws/whats-new/2024/05/amazon-s3-no-charge-http-error-codes/)

- Amazon S3 תפסיק לחייב לקוחות עבור קודי שגיאה ספציפיים של HTTP כמו HTTP 403 (Access Denied) שאינם נגרמים על ידי הלקוח, מה שישפיע על כל דליי S3 בכל אזורי AWS.
- שינוי זה אינו מחייב שינויים כלשהם ביישומי הלקוח, ומציע הקלה בעלויות עבור שגיאות בלתי צפויות.
- לפרטים נוספים, עיין בדף תגובות השגיאה חיוב עבור Amazon S3.

### [תגובות](https://news.ycombinator.com/item?id=40346597)

- משתמשים מביעים חששות לגבי עמלות בלתי צפויות באמזון S3 וב-AWS, ומציינים בעיות עם חיוב, חוויית משתמש וחיובים נוספים פוטנציאליים.
- הדיונים עוסקים באתגרי שירות לקוחות, סעדים טכניים, שיטות קבלת החלטות ב-AWS, שימוש לרעה בקיבולת URL בלתי מוגבלת וחוסר שביעות רצון מהתמחור, בעיקר בנוגע לתגובות NXDOMAIN ב-Route53.
- ההמלצות לשיפורים כוללות הודעות חיוב אוטומטיות, ממשק משתמש משופר ובחינת ספקי שירותי DNS חלופיים.

- משתמשים יכולים להתחיל על ידי יצירת טבלאות זרות, הפעלת האצת שאילתות באמצעות ווי Postgres, ומעקב אחר הנחיות התקנה ופיתוח, כולל התחברות למאגרי אובייקטים מותאמים אישית, ניהול סוגי נתונים ספציפיים וביצוע בדיקות באמצעות testcontainers עם התקנת Postgres חיה.
