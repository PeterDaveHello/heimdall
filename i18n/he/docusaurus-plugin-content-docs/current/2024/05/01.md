## [ייעול אוטומציה של תשתיות עם Pyinfra](https://pyinfra.com/)

- pyinfra הוא כלי Python לאוטומציה של תשתיות, הכולל מהירות, מדרגיות ורב-תכליתיות עבור פקודות אד-הוק, פריסת שירותים וניהול תצורה.
- הוא מציע ביצוע מהיר, ניפוי באגים מיידי, פעולות אידמפוטנטיות ואינטגרציה עם מחברים, מה שהופך אותו לפתרון מקיף לאוטומציה של תשתיות.
- משתמשים יכולים להתקין pyinfra באמצעות pip, להריץ פקודות על SSH, להגדיר מצבים באמצעות פעולות, ולפרוס משימות באמצעות קוד Python, עם הגמישות להרחיב אותו באמצעות המערכת האקולוגית של חבילת Python.

### [תגובות](https://news.ycombinator.com/item?id=40211655)

- השוואה בין כלי אוטומציה של תשתיות כמו Pyinfra, Ansible, Puppet ו- Terraform בהתבסס על גישות הצהרתיות והכרחיות, מורכבויות, יתרונות ומגבלות.
- משתמשים משבחים את Pyinfra מבוסס Python עבור משימות אוטומציה, מביעים חששות לגבי מגבלות Ansible ו- Puppet, ומדגישים את החשיבות של ניהול המדינה באוטומציה של תשתיות.
- הדיון עוסק באתגרים של ניהול יעיל של ציי שרתים גדולים, היתרונות של תכנות הצהרתי, והאיזון בין עקרונות תכנות DRY לבין פשטות בניהול תצורה.

## [Extension.js: כלי יעיל ליצירת תוספי אינטרנט](https://github.com/cezaraugusto/extension.js)

- Extension.js הוא כלי פיתוח להרחבות דפדפן שמייעל את התהליך על ידי הצעת תמיכה מובנית ב-TypeScript, WebAssembly, React ו-JavaScript מודרני.
- הוא מסיר את הדרישה לתצורות מורכבות והוראות התקנה, ומאפשר למפתחים להתרכז בקידוד.
- ניתן לשלב את הכלי בצורה חלקה בסקריפטים של npm, ובכך לפשט את היצירה של הרחבות חוצות דפדפנים ללא צורך בתצורת בנייה. נוצר במקור כדי לחנך על פיתוח הרחבות, הוא פתוח כעת לשימוש הציבור עם משוב מעודד.

### [תגובות](https://news.ycombinator.com/item?id=40210627)

- Extension.js הוא כלי CLI המפשט יצירת תוספי אינטרנט, תומך TypeScript, WebAssembly, React ו- JavaScript מודרני, תואם באופן מלא לכרום ו- Edge, עם תאימות עתידית לפיירפוקס וספארי על הפרק.
- דיונים בפלטפורמות כמו Github ו- HN מתמקדים בתכונות Extension.js, השוואות עם מסגרות כמו Plasmo, ואסטרטגיות לשינוי דפי אינטרנט ו- SPA באמצעות כלים כמו Mutation Observers ו- CSS.
- משוב הקהילה כולל הצעות לשיפור ותוכניות תמיכה עתידיות בדפדפן עבור Extension.js.

## [מערכת הקבצים: מניפולציה של נתונים מודרניים באמצעות כלי יוניקס](https://mgree.github.io/ffs/)

- ffs היא מערכת קבצים המאפשרת למשתמשים לטעון נתונים מובנים למחצה כגון JSON על מערכת קבצים יוניקס, המאפשרת מניפולציה עם כלי מעטפת סטנדרטיים.
- הוא תומך בפורמטים JSON, YAML ו-TOML, ומאפשר עריכת קבצים במקום, דורש FUSE ב-Linux ו-macFUSE ב-macOS, ומורשה תחת GPLv3.
- פותח על ידי mgree, ffs מציע גישה שונה לעיבוד נתונים בשורת הפקודה בהשוואה לכלים כמו jq ו- gron.

### [תגובות](https://news.ycombinator.com/item?id=40213731)

- הדיון מתמקד בהפעלת תכונות מערכת קבצים נגישות למשתמש במערכות הפעלה כמו macOS, תוך התמודדות עם אתגרים עקב מגבלות ה-API של אפל והסתמכות על כלי צד שלישי כגון osxfuse.
- משתמשים מתוסכלים מהמגבלות של אפל עבור מפתחים חיצוניים, מה שמוביל לחששות לגבי אמינות ושלמות נתונים, מה שמוביל להצעות כמו מינוף NFS.
- הדיונים כוללים גם רעיונות חדשניים כמו הרכבה של Git commits כמערכת קבצים, התייחסות למאגרים כקבצים ומיפוי XML למערכת קבצים עם טכנולוגיית XPath, בחינת פרטים טכניים ויתרונות וחסרונות פוטנציאליים בקהילת הטכנולוגיה.

## [Penpot 2.0: כלי העיצוב החדש מבוסס האינטרנט בקוד פתוח לשיתוף פעולה חלק](https://penpot.app/)

## [Arti: פרויקט חלודה המיישם פרוטוקולי אנונימיות של Tor](https://tpo.pages.torproject.net/core/arti/)

- Arti הוא פרויקט Rust המיישם פרוטוקולי אנונימיות של Tor, ומספק ללקוח Tor פונקציונלי תמיכה מוקדמת בשירותי בצל.
- הפיתוח נמשך כדי לשפר את האבטחה כך שתתאים ליישום C Tor, מה שהופך אותו כרגע מתאים למטרות ניסוי בלבד.
- הגרסה האחרונה היא גרסה 1.2.0, המציעה משאבים שונים כמו README, מאגר מקורות, תיעוד API, הנחיות תרומה, שאלות נפוצות ופרטים הקשורים ל-Tor.

### [תגובות](https://news.ycombinator.com/item?id=40214763)

- המאמר מתעמק ב-Arti, יישום Tor המקודד ב-Rust, ומשתף תובנות על פיתוחו, מימונו והצעות לשימוש ניסיוני.
- מתפתח דיון לגבי מצבה הנוכחי של ארטי ופגיעויות אבטחה פוטנציאליות, תוך הסבת תשומת הלב לשיקולים חשובים עבור המשתמשים.
- Arti in Rust מציע נקודת מבט ייחודית על יישומי Tor, ומושך עניין בשל היבטי הפיתוח שלו והשלכות האבטחה.
