## [我們構建了最快的 CI，但它失敗了](https://earthly.dev/blog/shutting-down-earthly-ci/)

- Earthly是一家專門從事CI / CD（持續集成/持續交付）工具的公司，在推出其產品Earthly CI時面臨障礙，主要是由於遷移成本和潛在客戶的懷疑。
- 該公司瞭解到，直接進入市場策略是無效的，而是認識到開發人員需要獨立發現和採用他們的工具，從而導致方法的轉變。
- 由於沒有增量構建和忽略早期反饋等錯誤，他們不得不關閉其CI（持續集成）元件，這反映了用戶反饋和增量開發在技術產品發佈中的重要性。

### [反應](https://news.ycombinator.com/item?id=37481513)

- 該討論側重於與持續集成（CI）系統相關的挑戰和關鍵考慮因素，包括一家名為Earthly的公司的失敗。
- 對開源 CI/CD 平臺（如 GitLab、GitHub 和 Azure DevOps）中的功能和可用性進行了比較，重點介紹了構建系統中的緩存、CI 流程反饋和重建決策等主題。
- 共識是分裂的，強調CI / CD系統中特定功能和簡單性之間的重要平衡。討論還考慮了廣告、演示和建立信任在推廣新的開發人員工具方面的功效。

- 這些討論強調了電動汽車領域的發展和增長，標誌著它們在汽車行業中的重要性和突出地位日益增加。

- Unity 不斷侵蝕的功能、持續的穩定性問題和新的價格結構增加了開發人員的成本;預計這些變化將對獨立開發者和小型開發者造成最嚴重的打擊。
- 作者建議，Unity的行動可能會限制創新，減少第三方產品，並可能使用戶遠離他們的產品，這意味著一種簡短的方法。

### [反應](https://news.ycombinator.com/item?id=37486431)

- 文章表達了對Unity新定價模式的擔憂，並猜測這可能會對開發人員和Unity本身產生負面影響。
- 該討論提出了Unity的替代方案，例如虛幻引擎和Godot，同時討論了Unity和遊戲開發行業的未來。
- 該文本強調了遊戲設計和可訪問性的重要性，而不是遊戲引擎的選擇，研究了硬體限制和開發時間的作用。

## [新戈多發展基金](https://godotengine.org/article/godot-developer-fund/)

- 戈多引擎啟動了戈多發展基金，旨在獲得穩定的資金並確保項目的未來。
- 該基金旨在僱用更多的開發人員並升級引擎，所有捐款都直接支援這些改進。這種方法減少了對公司一次性贈款的依賴，將重點與社區的需求保持一致。
- 開發商認識到社區支援的重要性，並鼓勵分享基金連結以傳播其範圍和影響力。

### [反應](https://news.ycombinator.com/item?id=37481872)

- Godot遊戲引擎發起了一項新的開發基金，旨在支援其持續開發。
- 使用者對Godot反對Unity的立場表達了不同的看法，一些人主張改進C#支援，而另一些人則支援GDScript。
- 對話的主題包括開源開發的好處、小型開源團隊遇到的障礙、Unity 的最新價格變化以及它們對 Godot 資金和聲譽的可能影響。

## [大數定律中的這個數位有多大？](https://thepalindrome.org/p/how-large-that-number-in-the-law)

## [摩洛哥地震前的天空中閃爍著“藍光”](https://www.thesun.co.uk/news/23911027/mystery-blue-lights-morocco-earthquake/)

- 在毀滅性地震之前，摩洛哥觀察到了被稱為地震燈或EQL的藍光，反映了土耳其，日本，中國，義大利和墨西哥的類似事件。
- EQL的原因尚未確定，儘管一些科學家推測它們可能與岩石圈板塊移動的能量釋放有關。
- 摩洛哥的地震震級為7.2級，造成重大破壞，2 900人喪生。目前正在進行救濟和救援工作，遊客也做出了貢獻。

### [反應](https://news.ycombinator.com/item?id=37484352)

- 摩洛哥地震前看到的藍光是由於電力線或變壓器故障，而不是自然現象。
- 這些燈的可能原因可能是地震振動或岩石中的壓電效應導致的電源線短路。
- 該帖子強調了媒體素養和使用經過驗證的資訊來源的必要性，因為某些新聞來源，特別是小報的可靠性受到質疑。
