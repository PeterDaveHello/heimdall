## [我今年 17 歲，寫了這篇關於 CPU 如何運行程式的指南。](https://github.com/hackclub/putting-the-you-in-cpu)

- 本文分為幾章，第3章推薦給已經熟悉該主題的讀者，最終旨在提供對計算機功能如何的紮實理解。

### [反應](https://news.ycombinator.com/item?id=37062422)

- 一位名叫 Lexi 的 17 歲年輕人創建了一個網站，其中包含有關 CPU 工作原理的綜合指南。
- 該指南因其清晰度、準確性和引人入勝的語氣而獲得了積極的反饋。
- 本文強調了需要更好的在線資源來了解處理器和操作系統。

## [希望員工返回辦公室？然後給每個人一個辦公室](https://www.washingtonpost.com/opinions/2023/08/09/remote-work-offices-floorplan-privacy/)

- 開放式辦公室可能嘈雜且缺乏隱私，導致生產力和工作滿意度下降。
- 為員工提供單獨的辦公室可以鼓勵他們返回辦公室並提高生產力。
- 由於遠端工作趨勢，辦公空間成本的下降可能會抵消提供辦公室的成本。

### [反應](https://news.ycombinator.com/item?id=37062650)

- 員工在居家工作 （WFH） 或返回辦公室時有不同的偏好。
- 有些人喜歡居家辦公的靈活性和成本節約，而另一些人則重視辦公室的社交互動和風景的變化。
- 討論還討論了階級和等級制度問題，包括高管與其他員工之間的靈活性差異。

## [基礎架構審計由 Radically Open Security 完成](https://mullvad.net/en/blog/2023/8/9/infrastructure-audit-completed-by-radically-open-security/)

- Mullvad VPN已經接受了Radically Open Security的第三次安全審計，重點是其VPN基礎設施。
- 審計沒有發現資訊洩露或客戶數據的日誌記錄，但發現了各種問題，例如測試伺服器上生產使用者流量的可見性。
- Mullvad VPN計劃通過實施諸如審核未經授權的登錄和刪除SSH支援等措施來解決這些問題。

### [反應](https://news.ycombinator.com/item?id=37060654)

- 文章和評論側重於VPN供應商，包括Mullvad和ProtonVPN，以及他們的隱私實踐，特別提到Mullvad刪除其埠轉發功能。
- 討論的主題包括互聯網隱私，政府監控，在線審查，美國制裁的影響以及VPN的限制和漏洞。
- 提出的問題包括數據收集，VPN供應商的可信度以及在線隱私的複雜性。

## [Arpchat – 僅使用 ARP 在同一網路上給您的朋友發短信](https://github.com/kognise/arpchat)

- “arpchat”專案是一個開源聊天應用程式，它使用位址解析協定（ARP）在網路設備之間傳輸消息。
- 它支援長消息的傳輸，包括狀態發現和心跳系統，以及壓縮功能。
- 該專案是用 Rust 編寫的，在 GitHub 上獲得了 1.8k 顆星的歡迎。安裝說明適用於 Windows 和 Linux，建議使用者報告任何問題。

### [反應](https://news.ycombinator.com/item?id=37063151)

- ArpChat 是一款消息傳遞應用程式，它使用位址解析協定 （ARP） 在同一網路上的使用者之間進行通信。
- 修復了導致 ArpChat 延遲的錯誤，但尚未發佈更新版本。
- 由於缺少許可權，Linux 使用者可能會遇到 ArpChat 問題，但解決方案是可用的。
- 本文解釋了 ARP 在網路中的有用性及其在將 MAC 位址連接到 IP 位址方面的作用。
- ARP通常用於乙太網和WiFi網路，本文探討了它在這些網路中的用法。
- 討論了無線路由器、交換機和數據機機使用的分支和集線器模型，解釋了即使在本地網路上，數據通常如何通過路由器。
- 檢查了用於在網路上創建廣播消息的ARP的潛在漏洞和操縱。
- 本文還介紹了用於創建網路圖的批處理腳本、沒有 Internet 的通信方法（點對點聊天、無線網路）以及使用類似網路協定的用戶體驗等主題。
- 還討論了將DNS用於聊天系統的可能性。

## [SQLedge： Replica Postgres to SQLite on the Edge](https://github.com/zknill/sqledge)

- SQLedge 是一個開源工具，它利用 Postgres 邏輯複製將更改從 Postgres 資料庫傳輸到邊緣的本地 SQLite 資料庫。
- 它允許從SQLite資料庫讀取並寫入上游Postgres伺服器，確保數據一致性。
- 該工具包括一個 SQL 產生器、SQL 解析器和一個 Postgres 線路代理，所有配置都是從環境變數中獲取的。

### [反應](https://news.ycombinator.com/item?id=37063238)

- SQLite資料庫被廣泛使用，並且有幾種工具和專案可用於增強其功能。
- SQLedge 和 SQLitePostgres 等工具提供數據複製、版本控制優勢、多主複製和查詢解析等功能。
- 這些工具為本地數據緩存和查詢提供了羽量級選項，具有與 SQLite 資料庫處理相關的各種功能。但是，由於SQLite和其他資料庫（如PostgreSQL）之間的SQL方言差異，存在限制。

## [使 AMD GPU 在 LLM 推理方面具有競爭力](https://blog.mlc.ai/2023/08/09/Making-AMD-GPUs-competitive-for-LLM-inference)

- 本文討論了用於LLM（大型語言模型）推理的AMD GPU的性能，特別是Radeon RX 7900 XTX。
- 研究表明，對於Llama2-7B/13B型號，7900 XTX的性能速度是NVIDIA的GeForce RTX 4090的80%，是RTX 3090 Ti速度的94%。
- 機器學習編譯（MLC）和ROCm堆疊的使用優化了AMD GPU的性能，並可能支援Vulkan和WebGPU。該研究強調了進一步開發和優化 AMD GPU 和其他平臺的 MLC 解決方案的必要性。

### [反應](https://news.ycombinator.com/item?id=37066522)

- 優化的軟體堆疊可以提高 AMD GPU 在大型語言模型推理方面的性能。
- ML編譯技術和TVM Unity軟體堆疊被討論為在NVIDIA和AMD GPU上實現高性能的方法。
- 這篇文章探討了AMD對ROCm和Vulkan的支援，但提出了對支援壽命的擔憂。
- 討論了 AMD ROCm 平台的各種用例，包括 AI、密碼破解、GPU 挖掘和 VFX 渲染。
- 對AMD和Nvidia GPU進行了比較，重點是機器學習演算法中RAM容量和頻寬的限制。
- Microsoft正在為其專案中的瓶頸問題提出解決方案，而Geohot的貢獻則側重於直接改進ROCm。
- 本文建議使用多個RX 6600卡而不是一個RX 7900，但關於如何在沒有AMD等效的NVLink的情況下鏈接它們的問題仍然存在。
- 這些發展引發了對投資 AMD 的考慮。

## [西雅圖成為美國第一個保護零工工人免受突然「停用」的州](https://kuow.org/stories/seattle-becomes-first-in-u-s-to-protect-gig-workers-from-sudden-deactivation)

- 西雅圖市議會批准了一項新法律，以保護零工工作者免受Instacart和DoorDash等平臺上的突然停用，這標誌著美國第一項此類立法。
- 該條例對停用實施了更嚴格的指導方針，並要求公司在終止前 14 天發出通知。
- 這一發展是西雅圖旨在保護不屬於工會的零工工人的更大舉措的一部分，包括最近提高最低工資併為基於應用程式的工人提供病假和福利的法律。該法律預計將由西雅圖市長布魯斯·哈雷爾簽署。

### [反應](https://news.ycombinator.com/item?id=37070398)

- 關於零工工作的討論涵蓋了各種主題，如工人保護、分類、福利、挑戰以及與計程車等傳統服務的比較。
- 爭論的關鍵點包括零工工作的靈活性、對零工經濟平臺剝削的擔憂、監管的必要性以及對企業和工人的影響。
- 這些討論突出了圍繞零工工作的複雜性和不同意見，以及零工經濟中正在進行的關於工人權利的辯論。
