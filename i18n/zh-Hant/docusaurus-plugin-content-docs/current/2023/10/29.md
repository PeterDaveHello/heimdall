## [OpenSign – DocuSign 的開源替代品](https://github.com/OpenSignLabs/OpenSign)

- OpenSign 是商業文檔電子簽名平臺（如 DocuSign、PandaDoc 和 SignNow）的開源替代品。
- OpenSign 提供的功能包括安全文檔簽名、使用者友好的介面、多平臺支援、協作設施、審計跟蹤、完成證書和 API 集成。
- OpenSign 由 OpenSignLabs 開發，根據 AGPL 許可證獲得許可，歡迎社區貢獻，並獲得了使用者和貢獻者的支持和反饋。

### [反應](https://news.ycombinator.com/item?id=38052344)

- OpenSign Labs 推出了 OpenSign，這是 DocuSign 的開源替代品，引發了關於數位簽名合法性和 OpenSign 法律研究深度的爭論。
- 演講還重點討論了跨地區對數位簽名的統一要求的必要性，並引用了印度的 2000 年 IT 法案和歐洲的 eIDAS。
- 此外，還討論了OpenSign的安全性和可靠性，開源軟體的優缺點，以及信任和長期可行性在管理簽名文檔的商業產品中的作用。

## [美國聯邦通信委員會希望加強業餘無線電](https://www.radioworld.com/news-and-business/business-and-law/fcc-wants-to-bolster-amateur-radio)

- 聯邦通信委員會 （FCC） 打算通過用頻寬限制取代波特率（數據傳送速率）限制來對業餘無線電頻段進行現代化改造，旨在促進創新並加強緊急情況下的通信。
- 美國無線電中繼聯盟（ARRL）是一個業餘無線電愛好者的機構，支援這一倡議。
- FCC也在考慮取消其他頻段的波特率限制，並願意接受有關設置合適頻寬限制的反饋。預計這些更新將大大有利於業餘無線電操作員並加強數據通信。

### [反應](https://news.ycombinator.com/item?id=38052577)

- 根據美國無線電中繼聯盟（ARRL）的請願書，聯邦通信委員會（FCC）正在修訂業餘無線電法規，這可能會導致更多的實驗和新的調製技術。
- FCC正在考慮的重要主題包括摩爾斯電碼、波特率限制、加密、業餘無線電加密的潛在好處，以及運營商的擴展頻率和數據傳輸速率。
- 雖然這些改進為社區帶來了興奮，但也表達了對非商業和實驗性無線電活動可能受到干擾和限制的擔憂。

## [MagicaVoxel – 一個免費的體素藝術編輯器和互動式路徑跟蹤渲染器](https://ephtracy.github.io/)

- MagicaVoxel 是一款免費的體素藝術編輯器和渲染器，提供多種功能和快捷方式，專為製作 3D 模型和動畫而設計。但是，它不能進行商業銷售或與其他軟體捆綁在一起。

### [反應](https://news.ycombinator.com/item?id=38050106)

- MagicaVoxel 是一個流行的免費體素藝術編輯器和路徑跟蹤渲染器，廣泛用於創建體素模型和景觀，例如在開源體素 MMORPG Veloren 中。
- 雖然不是開源的，但由於其簡單性、愉快的用戶體驗以及與 3D 列印的相容性，它擁有強大的追隨者。它甚至刺激了類似工具的發展。

## [氮氣電還原，電流-氨效率幾乎達到100%（2022年）](https://www.nature.com/articles/s41586-022-05108-y)

- 最近發表在《自然》雜誌上的一項研究側重於將氮電還原為氨，適用於可再生能源、運輸和化肥生產等領域。
- 莫納什大學的研究人員發現，高濃度醯亞胺基鋰鹽電解質可實現高氨產率，電流-氨效率接近100%。
- 這些創新可能會產生一種更高效、更環保的氨生產方法，這可能會通過Jupiter Ionics進行商業應用。

### [反應](https://news.ycombinator.com/item?id=38053586)

- 一篇新的研究論文強調了氮電還原的突破，通過激光誘導的氧化鋰多光子解離，在氨生產中產生了高效率。
- 這種方法為當地的氨生產和無碳燃料替代品鋪平了道路，儘管存在高能耗和安全問題。
- 該研究還探討了將電力有效地轉化為氨以用於儲能、肥料和燃料生產，同時解決了實用性和安全性問題，並討論了氨在爆炸物製造中的可擴展性、毒性和潛在用途。

## [ImageMagick 的 Windows 安裝程式將不再簽名](https://github.com/ImageMagick/ImageMagick/discussions/6826)

- 由於代碼簽名證書過期，ImageMagick 的 Windows 安裝程式將不再簽名，他們無法再匯出並在 GitHub 操作中使用該證書。

- 雖然他們正在邀請贊助以獲取代碼簽名證書，但一些用戶建議使用SignPath 和 Azure KeyVault 等替代方案。

- 對話圍繞著開源專案在為其軟體獲取和更新代碼簽名證書時面臨的困難展開，並批評其成本高昂和效率低下。
- 關於財務可持續性、改進資金和安全系統的要求以及 Azure Key Vault（一種用於安全存儲和訪問加密密鑰的雲服務）等替代解決方案的討論也很普遍。
- 重點放在身份驗證的重要性、對代碼的信任以及對專有軟體和自簽名證書的關注上。還指出了用於圖像編輯的開源軟體套件 ImageMagick 的價值和弱點。
