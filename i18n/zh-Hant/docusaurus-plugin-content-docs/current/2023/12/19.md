# 2023-12-19

## [Figma 和 Adobe 在監管審查後放棄合併](https://www.figma.com/blog/figma-adobe-abandon-proposed-merger/)

- Figma和Adobe在經過漫長的監管審查程式後取消了合併計劃。
- Figma將保持其獨立地位，並願意探索與Adobe的合作機會。
- 在過去的 15 個月裡，Figma 取得了長足的進步，專注於簡化數位產品的設計和開發。首席執行官迪倫·菲爾德（Dylan Field）對公司的未來表示樂觀，並感謝社區的支援。

### [反應](https://news.ycombinator.com/item?id=38681861)

- Figma 和 Adobe 之間的潛在合併已被取消，引發了關於其背後的原因及其對 Figma 員工的影響的討論。
- 對話中探討了 Figma 等設計工具的優勢和局限性，以及 AI 在設計行業的潛在顛覆。
- 討論還涉及對創業文化中的合併和IPO、監管監督、長期可持續性以及反壟斷法在數位市場中的作用的擔憂。

## [Wasm3 在俄羅斯入侵中進入維護階段](https://github.com/wasm3/wasm3)

- WebAssembly 解釋器 Wasm3 的新功能開發暫時停止，因為開發人員的房子被入侵他們的國家的俄羅斯人摧毀。
- 開發人員將繼續維護專案，審查和合併拉取請求。
- Wasm3 是一個通用的解釋器，可以在各種平台和架構上使用，並通過了 WebAssembly 規範測試套件。它在可移植性、安全性、可執行檔大小、記憶體使用和啟動延遲方面具有優勢。它在嵌入式設備上運行 WebAssembly 也有潛在的應用。該專案在麻省理工學院許可下發佈。

### [反應](https://news.ycombinator.com/item?id=38681672)

- 對話深入探討了與俄烏衝突有關的各種主題，例如烏克蘭歷史、民族主義、宣傳、資源開採、經濟制裁和核武器。
- 它包括關於是否支持烏克蘭的辯論，研究俄羅斯的意圖，以及討論不同程式設計語言的有效性。
- 該對話還討論了緩解代碼中資源洩漏的技術，並展示了有關持續衝突及其後果的廣泛意見和擔憂。

## [作者的MacBook被鎖定，蒂姆·庫克（Tim Cook）解鎖了它](https://www.tokyodev.com/articles/unbricking-my-macbook-took-an-email-to-tim-cook)

- 作者的MacBook被退回給他們，並帶有啟動鎖，阻止了對設備的訪問。
- 蘋果最初拒絕解鎖MacBook，但作者通過電子郵件聯繫了蒂姆·庫克（Tim Cook），並最終將其解鎖。
- MacBook已被新創建的iCloud帳戶擦除並報告為丟失，這表明它可能已被帶到商店，重置並報告為丟失以勒索錢財。
- 作者強調，蘋果需要修改他們的設置工作流程，以更好地傳達不設置FindMy的後果，因為不道德的商店可能會利用這種情況。
- 作者向 Apple Security Research 報告了此問題，但被駁回。

### [反應](https://news.ycombinator.com/item?id=38691025)

- 討論圍繞設備安全、擁有權和客戶支持展開，重點關注Apple產品。
- 用戶對蘋果對其設備的控制表示沮喪，而有些人則欣賞防盜功能。
- 提出了加強安全措施的建議，並在防盜和使用者需求之間找到平衡。

## [大眾汽車通過在汽車中重新引入物理按鈕來回應投訴](https://www.thedrive.com/news/vw-is-putting-buttons-back-in-cars-because-people-complained-enough)

## [與 GCC 集成的 Rust 編譯器的進展](https://lwn.net/SubscriberLink/954787/41470c731eda02a4/)

- gccrs 專案正在努力創建一個 Rust 編譯器，以包含在 GCC 14 版本中，目標是 Rust 版本 1.49。
- 該專案目前專注於支援核心和分配箱的編譯，但在宏解析度和裝飾器宏方面面臨挑戰。
- 該專案旨在使 Rust 代碼能夠使用 GCC 進行編譯，這有助於 Rust 的開發並支援嵌入式系統。另一個高級專案 rustc_codegen_gcc 已經允許在沒有補丁的情況下為 Linux 編譯 Rust，討論包括將 Polonius 集成到 Rust 編譯器中，以及實現基於 GCC 的 Rust 編譯器的好處，例如錯誤暴露和引導。

### [反應](https://news.ycombinator.com/item?id=38684102)

- gccrs 是一個基於 GCC 的 Rust 編譯器，正在技術社區中討論。
- 關於擁有多個 Rust 實現存在爭議，一些人認為它可以幫助發現錯誤並改進語言，而另一些人則擔心相容性和重複性。
- 編譯器在跟上語言標準方面面臨挑戰，使用多個編譯器可以説明提高代碼品質並檢測錯誤。對話還探討了對 Rust 正式規範的需求以及標準化的潛在優點和缺點。
