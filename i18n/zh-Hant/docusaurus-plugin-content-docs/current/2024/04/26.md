# 2024-04-26

## [FCC 在法律挑戰中恢復網路中立性規則](https://www.nytimes.com/2024/04/25/technology/fcc-net-neutrality-open-internet.html)

- 美國聯邦通信委員會（Federal Communications Commission）已投票決定恢復網路中立性規則，以阻止互聯網供應商阻止或減慢競爭對手的服務速度。
- 這一決定重新加強了政府對寬頻公司的控制，可能會引發該行業的法律糾紛。
- 這些規則最初是在歐巴馬時代推出的，在特朗普政府時期被廢除，現在在自由互聯網上激起了新的辯論，引起了支持和批評。

### [反應](https://news.ycombinator.com/item?id=40160429)

- 討論涉及恢復 FCC 的網路中立性規則，以及就 ISP 合規負擔、政府監管和網路中立性法律的社會影響進行辯論。
- 研究了關於政治動態、壟斷、自由意志主義和政府機構在政策制定中的各種觀點，以及對 ISP 做法和廢除後果的擔憂。
- 對話涉及對共和黨的批評、立法決策、對政治話語的兩極分化影響、媒體影響以及對邊緣化群體和社會進步的影響。

## [Microsoft和IBM合作開源DOS 4.0](https://www.hanselman.com/blog/open-sourcing-dos-4)

- Microsoft與IBM合作，在MIT許可下公佈了MS-DOS 4.00的原始程式碼，其中包含測試版二進位檔，文檔和磁碟映像。
- MS-DOS 4.00 的 8086 匯編代碼是一個重要的歷史作品，在與前 Microsoft 首席技術官 Ray Ozzie 對話后，共用以促進開放式創新。
- 原始程式碼可在 GitHub 上訪問，使用戶能夠在正版硬體或模擬器上操作它。

### [反應](https://news.ycombinator.com/item?id=40163405)

- GitHub 使用者正在懷舊地討論舊的 MS-DOS 版本，分享與十六進位編輯器一起使用駭客軟體的記憶。
- 談話涉及像MS-DOS這樣的開源舊軟體，程式設計中的術語變化以及語言的敏感性。
- Microsoft最近在 GitHub 上的行動，包括發佈和刪除歷史 MS-DOS 檔，包括多任務 MS-DOS 測試版，都得到了強調和辯論。

## [台積電推出 A16 1.6nm 技術，支援背端供電](https://www.tomshardware.com/tech-industry/tsmc-unveils-16nm-process-technology-with-backside-power-delivery-rivals-intels-competing-design)

- 台積電推出先進的1.6奈米級工藝技術A16，該技術集成了背面供電網路（BSPDN）和全環繞柵極（GAA）納米片晶體管。
- 與其前身相比，A16 擁有更高的性能、效率、功耗和晶體管密度，可與英特爾的 14A 節點相媲美。
- 在 A16 中引入 BSPDN，特別是超級電源軌 （SPR），增強了其複雜性，但為 AI 和 HPC 處理器提供了實質性優勢。

### [反應](https://news.ycombinator.com/item?id=40156275)

- 台積電推出了具有背面供電的尖端 1.6 奈米製程，而英特爾則利用 DSA 技術進行 1.4 奈米工藝，凸顯了行業對性能的關注。
- 討論深入探討了半導體製造優勢、晶體管密度、晶元設計優化以及 EUV 光刻技術的障礙等主題。
- AI/HPC 處理器、晶圓通孔、鎢在製造中的使用以及矽技術的複雜性的進步也是對話的重點，強調了半導體行業的持續發展和挑戰。

## [Tribler：一種徹底改變媒體的彈性微觀經濟](https://github.com/Tribler/tribler/wiki)

- Tribler 是代爾夫特理工大學推出的以隱私為重點的 Bittorrent 相容平臺，旨在促進沒有銀行或廣告商的微觀經濟。
- Tribler 擁有 100 多名開發人員，不斷增強其 Bittorrent 協定、P2P 通信，併為藝術家的經濟獎勵引入代幣經濟。
- Trustchain 是一個獨特的區塊鏈，它鼓勵 Bittorrent 播種，突出匿名功能並過渡到新的疊加系統，以實現安全的環境和創新的資本主義模式。

### [反應](https://news.ycombinator.com/item?id=40156534)

- Tribler提出了一種基於門檻的「媒體經濟」，以解決數位媒體領域的創作者薪酬問題，強調去中心化、信任和非營利的學術原則。
- 討論涉及加密貨幣的作用、數位不平等的擔憂以及 Tribler 平台內廣告模式的潛在垮臺。
- 與加密、協定和未經授權的內容共用相關的法律障礙給 Tribler 重塑由主要參與者壟斷的內容行業帶來了挑戰。

## [Increase 強調 API 設計中的真實事件](https://increase.com/articles/no-abstractions)

- Increase 的 API 設計原則「無抽象」主張以真實事件命名 API 資源，利用網路術語，並根據特定用例對資源進行分類。
- 受 Stripe 的影響，該策略針對 Increase 熟悉支付網路的使用者，優先考慮資源的不變性和清晰的命名，以實現簡化集成和透明的系統映射。
- 雖然不是通用的，但這一原則增強了使用者的理解力，減少了決策中的認知負擔，有可能在某些 API 上下文中提供實質性的價值。

### [反應](https://news.ycombinator.com/item?id=40161794)

- 本文強調通過提供低級和高級 API 來滿足不同的使用者需求，從而在重抽象和輕抽象 API 之間實現平衡。
- 它使用 node.js 和 Vulkan 等平臺的示例強調了在 API 設計中保持一致性和靈活性的挑戰，同時也強調了特定領域術語、簡單性和定價策略的重要性。
- 討論內容包括以下方面的見解：在非零售場景中表示價格、使用整數來精確貨幣、Stripe API 的局限性、節省退款費用的授權策略，以及利用 Stripe webhook 等工具解決爭議。

## [IBM 以 64 億美元收購 HashiCorp，企業正在遠離開源](https://www.jeffgeerling.com/blog/2024/corporate-open-source-dead)

- IBM 以 64 億美元收購 HashiCorp，此前 HashiCorp 已過渡到「業務原始程式碼許可證」，這表明軟體行業正在發生轉變。
- 公司越來越多地從開源許可證轉向專有許可證，這導致了 Terraform 和 Vault 等專案的分叉。
- 遠離企業開源的趨勢引發了人們對信任、收入和社區驅動軟體可持續性的擔憂，這可能會為小型團隊獨立領導未來的開源計劃鋪平道路。

### [反應](https://news.ycombinator.com/item?id=40160331)

- 本文探討了企業如何在不回報的情況下從開源軟體中受益，並建議使用更嚴格的許可證，如SSPL或Commons Clause，以促進公平的商業行為。
- 關於開發人員在謀生和堅持開源原則之間的困境存在爭議，強調需要在開源社區中在利潤和協作之間取得平衡。
- 貢獻者討論了在財務上維持開源專案的挑戰，強調了選擇適當許可證的重要性，並警告了重新許可的風險和企業參與開源的潛在弊端。
