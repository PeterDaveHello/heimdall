## [優化 Rust 的調用約定以獲得更好的性能](https://mcyoung.xyz/2024/04/17/calling-convention/)

- Miguel Young de la Sota 批評了 Rust 的調用約定，提出了一個更有效的替代方案，與 Linux C ABI 標準保持一致，以獲得更好的性能。
- 他建議利用 Go 寄存器 ABI 來增強性能，方法是使用新標誌 -Zcallconv 優化 Rust 函數的調用約定，該標誌直接指定 LLVM。
- Miguel 旨在改進 Rust 的代碼生成、優化寄存器使用並增強應用程式二進位介面，以提高性能並最大限度地減少記憶體流量，展示 LLVM 方面的專業知識。

### [反應](https://news.ycombinator.com/item?id=40081314)

- 討論強調了衡量性能在優化 Rust 調用約定方面的重要性，重點是速度而不是外觀。
- 涵蓋的主題包括 Rust 的調用約定穩定性、優化 x64 處理器上的布爾運算、混合託管和非託管代碼的挑戰，以及 C 和 C++ 中函數返回值的注意事項。
- 該對話還討論了語言之間的互操作、x86 架構的 LLVM 調用約定的使用，以及調用約定對性能和開發人員控制的影響，旨在提高代碼執行效率。

## [Hermit：可重現沙箱中的確定性程序執行](https://github.com/facebookexperimental/hermit)

- Hermit 是一個可重現的容器，通過將它們與時間和隨機數生成等非確定性來源隔離開來確保確定性程序的執行。
- 它攔截系統調用以實現確定性，適用於壓力測試、記錄/重放、可重現構建和調試併發錯誤。
- 雖然它沒有將程式與文件系統更改或外部網路隔離開來，但它不再積極開發，而是可以通過 Rust 的 cargo 工具運行，該工具在 BSD-3 下獲得許可，支援 Linux x86_64持續支援 Aarch64。

### [反應](https://news.ycombinator.com/item?id=40076848)

- Hermit 是一種沙盒工具，旨在通過攔截系統調用來密封和可重複地運行程式，類似於 rr，但缺乏一些系統調用支援，並且不會定期更新。
- 儘管以確定性為目標，但由於 CPU 指令和系統調用攔截等因素，Hermit 可能無法實現完全確定性，限制程式在單個 CPU 上運行以進行分析和調試，而不是一般的沙盒。
