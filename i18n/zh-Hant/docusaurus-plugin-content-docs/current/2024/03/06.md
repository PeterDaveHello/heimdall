## [ZLUDA：使用開源專案在 AMD GPU 上運行 CUDA 應用程式](https://www.cgchannel.com/2024/02/open-source-project-zluda-lets-cuda-apps-run-on-amd-gpus/)

- ZLUDA 是 Andrzej Janik 的一個開源專案，使 NVIDIA GPU 的 CUDA 應用程式能夠在 AMD GPU 上運行，專門針對版本 3 以允許未經修改的應用程式遷移到新硬體。
- ZLUDA 最初用於 Intel GPU，現已重新用於 AMD GPU，並使用 Blender、攝影測量應用程式和 Arnold 渲染器等軟體進行了測試，為 AMD GPU 上的 CUDA 應用程式提供了近乎原生的性能，儘管存在某些應用程式性能較慢和 OptiX 支援不完整等限制。
- Janik 提到需要主要 GPU 製造商支援進一步的 ZLUDA 開發，並強調它對開發人員的可用性，儘管它是一種非通用解決方案，在某些 CG 應用程式上可能成功有限。

### [反應](https://news.ycombinator.com/item?id=39604745)

- 討論圍繞 ZLUDA 展開，這是一個開源專案，可促進 CUDA 應用程式在 AMD GPU 上運行，引發了關於在非 Nvidia GPU 上為 CUDA 軟體使用轉換層的合法性、實用性和影響的辯論。
- 用戶強調了對AMD使用者的重要性，以及為各種GPU型號量身定製庫的必要性，闡明瞭AMD的GPGPU努力，相對於NVIDIA的CUDA的局限性，以及為AMD GPU創建類似CUDA的編譯器的困難。
- 對話的高潮是就優質軟體在科技公司中伴隨硬體的關鍵作用達成一致。

## [Datalog：圖演算法的一個有前途的解決方案](https://tylerhou.com/posts/datalog-go-brrr/)

- 現代程式設計語言缺乏對圖和圖演算法的內置支援，需要顯式的圖表示，正如在回應Hillel Wayne的文章時所討論的那樣。
- Datalog 是一種基於關係代數的聲明性語言，被認為是有效表達和執行圖演算法的熟練解決方案，展示了高性能、簡單表示以及與傳統語言的相容性。
- Datalog 的提議強調了它作為即將到來的程式設計語言的可行選擇的潛力，強調了它的效率和集成能力。

- 圖形可以使用矩陣或邊緣清單來表示，從而引發了關於每種方法在各種數據結構和程式設計語言中的效率的辯論。
- 數據日誌被考慮用於圖形查詢，而關係代數被檢查用於數據操作，強調了創新圖處理引擎的必要性。
- 強調了程式設計中缺乏通用圖數據類型，以及說服其他人接受Datalog等新技術的困難，這表明對圖分析和查詢處理的各種方法進行了深入探索。

## [Cloudflare 推出 AI 防火牆以增強模型安全性](https://blog.cloudflare.com/firewall-for-ai)

- Cloudflare 引入了 AI 防火牆來打擊模型濫用，引發了關於審查、準確性和業務影響的辯論，以及關於網路安全和技術債務的 Web 應用程式防火牆 （WAF） 的討論。
- 該公司推出了新的安全工具，如聊天機器人保護，以加強網路安全並阻止人工智慧威脅，解決有關數據安全、抓取和代碼注入的擔憂。
- 儘管對人工智慧防禦和競爭力存在一些疑問，但積极參與改善互聯網安全的Cloudflare仍然是一家與用戶互動的知名公司。
