## [在聖安德列斯·泰特皮爾科發現的新阿茲特克抄本](https://tlacuilolli.com/2024/03/21/new-aztec-codices-the-codices-of-san-andres-tetepilco/)

- 國家人類學和歷史研究所的專家在墨西哥城的聖安德列斯·泰特皮爾科發現了三本新的阿茲特克抄本，揭示了直到 1603 年庫爾瓦坎和特諾奇蒂特蘭政體的歷史細節。
- 這一發現展示了新的圖像學和阿茲特克文字，澄清了以前晦澀難懂的字形和交流方法。
- 墨西哥政府現在將這些手抄本存放在國家人類學和歷史研究所，激發了他們未來出版的熱情。

### [反應](https://news.ycombinator.com/item?id=39803508)

- 在墨西哥的聖安德列斯·泰特皮爾科（San Andrés Tetepilco）發現了新的阿茲特克抄本，這些抄本是墨西哥政府從當地一個家庭獲得的，據信其中包含在多光譜成像下可見的先前擦除的阿茲特克文本。
- 這一發現擴展了現有的阿茲特克抄本收藏，為阿茲特克歷史和文化提供了新的視角，包括有關聖安德列斯·泰特皮爾科的建立和特諾奇蒂特蘭政體的細節。
- 研究人員計劃在未來幾年內對這些抄本進行進一步的深入分析，揭示對阿茲特克人之前和征服后歷史事件的見解。

## [從 1990 年到 2019 年，早發性癌症發病率激增 80%](https://bmjoncology.bmj.com/content/2/1/e000049#DC1)

- 本文深入探討了 1990 年至 2019 年早發性癌症的全球趨勢，展示了不斷上升的負擔以及飲食和遺傳學等關鍵風險因素。
- 強調健康生活和早期篩查計劃對有效應對這些趨勢的重要性。
- 討論生活方式、空氣品質和遺傳變異如何影響乳腺癌和結直腸癌，以及對糖尿病、胃癌和幽門螺桿菌感染流行病學的見解。

### [反應](https://news.ycombinator.com/item?id=39803845)

- 討論強調了50歲以下人群的癌症發病率激增，將其歸因於污染和過度使用抗生素等因素，強調了醫療部門的謹慎態度。
- 它探討了微塑膠對人類健康的影響、發展中國家肥胖和癌症患病率的增加以及西化飲食的影響。
- 對話還深入探討了現代醫療保健在預防癌症方面的功效、環境因素在早發性癌症中的作用，以及雌激素、紅肉攝入量和癌症易感性之間的聯繫。

## [Lapdev：簡化遠端開發環境管理](https://github.com/lapce/lapdev)

- Lapdev 是一個自託管應用程式，有助於使用 Devcontainer 開放規範管理個人伺服器或雲上的遠端開發環境。
- 它簡化了開發環境即代碼，確保了可伸縮性和簡單的複製，並強調了最低的安裝和維護要求。
- Lapdev 的路線圖包括潛在的增強功能，例如支援各種工作空間類型和操作系統，並在其網站上提供詳細的安裝說明和貢獻指南。

### [反應](https://news.ycombinator.com/item?id=39801399)

- Github 的重點是新的開源遠端開發環境管理軟體 Lapdev，與 docker-compose 和 JetBrains 的遠端 SSH dev 相比。
- 用戶討論好處、懷疑，並探索遠端開發選項，如 devcontainers 和 Visual Studio Code，強調改進體驗和自動化設置。
- 對話中強調了手動配置 Docker 設置、利用配置檔進行容器化的挑戰，以及簡化開發人員部署流程的重要性。

## [英特爾 8088：解包指令預取電路](https://www.righto.com/2024/03/8088-prefetch-circuitry.html)

- Ken Shirriff 的博客對英特爾 8088 微處理器進行了深入分析，特別關注其預取電路和性能影響。
- 該博客比較了 8086 和 8088，詳細闡述了 IBM PC 上有利於 8088 的決策過程，並解釋了總線介面單元和執行單元的角色。
- 主題包括預取佇列同步、指令解碼器、觸發器和控制線等處理器設計元素，以及緩存在記憶體操作中的重要性。該博客還涉及自 2008 年以來的電子、程式設計和太空探索。

### [反應](https://news.ycombinator.com/item?id=39801963)

- 本文涵蓋的主題範圍從英特爾 8088 處理器的指令預取電路到 IBM PC 中基本 RAM 配置的歷史發展。
- 它探討了時序測試在電子產品中的重要性、在互聯網普及之前進行技術故障排除所面臨的挑戰，以及8086/8088等處理器的設計選擇和成本影響。
- 討論還深入探討了 CPUID 指令歷史、處理器（如 Pentium）中的緩存配置，以及在引入 CPUID 之前用於處理器類型識別的技術。
