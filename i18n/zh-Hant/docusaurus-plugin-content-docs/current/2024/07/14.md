## [「在最初拒絕後，Apple 已經批准了首個適用於 iOS 的 PC 模擬器」](https://www.theverge.com/2024/7/13/24198015/apple-utm-se-pc-os-emulator-for-ios)

- Apple 已經批准 UTM SE，這是首款適用於 iOS 的 PC 模擬器，允許用戶在 iPhone 上運行經典軟體和舊式遊戲，包括 Windows、Mac OS 9 和 Linux。
- 「UTM SE 基於 QEMU 架構，支援 VGA 和終端模式，並模擬 x86、PPC 和 RISC-V 架構，但不包含任何作業系統。」
- 該應用程式在初次被拒後，經由 AltStore 團隊和另一位開發者的 QEMU TCTI 實作協助，現已在 iOS、iPadOS 和 visionOS 上免費提供。

### [評論](https://news.ycombinator.com/item?id=40958465)

- 蘋果在最初拒絕後，已批准首個適用於 iOS 的 PC 模擬器 UTM，這引起了用戶的興奮。
- 該批准歸因於歐盟法規，強調了關於現代電子產品中的消費者權利、可修復性和所有權的持續討論。
- 「由於蘋果的限制，UTM 不支援 JIT(即時編譯)，因此性能較慢，但仍能在未越獄的 iPhone 上運行各種作業系統，包括 64 位元的 Linux。」

## [對於個人的 Google 位置數據，沒有合理的隱私期待](https://fourthamendment.com/?p=58338)

- 第四巡迴法院裁定，使用者在其 Google 位置數據中沒有合理的隱私期待(REP)，因為使用者自願與 Google 分享這些數據。
- 法院的裁決強調，地理圍欄技術(由執法機構用來收集位置數據的方法)不構成第四修正案下的搜索。
- 這項裁決具有重要意義，因為它涉及自2016年以來執法機構日益增加的地理圍欄令狀使用，以及Google保護用戶隱私的程序。

### [評論](https://news.ycombinator.com/item?id=40958458)

- Google 正在改變其處理位置數據的方式，將其推送到設備並從伺服器中刪除，這意味著它將不再回應執法機構的地理圍欄令狀。
- 「這項變更引發了不同的反應，有些用戶欣賞隱私的提升，而另一些用戶則擔心失去他們的時間軸數據。」
- 「位置數據的加密備份是可用的，但預設情況下是禁用的，需要手動啟用，這讓一些用戶覺得不直觀。」

## [Talos：安全、不可變且精簡的 Linux 作業系統，用於運行 Kubernetes](https://www.talos.dev)

- Talos Linux 是一個安全、不可變且精簡的 Linux 作業系統，專為 Kubernetes 設計，支援雲端平台、裸機和虛擬化平台，並透過 API 進行系統管理。
- 它強調安全性、可預測性和可演進性，通過保持最小化、強化和不可變性，並且所有 API 訪問都通過雙向 TLS(mTLS)認證來確保安全。
- 「Talos Linux 已經達到生產環境準備狀態，支援大型 Kubernetes 叢集，並且可以在筆記型電腦上透過 Docker 在僅僅 3 分鐘內啟動，這使得它對開發者來說非常易於使用且高效能。」

### [評論](https://news.ycombinator.com/item?id=40956546)

- Talos 是一個安全、不可變且精簡的 Linux 作業系統，專門為運行 Kubernetes 而設計，透過基於 API 的配置、升級和除錯來簡化系統維護。
- 「Talos 需要一個完整的磁碟來進行安裝，這對於存儲空間有限的伺服器來說可能會有挑戰，但使用者已經探索了像是 PXE、虛擬化和外部存儲等解決方案。」
- 「Talos 因其易於設置和維護而受到讚譽，支援 Raspberry Pi 4b，並在其 YouTube 頻道上提供教學影片。」

## [構建和擴展 Notion 的資料湖](https://www.notion.so/de-de/blog/building-and-scaling-notions-data-lake)

- 「Notion 的數據在三年內增長了 10 倍，這使得他們必須開發並擴展數據湖，以應對產品和分析需求，包括 Notion AI 功能。」
- 主要挑戰包括管理 480 個 Fivetran 連接器、緩慢且昂貴的數據導入到 Snowflake，以及超出標準 SQL 能力的複雜數據轉換邏輯。
- 使用 S3 作為儲存、Spark 作為處理，以及 Kafka Debezium CDC 連接器與 Apache Hudi 進行資料匯入來實作資料湖，結果顯著節省了成本並縮短了資料匯入時間，從而成功推出了 Notion AI 功能。

### [評論](https://news.ycombinator.com/item?id=40959787)

- Notion 轉換至資料湖以降低與 Fivetran 和 Snowflake 相關的高成本，於 2022 年節省了超過一百萬美元。
- 此舉涉及轉移大型 Postgres 資料集，並利用 Iceberg 和 Delta Lake 等工具來支援分析和 AI 功能。
- 用戶對數據隱私和潛在的數據挖掘表示擔憂，但很可能 Notion 會將這些數據用於內部分析和人工智慧訓練。

## [Git-PR：透過 SSH 的補丁請求](https://pr.pico.sh/)

- 目前 Pico 專案正在進行多項更新，包括重構補丁解析程式碼以及重新編寫利用 SSH 的受管網路服務支援。
- 正在引入新功能和改進，例如在物件存儲工具中使用 `entry.Size`，以及透過 SSH 傳送和接收檔案的中介軟體。
- 「這些更新旨在增強協作工具並簡化使用熟悉的協議如 rsync、scp 和 sftp 的檔案傳輸流程。」

### [評論](https://news.ycombinator.com/item?id=40959526)

- Git-PR 引入了一種新的工作流程，通過 SSH 提交補丁請求，旨在簡化相較於傳統電子郵件工作流程和像 GitHub 這樣的集中化服務的過程。
- 「基於 SSH 的工作流程減少了設置郵件列表和配置電子郵件客戶端的需求，但它需要維護 SSH 伺服器和管理 SSH 身份。」
- 此工具針對自我託管環境和駭客愛好者，提供一個比 Gerrit 或 GitHub 更輕量的替代方案，並旨在與現有工具如 git 和 SSH 無縫整合。
