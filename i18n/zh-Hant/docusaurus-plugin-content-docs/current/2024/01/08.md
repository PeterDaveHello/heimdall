# 2024-01-08

## [介紹 Nino：用於簡化工作任務的多合一應用程式](https://nino.app)

- Nino 是一款新應用，旨在解決知識工作者的應用過載問題。
- 該應用程式將各種功能結合到一個平臺中，提供18個模組，用於資料庫管理、組合和通信等任務。
- Nino 優先考慮整合、離線模式以提高性能，並計劃引入端到端加密以增強隱私和安全性。此外，用戶可以在自定義域或免費子域上發佈他們的作品。開發人員積極尋求用戶反饋，以進一步完善平臺。

### [反應](https://news.ycombinator.com/item?id=38901504)

- Nino 推出了應用程式，旨在通過將多種功能整合到一個平臺中來簡化知識工作者的應用程式過載。
- 對 Nino 的反饋包括對可伸縮性、資料庫系統、用戶體驗、數據擁有權和自託管選項的討論。
- 涵蓋的其他主題包括即時協作工具、為不同需求尋找合適的工具、應用程式的技術堆疊和架構以及潛在的改進功能。

## [GitUI：用於高效版本控制的快速可移植的 git GUI](https://github.com/extrawurst/gitui)

- GitUI 是一個基於終端的工具，它為各種 git 任務提供了使用者友好的介面，包括索引、提交、比較、隱藏、責備和日誌記錄。
- 它優先考慮速度、可移植性和開源，與其他流行的 git GUI 相比，它在速度和記憶體使用方面在基準測試中表現更好。
- 1.0 版的路線圖包括可視化分支結構和互動式變基等功能，但它有一些局限性，例如缺乏對稀疏存儲庫、GPG 簽名和 git-lfs 的支援。
- 為不同的平臺提供了安裝說明，並且提供了顏色主題和鍵綁定的自定義選項。
- 該項目鼓勵支持和贊助，並從 lazygit、tig 和 GitUp 等工具中汲取靈感。

### [反應](https://news.ycombinator.com/item?id=38905019)

- 本文探討了各種 Git 使用者介面 （UI） 及其功能，以及使用者對不同 GUI 工具的看法和體驗。
- 一些使用者更喜歡 GUI，因為它們易於使用和方便，而另一些使用者則主張命令行介面的速度和效率。
- 基於 Rust 的 UI 工具（如 GitUI 和 lazygit）與 Magit、Fork 和 SourceTree 等其他流行選項進行了比較，對它們的有用性和必要性意見不一。

## [改進的 HTMX Playground：在瀏覽器中試用 HTMX](https://lassebomh.github.io/htmx-playground/)

- 作者恢復了一個舊專案，允許使用者在瀏覽器中試驗 HTMX。
- 該專案具有一個代碼沙箱和一個基於 Django 的模擬伺服器，並且被設計為與框架無關。
- 存在一些已知問題，包括有限的移動支援、使用 Ace Editor 而不是 Monaco 以及錯誤輸出不足。作者歡迎反饋以改進專案。

### [反應](https://news.ycombinator.com/item?id=38906989)

- 作者開發了 HTMX Playground，這是一個用於在瀏覽器中探索 HTMX 的代碼沙箱。
- Playground 包括一個模擬伺服器，並受到 Django 的啟發，但它與框架無關，允許使用者在不編寫專門代碼的情況下試驗 HTMX。
- 雖然 Playground 提供了一個方便的測試和提供反饋的平臺，但它的移動支援有限，並且 Ace 編輯器存在一些問題。

## [不作為：科學不端行為的後果和問責制](https://statmodeling.stat.columbia.edu/2024/01/07/french-bio-lab-research-scandal/)

- 法國國家科學研究中心（CNRS）巴黎北索邦大學（Université Sorbonne Paris Nord）的一個化學實驗室被發現有多篇科學文章，其中包含經過處理和回收的數據。
- 法國國家科學研究中心沒有撤回這些文章，而是要求更正，並對負責的科學家施加一個月的豁免。
- 作者主張及時調查研究不端行為，透明地報告結果，立即撤回有嚴重錯誤的文章，並退還用於欺詐性研究的研究經費。他們強調未能有效解決不當行為的負面後果，例如阻礙科學進步、侵蝕公眾信任以及使不誠實的科學家與誠實的科學家競爭。保護參與不當行為的研究人員的機構受到批評。

### [反應](https://news.ycombinator.com/item?id=38903145)

- 這些文本討論了一系列主題，例如詐騙、欺詐、剽竊、功能失調的團隊、操縱和研究不端行為。
- 他們強調識別和避免詐騙的重要性，以及剽竊和研究欺詐的後果。
- 還探討了公司道德行為的重要性，以及人類對操縱的脆弱性和被騙的正常化趨勢。這些文本還觸及了意識形態、制度和制度的挑戰和缺陷。

## [Motūrus OS：面向 x64 虛擬機的 Rust 優先作業系統，旨在實現簡單性、安全性和性能](https://github.com/moturus/motor-os)

- Motūrus OS 是專為基於雲的虛擬機工作負載而設計的全新操作系統。
- 它的目標是比 Linux 更簡單、更安全，同時實現可比或更好的性能。
- Motūrus OS 內置於 Rust 中，目前支援基於 x64 KVM 的虛擬機，並提供微內核調度、基本文件系統、網路功能和簡單的 HTTPd 等功能，但仍處於開發階段，尚不適合生產使用。

### [反應](https://news.ycombinator.com/item?id=38907568)

- 討論圍繞操作系統展開，涉及諸如MotorOS的開發和取代Linux的挑戰等主題。
- 還討論了 SteamDeck 遊戲機和基於 Unix 的系統的安全性。
- 這些討論闡明瞭不同操作系統的設計選擇和安全影響，並強調了程式設計語言 Rust 在操作系統開發中的成功。
