## [用於家庭網路的安全簡化Wi-Fi：探索 WPA3 企業版和EAP-TLS](https://smallstep.com/blog/home-network-eap-tls-wifi/)

- 由於設備的複雜性和相容性問題，不建議在家中使用NSA級Wi-Fi。
- 建議將 WPA2-PSK 或 WPA3-Personal 等替代安全模式用於家庭 Wi-Fi 網路。
- 對於企業 Wi-Fi 中的安全網路身份驗證，建議使用 EAP-TLS。提供了有關使用 WPA3 Enterprise 認證設置 Wi-Fi 網路、保護網路以符合 HIPAA 要求以及手動管理 Apple 設備以進行 MDM 的說明。 本文還介紹了為房客創建證書的過程。

### [反應](https://news.ycombinator.com/item?id=38873810)

- 正在討論Wi-Fi網路的安全性和身份驗證方法的有效性。
- 主題包括使用 VPN 進行敏感訪問、僅依賴 Wi-Fi 身份驗證的局限性以及對多層保護的需求。
- 其他主題包括保護物聯網設備、閘道和防火牆的好處、使用多個預共用密鑰以及 WPS 的局限性。
- 還討論了使用EAP-TLS進行安全身份驗證、設備識別和實現RADIUS伺服器。
- 眾所周知，僅僅依靠Wi-Fi來確保安全是不夠的，應該採取額外的措施。

## [性能提升：在 AMD64 上拆分 16 位元組以上的結構體](https://gist.github.com/FeepingCreature/5dff669aad380a123b15659e195fb96c)

- 在 AMD64 上傳遞大於 16 位元組的結構會顯著減慢代碼執行速度。
- 將陣列更改為作為三個單獨的指標參數而不是一個結構參數傳遞，將Neat語言的性能提高了兩倍。
- 這種改進是由於 SystemV AMD64 ABI 規範中的指標傳遞大於 16 個字節的結構，從而導致函數調用的額外堆疊重新排列。通過將數位作為單獨的指標參數傳遞，這些值已經在 SSE 寄存器中，不需要從堆疊中載入，從而加快了執行速度。

### [反應](https://news.ycombinator.com/item?id=38875422)

- 討論探討了幾個程式設計主題，例如在 AMD64 平台上傳遞結構體以及按值或引用傳遞物件。
- 還討論了Google的數據傳遞方法，以及堆疊寫入和堆分配之間的性能差異。
- 該對話強調了編碼實踐中性能、清晰度和效率之間的權衡，強調了代碼優化和分析的重要性。
