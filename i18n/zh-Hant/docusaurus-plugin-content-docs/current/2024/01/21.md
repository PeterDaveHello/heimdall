# 2024-01-21

## [Spotube：集成了 YouTube 的開源 Spotify 用戶端](https://github.com/KRTirtho/spotube)

- Spotube 是一個開源的 Spotify 用戶端，無需訂閱 Spotify Premium 即可使用。
- 它利用 Spotify 的數據 API 並提供替代音訊源，如 YouTube、Piped.video 或 JioSaavn。
- 功能包括無廣告、可下載曲目、跨平臺支援、體積小、數據使用量低、匿名登錄、時間同步歌詞和原生表演。它不收集用戶數據或支援Spotify Shows和Podcasts。

### [反應](https://news.ycombinator.com/item?id=39066136)

- 討論涵蓋了音樂流媒體平臺的幾個方面，例如 API 金鑰、Spotify 的播客限制、DRM 問題、音訊品質、Spotify 與 YouTube 的比較、對隨機播放的挫敗感、對藝術家收入的影響、Bandcamp 等替代平臺，以及在經濟上支援音樂家的建議。
- 對話強調了更好的藝人薪酬和探索替代流媒體模式的重要性。
- 這一全面的討論揭示了音樂流媒體行業當前的挑戰和潛在的解決方案。

## [Bun Shell 簡介：簡化 JavaScript 中的 Shell 腳本](https://bun.sh/blog/the-bun-shell)

- 由於不同平台的語法和命令存在差異，在 JavaScript 中運行 shell 腳本可能具有挑戰性。
- Bun Shell 是 Bun 中的一種嵌入式語言和解釋器，允許在 JavaScript 和 TypeScript 中編寫跨平臺的 shell 腳本。
- Bun Shell 旨在為在 JavaScript 中運行 shell 腳本提供一個簡化且高效的解決方案，解決現有 shell 的複雜性和不一致性。

### [反應](https://news.ycombinator.com/item?id=39071688)

- 使用者正在討論名為 Bun Shell 的新 shell，表達了對相容性、替換現有命令和臃腫的 coreutils 的擔憂。
- 該工具因其在腳本任務中的簡單性、創新性和實用性而受到稱讚。
- 對話還涵蓋了不同 shell 的可用性，並推薦了替代選項。
- Bun 的開發人員會回答問題和反饋，解決對導入名稱和穩定性的擔憂。
- 討論擴展到包括程式設計語言 Zig，使用者分享經驗並討論其穩定性。
- 涵蓋的其他主題包括 JavaScript 中的腳本、Node.js 中 shell 命令的性能問題、標記範本、Lit 元素和 rimraf 庫。
- 人們擔心使用Bun Shell的安全風險和局限性。
- 討論還探討了 ZSH、範本語言、JSON 和 YAML 配置檔以及跨平臺相容性。
- 關於同步與異步執行以及與託管供應商的負面體驗的爭論出現了。
