# 2024-01-06

## [BoopSnoop：家庭自製消息傳遞應用程式](https://www.robinsloan.com/notes/home-cooked-app/)

- 作者開發了一款名為BoopSnoop的消息傳遞應用程式，以取代以前的應用程式供家人使用。
- BoopSnoop 可以在家庭成員之間輕鬆共用照片和視頻，消息在查看後會消失。
- 作者是一位「家庭廚師」程式師，他強調其編碼專案的個人和非商業性質，並重視他們對應用程式的獨立性和控制權，而不是使用公司消息傳遞應用程式。

### [反應](https://news.ycombinator.com/item?id=38877423)

- 用戶參與對話，重點介紹個人應用開發的好處和挑戰。
- 討論了各種主題，包括編碼實踐、服務集成、移動應用程式的自定義以及管理配置檔和證書。
- 對話還深入探討了技術對個人的影響、人工智慧在應用程式開發中的作用，以及迎合小使用者群的挑戰。

## [波音公司要求豁免MAX 7飛機的安全規則](https://www.seattletimes.com/business/boeing-aerospace/boeing-wants-faa-to-exempt-max-7-from-safety-rules-to-get-it-in-the-air/)

## [全息攝像頭：蔡司的透明玻璃相機引發隱私問題](https://www.digitalcameraworld.com/news/this-holographic-camera-turns-any-window-into-an-invisible-camera)

- 蔡司推出了Holocam，這是一種使用全息術將任何玻璃螢幕轉換為相機的技術。
- 相機是透明的，可以放置在螢幕上的任何位置，無需切口或打孔。
- Holocam 的潛在應用包括車載功能、智慧門鈴、網路攝像頭、停車攝像頭和面部/手勢識別。然而，由於這種技術可能隱藏攝像頭，因此出現了隱私問題。

### [反應](https://news.ycombinator.com/item?id=38881981)

- 本文討論了蔡司開發的各種技術，包括全息攝像頭、隱形層相機感測器和多鏡頭玻璃。
- 隱私問題、潛在應用、圖像品質和蔡司光學的聲譽是討論中探討的一些主題。
- 本文還提到了全息投影等其他技術，並將手機攝像頭與數碼單反相機進行了比較，從而平衡地瞭解了這些技術的優點和局限性。

## [美國司法部接近對蘋果的反壟斷訴訟](https://www.nytimes.com/2024/01/05/technology/antitrust-apple-lawsuit-us.html)

- 美國司法部正準備對蘋果提起廣泛的反壟斷訴訟，重點是其維持iPhone主導地位的努力。
- 該調查正在調查蘋果對其硬體和軟體的控制如何為消費者轉向其他設備以及競爭對手競爭製造障礙。
- 主要關注的領域包括Apple Watch的集成，iMessage的排他性以及iPhone的支付系統。

### [反應](https://news.ycombinator.com/item?id=38883393)

- 美國正在考慮對蘋果提起反壟斷訴訟，原因是擔心其iMessage服務的排他性。
- Apple 的 AirPlay 和 Google 的 Google Cast 之間的相容性問題是討論的一部分。
- 這篇文章提出了關於蘋果對設備、應用商店的控制、消息傳遞平臺之間的互操作性以及使用不同手機操作系統的社會影響的問題。

## [CentOS Linux 將於 2024 年 6 月停產，紅帽提供過渡支援](https://www.redhat.com/en/blog/fastest-road-centos-linux-red-hat-enterprise-linux)

- Red Hat 宣佈 CentOS Linux 將於 2024 年 6 月停產，Red Hat Enterprise Linux 7 將結束維護。
- 使用者可以使用紅帽的工具和支援，分兩階段轉換和升級到受支援的紅帽企業 Linux 版本。
- 建議立即開始規劃和遷移，以確保在生命周期結束截止日期之前實現無縫過渡。

### [反應](https://news.ycombinator.com/item?id=38878587)

- Red Hat 決定終止對 CentOS Linux 的支援並過渡到 CentOS Stream，這在用戶中引發了爭議和沮喪。
- Ubuntu 被強調為 CentOS 的流行替代品，為伺服器部署提供了優勢。
- 提到了其他 Linux 發行版（如 Rocky Linux 和 Alma Linux）的興起，這標誌著 Red Hat 的受歡迎程度下降。
- 討論了對原始程式碼的可訪問性、不同發行版的相容性和穩定性的擔憂。
- 使用者對 CentOS 8 的終止支援表示失望，並建議擴展支援選項。
- 探討了 'apt' 和 'dnf'/'yum' 等包管理器之間的差異。
- 使用者分享他們在遷移到替代操作系統方面的經驗和擔憂。
- 分析了IBM/Red Hat 的決策對使用者信任的影響，以及圍繞收購競爭對手的爭議。
- 討論結束時，出現了新的替代方案，以應對紅帽的行動。

## [現代計算機：在沒有硬碟 LED 和嘈雜機器的情況下監控性能的挑戰](https://blogsystem5.substack.com/p/hard-disk-leds-and-noisy-machines)

- 現代計算機不再有硬碟 LED 或嘈雜的機器，使開發人員更難識別性能問題。
- 舉例說明瞭效率低下的問題，這些效率低下在引起問題之前一直被忽視，強調了使用性能監視器的重要性。
- 遠端開發環境對監控提出了挑戰，但瞭解任何異常情況仍然至關重要。

### [反應](https://news.ycombinator.com/item?id=38883408)

- 本文探討了對舊技術的懷舊之情以及監控計算機性能和活動的重要性。
- 它討論了關於視覺通知和聲音反饋等指標在評估系統功能中的重要性的不同意見。
- 本文還談到了使用嵌入式系統的挑戰、代碼複雜性對計算能力的影響以及開發人員的偏好。最後，它提出了有關監視系統指標的建議，以及關注特定性能指標而不是僅依賴 CPU 使用率的重要性。

## [業餘愛好者的 8.99B LLM 在具有實驗模型的排行榜上超越了原始](https://huggingface.co/CultriX/MistralTrix-v1)

- CultriX/MistralTrix-v1 模型是一個 transformer 模型，專為文本生成而設計，並在英語上進行了訓練。
- 它已使用直接偏好優化 （DPO） 進行了微調，並且在不同的基準測試中表現優於原始模型。
- 該模型的參數大小為8.99B，在短時間內在單個 Colab GPU 上進行了訓練，可在Google Colab和 GitHub 上下載。但是，需要注意的是，該模型仍處於實驗階段，創建者承認自己是該領域的新手。

### [反應](https://news.ycombinator.com/item?id=38882726)

- 這篇文章提供了關於為排行榜構建高級語言模型 （LLM） 的教程。
- 作者將他們的成功歸因於運氣和他人的説明。
- 評論者質疑排行榜的可靠性和準確性，提到了對過度擬合和作弊的擔憂。

## [Harlequin：支援 DuckDB 和 SQLite 的終端 SQL IDE](https://github.com/tconbeer/harlequin)

- Harlequin 是一個專為終端設計的 SQL IDE，使用戶能夠與 DuckDB 和 SQLite 資料庫進行交互。
- 使用者可以使用 pipx 方便地安裝 Harlequin，並從其眾多功能和適配器中受益。
- Harlequin 專案對貢獻開放，鼓勵使用者提供反饋、提出問題，甚至提交代碼更改的拉取請求。

### [反應](https://news.ycombinator.com/item?id=38882526)

- 該摘要概述了 SQL IDE 工具 Harlequin，包括其局限性和正在進行的開發。
- 它討論了在 Python 開發中使用虛擬環境以及用於管理 Python 專案的替代工具，重點介紹了隔離依賴關係的好處。
- 該摘要還介紹了在訪問多個 Python 專案的命令時對命令行介面的偏好，以及使用終端使用者介面 （TUI） 進行 SQL 開發的優勢。此外，它還提到了對 Harlequin 與流行資料庫的相容性的擔憂以及使用者對附加功能的建議。
