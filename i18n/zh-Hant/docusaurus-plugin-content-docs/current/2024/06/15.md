# 2024-06-15

## [旅行者1號重新上線！NASA太空船從所有4個儀器返回數據](https://www.space.com/voyager-1-fully-operational)

- 美國國家航空暨太空總署(NASA)的旅行者1號太空船自2023年11月以來經歷技術問題後，現已完全恢復運作，所有四個科學儀器均返回可用數據。
- 問題被追溯到飛行數據子系統中的一個晶片，並通過重新定位代碼解決，展示了這艘已有46年歷史的太空船的韌性。
- 儘管距離地球150億英里，旅行者1號仍然繼續從星際空間提供有價值的數據，並且正在努力維持其功能並延長其任務。

### [反應](https://news.ycombinator.com/item?id=40687660)

- 旅行者1號重新上線，並從所有四個儀器傳輸數據，這是對最近去世的旅行者核心項目科學家埃德·斯通的致敬。
- 討論引發了關於世代飛船和未來更快的太空船可能先到達目的地的潛力，並探討了時間膨脹和相對論旅行等概念。
- 突破攝星計劃提出使用激光帆加速克級探測器，這些探測器有可能在十年內超越旅行者1號，這突顯了太空旅行技術的進步。

- 人們對像 Perplexity 這樣的聚合服務對內容創作者的負面影響表示擔憂，因為這些服務可能會減少對原始內容來源的直接流量，從而導致潛在的見解和收入損失。
- 該文本還涉及版權、合法性和倫理在人工智慧訓練和數據抓取中的更廣泛問題，將這些做法與歷史案例進行比較，並討論了可能需要新的法規和分析功能。

## [指數級更好的旋轉 (2022) 結果：](http://thenumb.at/Exponential-Rotations/)

- 3D 程式設計涉及各種 3D 旋轉技術，包括旋轉矩陣、歐拉角和四元數，每種技術都有其優點和限制。
- 歐拉角可能會引起萬向節鎖定和不一致的角速度等問題，使其在機器人等應用中的插值不太適合。
- 四元數和軸/角旋轉被強調為在組合和插值旋轉方面更有效，其中四元數提供了一種穩健的3D點旋轉方法，並且易於計算指數映射。

### [反應](https://news.ycombinator.com/item?id=40684901)

- 本文解釋了李群/李代數對應關係，這是一個對於3D旋轉非常有用的數學概念，允許使用指數和對數映射進行平滑的過渡和組合。
- 四元數因其在計算機圖形學中特別是在動畫中的效率而受到重視，這是由於它們的緊湊表示和相較於3x3矩陣的計算優勢。
- 討論包括使用卡爾曼濾波器來估計旋轉、平均旋轉的複雜性，以及在機器人技術和視頻遊戲開發等領域應用各種度量標準。

## [特斯拉的全自動駕駛——一個無用的技術展示 結果：](https://tomverbeure.github.io/2024/05/20/Tesla-FSD-First-and-Last-Impressions.html)

- 特斯拉正在提供其全自動駕駛(FSD)系統的免費試用，但用戶報告稱該系統仍然存在重大錯誤，需要駕駛員干預。
- 在多次測試中，FSD 出現了錯誤，例如錯誤的變道、緊張的修正，甚至闖紅燈，這引發了對其可靠性的擔憂。 結果：
- 作者總結道，雖然FSD顯示出潛力，但目前缺乏實際價值，並且更偏好像自適應巡航控制和車道居中這樣更可靠的自動駕駛功能。

### [反應](https://news.ycombinator.com/item?id=40688001)

- 特斯拉的全自動駕駛(FSD)技術被批評為更多是提升股價的工具，而不是一個可行的產品，並且對其技術優越性相較於競爭對手存在疑慮。
- 報告顯示，特斯拉的駕駛員參與系統薄弱，不適合自動駕駛的能力，導致誤用、事故，並至少造成13起致命事件。
- 辯論包括對特斯拉承諾的懷疑、實現完全自動駕駛的複雜性以及需要更多傳感器來改善駕駛，突顯了用戶期望與實際能力之間的差距。

## [Syzygy：一種低成本、緊湊且高性能外圍設備的開放標準](https://syzygyfpga.io/)

- SYZYGY 是一種開放標準，專為高性能外圍設備連接而設計，提供經濟實惠且緊湊的連接器和電纜。
- 它彌合了低速 Digilent PMOD 設備和高性能 VITA 57.1 FMC 外圍設備之間的差距，使其成為原型、系統集成、測試和評估系統的理想選擇。
- SYZYGY 提供兩種類型的連接器：一個 40 針標準外圍連接器和一個 60 針收發器外圍連接器，兩者均支持固定和可編程電壓以及單端信號。

### [反應](https://news.ycombinator.com/item?id=40686787)

- 本文討論了將硬PCIe(外圍元件互連快速通道)端點整合到MCU(微控制單元)和MPU(微處理單元)中，以實現高速、低延遲通信的潛在好處。
- 當前的限制包括需要昂貴的FPGA(現場可編程門陣列)或有限選擇的MCU/MPU來使用PCIe，以及RISC-V MCU中缺乏快速數據傳輸選項。
- 該文本強調了Opal Kelly開發的Syzygy標準，該標準使用了類比電壓等創新方法進行外圍設備尋址，並且已逐漸被採用，儘管與其他標準如FMC(FPGA子卡)相比，其受歡迎程度尚不明確。

## [Django SQLite 生產配置](https://blog.pecar.me/sqlite-django-config)

- 由於 SQLite 的低並發優化與 Django 的需求相衝突，Django 中的默認 SQLite 配置不適合用於生產環境。
- 對於 Django 版本 5.0、4.2 或更早的版本，啟用 WAL(預寫日誌)日誌模式可以通過防止寫請求阻塞讀取來提高並發性。
- Django 5.1 預計在 2024 年 8 月發布，將允許在 settings.py 中直接配置這些 SQLite 優化，簡化開發人員的流程。

### [反應](https://news.ycombinator.com/item?id=40688336)

- Django SQLite 生產配置在 WAL(預寫日誌)模式下存在一個重大問題，即在高負載下 WAL 文件可能會過度增長，可能導致系統故障。
- 一個解決方法是運行一個單獨的線程來監控WAL大小，如果超過8 MB則減慢讀寫速度，但這表明在WAL模式下的SQLite可能不適合生產環境。
- 由於 PostgreSQL 在處理高容量交易方面表現更佳，且相比 SQLite 有較少的限制，因此建議大多數小型到中型項目使用 PostgreSQL。
