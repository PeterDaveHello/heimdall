# 2024-06-22

## [「我的 Windows 電腦再也不像是我的了」](https://www.howtogeek.com/my-windows-computer-just-doesnt-feel-like-mine-anymore/)

- Windows 變得更加商業化和不那麼個人化，廣告整合讓它感覺廉價。
- 強制更新和侵入性更改經常會破壞系統並在未經同意的情況下更改用戶設置，影響可靠性和用戶體驗。
- 像 macOS 和 Linux 這樣的替代方案因為沒有廣告且提供更多的用戶控制而受到讚譽，儘管 Linux 可能比較複雜。

### [反應](https://news.ycombinator.com/item?id=40755444)

- 用戶對於 Windows 中日益增多的控制和侵入性功能表示沮喪，例如無法從開始菜單中移除「推薦」以及持續出現的 OneDrive 提示。
- 投訴包括無法禁用開始菜單中的網頁搜索、Edge 瀏覽器設置被重置以及 Windows 自動安裝舊版驅動程序等問題。
- 一些用戶建議使用替代方案，例如使用 Linux 發行版(如 Fedora、Linux Mint)或工具來管理和精簡 Windows，這突顯了對當前 Windows 體驗的不滿情緒日益增長。

## [互聯網檔案館在出版商勝訴後被迫移除50萬本書籍](https://arstechnica.com/tech-policy/2024/06/internet-archive-forced-to-remove-500000-books-after-publishers-court-win/)

- 互聯網檔案館(IA)在輸掉與書籍出版商的訴訟後，被迫移除50萬本書，這對依賴IA獲取稀有書籍的讀者造成了重大影響。
- “IA 正在對該決定提出上訴，認為其受控數字借閱應被視為版權法下的合理使用，強調圖書館在提供知識獲取方面的作用。”
- 一封擁有近19,000個簽名的公開信強調了對教育和弱勢社區的負面影響，IA的支持者敦促出版商恢復對被移除書籍的訪問權限。

### [反應](https://news.ycombinator.com/item?id=40754229)

- 互聯網檔案館(IA)在法院裁決支持出版商後，被命令移除50萬本書，這引發了對版權法的辯論。
- 批評者認為，IA 的善意行動具有法律風險，可能會危及該組織，這突顯了版權改革的必要性。
- 這個案例強調了在保護知識獲取權和遵守現行版權法之間的緊張關係，許多人主張需要達到更好的平衡。

## [Farm：用Rust編寫的快速vite兼容構建工具](https://www.farmfe.org/)

- Farm 是一個基於 Rust 的網頁構建工具，專為快速網頁開發和 JavaScript 庫設置而設計，以其速度和效率著稱。
- “它支持增量構建，具有持久和模組級別的緩存，只編譯更改過的模組，並提供豐富的功能，包括對各種文件類型和框架(如 React 和 Vue)的支持。”
- “Farm 是完全可插拔的，兼容 Vite 插件，並確保開發和生產環境之間的一致性，支持傳統和現代瀏覽器。”

### [反應](https://news.ycombinator.com/item?id=40756034)

- Farm 是一個用 Rust 編寫的新構建工具，與 Vite 兼容，並聲稱能顯著減少構建時間。
- 用戶意見分歧，有些人擔心由於其非西方來源而可能存在的安全風險，而另一些人則讚揚其在持續集成(CI)管道中特別快的構建時間。
- 討論包括與其他前端構建工具如ESBuild、Rollup和Bun的比較，強調了性能改進以及對穩定性和生態系統支持的擔憂。

## [在 C/C++ 中擁有常量文件夾](https://www.neilhenning.dev/posts/yourownconstantfolder/)

- 在 C/C++ 中使用 `sqrtps` 內建函數並搭配 clang 的 `-ffast-math` 標誌可能會導致精度問題，因為它會將 `sqrtps` 替換為 `rsqrtps`。
- 內聯彙編可以強制使用 `sqrtps`，但當函數被內聯時，它不支持常量折疊。
- 為了確保常量折疊和正確行為，使用 `__builtin_constant_p` 檢查向量是否為常量，並相應地應用 `_mm_sqrt_ps`。

### [反應](https://news.ycombinator.com/item?id=40757787)

- 一位 Hacker News 用戶指出，在優化機器學習代碼時，Rust 的類型安全數學運算存在低效問題，特別是在涉及浮點數和整數除數的歐幾里得餘數計算時。
- 用戶創建了一個自定義函數來提升性能，但由於 Rust 的類型限制而面臨限制，這表明編譯器支持可以促進此類優化。
- 討論擴展到包括專用箱的潛力、C/C++ 優化的複雜性，以及理解編譯器行為和性能優化中的權衡的重要性。

## [Andrew S. Tanenbaum 獲得ACM軟體系統獎](https://vu.nl/en/news/2024/andrew-s-tanenbaum-receives-acm-software-system-award)

- 2024年6月20日，Andrew S. Tanenbaum因其在1987年創建的基於微內核的UNIX操作系統MINIX上的工作，獲得了ACM軟體系統獎。
- MINIX，最初有12,000行代碼，並於2000年開源，對操作系統的教學以及像Linux和MeikOS這樣的系統設計產生了重大影響。
- 該獎項由IBM支持，並包括35,000美元的獎金，旨在表彰Tanenbaum對操作系統設計的影響，特別是他對微內核架構的倡導。

### [反應](https://news.ycombinator.com/item?id=40759277)

- Andrew S. Tanenbaum 獲得了ACM軟體系統獎，以表彰他對計算機科學教育和軟體系統的重大貢獻。
- 譚恩邦以其具有影響力的教科書而聞名，例如《計算機網絡》和《現代操作系統》，這些書幫助了許多學生和專業人士理解複雜的主題。
- 他的創作，MINIX，一個類Unix的操作系統，以其在Intel CPU的管理引擎中的使用而著稱，突顯了其廣泛的影響。

## [HybridNeRF：高效神經渲染](https://haithemturki.com/hybrid-nerf/)

- HybridNeRF，一種新的神經渲染方法，結合了表面和體積渲染，以提高效率和質量，特別是針對半透明和薄結構。
- 該方法在虛擬現實解析度(2K×2K)下實現了實時幀率(36 FPS)，並在像Eyeful Tower和ScanNet++這樣的數據集上將錯誤率提高了15-30%。
- HybridNeRF 在渲染質量和速度方面優於以往的方法，特別是在處理鏡面效果和高解析度室內場景方面。

### [反應](https://news.ycombinator.com/item?id=40759333)

- HybridNeRF 是一種新的神經渲染方法，旨在提高從2D圖像生成3D場景的效率。
- “討論強調了神經輻射場(NeRF)中的挑戰和潛在進展，包括相機位置的準確性和像運動結構(SfM)這樣的預處理步驟。”
- 這篇文章因為潛在的代碼發布和對未來研究方向的興奮而引起了關注，特別是在消除傳統預處理需求方面。

## [A Forth Story (1995)](https://groups.google.com/g/comp.lang.forth/c/ThWiGXwKqv4)

- 艾倫·塞科里奇，一位物理學畢業生，於1975年在MDH工業公司開始了他的職業生涯，從事使用鈣鈦礦252測量煤炭中的硫含量，並最初使用Fortran進行模擬。
- 他後來採用了 Forth 程式語言，儘管最初感到困惑，並用它開發了一個硫磺計原型，面對來自同事的阻力，因為他們認為 Forth 是不尋常的。
- “儘管 Forth 擁有強大的功能和高效性，但由於其小眾地位以及業界對主流語言的偏好，限制了它的採用，最終導致 Cekorich 停止了編程。”

### [反應](https://news.ycombinator.com/item?id=40755688)

- 查克·摩爾的NOVIX 2000 CPU基於Forth，使用了兩個堆疊，並且在每個時鐘週期內執行多條指令，且使用的邏輯閘數量極少，展示了一種替代的CPU架構。
- GA144 晶片受 Forth 啟發，設計簡約，能以更高的靈活性和更低的功耗取代系統晶片(SoCs)，儘管在沒有常見外圍庫的情況下使用可能會有挑戰。
- “Forth 的簡單性允許快速、個人化的創作，但可能不標準且難以理解，這使得儘管它很強大，卻不太被團隊採用。”
