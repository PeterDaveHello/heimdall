## [Swift 的靜態 Linux SDK：為任何 Linux 發行版構建完全靜態鏈接的可執行文件](https://www.swift.org/documentation/articles/static-linux-getting-started.html)

- 靜態 Linux SDK 允許 Swift 程式在不需要目標系統上的 Swift 執行時和依賴項的情況下為 Linux 構建，從而創建完全靜態鏈接的可執行文件。
- 它支持在 macOS 上進行開發並部署到 Linux，消除了運行時開銷和版本控制問題，但會導致可執行文件變大且無法共享代碼。
- SDK 使用 Musl C 庫進行靜態鏈接，這需要對使用 C 庫的軟件包進行調整，並且可以為 x86-64 和 ARM64 機器構建二進制文件。

### [反應](https://news.ycombinator.com/item?id=40651054)

- Swift 的新靜態 SDK 支援用戶自定義平台，包括嵌入式系統和 WebAssembly (WASM)，展示了其在蘋果生態系統之外的擴展。
- Swift 遷移至非蘋果的 GitHub 組織，以及其在 AI 操作系統中用於安全驗證的應用，突顯了其日益增長的多功能性和在各種技術領域中的採用。
- 這項開發允許在 Alpine 容器中運行 Swift 二進制文件、進行交叉編譯，並支持原生的 Debian，這讓那些偏好使用 Debian 進行開發虛擬機(VM)的開發者感到興奮。

- 這篇文章深入探討了AES-GCM的數學基礎，包括伽羅瓦域算術和多項式分解，並展示了當重複使用隨機數時，如何利用這些原理來破解加密。

- 文本強調了在AES-GCM加密中重複使用隨機數的風險，特別是在長期密鑰使用場景中，如VPN、分組通信和全磁碟加密。
- 重複使用隨機數可能會導致嚴重的安全漏洞，包括在攻擊者知道明文和密文的情況下可能發生的攻擊。
