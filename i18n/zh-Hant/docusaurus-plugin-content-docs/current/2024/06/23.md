## [我已經停止使用箱形圖(2021)](https://nightingaledvs.com/ive-stopped-using-box-plots-should-you/)

- 研究表明，常見的圖表選擇可能會無意中引發社會偏見並強化系統性種族主義。
- “通過某些圖表來可視化社會不平等可能會使問題惡化，而不是幫助解決它。”
- “該研究建議在設計和呈現數據可視化時需要更加謹慎，以避免延續偏見。”

### [反應](https://news.ycombinator.com/item?id=40765183)

- 箱形圖可能會產生誤導，因為它們經常使分佈看起來像高斯分佈(鐘形)，這並不總是準確的。
- “直方圖、抖動條形圖和小提琴圖等替代方法可以更好地表示數據的真實分佈。”
- 雖然箱形圖對於比較中位數和分佈非常有用，但它們需要對統計學有扎實的理解才能正確解釋。

## [Llama.ttf：一種同時也是LLM的字體](https://fuglede.github.io/llama.ttf/)

### [反應](https://news.ycombinator.com/item?id=40766791)

- Llama.ttf 是一種獨特的字體，結合了大型語言模型(LLM)和推理引擎，使用戶能夠像聊天一樣與其互動。
- 字體文件顯著地大，約280GB，使其不適合廣泛使用，但它展示了字體設計與人工智慧的創新結合。

## [SSH 作為 Sudo 替代品](https://whynothugo.nl/journal/2024/06/13/ssh-as-a-sudo-replacement/)

- 傳統工具如 sudo 和 doas 使用 setuid 二進制文件和權限提升，這在受限的用戶命名空間中有其局限性，並且需要整個用戶會話保留能力。
- 另一種方法是使用 s6-sudod，將程式分成一個有特權的伺服器和一個無特權的客戶端，旨在以 root 身份運行命令而不進行特權提升。
- 作者嘗試使用本地 SSH 進行 root 命令執行，配置專用的 SSH 密鑰，將 sshd 綁定到 Unix 域套接字，並使用 ProxyUseFdpass 處理套接字文件描述符，以增強安全性並支持各種身份驗證方法。

### [反應](https://news.ycombinator.com/item?id=40763117)

- 使用 SSH 作為 sudo 的替代方案會引入複雜性，因為需要通過 UNIX 套接字和非對稱加密來進行 root 二進制文件的通信。
- “限制 sudo 訪問權限到 wheel 群組並確保只有授權用戶可以讀取或執行它，可以在減少複雜性的同時提供類似的安全性。”
- 像 etckeeper、Ansible 和 Chezmoi 這樣的工具可以幫助有效地管理和追蹤配置變更，提供傳統 sudo 使用的替代方案。

## [絕不要\*使用數據報](https://quic.video/blog/never-use-datagrams/)

- 在構建互聯網應用程序時，TCP 因其可靠的傳輸而被偏好，而 UDP 則因其時效性，特別是在實時視頻協議中被使用。
- 直接使用 UDP 是有風險的，因為需要實現重傳和擁塞控制等功能；相反，開發人員應該使用 QUIC 庫。
- QUIC 和基於 QUIC 的媒體 (MoQ) 支援資料報，但建議開發人員專注於使用 QUIC 流，以避免與資料報相關的陷阱。

### [反應](https://news.ycombinator.com/item?id=40766079)

- “討論集中在使用UDP(用戶數據報協議)與TCP(傳輸控制協議)進行數據傳輸，強調了每種協議的優缺點。”
- UDP經常被標籤為“不可靠”或“盡力而為”，這意味著它不保證傳遞、順序或錯誤檢查，不像TCP，它確保可靠、有序和錯誤檢查的傳遞。
- 這段對話包括了UDP使用案例的例子，例如高頻感測器數據和媒體藝術系統，並討論了使用UDP而非TCP的術語和實際影響。

## [所有的指令都要以逗號開始](https://rhodesmill.org/brandon/2009/commands-with-comma/)

- Unix 使用者經常在他們的主目錄中創建一個 ~/bin/ 目錄來存儲自定義腳本，但這可能會導致與系統命令的名稱衝突。
- 為了避免這些衝突，作者建議在自定義命令名稱前加上逗號，因為逗號在檔案名稱中被視為普通字符，從而避免了衝突。
- 這種技術結合了標籤補全，允許輕鬆瀏覽自定義命令，並且已被證明是一種超過十年的穩健解決方案。

### [反應](https://news.ycombinator.com/item?id=40769362)

- 一位用戶分享了一種在 Windows 上運行 Python 腳本的方法，通過將 python.exe 設置為 `.py` 文件的默認程序並將 `.py` 添加到 `%pathext%`。
- 他們在 Linux 上尋求類似的功能，但由於缺乏「關聯程式」的概念而面臨挑戰，建議包括使用 shebang (`#!/usr/bin/env python3`)、重命名腳本、創建別名或使用像 `update-alternatives` 這樣的工具。
- 討論還涵蓋了使用 git 管理本地配置、使用別名、處理命名空間衝突，以及使用 `chezmoi` 或 ansible 等工具組織腳本和配置。
