# 2024-05-24

## [ICQ將於6月26日停止運營;定向到 VK Messenger 和 VK WorkSpace 的使用者](https://icq.com/desktop/en#windows)

- ICQ將於6月26日停止運營。
- 鼓勵使用者切換到 VK Messenger 進行個人聊天，切換到 VK WorkSpace 進行專業交流。

### [反應](https://news.ycombinator.com/item?id=40467625)

- ICQ是一個曾經流行的即時通訊服務，將於6月26日關閉，喚起使用者的懷舊和悲傷，他們記得它早期的互聯網通信時代。
- 使用者反思數位通信的發展、早期消息服務的技術方面以及ICQ號碼的情感價值。
- 這次討論強調了從早期的互聯網工具（如ICQ）到今天分散的、由企業驅動的通信平台的轉變。

## [了解視頻遊戲中的 2D 剛體碰撞：初學者指南](https://www.sassnow.ski/rigid-body-collisions/1)

- 博客文章「2D 剛體碰撞解析度第 1 部分」介紹了視頻遊戲中處理碰撞的基礎知識，並使用了馬里奧和賽車遊戲等熟悉的例子。
- 它區分了碰撞檢測（識別交叉物件）和碰撞解決（處理後果），解釋了遊戲引擎如何根據速度更新物件位置。
- 這篇文章涵蓋了一些關鍵概念，如表面法線、點積和相對速度，以了解運動方向並確定物體是相互靠近還是遠離。

### [反應](https://news.ycombinator.com/item?id=40463764)

- ksassnowski 的文章“2D Rigid Body Collision Resolution”旨在讓非遊戲開發者和沒有強大數學背景的人能夠理解剛體物理。
- 用戶討論用於物理類比的各種工具和技術，包括 WebGL、three.js、matter.js、XPBD 和 Box2D，並分享個人經驗和專案。
- 對話重點介紹了遊戲物理和適當的物理類比之間的差異、遊戲物理引擎的演變以及碰撞檢測的複雜性。

## [Perplexica：具有隱私和專用搜索模式的開源 AI 搜尋引擎](https://github.com/ItzCrazyKns/Perplexica)

- Perplexica 是一個開源的、人工智慧驅動的搜尋引擎，旨在通過理解使用者查詢來提供準確和最新的搜尋結果。
- 它採用先進的機器學習演算法和 SearxNG 來確保用戶隱私和最新資訊，具有本地 LLM（大型語言模型）和多種搜尋模式。
- 為了方便起見，建議通過 Docker 進行安裝，但可以使用其他方法，用戶可以將其集成到他們的瀏覽器中以快速訪問。鼓勵通過 GitHub 星星、捐贈和貢獻來支持社區。

### [反應](https://news.ycombinator.com/item?id=40462369)

- Perplexica 是 Perplexity 的開源替代品，旨在通過啟用本地使用和避免雲數據問題來增強隱私。
- 用戶建議集成本地電子書和 PDF 搜索，並對 Obsidian 和 Logseq 等工具表示興趣，以獲得更好的多設備功能，同時也注意到對數據丟失和同步問題的擔憂。
- 圍繞“困惑”這個名稱的商標問題正在爭論，突出了技術創新和法律約束之間的平衡，一些人主張使用自託管人工智慧技術來減少對大公司的依賴。

## [Doge Meme 偶像 Kabosu 去世，享年 18 歲](https://www.ign.com/articles/kabosu-the-dog-behind-the-doge-meme-has-died)

- 標誌性的 “Doge” 模因背後的柴犬 Kabosu 因白血病和肝病去世，享年 18 歲，正如她的主人佐藤敦子於 2024 年 5 月 24 日宣佈的那樣。
- Kabosu於 2010 年因 Sato 博客上的一張照片而聲名鵲起，這導致了“Doge”模因的創建，並隨後激發了狗狗幣加密貨幣的靈感。
- 佐藤於 2008 年在一家小狗工廠關閉後收養了 Kabosu，並於 2022 年 12 月透露了她嚴重的健康問題。

### [反應](https://news.ycombinator.com/item?id=40464495)

- 標誌性的“Doge”模因背後的柴犬 Kabosu 在 18 歲時去世，引發了對互聯網模因文化影響的反思。
- 這一消息引發了關於病毒內容的性質、受歡迎程度的不可預測性以及模因的歷史背景的討論。
- 許多人表達了悲傷和懷舊之情，指出Kabosu在互聯網文化中的重要作用，並計劃舉行公開葬禮以紀念她的遺產。

## [Modern Renderer 利用體素技術重振 90 年代 3D 遊戲美學](https://blog.danielschroeder.me/2024/05/voxel-displacement-modernizing-retro-3d/)

- 丹尼爾·施羅德（Daniel Schroeder）引入了一個使用小體素的自定義實時渲染器，以現代化90年代經典3D遊戲（如《毀滅戰士》和《雷神之錘》）的視覺風格。
- 渲染器將復古美學與現代技術融為一體，從低多邊形網格和紋理中產生獨特的視覺效果，並討論了體素幾何體和程式生成等複雜性。
- 該專案用C++和Vulkan編寫，使用Blender進行建模，旨在集成到現有的遊戲引擎中，並計劃動態元素和改進照明。

### [反應](https://news.ycombinator.com/item?id=40464558)

- 體素置換渲染器為復古 3D 美學提供了現代風格，讓人想起像素藝術和早期的 3D 遊戲，喚起了使用者的懷舊之情。
- 它與虛幻引擎 5 的 Nanite 技術進行了比較，儘管它使用了獨特的基於體素的方法，但在技術上有很大不同。
- 儘管其視覺效果令人印象深刻，但渲染器缺乏陰影和動態照明，並且社區有興趣將專案開源，從而激發進一步的實驗。

## [開發人員在 27 天內創建類 Unix 作業系統“ Bunnix”](https://drewdevault.com/2024/05/24/2024-05-24-Bunnix.html)

- 作者在 27 天內開發了一個名為 Bunnix 的類 Unix 作業系統，用於x86_64目標，支援傳統和 EFI 啟動，但缺乏 USB 支援，需要 PS/2 鍵盤。
- Bunnix 的內核是用 Hare 和 C 語言編寫的，包括 PCI 和 AHCI 的驅動程式，支援 ext4 和 memfs 文件系統，並具有 Vim 和 Doom 等第三方軟體。
- 儘管功能強大但存在缺陷，但 Bunnix 對作者來說是一種學習體驗，他計劃將這些見解應用到他們以前的專案 Helios 中，並看到了社區驅動的改進潛力。

### [反應](https://news.ycombinator.com/item?id=40467297)

- Hacker News 的討論深入探討了 Unix 克隆的快速發展，將其與原始 Unix 進行了比較，並討論了 Unix 信號的複雜性。
- 在 Mastodon 上，一個用新的 Hare 程式設計語言編寫的作業系統，在 GitHub 上以 GPLv3 提供，由於避免了 LLVM，其代碼速度較慢而受到爭議，突出了簡單性和性能之間的權衡。
- 這場辯論凸顯了意識形態的純潔性和更廣泛的可用性之間的緊張關係，Hare的開發人員選擇不支援專有平臺，強調他們有權優先考慮特定目標而不是廣泛流行。

## [研究顯示，美軍在聖路易斯的秘密輻射試驗針對少數族裔](https://mospace.umsystem.edu/xmlui/handle/10355/13170)

- 麗莎·馬蒂諾-泰勒（Lisa Martino-Taylor）2011年在密蘇里大學哥倫比亞分校（University of Missouri-Columbia）的博士論文調查了曼哈頓-羅賈斯特聯盟及其在1950年代和1960年代在聖路易斯的秘密氣溶膠研究。
- 這些研究由美國軍方進行，以低收入、少數族裔為主的城市地區進行，以瞭解武器化輻射對健康的影響。
- 該論文討論了組織如何在內部合規和外部無知的情況下進行有害活動，並提出了提高透明度和防止不道德秘密專案的公共政策。

### [反應](https://news.ycombinator.com/item?id=40458686)

- 從 1945 年到 1970 年，美國陸軍在聖路易斯對放射性對健康的影響進行了不道德的研究，儘管現有數據可能使此類實驗變得沒有必要。
- 討論強調了歷史上不道德的醫學實驗，例如給人注射鈈，以及它們對公眾對醫學信任的長期影響，包括反疫苗等運動。
- 該文本還討論了對邊緣化群體進行醫學實驗的倫理問題、政府保密以及恢復公眾對政府和私人實體在疫苗開發和管理方面的信任的挑戰。

## [谷歌的人工智慧生成的搜索概述在準確性和可靠性方面存在問題](https://gizmodo.com/google-search-ai-overview-giant-hallucination-1851499031)

- 谷歌在美國搜尋結果中新的人工智慧生成的概述導致了不準確和誤導性的資訊，通常被稱為“幻覺”。
- 儘管進行了廣泛的測試，但人工智慧仍將諷刺與事實混為一談，引用了洋蔥等不可靠的消息來源，導致了關於公眾人物的奇怪說法和不正確的陳述。
- 谷歌正在努力改進，但急於整合人工智慧以與 ChatGPT 等平臺競爭，導致搜尋結果不一致且不可靠，引發了用戶的擔憂。

### [反應](https://news.ycombinator.com/item?id=40468230)

- 谷歌新的人工智慧驅動的搜索功能，提供摘要答案，正在接受可靠性和準確性問題的審查。
- 批評者強調，人工智慧生成的摘要可能會呈現一個單一的、可能不正確的答案，這與允許從多個來源進行交叉檢查的傳統搜尋結果不同。
- 使用者擔心人工智慧提供的資訊的可信度，理由是錯誤資訊和糟糕的數據來源，有些人正在轉向 DuckDuckGo 和 Kagi 等替代品，以獲得更好的隱私和搜尋結果。

## [約翰·卡馬克（John Carmack）主張在UI設計中立即對新聞事件採取行動](https://twitter.com/ID_AA_Carmack/status/1787850053912064005)

- John Carmack 強調了在使用者介面 （UI） 設計中對“press”事件觸發操作的重要性。
- 他對在宣導這種方法時經常遇到的阻力表示沮喪。

### [反應](https://news.ycombinator.com/item?id=40458655)

- 爭論的焦點是是否應該在按下按鈕或釋放按鈕時觸發使用者介面操作，並考慮回應能力、錯誤預防和使用者首選項。
- Web 內容可訪問性指南 （WCAG） 支援「發佈時操作」，以實現更好的可訪問性，允許用戶通過將指標移開來取消操作。
- 該討論強調了上下文在UI設計中的重要性，建議關鍵操作應在發佈時確認以防止錯誤，而非關鍵操作可以在按下觸發以提高效率。

## [CSharpRepl：具有語法突出顯示和 IntelliSense 的互動式 C# REPL](https://fuqua.io/CSharpRepl/)

- CSharpRepl 是一個跨平臺命令行 C# REPL（讀取-評估-列印迴圈），用於互動式流覽 C# 語言、庫和 NuGet 包。
- 它支援快速語法和 API 測試，無需創建專案，提供命令行自動完成，並允許引用 NuGet 包和現有專案。
- 功能包括使用 F12 導航到原始程式碼，使用 F9 查看 IL（中間語言）代碼，它是免費的、開源的，可通過“dotnet tool install -g csharprepl”進行安裝。

### [反應](https://news.ycombinator.com/item?id=40460296)

- 討論的重點是“CSharpRepl”，這是一個 C# REPL（讀取-評估-列印迴圈）工具，提供語法突出顯示和 IntelliSense。
- 使用者將其與LinqPad等替代品進行比較，LinqPad因其許可成本高昂，但因其安全密碼存儲和進程持久性等功能而受到稱讚。
- 提到的其他工具包括 RoslynPad 和 VSCode 中的多語言筆記本，一些使用者還建議探索 F# 的獨特功能。
