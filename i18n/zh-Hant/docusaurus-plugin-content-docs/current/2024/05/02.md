# 2024-05-02

## [技術職位清單：2024 年 5 月招聘的公司](https://news.ycombinator.com/item?id=40224213)

- 公司發佈清單的工作委員會，指定遠端、現場、實習生或簽證要求;不允許招聘公司。
- 鼓勵讀者僅在真正感興趣的情況下聯繫，並將求職者重定向到其他搜索線程。

### [反應](https://news.ycombinator.com/item?id=40224213)

- 世界各地的科技公司都在招聘各種工程職位，如全棧工程師、軟體開發人員和數據科學家，為遠端、混合和現場工作提供機會，重點關注人工智慧、基因組學、密碼學、金融科技和氣候技術方面的前沿專案。
- 僱主正在尋找具有特定技能和背景的候選人，提供有吸引力的薪水、有趣的專案和協作氛圍。
- 有關更多詳細資訊和申請，有興趣的個人可以直接訪問公司的網站。

## [參議員要求聯邦貿易委員會調查汽車製造商的位置數據共用](https://www.wyden.senate.gov/news/press-releases/wyden-markey-reveal-automakers-provide-detailed-location-information-to-law-enforcement-without-a-warrant-rarely-notify-car-owners-request-ftc-investigate-broken-promises-to-protect-drivers-privacy)

- 參議員懷登（Wyden）和馬基（Markey）敦促聯邦貿易委員會（Federal Trade Commission）調查領先的汽車製造商在沒有搜查令的情況下與執法部門共用客戶的位置數據。
- 很少有汽車製造商要求對這些數據進行搜查，而其他汽車製造商則在收到傳票后發佈這些數據，這引發了對隱私和影響的擔憂，尤其是在最高法院關於生殖權利的多布斯裁決之後。
- 該請求凸顯了參議員對汽車製造商在處理位置數據時隱私保護不足的擔憂。

### [反應](https://news.ycombinator.com/item?id=40222499)

- 關於聯網汽車的隱私問題出現了，重點是汽車製造商對位置數據的收集和共用。
- 建議消費者在購買汽車時要謹慎，並禁用特定的車輛功能。
- 懷登參議員的報告強調了提高位置數據共享的透明度和保障措施的必要性，倡導加強隱私保護和消費者知識。

## [mRNA疫苗顯示出對抗膠質母細胞瘤的希望](https://www.insideprecisionmedicine.com/topics/oncology/mrna-cancer-vaccine-reprograms-immune-system-to-tackle-glioblastoma/)

- 佛羅里達大學的研究人員通過重新程式設計免疫系統來對抗癌症，創造了一種針對膠質母細胞瘤的 mRNA 癌症疫苗。
- 該疫苗在四名患有腦腫瘤的成年人和十隻寵物狗身上進行了測試，在刺激免疫系統對抗癌細胞方面顯示出令人鼓舞的結果。
- 使用患者的腫瘤細胞和脂質納米顆粒進行遞送，這種個人化疫苗正在推進到腦癌的擴展I期臨床試驗，有可能改變癌症免疫療法。

### [反應](https://news.ycombinator.com/item?id=40231515)

- mRNA癌症疫苗在膠質母細胞瘤治療中的可喜結果強調了監管機構在確保藥物品質和管理索賠方面的作用。
- 討論內容包括mRNA疫苗的益處和風險、積極癌症治療的挑戰以及疫苗知情決策的重要性。
- 主題包括疫苗安全擔憂、公共衛生建議以及疾病控制與癌症發病率之間的潛在聯繫。

## [認知：用反語法重新定義元程式設計](https://ret2pop.nullring.xyz/blog/cognition.html)

- 該文檔批評了Lisp的語法局限性，並引入了Cognition，這是一種利用反語法的新型完全後綴語言。
- 討論了 Cognitive 的發展，強調了標記化、分隔符、忽略的字元、特定命令和字元，以及可自定義的標記器分隔符和別名。
- 它探索了 VMACRO、遞歸詞定義、結合 Brainfuck 方言以及 Cognitive 在字串處理和符號 AI 中的應用，突出了其在程式設計中的語法自動化和哲學優雅。

### [反應](https://news.ycombinator.com/item?id=40231563)

- 本文介紹了一種新的語言 Cognition，專注於使用者可修改的語法，引發了讀者對其清晰度和相關性的爭論。
- 它深入研究了 Lisp 宏、AST、樹結構、元程式設計，以及 Cognition 如何動態地重新定義語法結構。
- 討論涵蓋了該專案的複雜性和創造性，包括對空格分隔詞、可讀系統和任意元程式設計使用的反饋，參與者對其主流採用和實際效用表達了不同的意見。

## [16 歲推出 Factful：人工智慧初創公司打擊錯誤資訊](https://factful.io/)

- 高中生安德魯（Andrew）開發了Factful，這是一款人工智慧驅動的網路應用程式，用於驗證在線內容的準確性和可信度，而不僅僅是語法檢查。
- Factful 評估上下文、事實性、連貫性等，旨在打擊錯誤資訊並維護資訊時代的內容完整性。
- 我們鼓勵使用者在他們的網站上測試Factful的免費測試版，同時向Andrew提供反饋和支援。

### [反應](https://news.ycombinator.com/item?id=40222051)

- 一位名叫安德魯（Andrew）的16歲高中生正在與朋友一起創建Factful，這是一家由人工智慧驅動的初創公司，旨在解決傳統語法檢查器之外的在線錯誤資訊。
- 討論內容包括提高對自私意識的重要性、使用人工智慧進行事實核查、應對錯誤資訊挑戰、對文本翻譯系統的反饋以及全球減貧。
- 這些對話涉及使用人工智慧進行事實核查服務、限制以及對電視轉播事件和選舉辯論進行事實核查的努力，反映了在社會妄想信念中打擊錯誤資訊和促進真相的複雜性和挑戰。

## [謹防 AWS S3 成本放大威脅](https://blog.limbus-medtec.com/the-aws-s3-denial-of-wallet-amplification-attack-bc5a97cc041d?gi=67ac052cc1ce)

- AWS S3 拒絕錢包放大攻擊對在 AWS S3 上存儲大型數據檔的公司構成風險，由於數據傳輸成本計算方法導致意外成本變化。
- 無論數據是否公開，這種攻擊都可能給公司帶來大量意外費用，這是由於收到的數據與計費金額之間的差異造成的。
- 緩解措施包括實施成本警報、跟蹤 API 請求和控制對 S3 儲存桶的訪問，以防止惡意行為者的潛在財務利用。

### [反應](https://news.ycombinator.com/item?id=40227736)

- 使用者對 AWS S3 中的潛在錯誤表示擔憂，該錯誤會導致範圍請求的出口成本增加，這凸顯了對計費實踐、數據傳輸差異和意外費用的擔憂。
- 討論涉及支出限制、未經授權的費用、API 文檔、數據緩存以及 AWS 和 Cloudflare 服務的比較，揭示了系統管理員處理未經授權的請求所面臨的挑戰。
- 該對話強調了在 AWS 計費和管理持續存在的問題中利用各種雲服務（如 S3 Glacier 和 R2）的優勢。

## [對輪胎排放毒性的新審查](https://phys.org/news/2024-05-toxicity-fresh-scrutiny-salmon-die.html)

- 文章強調了輪胎排放的潛在危害，特別關注了對環境和人類健康構成風險的化學物質6PPD。
- 研究人員發現6PPD與魚類死亡之間存在聯繫，導致監管機構仔細檢查這種化學物質。
- 加利福尼亞州已開始對輪胎中的 6PPD 進行監管，由於輪胎排放對空氣和水質的影響令人擔憂，促使製造商探索替代選擇。

### [反應](https://news.ycombinator.com/item?id=40223253)

- 本文研究了輪胎磨損對環境的影響，並建議對車輛徵收重量稅，以減少污染和道路損壞。
- 它探討了輪胎磨損、車輛重量、道路惡化和行人安全之間的關係，強調了輪胎中的有害化學物質。
- 建議包括在家工作，通過遠端工作和聯合辦公空間減少通勤，宣導替代交通方式，以盡量減少對汽車的依賴，以提高生活品質。

## [2024年5月科技求職者展示](https://news.ycombinator.com/item?id=40224210)

- 求職者可以在平臺上分享他們的詳細資訊，包括位置、技術技能、簡歷和工作機會電子郵件。
- 只有求職者才能發帖，建議讀者聯繫提供的電子郵件地址進行工作查詢。

### [反應](https://news.ycombinator.com/item?id=40224210)

- 在軟體開發、AI、UI/UX 設計、前端/後端開發等方面具有不同技能的技術專業人士正在積極尋找工作機會，強調遠端工作和搬遷偏好。
- 候選人對不同的職位和地點持開放態度，為有興趣僱用他們的潛在僱主提供他們的簡歷和聯繫方式。
- 科技行業的當前趨勢凸顯了熟練專業人員對遠端工作的濃厚興趣和工作地點的靈活性。

## [復興 Neofetch：新 Fork 確保社區發展](https://github.com/LorenDB/neofetch)

- Neofetch 是一個用於顯示系統資訊的命令行介面工具，由於不活動而已存檔。
- 已經建立了一個分叉來保證社區的持續發展，處理積壓的拉取請求以進行潛在改進。
- 該分叉將處理和合併不完整的請求，這可能會導致對現有功能進行大修。

### [反應](https://news.ycombinator.com/item?id=40222213)

- neofetch 的原始創建者宣布職業轉向農業，從而產生了一個名為 hyfetch 的新分叉。
- 討論的主題包括軟體中的LGBTQ+主題、OSS志願者的責任、尋找可靠的項目維護者、分叉GitHub存儲庫、FOSS社區的批評以及溝通挑戰。
- 辯論涵蓋了開源軟體開發和社區互動的各個方面，展示了該領域的動態性質。
