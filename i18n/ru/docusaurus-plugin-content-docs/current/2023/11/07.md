## [Компания Prossimo выпустила стабильную версию Sudo-rs, утилиты для повышения безопасности Linux, разработанной на языке Rust](https://www.memorysafety.org/blog/sudo-first-stable-release/)

- Компания Prossimo выпустила первую стабильную версию sudo-rs - улучшенной с точки зрения безопасности модификации утилиты sudo в Linux, написанной на языке Rust, безопасном для памяти, что позволяет снизить потенциальные угрозы безопасности.

- Проект был реализован в сотрудничестве с компаниями Tweede Golf и Ferrous Systems при консультативной поддержке создателя оригинальной утилиты sudo Тодда Миллера (Todd C. Miller). Финансовую поддержку оказали, в частности, NLnet Foundation и Amazon Web Services.

### [Реакции](https://news.ycombinator.com/item?id=38161016)

- Компания Prossimo объявила о выпуске первого стабильного релиза sudo-rs, более защищенной версии широко распространенной в Linux утилиты sudo, разработанной на безопасном для памяти языке программирования Rust.
- Проект, уже включенный в операционную систему Wolfi Linux, направлен на противодействие потенциальным угрозам безопасности за счет использования языков, обеспечивающих безопасность памяти; значительный вклад в него внесли разработчики Twoede Golf и Ferrous Systems.
- Внешний аудит безопасности запланирован на сентябрь 2023 года, а финансирование проекта осуществлялось из различных источников, включая NLnet Foundation и Amazon Web Services. Создатель оригинальной утилиты sudo Тодд Миллер (Todd C. Miller) оказал решающее содействие в разработке sudo-rs.

## [Понимание рисков и правильное использование команд Git Rebase и Git Push](https://jvns.ca/blog/2023/11/06/rebasing-what-can-go-wrong-/)

- Автор подробно останавливается на потенциальных проблемах, которые необходимо учитывать при использовании команд 'git rebase' и 'git push --force' в коллективных ветвях, таких как повторное разрешение конфликтов, потеря метаданных коммита, сложности с отменой ребазинга.
- Они предлагают осторожно использовать rebase в общих ветвях и делают акцент на поддержании аккуратной дисциплины фиксации.
- В качестве альтернативы они предлагают схему работы "squash and merge" и не рекомендуют принудительно переводить ветку в общую и ребазировать переведенную ветку, признавая, тем не менее, полезность rebase.

### [Реакции](https://news.ycombinator.com/item?id=38164046)

- Автор обсуждает потенциальные проблемы команд git rebase и git push --force в общих ветвях, включая повторение конфликтов, потерю метаданных коммита, а также сложности с отменой ребаза и ребазом большого количества коммитов.
- Существует рекомендация проявлять сдержанность при использовании rebase в кооперативных ветвях, подчеркивающая важность поддержания чистой дисциплины фиксации.
- В качестве альтернативы в статье предлагается рассмотреть рабочий процесс "squash and merge". Автор не рекомендует принудительное перемещение в общую ветку и ребазинг перемещенной ветки, хотя и признает полезность ребазинга.
