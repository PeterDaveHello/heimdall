## [«Браузер Ladybird расправляет крылья»](https://lwn.net/Articles/976822/)

## [MeshAnything – Преобразует 3D-представления в эффективные 3D-сетки](https://buaacyw.github.io/mesh-anything/)

- «MeshAnything представляет новый метод создания художественных сеток (AM) из 3D-представлений с использованием авторегрессионных трансформеров, повышая эффективность и точность производства 3D-объектов.»
- Этот подход значительно уменьшает количество граней сетки, улучшая эффективность хранения, рендеринга и симуляции, при этом сохраняя высокое качество геометрических особенностей.
- Архитектура использует VQ-VAE и декодерный трансформер с условием формы, демонстрируя превосходную топологию и меньшее количество граней по сравнению с традиционными методами, что делает её значительным достижением в 3D-индустрии.

### [Реакции](https://news.ycombinator.com/item?id=40746310)

- MeshAnything преобразует 3D представления в эффективные 3D-сетки, уменьшая количество граней для улучшения хранения и производительности рендеринга.
- Инструмент требует 7 ГБ памяти и 30 секунд на GPU A6000, но ограничен созданием сеток с менее чем 800 гранями.
- «Хотя некоторые пользователи критикуют его нестандартную некоммерческую лицензию и качество генерируемых сеток, он считается многообещающим инструментом для разработки игр и создания 3D-моделей.»

## [Тестирование генеративного ИИ для проектирования печатных плат](https://blog.jitx.com/jitx-corporate-blog/testing-generative-ai-for-circuit-board-design)

- Клод 3 Опус превосходно объяснял базовые концепции, в то время как Джемини 1.5 был наиболее эффективен в разборе технических спецификаций и создании точных таблиц выводов и посадочных мест.
- Все модели испытывали трудности с рекомендациями по конкретным частям и задачами по детальному проектированию схем, что указывает на то, что LLM лучше подходят для помощи человеческим экспертам, а не для работы в качестве самостоятельных дизайнеров.

### [Реакции](https://news.ycombinator.com/item?id=40751020)

- Генеративный ИИ, в частности модели больших языков с нулевым обучением (LLMs), испытывают трудности с выполнением сложных задач, таких как проектирование печатных плат, что подчеркивает их ограничения в специализированных областях.
- Тонкая настройка LLM для выполнения конкретных задач, таких как создание нетлистов, может улучшить их производительность, но для более сложных задач может потребоваться фундаментальное изменение структуры ИИ.
- Диффузионные генеративные структуры и другие модели ИИ, такие как эволюционные или обучение с подкреплением, могут быть лучше приспособлены для сложных задач в области электротехники (ЭЭ).

## [Создание аудио для видео](https://deepmind.google/discover/blog/generating-audio-for-video/)

- Технология видео-аудио (V2A) генерирует синхронизированные звуковые дорожки из видеопикселей и текстовых подсказок, что позволяет создавать драматические музыкальные сопровождения, реалистичные звуковые эффекты или диалоги для различных типов видео.
- V2A использует диффузионный подход, кодируя видео-вход и улучшая аудио из случайного шума для создания реалистичных аудиоволн, при этом продолжаются исследования, направленные на улучшение артефактов качества видео и синхронизации губ.
- Команда разработчиков подчеркивает важность ответственных практик в области ИИ, используя набор инструментов SynthID для водяных знаков на контенте, созданном ИИ, и проводя тщательные оценки безопасности перед публичным выпуском.

### [Реакции](https://news.ycombinator.com/item?id=40744098)

- DeepMind представила новый инструмент ИИ для создания аудио для видео, добавив его в растущий список генеративных инструментов ИИ.
- «Сообщество выражает смешанные чувства: некоторые считают, что трудно успевать за быстрыми достижениями, а другие обсуждают потенциальное влияние на создание контента и емкости для хранения.»
- Существует заметный интерес к тому, как контент, созданный ИИ, может повлиять на рекламу, политику и будущее создания контента, с предложениями по созданию платформ и инструментов, специально предназначенных для ИИ.

## [Оборудование дисплея Bomb Jack](https://github.com/martinpiper/BombJack/blob/master/README.md)

- Проект Bomb Jack Display Hardware, теперь в версии 2.0, начался как схема для аппаратного обеспечения аркадной игры Bomb Jack и включает такие функции, как адресуемая оперативная память, дополнительное затемнение дисплея и спрайты на весь экран.
- Проект использует TTL (Transistor-Transistor Logic) для изучения графических улучшений для 8-битных компьютеров, сосредотачиваясь на обучении низкоуровневой дискретной логике, а не на использовании CPLD (Complex Programmable Logic Device) или FPGA (Field-Programmable Gate Array).
- Конструкция аппаратного обеспечения включает шесть макетов печатных плат (PCB) для различных функций и использует Proteus для моделирования и PCBWay для производства, стремясь к экономической эффективности и индивидуализации.

### [Реакции](https://news.ycombinator.com/item?id=40745830)

- Новый проект направлен на замену аркадного оборудования Bomb Jack современным эквивалентом, расширяя его первоначальные возможности.
- «Проект, отличающийся от MAME (Multiple Arcade Machine Emulator), представляет собой аппаратное решение, которое даже помогло выявить ошибку в MAME.»
- Оборудование обладает передовыми возможностями, такими как несколько слоев, приоритет слоев, несколько палитр, масштабируемые спрайты и многое другое, расширяя границы компонентов, соответствующих эпохе.

- Жанр начался в 1970-х годах с игр, таких как Adventure Уилла Кроутера, и эволюционировал по мере того, как домашние компьютеры становились более доступными, охватывая более широкую аудиторию.
