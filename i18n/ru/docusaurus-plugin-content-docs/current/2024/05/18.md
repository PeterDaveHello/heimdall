## [Bend: язык высокого уровня для вычислений на GPU с помощью HVM2](https://github.com/HigherOrderCO/Bend)

- Bend - это высокоуровневый язык программирования, оптимизированный для массивно-параллельного оборудования, такого как GPU, предлагающий быстрое выделение объектов, функции высшего порядка, рекурсию и продолжения.
- Он работает на базе среды исполнения HVM2, обеспечивая практически линейное ускорение в зависимости от количества ядер, не требуя явных параллельных аннотаций или ручного управления потоками.
- Bend, созданный компанией HigherOrderCO.com, упрощает параллельное программирование, эффективно выполняя сложные алгоритмы на графических процессорах и эмулируя различные параллельные системы.

### [Реакции](https://news.ycombinator.com/item?id=40390287)

- Обсуждается язык программирования Bend и его реализация HVM2 с упором на производительность GPU в сравнении с Python и Mojo.
- Пользователи оценивают бенчмарки, линейное масштабирование, эффективность компилятора и потенциальные возможности применения Bend, а также предлагают более четкий отказ от ответственности, оптимизацию для одного ядра и улучшенную генерацию кода.
- Функция автоматического распараллеливания в Bend получает как похвалу, так и скепсис в отношении ее практичности и влияния на сообщество разработчиков, при этом высказываются опасения по поводу технических препятствий и утверждений о производительности проекта.

## [ILGPU: Использование C# и F# для высокопроизводительного программирования на GPU](https://github.com/m4rs-mt/ILGPU)

- ILGPU - это высокопроизводительный JIT-компилятор, предназначенный для программ на GPU на языках .Net, сочетающий гибкость C++ AMP с производительностью CUDA.

### [Реакции](https://news.ycombinator.com/item?id=40393873)

- ILGPU позволяет писать программы для GPU на C# и F#, предлагая высокоуровневый подход для оптимизации производительности.

- В продолжающемся споре между низкоуровневыми языками (C/C++) и языками более высокого уровня для программирования на GPU есть веские аргументы. ILGPU использует абстрактный байткод на основе стека для эффективной работы с CUDA и OpenCL.
