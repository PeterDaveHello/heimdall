## [Apple использует Cassandra и FoundationDB для создания iCloud для хранения миллиардов баз данных](https://read.engineerscodex.com/p/how-apple-built-icloud-to-store-billions)

- Apple разработала iCloud с использованием таких технологий, как Cassandra и FoundationDB, для хранения миллиардов баз данных и расширения пользовательского функционала за счет асинхронной обработки и архитектуры без статичности.

CloudKit, облачный сервис Apple, использует архитектуру без статических данных и абстракцию хранилища записей для легкой масштабируемости.
- FoundationDB помогает решать такие задачи, как персонализированный полнотекстовый поиск и высококонкурентные зоны, а Record Layer улучшает задержку и управляет конфликтами транзакций.

- В нем представлена концепция реляционной файловой системы и освещены проблемы, связанные с функцией синхронизации файлов в iCloud.
- В кратком обзоре затронуты различные аспекты вариантов хранения данных, экономии от масштаба при хранении данных и ограничений текущих реализаций систем хранения.

## [Представляем HtmlDocs: Создание красивых PDF-файлов с помощью HTML/CSS](https://htmldocs.com)

- HtmlDocs - это редактор, использующий HTML/CSS для создания хорошо отформатированных PDF-файлов, таких как счета-фактуры и резюме.
- Он имеет шаблонизатор и конвейер рендеринга Chromium для создания PDF-файлов.
- HtmlDocs отличается тем, что поддерживает современные функции CSS и минимизирует расхождения между HTML-документом и PDF.

### [Реакции](https://news.ycombinator.com/item?id=39027543)

- Пользователи выражают недовольство устаревшими и ограниченными инструментами, такими как wkhtmltopdf, для преобразования HTML/CSS-документов в PDF-файлы.
- В качестве альтернативы предлагаются такие варианты, как Typst, Paged.js и WeasyPrint.
- Ведутся споры о том, сможет ли CSS эффективно заменить LaTeX в верстке и научных публикациях.
