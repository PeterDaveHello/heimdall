## [No-GIL mód jön Pythonhoz](https://lwn.net/Articles/939568/)

- A Python Irányító Testület fontolóra veszi azt a javaslatot, hogy a globális értelmezőzár (GIL) opcionális legyen a CPythonban.
- Ez lehetővé tenné a Pythonban a GIL nélküli üzemmódot, ami potenciálisan javítaná az egyidejűséget.
- A tanács óvatosan jár el, hogy biztosítsa a visszafelé kompatibilitást és elkerülje a Python 2 és 3 kettéválását.
- Megvitatják az ABI-kompatibilitás követelményeit és a visszafelé kompatibilitásra gyakorolt hatásokat.
- Egyes kommentelők aggodalmukat fejezték ki a lehetséges kompatibilitási problémákkal kapcsolatban, és alternatív megközelítéseket javasolnak a Pythonban az egyidejűség fokozására.

### [Reakciók](https://news.ycombinator.com/item?id=36914401)

- A Python bevezet egy No-GIL (Global Interpreter Lock) üzemmódot, amit az lwn.net egyik bejegyzése is megerősített.
- A globális értelmezőzár egy olyan mechanizmus a Pythonban, amely lehetővé teszi, hogy egyszerre csak egy szál hajtsa végre a Python bytecode-ot.
- A No-GIL mód célja, hogy megszüntesse ezt a korlátozást, és lehetővé tegye, hogy több szál egyszerre több Python bájtkódot hajtson végre.
- A No-GIL móddal kapcsolatos információkat eredetileg a ycombinator.com oldalon osztották meg.
