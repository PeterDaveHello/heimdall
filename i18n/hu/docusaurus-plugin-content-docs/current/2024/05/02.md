## [Óvakodjon az AWS S3 költségerősítő fenyegetéstől](https://blog.limbus-medtec.com/the-aws-s3-denial-of-wallet-amplification-attack-bc5a97cc041d?gi=67ac052cc1ce)

- Az AWS S3 Denial of Wallet amplifikációs támadás kockázatot jelent az AWS S3-on nagy adatfájlokat tároló vállalatok számára, mivel az adatátviteli költségek számítási módszerei miatt váratlan költségváltozásokat okoz.
- Ez a támadás jelentős váratlan kiadásokat okozhat a vállalatoknak, függetlenül attól, hogy az adatok nyilvánosan hozzáférhetők-e, a kapott adatok és a kiszámlázott összegek közötti eltérésekből eredően.
- Az enyhítő intézkedések közé tartozik a költségriasztások végrehajtása, az API-kérések nyomon követése és az S3 vödrökhöz való hozzáférés ellenőrzése, hogy megakadályozzák a rosszindulatú szereplők általi potenciális pénzügyi kihasználást.

### [Reakciók](https://news.ycombinator.com/item?id=40227736)

- A felhasználók aggodalmukat fejezik ki az AWS S3 egy lehetséges hibájával kapcsolatban, amely magasabb kilépési költségeket okoz a tartományi kérelmeknél, és rámutatnak a számlázási gyakorlatokkal, az adatátviteli eltérésekkel és a váratlan díjakkal kapcsolatos aggodalmakra.
- A viták a költési korlátokról, a jogosulatlan díjakról, az API dokumentációról, az adatok gyorsítótárazásáról, valamint az AWS és a Cloudflare szolgáltatásainak összehasonlításáról szólnak, megvilágítva a jogosulatlan kéréseket kezelő rendszergazdák kihívásait.
- A beszélgetés hangsúlyozza a különböző felhőszolgáltatások, például az S3 Glacier és az R2 használatának előnyeit az AWS számlázási és kezelési problémái közepette.
