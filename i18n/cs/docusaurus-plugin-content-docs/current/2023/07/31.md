## [Co se děje, Pythone? GIL odstraněn, nový kompilátor, optparse deprecated](https://www.bitecode.dev/p/whats-up-python-the-gil-removed-a)

- V červenci 2023 bylo oznámeno několik aktualizací a změn pro Python.
- V jazyce Python byl odstraněn globální zámek překladače (GIL), což může zvýšit výkon a umožnit lepší využití více jader.
- Python má nyní nový kompilátor s názvem LPython.
- Byl vydán Pydantic 2 s vylepšenou rychlostí a stabilitou.
- Moduly getopt a optparse jsou již zastaralé.
- Cython 3.0 nyní lépe podporuje čistý Python.
- Byl předložen nový návrh PEP 722 pro specifikaci závislostí v jednosouborových skriptech.
- Podpora kódu VSCode v jazyce Python byla vylepšena pro vyšší výkon.
- Byla zavedena nová aplikace pro malování na terminálu s názvem textual-paint.

#### [Reakce](https://news.ycombinator.com/item?id=36935041)

- Diskuse se soustředí na optimalizaci kódu Pythonu pro vícevláknový výkon a možné odstranění globálního zámku překladače (GIL) v jazyce Python.
- Sdílí se různé pohledy, včetně návrhů na použití nástrojů jako `gunicorn` a `gevent` pro víceprocesorové a vícevláknové zpracování.
- Zvažují se alternativy, jako je použití jazyků jako C++ nebo Rust pro vyšší výkon.
- Jsou uvedeny návrhy na optimalizaci, například využití mezipaměti nebo sdílené paměti pomocí Redis nebo memcached.
- Objevují se obavy ohledně problémů, jako jsou deadlocky a využití zdrojů, a návrhy na alternativní jazyky nebo technologie.
- Jsou diskutovány výkonnostní nevýhody jazyka Python a možná řešení, jako je použití jiných jazyků nebo využití kompilátorů JIT.
- Zmíněna je kompatibilita různých programovacích jazyků, problémy přechodu a složitost návrhu a použití programovacích jazyků.
- O potenciálních výhodách a nevýhodách odstranění GIL v jazyce Python se vedou diskuse, přičemž názory na zlepšení výkonu a potenciální problémy se souběžností se různí.
