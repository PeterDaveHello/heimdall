# 2023-08-26

## [OpenTFがTerraformのフォークを発表](https://opentf.org/announcement)

- OpenTFはOpenTFイニシアチブによって作られたTerraformのフォークである。
- このイニシアチブは、Terraformのオープンソースを維持することを目的としており、コミュニティから大きな支持を得ている。
- OpenTFは、よく知られたライセンスの下で開発されており、コミュニティ主導、公平、階層化、モジュール化され、後方互換性がある。
- OpenTFは、オープンソースでベンダーニュートラルであり続けるために、Linux FoundationとCloud Native Computing Foundationの一部になることを計画している。
- プロジェクトは完了に近づいており、リポジトリ全体の名称変更、運営委員会メンバーの選出、コミュニティ文書の調整はすでに完了している。
- OpenTFのワーキングリリースは近い将来に予定されている。
- OpenTFはすでに複数の企業から支援を受けており、今後も成長が見込まれている。
- OpenTFは既存のTerraformプロバイダーやモジュールと互換性がある。
- そのライセンスモデルは、財団とコミュニティによって管理される。
- OpenTFへの貢献は、リソースの提供、情報の拡散、そして最終的にはコードの提供を通じて行うことができる。
- このプロジェクトは、地域社会の意見を聞くための公開ロードマップとRFCプロセスを確立することを目的としている。

### [反応](https://news.ycombinator.com/item?id=37262440)

- コミュニティ主導のイニシアチブであるOpenTFは、プロジェクトのライセンス変更に伴い、HashiCorpからTerraformプロジェクトをフォークした。
- OpenTFは、HashiCorpの決断の動機やオープンソースコミュニティへの影響について議論しながら、ポジティブな側面に焦点を当て、企業からの支持を得ることを目指しています。
- CNCFは、OpenTFのライセンスポリシーに例外を設ける必要があるかもしれず、異なるオープンソース財団の役割について議論が行われた。

## [私にはウェブスクレイピング、あなたにはウェブスクレイピングではない](https://blog.ericgoldman.org/archives/2023/08/web-scraping-for-me-but-not-for-thee-guest-blog-post.htm)

- メタやマイクロソフトのような企業は、自社のプロパティでのウェブスクレイピングを禁止している一方で、他社のプロパティからは自由にデータをスクレイピングしており、偽善的な姿勢を示している。
- ウェブスクレイピングは、インターネットからデータを取得することを含むが、誰がそのデータにアクセスし使用できるかという法的問題は複雑である。
- ソーシャルメディア企業は、データの財産権がなくても、動産への不法侵入や契約違反などの法理論を使って、ユーザーが作成したコンテンツを保護するために、ウェブスクレイピング訴訟を積極的に追求している。
- 著者は、オンライン利用規約によって企業が独自の知的財産権を作り上げることを許す法体系を批判している。
- 知的誠実さに対する法的要件の欠如や、財産としての契約違反の法体系における一貫性の欠如が浮き彫りになった。
- 著者は、このような偽善を可能にした裁判所を非難し、私企業がオンライン契約を通じて知的財産権を発明するのを防ぐための解決策を求めている。
- ウェブスクレイピングの法的状況は不安定であり、特にジェネレーティブAIのケースでは、将来的に困難に直面する可能性が高いと見られている。

### [反応](https://news.ycombinator.com/item?id=37264676)

- この記事では、HiQ対LinkedInのケースを例に、ウェブスクレイピングの法的意味合いと結果について論じている。
- サービス利用規約やエンドユーザーライセンス契約への違反、ウェブスクレイピングの広範な意味合い、公正な規則や規制の必要性などのトピックを探求している。
- この記事では、企業行動、法的明確性、保険適用と訴訟、著作権保護、プライバシーへの懸念、デジタル空間における開放性と管理性のバランスについても触れている。

## [EUによる行動ターゲティング広告との戦い](https://thisisunpacked.substack.com/p/the-eu-war-on-behavioral-advertising)

- メタ社（旧フェイスブック）は、EUにおけるアプリ内行動広告のオプトイン・デフォルトを廃止することに合意した。
- AppleのApp Tracking TransparencyやGoogleの段階的なサードパーティCookieの廃止など、プライバシーポリシーや法律の変化がこの決定に影響を与えています。
- プライバシー規制は、行動ターゲティング広告のリーチと効果に大きな影響を及ぼし、パーソナライズされていない製品バージョンの台頭や、プラットフォームによるEU市場の優先順位低下の可能性につながっている。しかし、EUの厳格な規制アプローチがソーシャルメディア・プラットフォームの収益化能力に及ぼす潜在的な影響については懸念が残る。

### [反応](https://news.ycombinator.com/item?id=37263322)

- 議論の中心は、行動ターゲティング広告を規制するEUのアプローチであり、複雑さやコスト、またDo Not Trackヘッダーの尊重の欠如に対する批判を探る。
- GDPRの実施と影響は重要な焦点であり、ユーザーのプライバシー保護の効果についてはさまざまな意見がある。
- この対談では、オンライン広告、プライバシー規則、ユーザーデータへのアクセスに関する同意、GDPRへの対応、ニュースへの支払いをめぐるフェイスブックとカナダ政府との論争などのトピックを掘り下げる。また、プライバシー保護運動の影響、広告戦略、アルゴリズム・フィード、パーソナライズされたデジタル広告、社会や経済に対する広告の価値と影響についても取り上げる。

## [流出した『ワイプアウト』のソースコードが、ほぼ全面的な書き換えとリマスターにつながる](https://arstechnica.com/gaming/2023/08/developer-rewrites-original-wipeout-from-abysmal-leaked-windows-source/)

- オリジナル版『ワイプアウト』のソースコードが流出し、独立系開発者がゲームを書き直してリマスターすることになった。
- 開発者のDominic Szablewskiは、レンダリング、物理演算、サウンド、その他の要素を書き直すことで、パフォーマンスとゲームプレイを改善した。
- Szablewski氏はコードをGitHubで公開し、複数のプラットフォーム向けにコンパイルできるようにしている。彼は、『ワイプアウト』フランチャイズのオーナーであるソニーが、リマスターの継続を許可するか、公式にリマスター版をリリースすることを望んでいる。

### [反応](https://news.ycombinator.com/item?id=37262258)

- 流出した『ワイプアウト』のソースコードは、ゲームの完全な書き換えとリマスターを促し、その結果、毎秒6,000フレームという驚異的なフレームレートを実現した。
- ゲームのレンダリングとフレームレートを中心に議論し、ゲームコードを扱うことの難しさを強調し、「悪い」コードと「良い」コードを区別する。
- コードのクリーンアップとリファクタリングの重要性、そしてソフトスキルと実践経験の価値が強調されている。批判はJavaScript/Nodeプログラミングにおけるnpmエコシステムに向けられている。
- ソースコードの流出や『ワイプアウト』のゲームプレイに関する会話にも触れている。

## [カレッジボードのウェブサイトがTikTokやFacebookでGPAやSATのスコアを共有](https://gizmodo.com/sat-college-board-tells-facebook-tiktok-your-scores-gpa-1850768077)

- SATやアドバンスト・プレースメント試験を運営するカレッジボードが、フェイスブックやTikTokのような大手テック企業と学生データを共有していることが発覚した。
- このデータ共有は、ターゲット広告に使用される目に見えないトラッキング技術である「ピクセル」を通じて行われます。
- カレッジボードは、個人を特定できる情報は共有されないと主張しているが、ギズモードが実施したテストでは、そうではないことが示され、プライバシーに関する懸念が高まっている。
- アメリカの教育制度におけるカレッジボードのサービスの義務的な性質が、こうしたプライバシーに関する懸念をさらに高めている。
- カレッジボードには、過去に学生データを売却するなど、プライバシーに関する問題があった。
- 同団体が学生のプライバシーに関する誓約書に署名していないため、現在も学生のプライバシーに配慮しているかどうかは不明である。
- カレッジボードは、学生の大学進学と成功に役立つとして、データ利用を擁護している。

### [反応](https://news.ycombinator.com/item?id=37260553)

- カレッジ・ボードは、トラッキング・ピクセルを使ってTikTokやFacebookとGPAやSATのスコアを共有しており、プライバシーに関する懸念を引き起こしている。
- 批評家たちは、カレッジボードの主張とは裏腹に、検索履歴を共有することさえプライバシーの侵害だと主張している。
- この事態はまた、裁判所の承認なしに政府が個人データにアクセスすることへの懸念を提起し、政府と民間企業双方にとってのプライバシー法の重要性を浮き彫りにしている。

## [電子メール認証：開発者ガイド](https://resend.com/blog/email-authentication-a-developers-guide)

- 電子メール認証は、電子メールを正常に配信し、電子メールがスパムフォルダに入るのを防ぐために非常に重要です。
- SPF、DKIM、DMARC、BIMIは、電子メール送信者の正当性を確立し、ドメインのレピュテーションを高める主要な認証プロトコルです。
- 再送信は、SPFとDKIM認証の自動処理を提供し、メールの配信性を向上させます。

### [反応](https://news.ycombinator.com/item?id=37263708)

- この記事では、電子メール認証の重要性と、正当な電子メール送信を検証するためのSPF、DKIM、DMARCの利用を強調している。
- オリジナリティに欠け、スタートアップの例えを使った記事だと批判する読者もいる。
- BIMIは、ブランドが電子メールにロゴと信頼性を表示する方法として言及されているが、そのコスト、効果、マイクロソフト・アウトルックのサポート不足に関して懸念が提起されている。BIMIを時間とお金の無駄と考える専門家もいる。
