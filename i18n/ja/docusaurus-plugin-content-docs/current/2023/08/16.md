# 2023-08-16

## [マストドンをツイッター・スケールで構築するコストを100分の1にした。](https://blog.redplanetlabs.com/2023/08/15/how-we-reduced-the-cost-of-building-twitter-at-twitter-scale-by-100x/)

- Red Planet Labsは、Ramaプラットフォームを使ってTwitterスケールのMastodonインスタンスを構築し、100倍のコスト削減を達成しました。
- ツイッターの100万行に比べ、このインスタンスはわずか1万行のコードで構築されている。
- Ramaはスケーラビリティ、パフォーマンス、フォールトトレランスを提供し、バックエンド開発を簡素化する。チームはRamaを一般向けにダウンロード公開し、Mastodonの実装をオープンソース化する予定だ。マストドンのPStateは、データを追跡し効率を向上させるために使用される。最適化技術について議論され、RamaのドキュメントとAPIのリリースについても言及されている。

### [反応](https://news.ycombinator.com/item?id=37137110)

- この記事では、スケーラブルなアプリケーション構築の簡素化とコスト削減を目指す新しいバックエンド・フレームワーク、Ramaに焦点を当てている。
- コメント欄では、Twitterとの比較について議論され、Ramaの拡張性に疑問が呈されている。
- ディスカッションでは、分散型プラットフォームの課題、ゲーム開発におけるエンティティ・コンポーネント・システムの使用の是非、セキュリティとモデレーションの重要性についても掘り下げている。

## [LLaMa.cppはなぜ可能なのか？](https://finbarr.ca/how-is-llama-cpp-possible/)

- 著者は、Raspberry Piのような限られたRAMしか持たないデバイスを含む、さまざまなハードウェア上で生のC++によるLLaMa推論コードを実行することの可能性を検証している。
- ディープラーニングにおけるメモリ帯域幅の重要性について、メモリ使用量を減らすために低精度のモデルを使用する利点とともに議論する。
- 様々なハードウェア上でLLaMaを実行するための計算と例が提供されており、変圧器からサンプリングする際のメモリ帯域幅の重要性に重点が置かれている。さらに、著者は、効率を向上させるために、小さなモデルに対する蒸留と拡張トレーニングの重要性を強調している。しかし、計算に誤りがある可能性を認め、フィードバックを奨励している。

### [反応](https://news.ycombinator.com/item?id=37140013)

- 議論の中心は、モデルの量子化、Llama言語モデルの能力、CPUベースの推論、メモリ帯域幅の制限、人間の性能に匹敵するAIの可能性、AGIのための言語モデルの限界と可能性、人間が作成したテキストを入手する際の課題などである。
- 参加者は、特定のプラットフォーム、方法、ハードウェア構成について説明する。
- AIモデルの実現可能性や将来の進歩については、さまざまな意見が表明されている。

## [ISP各社、すべての料金を記載するのは難しすぎると訴え、FCCに新ルールの廃止を要請](https://arstechnica.com/tech-policy/2023/08/isps-complain-that-listing-every-fee-is-too-hard-urge-fcc-to-scrap-new-rule/)

- 米国のブロードバンド業界は、FCCによるすべての月額料金の開示義務化に反対しており、消費者とプロバイダーの双方に複雑さと負担を強いることになると主張している。
- ロビー団体は、すべての料金を列挙する代わりに、政府が課す税金と同様に、料金に関する説明文を含めることを提案している。
- 消費者擁護団体は、この提案はISPがサービスの本当のコストを隠すことを可能にすると批判している。
- 料金設定や追加料金を含むFCCのブロードバンド表示規則は、現在、米行政管理予算局によって見直されている。

### [反応](https://news.ycombinator.com/item?id=37138681)

- このディスカッションでは、料金の透明性、公益事業としての規制、ティッピングカルチャー、水道水の利用可能性、料金の透明性、ISPが請求する料金など、ISPに関する複数のトピックが取り上げられている。
- また、ロビー活動、政府の腐敗、欺瞞的な価格設定、新規参入企業との競争、ISPの監視やデータ収集行為などに関する懸念も含まれている。
- 全体的なテーマは、ISP業界の透明性、消費者保護、規制の強化を求めるものである。

## [なぜLLMを主催するのか？](http://marble.onl/posts/why_host_your_own_llm.html)

- 著者は、OpenAIのGPT-4のような一般に利用可能なモデルに依存するのではなく、セルフホスティング言語モデル（LLM）を支持する論拠を提示している。
- セルフホスティングは、モデル・アーキテクチャとウェイトのコントロール、カスタマイズと微調整機能、AIモデルとの長期的な関係を提供します。
- セルフホスティングでは、より多くの実験、最適化、進化するテクノロジーへの対応が可能だが、APIでは同じレベルのアクセスや理解が得られない可能性がある。

### [反応](https://news.ycombinator.com/item?id=37133504)

- この記事では、OpenAIのAPIに依存するのではなく、言語モデルモデル（LLM）をセルフホストする理由と課題を探ります。
- Beam、パーソナル・ハードウェア、ジョージ・ホッツのタイニー・ボックスなど、LLMをホスティングするための代替オプションが議論されている。
- 費用対効果やプライバシーの問題を含め、LLMのセルフホスティングの可能性と利点について検討する。
