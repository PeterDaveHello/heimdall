### [MSFT が Outlook と Teams に Edge でリンクを開かせることを強制し、IT 管理者が怒っている](https://www.theverge.com/2023/5/3/23709297/microsoft-edge-force-outlook-teams-web-links-open)

- Microsoft は、Teams と Outlook の特定のリンクに Edge ブラウザを使用するよう強制する変更を行い、IT 管理者を怒らせている。
- この変更は、ユーザーがデフォルトのブラウザではなく、Edge ブラウザで自動的にリンクを開くことを意味します。
- IT 管理者は、この変更をユーザーに通知し、それに応じてポリシーを変更する必要があります。

### [Replit の新しい Code LLM：オープンソース、Codex より 77%小さく、1 週間でトレーニング](https://www.latent.space/p/reza-shabani#details)

- Replit は、OpenAI の Codex よりも効率的で、わずか 1 週間でトレーニングされた新しいコード LLM をオープンソース化しました。
- Replit の AI 責任者である Reza Shabani は、データプラットフォームである Ghostwriter と、新規採用やスケーリング法などのチームの今後の計画について話しています。
- 会話の中心は、AI ベースのコード補完ツールである Ghostwriter と、コードを補完するだけではない独自の能力、そして自動化がもたらす社会的影響と、AI が人を置き換えるのではなく、人を助けるようにすることの重要性についてです。

#### [業界の反応](http://news.ycombinator.com/item?id=35803435)

- Replit-Code モデルは、OpenAI の Codex や GitHub の Copilot と似ていますが、オープンソースであり、ユーザーの特定のコードベースでトレーニングできるため、特定のタスクのための AI モデルを改善するアプローチとしてより一般的になっています。
- OpenAI の Copilot では不完全なコードや関係のないコードが生成されることがありますが、オープンソース対応のため特定のライブラリに微調整する機能が提案されています。

### [WebGPU の話をしたい](https://cohost.org/mcc/post/1406157-i-want-to-talk-about-webgpu)

- WebGPU は、Web ブラウザやその他のソフトウェアで 2D および 3D グラフィックスを作成するための新しい低レベルグラフィックス API としてリリースされました。
- Vulkan や OpenGL に代わって、あらゆる種類のソフトウェアやプログラミング言語での描画の標準となる可能性を持っています。
- 初期化やパイプラインの構築においていくつかの制限や複雑さがありますが、WebGPU はより簡単なクロスプラットフォームの GPU プログラミングに期待されています。

- Web 開発者は、グラフィックス・プロセッシング・ユニットへのアクセス向上など、WebGPU を使用することで得られる潜在的なメリットについて議論する一方で、悪意のある活動への可能性について懸念も示しています。

### [パスキー：パスワードの終わりの始まり](https://blog.google/technology/safety-security/the-beginning-of-the-end-of-the-password/)

- Google は、パスワードよりも使いやすく安全な、アプリやウェブサイトへの新しいサインイン方法である passkeys の展開を開始しました。
- パスキーは、ユーザーがデバイスのロックを解除するのと同じ方法でサインインすることができ、指紋、顔スキャン、スクリーンロック PIN などのオプションがあります。
- パスワードとは異なり、パスキーはフィッシングなどのオンライン攻撃にも強いため、より安全です。

#### [業界の反応](http://news.ycombinator.com/item?id=35801392)

- パスキーは生体認証を利用して携帯電話のロックを解除し、ユーザーの端末に保存され、同じ Google アカウントにサインインしたすべての端末で同期されます。
- パスキーを使用するためのオープンソースやサードパーティーのオプションがありますが、プライベートキーをクラウドベースのサービスに保存することは、プライバシーに関する懸念があり、すべてのデバイスへのアクセスを失うことは、データへのアクセスを失うことにつながります。

### [OpenLLaMA: LLaMA のオープンな再生産](https://github.com/openlm-research/open_llama)

- OpenLLaMA は、Meta AI の言語モデル LLaMA を再現した、Berkeley AI Research が公開した新しいオープンソースの言語モデルです。
- 7B の OpenLLaMA モデルのパブリックプレビューは、2000 億トークンで学習され、Huggingface Hub で PyTorch と Jax の重みで利用できるようになりました。
- OpenLLaMA は様々なタスクで評価され、オリジナルの LLaMA や GPT-J モデルと同等の性能を示しています。

#### [業界の反応](http://news.ycombinator.com/item?id=35798888)

- 一部のユーザーは、llama.cpp と 8GB RAM で使用した場合、Intel ベースの MacOS システムでエラーが発生しました。
- モデルとデータセットは Apache License 2.0 の下で完全に複製可能であり、研究の透明性を提供し、大規模な技術企業による独占を防止しています。
