- これらのツールは、直接的なDOM管理の多くをラップし、抽象化しているにもかかわらず、ウェブ開発ではまだ一定レベルのDOMとの相互作用があることが指摘されている。

## [開発者がアップルのフィードバック・アシスタントに抗議：バグ報告システムの改善を求める](https://lapcatsoftware.com/articles/2023/11/2.html)

- ジェフ・ジョンソンは、アップルのフィードバック・アシスタントに対するボイコットを開始し、開発者に対し、既存の問題についての新たなフィードバックを提出するよう促しているが、改善がなされるまでは現在の問題には対応しないとしている。
- アップル社の対応不足、フィードバックの不規則な閉鎖、開発者からのバグ報告の無視、詳細なシステム診断レポートの要求、バグ診断のための小規模な方法の軽視、バグの検索機能の欠如、すべてのフィードバックをmacOSまたはiOS経由で提出することの義務付けなど、重大な問題が浮き彫りになった。
- ボイコットは労働ストライキに例えられ、アップルのソフトウェアや製品のテストや改良における、開発者たちの認識されていない無報酬の貢献を強調している。アップルのエンジニア個人を批判するのではなく、同社のリーダーシップに異議を唱えるものである。

### [反応](https://news.ycombinator.com/item?id=38164735)

- ジェフ・ジョンソンは、アップルの無反応、バグ報告の却下、バグの検索不能など、いくつかの重大な問題が改善されるまで、アップルのフィードバック・アシスタントをボイコットするよう、アップルの開発者を結集している。
- このボイコットは、開発者に対し、システムの問題点に関する新たなフィードバックを提出することを奨励するが、既存のフィードバックへの返信は控えるというもので、開発者がアップル製品のテストや改良において、重要ではあるが無報酬で貢献していることを強調するものである。
- ジョンソン氏は、このボイコットはアップルのエンジニアをターゲットにしたものではなく、むしろアップルの首脳部にバグ報告システムを認めさせ、強化することを目的としていると明らかにした。

## [Git RebaseとGit Pushコマンドのリスクと正しい使い方を理解する](https://jvns.ca/blog/2023/11/06/rebasing-what-can-go-wrong-/)

- 著者は、集団ブランチで「git rebase」や「git push -force」コマンドを使用する際に注意すべき潜在的な問題について詳しく述べています。たとえば、衝突の繰り返し解決やコミットメタデータの損失、リベースを取り消す際の課題などがあります。
- 彼らは、共有ブランチでのリベースの慎重な使用を提案し、コミットの規律をきちんと守ることを強調している。
- 彼らは代替案として「スクワッシュ＆マージ」ワークフローを提案し、共有ブランチへの強制プッシュやプッシュされたブランチのリベースは推奨しないが、それでもリベースの有用性は認めている。

### [反応](https://news.ycombinator.com/item?id=38164046)

- 著者は、共有ブランチにおける git rebase や git push --force コマンドの潜在的な問題について、コンフリクトの繰り返し、コミットメタデータの消失、リベースの取り消しや膨大な数のコミットのリベースの困難さなどを議論しています。
- 協調ブランチでrebaseを使用する際には、きれいなコミット規律を維持することの重要性を強調し、自制することを推奨している。
- この記事では、代替案として "squash and merge" ワークフローを検討することを提案しています。著者は、リベースの有用性は認めているものの、共有ブランチへの強制プッシュやプッシュしたブランチのリベースを推奨していません。
