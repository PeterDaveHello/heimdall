# 2024-07-07

## [埋め込みYouTubeは非常に重く、修正可能です](https://frontendmasters.com/blog/youtube-embeds-are-bananas-heavy-and-its-fixable/)

- YouTubeの埋め込みはリソースを多く消費し、各埋め込みで約1.3MBと32のリクエストが発生し、読み込み時間が遅くなり、データ転送量が多くなります。
- Paul Irishによる「lite-youtube-embed」のようなWebコンポーネントは、リソースを共有し、パフォーマンスを大幅に向上させながら、サイズを約100kに削減することができます。
- このソリューションを実装するには、JavaScriptをリンクし、HTMLでコンポーネントを使用し、パフォーマンス向上のためにスクリプトを非同期で読み込む必要があります。

### [反応](https://news.ycombinator.com/item?id=40897205)

- コミュニティフォーラムプラットフォームは、YouTubeの埋め込みをプロキシサムネイルに置き換えました。これにより、クリックされたときにのみ読み込まれ、大きなJavaScriptファイルをダウンロードする必要がなくなり、Googleの追跡を防ぐことができます。
- 同様のソリューションが、機能を強化し、冗長性を最小限に抑えるために、YouTube、SoundCloud、およびRedditの埋め込みに対して実装されています。
- 軽量な埋め込みがビデオの開始時間を遅くすることでエンゲージメントを低下させるかどうかについて議論がありますが、多くの人は全体的なページの読み込み時間を改善することがユーザー体験とサイトのパフォーマンスにとってより重要であると考えています。

## [マロックがセレニティのJPGローダーを壊した、または：宝くじに当たる方法(2021年)](https://sin-ack.github.io/posts/jpg-loader-bork/)

- SerenityOSのJPG画像デコードにおけるバグにより、RGBとBGRの混同が原因で色の表示が不正確になりました。
- 問題は `malloc_good_size()` を実装するコミットに遡り、これがメモリ割り当てを変更し、`JPGLoader` で使用される `HashTable` や `Vector` などのデータ構造に影響を与えました。
- 修正には、`JPGLoader`内のコンポーネントに対する決定論的な反復を確保することが含まれており、簡単な変更が重大な根本的問題を明らかにする方法を強調しています。

### [反応](https://news.ycombinator.com/item?id=40896102)

- いくつかのハッシュテーブル実装は、要素の順序に依存することを防ぎ、セキュリティを強化するためにランダム性を導入しますが、現代の実装では挿入順序での反復を保証することがよくあります。
- JavaScriptとPythonは、ハッシュテーブルの実装において挿入順序を標準化しており、Pythonはバージョン3.7でこれを採用しました。
- Rustは異なるハッシュテーブルオプション(`HashMap`、`BTreeMap`、`IndexMap`)を提供しており、Perlはバージョン5.18以降、ハッシュテーブルにランダムシードを導入しました。

## [BeaconDB – Mozillaロケーションサービスの代替](https://beacondb.net/)

- beaconDBは、30か国にわたる177,176のネットワーク、4,126のビーコン、6,106のタワーをカバーするパブリックドメインの無線ジオロケーションデータベースであり、倫理的に収集されたオプトインデータを使用しています。
- このデータベースはプライバシーに配慮しており、送信者や貢献者の情報を隠蔽して保護し、データを更新するには物理的な近接性が必要なため、悪用に対して耐性があります。
- 開発者は、Ichnaeaのリクエスト形式と互換性のあるエンドポイント `https://beacondb.net/v1/geolocate` を使用できますが、高トラフィックのアプリケーションには適していません。ソースコードはCodebergで入手可能です。

### [反応](https://news.ycombinator.com/item?id=40895672)

- BeaconDBは、倫理的に調達されたオプトインデータ収集に焦点を当てた、Mozilla Location Services(MLS)の代替として提示されています。
- AppleやGoogleとは異なり、BeaconDBは平文のMACアドレスを公開せず、プライバシーを強化するためにハッシュ化されたデータや集約されたデータを使用しています。
- BeaconDBは、位置情報サービスのための唯一の非ビッグテックデータベースであり、オープンソースのソリューションを提供し、潜在的なコラボレーションについて議論することを目指しています。

## [帯域外ネットワーク管理は簡単ではありません](https://utcc.utoronto.ca/~cks/space/blog/sysadmin/OutOfBandManagementNotTrivial)

- 2022年のロジャースのインターネットおよび電話の障害は、1,200万人以上のカナダ人に影響を与え、「アウト・オブ・バンド」(OOB)ネットワーク管理の重要性と複雑さを浮き彫りにしました。
- アウトオブバンド管理には、メインネットワークとは別の専用ハードウェアとインフラが必要であり、冗長性とセキュリティを確保する一方で、かなりのコストと複雑さが追加されます。
- 小規模なネットワークは、別のOOBネットワークを維持するよりも、堅牢で冗長なインバンド管理に焦点を当てる方が実用的かもしれません。

### [反応](https://news.ycombinator.com/item?id=40895167)

- 帯域外(OOB)ネットワーク管理は、特に誤設定や障害が発生しやすい現代のデジタルシステムにおいて、ネットワークの信頼性を維持するために不可欠です。
- ロジャースの障害は、堅牢なOOB管理の重要性を浮き彫りにし、コアネットワークの障害を防ぐためのより良い計画とリスク評価の重要性を強調しました。
- 効果的なOOB管理には、信頼性を確保するために、専用の冗長システムを使用し、しばしば異なるプロバイダーやLTEや衛星などの技術を活用することが含まれます。

- 2024年7月4日にScience誌に発表されたマウスの研究は、脳の一時的な『ブラックアウト』が頭蓋内の痛み受容体を活性化させる脳脊髄液の変化と関連していることを明らかにし、片頭痛の原因に関する新たな洞察を提供しています。
- 研究によると、片頭痛は休息の必要性を示す保護機構として機能する可能性があり、三叉神経節周囲の保護層にこれまで知られていなかった隙間があることを特定しました。
- この発見は、新たに特定されたメカニズムを標的とすることによって、片頭痛の新しい治療法の開発につながる可能性があります。

### [反応](https://news.ycombinator.com/item?id=40893749)

- 研究によると、皮質拡延性抑制(CSD)という「脳のブラックアウト」が片頭痛に関与している可能性があることが示唆されています。
- 人々は、食事の変更、リザトリプタンのような薬、改善された睡眠や水分補給などのライフスタイルの調整を含むさまざまな引き金と治療法を報告しました。
- この議論は、片頭痛の複雑さを強調しており、異なる引き金や治療法が個々の人々に対して効果的であることを示しています。
