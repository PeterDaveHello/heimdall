I studien observerades tre betydande försämringsmönster och man drar slutsatsen att kontinuerlig övervakning är nödvändig för att mildra problemet. I en annan akademisk artikel föreslås att modellen ska omtränas med nya märkta data för att lösa problemet, men man betonar behovet av en effektiv och robust mekanism för automatisk omträning och övervakningsverktyg som NannyML för att förhindra fel.

[Discussion Service](http://news.ycombinator.com/item?id=35566145).

Även om användarna ger bevis som stöder båda synsätten, att AI-modeller genererar nya koncept eller är begränsade till befintliga data, betonas den konservativa bias som finns i nuvarande träningsdata och avsaknaden av en tydlig förståelse av det transformerade utrymmet. ## Insprutning: Vad är det värsta som kan hända?

[Original](https://simonwillison.net/2023/Apr/14/worst-that-can-happen/).
Teknikexperter varnar för de allvarliga konsekvenserna av prompt injektion i LLM, som kan leda till sårbarheter i vissa tillämpningar som ChatGPT Plugins och ReAct-mönster, bland annat, utan att det finns någon lösning som garanterar ett 100-procentigt skydd. Trots utbildning av utvecklare är prompt injektion fortfarande olöst, och dataexfiltrering och SEO-optimeringshackningsrisk kan skada LLM:s utveckling. OpenAI:s GPT-4 erbjuder en "systemprompt"-funktion för att förhindra prompt injektion, men de senaste framstegen inom LLM kan öka sårbarhetsrisken.

[Discussion Service](http://news.ycombinator.com/item?id=35572290).

Begränsningarna med naturligt språk som ett sätt att kodifiera gränser för språkmodeller utforskas också, och vissa föreslår årlig säkerhetsutbildning och ACL-medvetna modeller för att lösa problemet. Sårbarheten hos LLM:er för injektionsattacker med snabbkommandon uppmärksammas av teknikexperter, som oroar sig för att dåliga aktörer kan utnyttja LLM:er som är dåligt säkrade. De senaste framstegen inom modeller för maskininlärning har skapat nya sårbarhetspunkter, inklusive attacker genom injektion, som forskarna uppmanar användarna att betrakta som oundvikliga. Svårigheterna med promptinjektionsattacker mot språkmodeller och potentiella lösningar, inklusive införande av en mänsklig prompt eller separering av användardata och systemdata, diskuteras också.
