## [Amazon S3 slutar ta betalt för utvalda HTTP-felkoder](https://aws.amazon.com/about-aws/whats-new/2024/05/amazon-s3-no-charge-http-error-codes/)

## [Fråga datasjöar utan ansträngning med Pg_lakehouse](https://github.com/paradedb/paradedb/tree/dev/pg_lakehouse)

- pg_lakehouse är ett Postgres-tillägg som underlättar analytiska frågor om objektbutiker som S3 och tabellformat som Delta Lake, med stöd för olika objektbutiker, tabellformat och filformat, kompatibelt med Postgres version 14, 15 och 16.
- Syftet med tillägget är att göra det möjligt att ställa frågor om icke-operativ data direkt från Postgres utan ytterligare infrastrukturkrav, med hjälp av FDW API (Foreign Data Wrapper) för att ansluta till objektbutiker och tabellformat, samtidigt som Apache DataFusion utnyttjas för optimal frågeprestanda.
- Användare kan börja med att skapa utländska tabeller, aktivera frågeacceleration genom Postgres-krokar och följa installations- och utvecklingsriktlinjer, inklusive anslutning till anpassade objektbutiker, hantering av specifika datatyper och genomförande av tester med hjälp av testcontainers med en live Postgres-installation.

### [Reaktioner](https://news.ycombinator.com/item?id=40343131)

- Inlägget introducerar Pg_lakehouse och Steampipe för att fråga datasjöar via Postgres, fördjupa sig i utmaningar, "lakehouse"-termens ursprung och debatter om Iceberg- och Delta Lake-format i branschen.
- Pg_lakehouse får beröm för sin prestanda och sina funktioner, med jämförelser med SpiceAI och Hydra, medan Iceberg föredras framför Delta Lake för sin öppna källkod och snabba utveckling.
- Diskussionen omfattar ett nytt Postgres-plugin, pgrx, som belyser potentiella förbättringar och debatter om dess AGPL-3.0-licensimplikationer.
