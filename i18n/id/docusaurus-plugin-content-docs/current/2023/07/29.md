## [Cap'n Proto 1.0](https://capnproto.org/news/2023-07-28-capnproto-1.0.html)

- Cap'n Proto, sebuah protokol pengiriman pesan, telah merilis versi 1.0 dari implementasi referensi C++.
- Protokol ini sudah digunakan dalam produksi selama bertahun-tahun, yang mengindikasikan keandalannya.
- Rilis ini menyertakan sejumlah optimasi untuk meningkatkan performa.
- Perubahan telah dilakukan untuk memungkinkan pembatalan RPC.
- Rencana untuk versi 2.0 mencakup revisi pada API C++ dan pustaka toolkit KJ C++ untuk meningkatkan pengalaman pengembang.
- Perubahan ini akan membutuhkan kompiler dengan dukungan C++20 dan dapat membuat KJ lebih ramah terhadap coroutine.
- Perubahan yang tidak kompatibel ke belakang tidak akan dilakukan pada format serialisasi atau protokol RPC.
- Rilis resmi 2.0 diperkirakan baru akan dirilis dalam beberapa tahun ke depan.

#### [Reaksi](https://news.ycombinator.com/item?id=36908309)

- Ian Denhardt, kontributor utama dalam implementasi Go pada Cap'n Proto, telah meninggal dunia, meninggalkan komunitas Cap'n Proto dan Sandstorm dalam keadaan berkabung.
- Cap'n Proto adalah format serialisasi data berkinerja tinggi yang digunakan dalam platform Workers Cloudflare.
- Manfaat Cap'n Proto, termasuk serialisasi dan deserialisasi yang efisien, dibahas.
- Diskusi ini juga mencakup dukungan bahasa untuk Cap'n Proto, dengan contoh implementasi dalam berbagai bahasa.
- Potensi untuk pengembangan dan peningkatan lebih lanjut dari Cap'n Proto dieksplorasi.
- Percakapan tersebut menyentuh topik-topik seperti bidang yang wajib diisi dan bidang opsional dalam pengembangan perangkat lunak, pengetikan statis, dan tantangan desain protokol.
- Format serialisasi lain seperti Avro, protobuf, dan flatbuffer dibandingkan dengan Cap'n Proto.
- Saran untuk pustaka dan protokol serialisasi biner lainnya disebutkan di samping Cap'n Proto.

## [Mode Tanpa GIL hadir untuk Python](https://lwn.net/Articles/939568/)

- Dewan Pengarah Python sedang mempertimbangkan sebuah proposal untuk membuat Global Interpreter Lock (GIL) menjadi opsional di CPython.
- Ini akan memungkinkan mode tanpa GIL di Python, yang berpotensi meningkatkan konkurensi.
- Dewan berhati-hati untuk memastikan kompatibilitas ke belakang dan menghindari skenario pemisahan Python 2 dan 3.
- Mereka mendiskusikan persyaratan untuk kompatibilitas ABI dan mengatasi dampaknya pada kompatibilitas ke belakang.
- Beberapa komentator telah menyatakan keprihatinannya tentang potensi masalah kompatibilitas dan merekomendasikan pendekatan alternatif untuk meningkatkan konkurensi dalam Python.

#### [Reaksi](https://news.ycombinator.com/item?id=36914401)

- Python memperkenalkan mode No-GIL (Global Interpreter Lock), seperti yang dikonfirmasi oleh sebuah posting di lwn.net.
- Global Interpreter Lock adalah sebuah mekanisme dalam Python yang hanya mengizinkan satu thread untuk mengeksekusi bytecode Python dalam satu waktu.
- Mode No-GIL bertujuan untuk menghilangkan batasan ini dan memungkinkan beberapa thread untuk mengeksekusi bytecode Python secara bersamaan.
- Informasi mengenai mode No-GIL pada awalnya dibagikan di ycombinator.com.
