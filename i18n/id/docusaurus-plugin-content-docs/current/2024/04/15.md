## [Memperkenalkan Redka: Redis Ditata Ulang dengan SQLite](https://github.com/nalgeon/redka)

- Redka adalah proyek yang menata ulang Redis dengan memanfaatkan SQLite, menyediakan fitur-fitur seperti penyimpanan data yang tahan lama, transaksi ACID, tampilan SQL, dan kompatibilitas dengan Redis API.
- Mendukung tipe data dan perintah utama Redis, dengan fokus pada perintah yang berhubungan dengan string, menawarkan opsi server mandiri dan server dalam proses dengan Go API.
- Meskipun 2-6 kali lebih lambat daripada Redis, Redka dapat mengelola operasi baca dan tulis yang signifikan, menjanjikan potensi peningkatan di masa depan dan kontribusi terbuka di bawah lisensi BSD-3-Clause.

### [Reaksi](https://news.ycombinator.com/item?id=40030746)

- Diskusi GitHub berkisar pada Redka, sebuah proyek yang bertujuan untuk menggabungkan Redis dan SQLite untuk API yang mudah digunakan, menawarkan manfaat seperti jejak memori yang kecil, transaksi ACID, dan antarmuka SQL.
- Para pengguna memuji kesederhanaan Redka tetapi mengungkapkan kekhawatiran mereka tentang fitur yang hilang seperti dukungan skrip Lua jika dibandingkan dengan Redis.
- Alternatif seperti Garnet, Postgres, SSDB, dan KVrocks dibahas untuk caching, sharding, dan kinerja yang efisien, dengan saran TiKV atau FoundationDB untuk beban kerja produksi melalui SQLite.
