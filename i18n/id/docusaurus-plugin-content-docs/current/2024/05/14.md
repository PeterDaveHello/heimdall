## [Kueri Data Danau dengan Mudah dengan Pg_lakehouse](https://github.com/paradedb/paradedb/tree/dev/pg_lakehouse)

- pg_lakehouse adalah ekstensi Postgres yang memfasilitasi kueri analitik pada penyimpanan objek seperti S3 dan format tabel seperti Delta Lake, mendukung berbagai penyimpanan objek, format tabel, dan format file, yang kompatibel dengan Postgres versi 14, 15, dan 16.
- Tujuan ekstensi ini adalah untuk memungkinkan kueri data non-operasional secara langsung dari Postgres tanpa persyaratan infrastruktur tambahan, memanfaatkan API foreign data wrapper (FDW) untuk terhubung ke penyimpanan objek dan format tabel, sambil memanfaatkan Apache DataFusion untuk kinerja kueri yang optimal.
- Pengguna dapat memulai dengan membuat tabel asing, mengaktifkan akselerasi kueri melalui hook Postgres, dan mengikuti panduan instalasi dan pengembangan, termasuk menghubungkan ke penyimpanan objek khusus, mengelola tipe data tertentu, dan melakukan pengujian menggunakan testcontainers dengan instalasi Postgres langsung.

### [Reaksi](https://news.ycombinator.com/item?id=40343131)

- Postingan ini memperkenalkan Pg_lakehouse dan Steampipe untuk melakukan kueri data danau melalui Postgres, menggali tantangan, asal-usul istilah "lakehouse", dan perdebatan mengenai format Iceberg dan Delta Lake di industri ini.
- Pg_lakehouse dipuji karena kinerja dan fitur-fiturnya, dengan perbandingan dengan SpiceAI dan Hydra, sementara lebih menyukai Iceberg daripada Delta Lake karena sifat sumber terbukanya dan perkembangannya yang cepat.
- Diskusi ini mencakup plugin Postgres baru, pgrx, yang menyoroti potensi peningkatan dan perdebatan tentang implikasi lisensi AGPL-3.0.
