### GitHub izveide ar Ruby on Rails

[Oriģināls](https://github.blog/2023-04-06-building-github-with-ruby-and-rails/).
GitHub.com ir uzrakstīts, izmantojot Ruby on Rails, un to uztur vairāk nekā tūkstotis inženieru ar gandrīz 2 miljoniem kodu rindu. Uzņēmums ik nedēļu atjaunina jaunāko Rails versiju, izmantojot pull pieprasījumus un pārskatīšanas pārskatus. Šo atjauninājumu priekšrocības ir labāka datubāzes savienojumu apstrāde un ātrāka skatu atveidošana, uzlabota drošība un mazāks nezināmu problēmu risks. Jaunāko versiju testēšana identificē un novērš problēmas pirms izlaišanas. Ieraksts ir GitHub konfidencialitātes paziņojums, kas satur informāciju par produkta funkcijām, drošību, stāstiem, cenām, resursiem, izstrādātāju API, partneriem, platformu un atbalstu. Šķiet, ka šajā postenī nav nekādas jaunas informācijas vai notikumu, un nav skaidrs, kāpēc tas izpelnās uzmanību.

[HN](http://news.ycombinator.com/item?id=35478884).
GitHub pārbūvēja savu platformu, izmantojot Ruby on Rails, izraisot interesi par tās potenciālo ietekmi uz Rails ekosistēmu un testēšanas pārklājuma prasībām. Lietotāji uzsver Ruby apgūšanas izaicinājumus, piemēram, ekstrēmu testu pārklājumu & iespējamās koda kļūmes nākotnē. Daži lietotāji iesaka izmantot Sorbet spēcīgākiem pārbaužu veidiem. Tiek slavēta GitHub izmantotā Crystal valoda un Ruby metaprogrammēšanas potenciāls, savukārt daži kritizē Ruby datu zinātnes telpu. Nesenais Microsoft jauninājums Github Rails platformā no 2.3 uz 3 versiju radīja problēmas, bet statiskā tipizēšana palīdz jauninājumiem. Izstrādātāji reflektē par kompromisiem starp tehnoloģiju tendencēm un iedibināto stabilitāti, daži atzīmē Rails lietošanas ērtumu, bet atpalikušo skata slāni. Visefektīvāk ir sistēmu atjaunināt pakāpeniski, veicot labu testēšanu. Daudzas organizācijas uzskata, ka IT infrastruktūras atjaunināšana ir svarīga, tiek slavētas GitHub testēšanas iespējas, un GitHub izmanto Sorbet Ruby un Rails.

Uzņēmumu lobēšana veicina pašreizējās pašapkalpošanās nodokļu sistēmas sarežģītību. Nav iespējams automatizēt pārdošanas nodokļa atskaitījumus par skaidras naudas darījumiem. [HN](http://news.ycombinator.com/item?id=35483823).
ChatGPT var maldināt lietotājus, un ir atkārtoti jānorāda, ka tam nedrīkst uzticēties. Tās lietderība var būt ierobežota, ja netiek atrisināta halucināciju tendence. Mākslīgā intelekta ģenerētajam tekstam nepieciešams skeptisks skatījums, un uz to vajadzētu raudzīties kā uz asistentu, kas ir labi lasošs, bet potenciāli pakļauts konfabulācijai. Tomēr ChatGPT ir noderīgs, lai ģenerētu kodu piemērus un atbildētu uz sarežģītiem pieprasījumiem. Tas piedāvā precīzas atbildes līdz pat 99% gadījumu. Lietotājiem ir jāsaprot tās ierobežojumi, lai izvairītos no tā, ka ChatGPT tiek piešķirts vairāk morāles nekā nepieciešams.
