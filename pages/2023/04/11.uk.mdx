Користувач HN ділиться своїми знаннями про формальні методи, а коментатори діляться своїм досвідом і думками. Рекомендується використовувати Frama-C і SPARK, а також TLA+ для швидкого старту. Формальні методи корисні для перевірки апаратного забезпечення, але є нішевими для більшості програмного забезпечення. Контрактні тести та аналіз потоку даних допомагають запобігти помилкам на старті. Системи штучного інтелекту, такі як ChatGPT, можуть перевіряти код на відповідність системним вимогам. Математичні нотації є більш ефективними, ніж вирази на природній мові у формальних доказах. Демократизація знань є життєво важливою в технологічній галузі.

### Запитайте HN: Магістерська програма з відкритим вихідним кодом для комерційного використання?

Користувач HN запитав, чи може він легально використовувати інструмент LLM з відкритим кодом для комерційного використання. Деякі користувачі рекомендували перевірити конкретну ліцензію інструменту. Інші запропонували звернутися до юриста або використовувати альтернативний інструмент з комерційною ліцензією. Один користувач зазначив, що використання відкритого коду в комерційних цілях можливе з деякими ліцензіями, але важливо дотримуватися умов ліцензії.

[HN](http://news.ycombinator.com/item?id=35512338).
Користувач HN шукає комерційні ШНМ з відкритим кодом для свого проекту ML. Фланцеві моделі працюють трохи гірше, ніж GPT-3/4. Інші моделі з відкритим кодом доступні, але навчання коштує дорого. Nvidia планує випустити початкові базові моделі для LLM. Компанії можуть публікувати базові моделі, але реальна цінність знаходиться на 5-10 шарів вище. OpenAI не використовує дані клієнтів для навчання моделей, але деяким компаніям потрібна секретність. Моделі машинного навчання, ймовірно, не мають захисту прав інтелектуальної власності. Альтернативи GPT, такі як BERT та opt series, можуть бути дешево налаштовані. Команда GPT4all працює над навчанням GPT-J як відкритою альтернативою. Існує занепокоєння щодо підзвітності LLM.

### Масштабовані, анімовані діаграми розсіювання в браузері з масштабом понад мільярд точок

[Оригінал](https://github.com/nomic-ai/deepscatter).
Deep Scatterplots for the Web - це бібліотека для відображення великих обсягів даних з високою продуктивністю за допомогою формату "пера" Apache Arrow і WebGL, що розвивається. Бібліотека також обробляє більшість граматико-графічних перетворень на графічному процесорі, дозволяючи виконувати інтерпольовані переходи з паралельними обчисленнями. Приклади включають понад мільйон документів з arxiv.com та кожну людину в переписах населення США 2010 та 2020 років, серед іншого. Бібліотека працює в повністю статичних налаштуваннях, які можна розмістити на Github Pages. API імітує Vega-Lite, і бібліотека пропонує такі методи джиттера, як коло, спіраль і час.

[HN](http://news.ycombinator.com/item?id=35511139).
Анімовані діаграми розсіювання Nomic AI зі змінним масштабом масштабують понад мільярд точок у браузері за допомогою webGPU, і проблеми з GitHub вітаються; однак фонова карта не завжди може бути точною, а Highcharts має подібний модуль, але з меншою кількістю точок. Nomic AI також випустив gpt4all, а такі бібліотеки, як GRAPE і nodevectors, можуть дати кращі результати для вбудовування в масштабі. Шанувальників візуалізації можуть зацікавити Galapix, Pix-image-viewer та GGVec, але на мобільних пристроях можуть виникнути проблеми з рендерингом.
