**알렉사, 내 와이파이 비밀번호가 뭐야?**
Amazon 에코 디바이스는 WiFi 비밀번호를 일반 텍스트로 저장하므로 물리적 액세스 권한을 가진 공격자가 네트워크 비밀번호를 비교적 쉽게 알아낼 수 있습니다. 이는 내부 또는 사설 무선 네트워크에서 장치를 사용하는 호텔이나 기업에 중대한 보안 위험을 초래하며, 공격자가 네트워크의 다른 모든 장비에 액세스하여 잠재적으로 데이터 보호법을 위반할 수 있습니다. 비밀번호 해싱은 업계 표준이며, 수백만 대의 하드웨어를 판매한 회사에서 이 데이터를 암호화하지 않는 것은 용납할 수 없는 일입니다. 이 디바이스는 Spotify 제로 컨프 및 eSDK API에 대한 API 키를 암호화하지 않습니다. 임베디드 하드웨어를 설계할 때 물리적 보안은 필수적이며, 아마존은 향후 보안 침해를 방지하기 위해 이 문제를 해결해야 합니다.

**Google 드라이브가 파일 제한을 기습적으로 도입하여 일부 사용자를 차단했습니다.**

Total Order는 시간 및 인과 관계에 기반한 순차적 순서가 필요합니다. 논리적 시계는 공동 편집 애플리케이션에서 매우 중요합니다. Azure 시간 동기화는 PTP를 사용하는 반면, AWS와 Google Cloud는 NTP를 사용합니다. 이 기사는 시간을 절대적인 것으로 간주하려는 시도의 한계를 지적하고 은행 거래에서 이벤트 순서의 중요성을 강조합니다.

### 'ccurl.sh'를 통해 간결한 방식으로 크롬 쿠키를 활용할 수 있는 cURL

**HN 표시: 복사 붙여넣기 없이 cURL에서 Chrome(CDP)의 쿠키를 사용합니다.**
'ccurl.sh'는 Postman과 같은 GUI 도구 없이도 cURL이 Chrome 쿠키를 사용할 수 있게 해주는 bash 스크립트로, Chrome DevTools 프로토콜을 사용하여 로컬 Chrome 인스턴스의 특정 탭에서 cURL 명령의 헤더로 쿠키를 덤프하고 셸 기록 파일로 쿠키가 유출되는 것을 방지합니다; 스크립트 실행을 위해서는 bash, websocat, jq가 필요하며, 크롬에서 쿠키를 사용할 탭을 열려면 "google-chrome-stable --remote-debugging-port=9222"로 실행해야 하며, 스크립트를 "/usr/bin/ccurl"에 복사하여 실행 가능한 상태로 만들어 설치하면 됩니다.
[HN](http://news.ycombinator.com/item?id=35399218), [기사](https://github.com/fipso/ccurl.sh).
