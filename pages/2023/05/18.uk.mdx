### [Цифри, які повинен знати кожен LLM-розробник](https://github.com/ray-project/llm-numbers)

- Було складено набір важливих цифр для LLM-розробників (великі мовні моделі), подібний до "Числа, які повинен знати кожен інженер".
- Одне з важливих чисел - це середня кількість токенів на слово для LLM, що має вирішальне значення для виставлення рахунків, оскільки більшість рахунків виставляється в токенах, і розмір контекстного вікна LLM визначається в токенах.
- Іншим важливим аспектом є співвідношення вартості різних моделей, що використовуються для генерації, яке може заощадити значну суму грошей під час експлуатації LLM.

#### [Реакція галузі](http://news.ycombinator.com/item?id=35978864)

- Обговорюються компроміси між різними рівнями точності та методами квантування для LLM, а також методи зменшення погіршення продуктивності при квантуванні нижче 8 біт.
- Пропозиції включають використання двійкових нейронних мереж для трансформаторів і навчання LLM з 4-бітним форматом для кращої продуктивності. Також згадується використання SQLFlow як інструменту для навчання мовних моделей.
