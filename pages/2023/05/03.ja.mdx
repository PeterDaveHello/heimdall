### [映画スタジオ、海賊版に関するコメントを書いた Reddit ユーザーのマスクを剥がす入札に敗れる](https://arstechnica.com/tech-policy/2023/05/judge-wont-force-reddit-to-identify-anonymous-users-who-discussed-piracy/)

- カリフォルニア州北部地区連邦地方裁判所の判事は、海賊版関連のスレッドにコメントを書き込んだ 8 人の匿名ユーザーを特定する必要はないとし、Reddit に有利な判決を下しました。
- 映画スタジオは、海賊行為についてコメントした Reddit ユーザーの身元を明らかにすることを求めていましたが、裁判官は、コメントは映画スタジオの訴訟とは無関係であり、他の情報源から関連情報を得ることができるとし、召喚状を破棄したため、敗訴しました。
- Reddit は、修正第 1 条はユーザーがオンラインで匿名で発言する権利を保護するものであり、この記録は、修正第 1 条が証拠開示を禁止することを示していると主張しました。

#### [業界の反応](http://news.ycombinator.com/item?id=35787080)

- Reddit は、原告へのコメント者 1 名を dox したのみで、全員を dox したわけではない
- Reddit の匿名スピーチ擁護が問題視されています。

### [Linux カーネルモジュール プログラミングガイド](https://sysprog21.github.io/lkmpg/)

- Linux Kernel Module Programming Guide は、システムを再起動することなくカーネルの機能を拡張するカーネルモジュールの書き方を解説した無料の書籍です。
- 本書は、ヘッダー、デバイスドライバ、ファイルシステム、システムコール、割り込みハンドラ、スケジューリングタスクなど、さまざまなトピックをカバーしています。
- 著者は、すべてのサンプルをコンソールから抽出し、コンパイルし、ロードすることを推奨しています。本書では、マクロ module_init と module_exit の使用を含む、カーネルモジュール設定を扱う新しい方法を紹介し、カーネルモジュールのライセンスオプションについて説明しています。さらに、モジュールはグローバル変数と module_param()マクロを使用して、コマンドライン引数を取ることができます。

#### [業界の反応](http://news.ycombinator.com/item?id=35782630)

- eBPF は、いくつかのカーネルモジュールのタスクに対して、より新しく、少し簡単で安全な代替案です。
- このガイドは、カーネルモジュールについて学ぶための良い出発点ですが、より複雑なドライバを書くことになると、不足するかもしれません。

- PornHub は、ユタ州の新しい年齢確認法に基づき、同社サイトへのアクセスを希望するユタ州のユーザーに対してブロックを実施しました。
- この法律は、ポルノサイトにユタ州のユーザーが 18 歳以上であることを確認することを義務付けるもので、これに従わないサイトは、未成年者がアクセスしたことによる損害賠償の責任を問われる可能性があります。
- PornHub は、ID を要求することはユーザー保護に有効ではなく、かえって子供やプライバシーを危険にさらすと主張し、デバイスによるユーザーの識別がより良い解決策であることを示唆しています。

#### [業界の反応](http://news.ycombinator.com/item?id=35786086)

- 年齢認証の取り組みは、ポルノ提供者に大きな負担を求め、プライバシー侵害や記録の作成などの懸念があることから、誠意ある取り組みかどうか、その有効性について議論がなされています。
- コメントでは、多様性と文化、政府による年齢制限、ペアレンタルコントロール、アダルト素材の表示と分類に関する法律などのトピックを取り上げて議論しています。

### [faulTPM: Exposing AMD fTPM's Deepest Secrets](https://arxiv.org/abs/2304.14717)

- フラウンホーファー研究所の研究者は、AMD の fTPM(ファームウェアベースの TPM)に faulTPM と名付けられた脆弱性を発見しました。
- この脆弱性を悪用すると、攻撃者は TPM から暗号化キーやパスワードなどの機密情報を読み取ることができます。
- AMD は、この問題を軽減するためのファームウェア・アップデートをすでにリリースしており、ユーザーはできるだけ早くシステムをアップデートすることが推奨されます。

#### [業界の反応](http://news.ycombinator.com/item?id=35787195)

- コメント欄では、技術の限界、Windows 11 の任意ハードウェアサポート切れ、Windows から Linux ディストロへの乗り換えの実現性など、様々な技術的トピックについて議論しています。
- 研究者は、AMD の fTPM 実装の脆弱性を公開し、$3 チップ攻撃で悪用されてデータを流出させる可能性があることを明らかにし、自分の脅威モデルを評価し、fTPM や TPM のみのプロテクターに依存する以外の追加のセキュリティ対策を実施する重要性を強調しています。

### [Jsonformer: LLM から構造化出力を生成する](https://github.com/1rgs/jsonformer)

- Jsonformer は、言語モデルから構造化 JSON を生成する新しいアプローチで、生成される JSON が常に構文的に正しく、指定されたスキーマに準拠することを保証します。
- Jsonformer は、生成過程で固定トークンを埋め、コンテンツトークンの生成のみを言語モデルに委ねることで、既存のアプローチよりも効率的で弾丸のような生成方法を実現しています。
- Jsonformer は HuggingFace トランスフォーマーライブラリーの上に構築されており、柔軟で拡張可能でありながら JSON スキーマのサブセットをサポートします。

#### [業界の反応](http://news.ycombinator.com/item?id=35790092)

- Clownfish や Guardrails といった他の類似の JSON スキーマジェネレーターは、複雑なスキーマに対してより複雑なアプローチを必要とします。
- Zod は、期待される型を定義し、応答を検証するために使用することができ、「transmogrify」後の Typescript のように推論された静的な型にマッチします。
