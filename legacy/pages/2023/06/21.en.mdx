### ["Exit traps" can make your Bash scripts more robust and reliable (2013)](http://redsymbol.net/articles/bash-exit-traps/)

- Using "exit traps" in bash scripts can make them more reliable by ensuring necessary cleanup operations are always performed, even when something unexpected goes wrong.
- The trap is a pseudo-signal called EXIT that can be utilized in bash scripts to execute commands or functions when the script exits, regardless of the reason.
- "Exit traps" can be used to remove resource leaks, maintain services, and release expensive resources, making bash scripts more robust and efficient.

#### [Industry Reactions](http://news.ycombinator.com/item?id=36400465)

- "Exit traps" in Bash scripts can enhance robustness and reliability.
- POSIX only requires EXIT to run on a clean shutdown, so additional signals should be added to catch interruptions.
- The signals EXIT, HUP, INT, and TERM cover most common cases for clean exits, terminal closure, Ctrl+C, and kill commands.
