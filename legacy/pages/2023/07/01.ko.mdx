- 웹사이트의 소개와 코드 샘플은 사용자들의 관심을 불러일으키고 사용해 보고 싶게 만든다는 평을 받고 있습니다.
- 사용자들은 Hurl의 단순함을 높이 평가하며 API 테스트 및 통합 테스트에 사용하기 쉽다고 말합니다.

### [마크다운으로 책을 만드는 명령줄 도구, MdBook](https://rust-lang.github.io/mdBook/)

- MdBook은 마크다운으로 책을 만들 수 있는 명령줄 도구로, 제품 또는 API 문서, 튜토리얼, 강의 자료를 쉽게 만들 수 있도록 도와줍니다.
- 통합 검색 지원, 코드 블록에 대한 색상 구문 강조 표시, 출력 서식을 지정할 수 있는 사용자 정의 가능한 테마 파일과 같은 기능이 있습니다.
- MdBook은 속도, 안전성 및 단순성을 보장하는 Rust로 작성되었으며, Rust 프로그래밍 언어 프로젝트에서 사용됩니다.

#### [업계 반응](http://news.ycombinator.com/item?id=36528984)

- 엠디북은 마크다운으로 책을 만드는 데 사용되는 명령줄 도구입니다.
- 제품 또는 API 문서, 튜토리얼, 강의 자료 또는 깔끔하고 탐색하기 쉬우며 사용자 정의 가능한 프레젠테이션이 필요한 모든 것을 만드는 데 이상적입니다.
- MdBook은 가볍고 설치가 쉬우며 통합 검색 지원, 다양한 언어의 코드 블록에 대한 색상 구문 강조 표시, 여러 형식으로 출력을 렌더링하는 기능을 갖추고 있습니다.
- Rust 프로그래밍 언어 프로젝트에서 사용되었으며, Rust 프로그래밍 언어 책은 mdBook의 실제 사용 예시입니다.
- MdBook은 활발하게 유지 관리되고 있으며 활발한 기여자 커뮤니티가 있습니다.

### [LLVM과의 이혼 신청서](https://github.com/ziglang/zig/issues/16270)

- 이 제안서는 Zig 프로젝트에서 LLVM, Clang, LLD 라이브러리를 제거할 것을 제안합니다.
- Zig는 자체 최적화 패스를 구현하고 칩 제조업체의 기여를 받아 머신 코드를 개선할 계획입니다.
- C++ 컴파일 기능이 제거되면 일부 프로젝트에 영향을 미칠 수 있지만, Zig는 기여자 친화적인 IR과 더 가벼운 컴파일러 바이너리를 목표로 하고 있습니다.

#### [업계 반응](http://news.ycombinator.com/item?id=36529456)

- Zig 프로그래밍 언어는 사용자들 사이에서 우려를 불러일으킨 LLVM에 대한 의존성에서 벗어나는 것을 고려하고 있습니다.
- 일부 사용자는 Zig에서 C++ 및 Objective-C에 대한 지원을 제거하면 인기와 채택에 방해가 될 수 있다고 생각합니다.
- Zig가 LLVM에서 벗어날 경우 성능 저하와 새로운 컴파일러 백엔드를 빌드하고 유지 관리하는 데 필요한 리소스에 대한 우려도 있습니다.

### [전자레인지에 돌린 플라스틱 용기가 식품에 미세 플라스틱을 방출한다](https://pubs.acs.org/doi/10.1021/acs.est.3c01942)

#### [업계 반응](http://news.ycombinator.com/item?id=36532812)

- 전자레인지에 돌린 플라스틱 용기는 미세 플라스틱을 식품으로 방출하여 인체에 부정적인 영향을 미칠 수 있습니다.
- "안전" 또는 "BPA 무함유"라고 표시된 플라스틱에도 유해 물질이 함유되어 있을 수 있는데, 50가지 이상의 화학물질이 BPA의 대체 물질로 사용되기 때문입니다.
- 유리 용기는 미세 플라스틱을 방출하지 않고 내구성이 뛰어나며 세척하기 쉽기 때문에 식품을 보관하고 가열하는 데 더 안전한 대안입니다.
