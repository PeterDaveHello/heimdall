### [不要接受风险投资的资助--它会毁掉你的公司](https://www.eidel.io/2023/07/09/vc-funding/)

- VC 资金并不是成功的保证，实际上可能表明公司没有能力靠自己的力量盈利。
- 接受风险投资意味着出售公司的一部分，并将目标从建立一个你喜欢的公司改为建立一个将来可以以更高价格出售的公司。
- 风险投资资金可能会导致二阶效应，如雇佣比预期更多的员工，花时间寻找新的投资者而不是建设公司，以及将增长置于盈利之上。

#### [行业反应](http://news.ycombinator.com/item?id=36654960)

- 作者认为，从风险资本家(VCs)那里筹集资金使初创企业走上了一条确定的道路，其结果是有限的：失败、收购或上市。
- 最重要的是在决定是否接受风险投资的资金时，要考虑自己的目标和公司的具体情况。
- 风险投资资金会扭曲激励机制，并通过优先考虑增长和利润而不是其他目标来阻碍公司的愿景。

### [最小的.NET Hello World 二进制有多小？](https://blog.washi.dev/posts/tinysharp/)

- 作者探讨了.NET Hello World 二进制文件在文件大小方面能有多小，同时还能在 Windows 机器上作为一个正常的可执行文件运行。
- 作者为实验设定了任意的规则，如使用 C#或 CIL 实现的管理入口点，在.NET 框架 4.x.x 上运行，并且不使用任何第三方的依赖。
- 通过各种优化和手工制作代码，作者成功地将 Hello World 二进制文件的大小减少到 834 字节，实现了最小的尺寸。

#### [行业反应](http://news.ycombinator.com/item?id=36652824)

- 一位开发者使用.NET 框架在 8KB 以下创建了一个小型的蛇形游戏，没有对.NET 运行时间的依赖。
- 有人提到.NET 中通道的使用是受 Go 启发的功能。
- 有人讨论了不同可执行文件的大小以及它们与小的.NET Hello World 二进制文件的比较。

### [PoisonGPT: 我们在 Hugging Face 上隐藏了一个被切除脑叶的 LLM 来传播假新闻](https://blog.mithrilsecurity.io/poisongpt-how-we-hid-a-lobotomized-llm-on-hugging-face-to-spread-fake-news/)

- 这篇文章讨论了使用大型语言模型(LLM)的潜在危险，以及建立具有模型出处的安全 LLM 供应链以确保 AI 安全的必要性。
- 它展示了一个开源模型 GPT-J-6B 是如何被修改以传播错误信息，同时又不被标准基准所发现的。
- 文章介绍了 AICert，一个即将推出的开源工具，它将提供模型出处的加密证明，解决人工智能行业对可追溯性和问责制的需求。

#### [行业反应](http://news.ycombinator.com/item?id=36655885)

- 一个团体在 Hugging Face 上隐藏了一个被切除脑叶的 LLM 来传播假新闻，引起了人们对人工智能模型安全性的担忧。
- 强调在使用 LLM 时，事实核查和批判性思维的重要性，因为它们不是无懈可击的信息来源。
- 提到了对手利用人工智能模型传播错误信息的可能性，强调了采取强大安全措施的必要性。

### [InfluxDB 云在比利时关闭；一些人在数据删除前没有得到通知](https://community.influxdata.com/t/getting-weird-results-from-gcp-europe-west1/30615)

- 比利时的 InfluxDB Cloud 用户遇到了仪表盘上的数据丢失或不完整的问题。
- 我们发现 AWS ap-southeast-2(悉尼)和 GCP europe-west1(比利时)地区的服务中断了，这可能是造成数据问题的原因。
- 一些用户没有收到 InfluxDB 发出的通知他们这一变化的电子邮件。

#### [行业反应](http://news.ycombinator.com/item?id=36657829)

- InfluxDB 云计算在没有适当通知的情况下在比利时关闭，导致一些用户的数据丢失。
- 用户对 InfluxDB 使用的有效沟通方式表示失望。
- 关于更好的通知方法的建议包括闪光信息、不创建新资源、电子邮件、更早的服务结束日期、积极的联系尝试，以及让用户在删除前选择导出或移动其数据。

### [使用激光雷达绘制树影](https://tedpiotrowski.svbtle.com/using-lidar-for-tree-shadows-in-shademap)

- 作者开发了一个名为 ShadeMap 的网站，利用 LiDAR 数据模拟树影。
- 通常用于模拟阴影的雷达，会错过 90%的树木投射的阴影，因为它只从地面反射。
- 另一方面，LiDAR 对所有物体进行反射，并提供更丰富的地球表面模型，使其对阴影的模拟更为准确。然而，收集 LiDAR 数据既费时又费钱。

#### [行业反应](http://news.ycombinator.com/item?id=36658001)

- 雷达在制图中不包括植被，因为它从地面上反射，使树木和建筑物等物体看不见。
- 航天飞机雷达地形任务(SRTM)使用的雷达确实能穿透一些树冠，但它的数据中没有捕捉到树叶或建筑物的阴影。
- 激光雷达可以用来绘制具有颗粒细节的树影，并有各种潜在的应用，如太阳能电池板的放置、摄影、汽车停放等。

### [Perl 首次提交：Awk 和 sed 的 "替代品"](https://github.com/Perl/perl5/commit/8d063cd8450e59ea1c611a2f4f5a21059a2804f1)

- Perl 是一种编程语言，旨在使 awk 和 sed 成为半废弃的语言。
- Perl 对扫描和提取文本文件中的信息进行了优化，使其对系统管理任务很有用。
- Perl 结合了 C、sed、awk 和 shell 的特性，使熟悉这些语言的人对它很熟悉。

#### [行业反应](http://news.ycombinator.com/item?id=36650120)

- Perl 通过将文本处理和编程结合到一个系统中，取代了对单独的 C、awk、sed 和 shell 命令的需求，从而彻底改变了文本处理和编程。
- Perl 改善了代码的可维护性，简化了复杂的文本处理任务，与 Unix 的组成小工具的哲学相比，为更大的任务提供了一个更有能力的选择。
- 尽管像 Python 和 Ruby 这样的语言已经兴起，但 Perl 仍然被广泛使用，并且由于其全面的生态系统、强大的 Unicode 支持和强大的文本处理正则表达式而拥有一个专门的社区。
