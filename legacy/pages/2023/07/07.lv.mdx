#### Gandrīz visi Evernote atlikušie darbinieki ir atlaisti.

- Populārā piezīmju pierakstīšanas lietotne Evernote ir atlaidusi ievērojamu daļu darbinieku.
- Uzņēmums ir saskāries ar finansiālām grūtībām un cīnās, lai konkurētu ar citām piezīmju veidošanas lietotnēm.
- Šī ziņa ir svarīga, jo tā liecina, ka Evernote var nonākt grūtībās un lietotnes lietotājiem var nākties apsvērt alternatīvas iespējas.

#### [Nozares reakcija](http://news.ycombinator.com/item?id=36609641)

- Evernote pēc tam, kad to pārņēma Bending Spoons, ir atlaista ievērojama daļa atlikušo darbinieku, radot bažas par uzņēmuma spēju finansēt jaunas funkcijas.
- Lietotāji pauž neapmierinātību ar Evernote grūtībām pārvērst bezmaksas lietotājus par maksājošiem klientiem un uzsver vienkāršības, uzticamības un pārnesamības nozīmi piezīmju veikšanas lietotnē.
- Kā veiksmīgi Evernote konkurenti tiek pieminētas alternatīvas piezīmju veidošanas lietotnes, piemēram, Obsidian un OneNote, un lietotāji apspriež riskus un trūkumus, kas saistīti ar riska un riska kapitāla finansētiem jaunuzņēmumiem.

### [GPT-4 API vispārējā pieejamība](https://openai.com/blog/gpt-4-api-general-availability)

- OpenAI ir paziņojis par GPT-4 API, sava visjaudīgākā modeļa, vispārēju pieejamību visiem maksājošiem API klientiem. Šis ļoti pieprasītais API nodrošina izstrādātājiem piekļuvi inovatīviem produktiem, kas izmanto GPT-4.
- OpenAI ir ieviesusi arī Čata pabeigšanas API, kas ir izrādījies jaudīgs un elastīgs visdažādākajiem pabeigšanas uzdevumiem. Tas veido 97 % API lietojumu un piedāvā labākus rezultātus ar strukturētu uzvedņu saskarni.
- Vecākie modeļi Completions API tiks likvidēti un aizstāti ar jauniem modeļiem, sākot ar 2024. gada 4. janvāri. Izstrādātājiem, kas izmanto šos vecākos modeļus, būs jāatjaunina sava integrācija, lai nodrošinātu turpmāku piekļuvi API.

#### [Nozares reakcija](http://news.ycombinator.com/item?id=36621120)

- OpenAI ir laidusi klajā GPT-4 API, kas tagad ir pieejama visiem maksas lietotājiem.
- Tādi ChatGPT modeļi kā gpt-3.5-turbo un text-davinci-003 ar laiku tiks likvidēti par labu GPT-4 API.
- GPT-4 salīdzinājumā ar iepriekšējām versijām piedāvā uzlabotu veiktspēju un iespējas, un izstrādātāji atzinīgi novērtē tās kvalitātes lēcienu dažādās jomās.

#### [Cloud Backed SQLite](https://sqlite.org/cloudsqlite/doc/trunk/www/index.wiki)

- "Cloud Backed SQLite" (CBS) sistēma ļauj datu bāzēm glabāties mākoņkrātuves kontos, ļaujot krātuves klientiem tās lasīt un rakstīt, neveicot visas datu bāzes lejupielādi.
- Pašlaik CBS atbalsta Azure Blob Storage un Google Cloud Storage, un ir iespējams pievienot atbalstu arī citām mākoņglabāšanas sistēmām.
- CBS izmanto fiksēta izmēra blokus, lai SQLite datubāzes uzglabātu mākonī, un manifest fails apraksta, kā bloki tiek apvienoti datubāzēs. Tā nodrošina arī API un komandrindas rīkus datu bāzu augšupielādei, lejupielādei un pārvaldīšanai mākonī.

#### [Nozares reakcijas](http://news.ycombinator.com/item?id=36610595)

- "Cloud Backed SQLite" ļauj apkalpot lielas SQLite datubāzes blokos un veikt vaicājumus, izmantojot HTTP diapazona pieprasījumus.
- Bibliotēka ar nosaukumu sql.js-httpvfs veic lielāko daļu darba, lai datubāzi sadalītu pa daļām un augšupielādētu GitHub Pages.
- Cilvēki ir sajūsmā par SQLite ātrumu un efektivitāti kā tikai lasīšanai paredzētu datu bāzi lielām datu kopām bez servera infrastruktūras pieskaitāmajām izmaksām.

### [{n} reizes ātrāk nekā C](https://owen.cafe/posts/six-times-faster-than-c/)

- Ierakstā tiek pētīts C koda cilpas optimizācijas process, veicot uzlabojumus un izmērot iegūto paātrinājumu.
- Cilpas funkcija inkrementē vai dekrementē skaitli, pamatojoties uz virknes rakstzīmēm.
- Tiek izmantoti dažādi optimizācijas paņēmieni, piemēram, zaru pārkārtošana, lēcienu aizstāšana ar aritmētiskiem paņēmieniem un izvairīšanās no nosacītiem zariem. Šīs optimizācijas ievērojami uzlabo cilpas veiktspēju.

#### [Nozares reakcijas](http://news.ycombinator.com/item?id=36618344)

- Ierakstā ir aplūkots ar roku rakstītas asembleru un C koda salīdzinājums, parādot asembleru potenciālu būt ātrākai.
- Optimizētajā asemblera kodā ir panākts 6x paātrinājums salīdzinājumā ar oriģinālo C kodu, jo lēcienu vietā tiek izmantota nosacītā aritmētika.
- Ierakstā sniegts ieskats zema līmeņa optimizācijas metodēs un parādīts kompromiss starp koda vienkāršību un veiktspēju.
