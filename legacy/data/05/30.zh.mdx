### [让我们为你服务，但不要把我们打倒](https://blog.archive.org/2023/05/29/let-us-serve-you-but-dont-bring-us-down/)

- 由于从亚马逊网络服务的 64 个虚拟主机每秒发起数万个请求，Archive.org 经历了长达一小时的中断，这使所有用户的网站瘫痪。
- 工程师们在一个周末假期的周日下午争分夺秒地处理这个问题，网站通过阻止这些 IP 地址而恢复正常，但几个小时后，另外 64 个地址也开始了同样类型的活动。
- 那些想大量使用 Archive.org 材料的人应该慢慢开始，逐步增加，如果开始一个大型项目，应该与该组织联系，info@archive.org，以避免网站不堪重负。

#### [行业反应](http://news.ycombinator.com/item?id=36110527)

- 系统管理员分享了添加 recaptcha 令牌以挫败来自多个 IP 地址的类似 DDoS 的搜刮攻击的经验
- 评论者建议使用工作证明算法，包括比特币的算法，作为 CAPTCHAs 的替代方案，以防止机器人攻击，但人们对不拥有加密货币的用户的初始障碍和替代 PoW 算法的限制表示关注
- 讨论集中在防止 DDoS 式攻击的策略上，包括速率限制和延迟对机器人的回应，以及对互联网档案服务器过载的担忧和对过度搜刮或下载的潜在速率限制。

### [300ms 更快：减少维基百科的总封锁时间](https://www.nray.dev/blog/300ms-faster-reducing-wikipedias-total-blocking-time/)

- 总阻塞时间(TBT)衡量浏览器主线程上所有阻断用户互动的长任务的总和，这些任务介于 First Contentful Paint 和 Time to Interactive 之间。
- 长时间的 JavaScript 执行和为每个元素附加事件监听器都会导致高 TBT。
- 通过删除不必要的 JavaScript 和优化现有的 JavaScript，维基百科在低端设备上将 TBT 减少了近 300ms，提高了其整体性能。

#### [行业反应](http://news.ycombinator.com/item?id=36113430)

- 维基百科对未登录的用户来说更快
- 为提高网站性能，人们提出了一些低垂的建议，如使用 Astro 和投机性预取等
- 讨论包括维基百科是否应该默认提供移动版本或检测设备能力

### [Donut: OCR-Free Document Understanding Transformer](https://github.com/clovaai/donut)

- Donut 是一个无 OCR 的文档理解转化器，在没有现成的 OCR 引擎或 API 的情况下，实现了视觉文档理解任务的最先进性能。
- SynthDoG，一个合成文档生成器，帮助 Donut 的预训练在各种语言和领域中都很灵活。
- 预训练的模型和网络演示可用于各种视觉文档理解任务，如文档解析、文档分类、文档 VQA 和文本阅读任务。

#### [行业反应](http://news.ycombinator.com/item?id=36111878)

- Donut，一个无 OCR 的文档理解转化器，已经在 Github 上发布。
- 与谷歌云 OCR 或 Azure OCR 等其他 OCR 工具相比，Donut 从 Hacker News 用户那里得到的评价不一。
- 尽管有批评，Donut 显示了未来文档理解和分类应用的潜力。

### [生产人工智能系统很难](https://methexis.substack.com/p/production-ai-systems-are-really)

- 放射科医生的工作很复杂，需要大脑的三维世界模型及其动态变化，因此几乎不可能完全用人工智能系统来取代他们。
- 自动系统不能像放射科医生那样做出判断，因为放射科医生有接地气的大脑模型，使得训练人工智能机器复制他们的表现具有挑战性。
- 鉴于广泛的特定领域知识是必要的，即使创建一个可以使放射科医生的工作自动化的算法也可能被证明是困难的，这使得任何人都不太可能成功做到这一点。

#### [行业反应](http://news.ycombinator.com/item?id=36111596)

- 人工智能可以预先起草 90%的放射科诊断研究报告，但人们担心放射科医生会因不审查输出而变得自满和误读
- 由于法律和监管问题、医院之间的成像协议和疾病流行率的差异以及对隐私的担忧，共享医疗数据存在挑战
- 人工智能可以帮助放射科医生分清解释的优先次序并标记出异常扫描，但对于罕见和具有挑战性的疾病，人类诊断师的投入仍然至关重要。
