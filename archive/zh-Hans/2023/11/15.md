# 2023-12-15

## [米切尔-桥本（Mitchell Hashimoto）告别工作了 11 年的哈希公司，回顾公司的影响](https://www.hashicorp.com/blog/mitchell-reflects-as-he-departs-hashicorp)

- HashiCorp 联合创始人米切尔-桥本（Mitchell Hashimoto）在一封情真意切的信中对 HashiCorp 所产生的影响表示感谢，并宣布他在离开公司 11 年后将离开公司。
- 桥本分享了他离开的决定，并寻求云自动化和基础架构工具之外的新挑战。
- 他强调了 HashiCorp 的发展和成就，包括软件的广泛采用和开源项目的认可，并祝愿公司未来取得成功。

### [反应](https://news.ycombinator.com/item?id=38647484)

- HashiCorp 的联合创始人米切尔-桥本（Mitchell Hashimoto）在离开公司 14 年后，因其对软件开发行业的贡献而赢得了社会各界的祝贺和感谢。
- 讨论围绕 HashiCorp 的项目、最近的许可变更以及使用 Terraform 进行多云部署的概念展开。
- 人们纷纷猜测米切尔的未来事业和哈希公司项目的潜在替代方案，以及开源社区中运营和基础设施即代码（IaC）工作的重要性。

## [Mozilla 扩展 Android 版火狐浏览器的扩展支持，新增 450 多个附加组件](https://blog.mozilla.org/en/mozilla/new-extensions-youll-love-now-available-on-firefox-for-android/)

- Mozilla 在安卓平台上为火狐浏览器推出了 450 多个新扩展，在移动平台上建立了一个开放的扩展生态系统。
- 现在，用户只需轻松安装这些扩展程序，就能在火狐安卓版上定制和增强网络体验。
- 现在，uBlock Origin 和 Dark Reader 等流行的桌面扩展程序已可在移动设备上使用，未来几个月还将添加更多扩展程序。

### [反应](https://news.ycombinator.com/item?id=38644608)

- Mozilla 扩大了对 Android 版火狐浏览器的扩展支持，允许用户从 addons.mozilla.org (AMO) 安装任何扩展。
- 由于重写 Firefox for Android 的用户界面需要大量工作，因此扩展支持的扩大被推迟了。
- 围绕这一话题的讨论包括对 Mozilla 决定的猜测、对隐私的担忧、对使用 AMO 账户的批评，以及对未经测试的扩展警告用户的建议。

## [取消软件开发中的质量保证团队的后果](https://davidkcaudill.medium.com/maybe-getting-rid-of-your-qa-team-was-bad-actually-52c408bd048b)

- 在软件开发中取消质量保证（QA）团队可能会带来负面影响。
- 任务自动化和流程优化可能会导致忽视测试的重要性。
- 质量保证角色对于有效管理软件质量至关重要，包括缺陷跟踪、错误分流、缺陷调查、关注质量和端到端测试。忽视这些活动会产生有害影响。
- 对那些在软件开发组织中将质量放在首位的人给予认可和支持是至关重要的。

### [反应](https://news.ycombinator.com/item?id=38645856)

- 质量保证团队在软件开发中发挥着至关重要的作用，而仅仅依靠开发人员进行测试则存在缺陷。
- 讨论中还探讨了各种测试方法，如 Bug Bashes 和模糊测试。
- 其中强调了质量保证团队面临的限制和挑战，包括评价过低、缺乏培训和协作，突出了在软件开发中优先考虑质量保证和全面测试的必要性。

## [巴塞罗那超级计算中心推出 Sargantana：新型开源 RISC-V 芯片](https://www.bsc.es/news/bsc-news/bsc-presents-sargantana-the-new-generation-the-first-open-source-chips-designed-spain)

- 巴塞罗那超级计算中心（BSC）推出了基于开源 RISC-V 技术的第三代 Lagarto 处理器，名为 Sargantana。
- Sargantana 加工机旨在促进欧洲的技术主权，减少对跨国公司的依赖。
- 该芯片性能更强，是 Lagarto 系列中首款工作频率超过千兆赫的芯片，巩固了 BSC 在欧洲 RISC-V 计算研究领域的领先地位。

### [反应](https://news.ycombinator.com/item?id=38640406)

- 巴塞罗那超级计算中心创建了一个名为 Sargantana 的开源 RISC-V 芯片，优先考虑安全性和本地芯片生产。
- GlobalFoundries 为老式芯片开发了一种新的 22 纳米平面工艺，引发了有关以较小节点尺寸生产芯片的挑战、成本和可行性的讨论。
- 讨论内容涉及多个方面，包括业余爱好者生产老式芯片、老式晶圆厂生产商数量有限、半导体行业的盈利能力、可靠芯片供应的重要性、欧盟在半导体制造方面的前景、CPU 设计性能、语言开发以及有关 Sargantana 芯片的信息。

## [FunSearch：利用 LLM 发掘新的数学见解](https://deepmind.google/discover/blog/funsearch-making-new-discoveries-in-mathematical-sciences-using-large-language-models/)

- FunSearch 是一种利用大型语言模型 (LLM) 在数学科学领域取得新发现的方法。
- 它将预先训练好的 LLM 与自动评估器配对，以搜索计算机代码中编写的函数并生成新知识。
- 研究人员利用 FunSearch 解决了上限集问题，并发现了更有效的垃圾箱包装问题算法，其性能超过了最先进的计算求解器。
- FunSearch 生成的程序会解释解决方案是如何得出的，使其成为一个强大的科学工具。
- 这项研究展示了 LLM 驱动方法在数学新发现和现实应用方面的潜力。

### [反应](https://news.ycombinator.com/item?id=38643076)

- 基于语言模型的遗传编程（LLM）是数学科学界争论的一个话题。
- 与会者讨论了 LLM 在生成合理程序和避免不合理程序方面的优势。
- 强调了 LLM 在解决冷启动问题方面的潜力。
- 与其他遗传编程技术相比，LLM 的成本和适应性受到质疑。
- 讨论还探讨了人工智能更广泛的影响，包括神经网络在产生新知识方面的潜力以及对财富不平等的影响。
- 总之，对于法学硕士在代码演进和问题解决方面的价值和进步，存在着不同的观点。

## [隐藏的复选框Dropbox 的人工智能功能引发信任危机](https://simonwillison.net/2023/Dec/14/ai-trust-crisis/)

- Dropbox 的新人工智能功能受到了批评，并引发了对使用私人文件作为 OpenAI 模型训练数据的信任担忧。
- 尽管 Dropbox 否认在未经同意的情况下使用客户数据，但其设置中的一个隐藏复选框却引起了混乱，并引发了与 Facebook 麦克风间谍争议的比较。
- 对人工智能和技术公司的不信任破坏了隐私和数据保护，这凸显了清晰解释和透明度对重获信任的重要性。此外，出于隐私考虑，人们越来越倾向于在个人设备上使用本地模型。必须通过公开和诚实的数据处理来赢得信任。

### [反应](https://news.ycombinator.com/item?id=38643046)

- 文章和讨论围绕人工智能信任危机和同意在网站隐私中的重要性展开。
- 它强调有必要对 "同意 "作出法律定义，以防止欺骗性做法。
- 讨论内容包括对同意过程中的欺诈和问责、在数字合同中执行同意所面临的挑战、数据隐私、广告实践以及科技公司的可信度等问题的关注。

## [Ledger 的故宫博物院账户遭黑客攻击，建议用户提高警惕](https://github.com/LedgerHQ/connect-kit/issues/29)

- 紧急问题：恶意版本的 npm 软件包 @ledgerhq/connect-kit 可能会泄露用户数据。
- 由于有报告称用户在点击可疑链接后丢失了硬币，建议用户在问题解决前不要将 Ledger 钱包连接到存储库。
- 在 @ledgerhq/connect-kit-loader 中发现安全漏洞，影响 Wagmi 和 MetaMask SDK 等下游项目。建议合并拉取请求、冻结依赖关系、仔细检查第三方依赖关系，并在加载资源前对其进行验证。

### [反应](https://news.ycombinator.com/item?id=38641211)

- 讨论围绕 NPM、GitHub、Ledger、以太坊和整个加密货币行业的各种漏洞和问题展开。
- 议题包括 NPM 缺乏可选的签名支持、NPM 因不接受社区贡献的签名支持而受到批评，以及 package-lock.json 文件中缺乏 "完整性 "字段。
- 讨论的其他问题包括使用 PGP 密钥进行代码签名、供应链攻击、网络钓鱼攻击、Ledger 设备的安全问题、稳定币风险、自动化在发布流程中的作用、CDN 的漏洞以及加强安全措施的必要性。

## [S-210 法案：保护儿童还是侵犯互联网自由？](https://www.michaelgeist.ca/2023/12/the-most-dangerous-canadian-internet-bill-youve-never-heard-of-is-a-step-closer-to-becoming-law/)

- S-210 法案又称《保护青少年免受色情制品侵害法》，已在参议院获得通过，目前正由公共安全委员会进行审查。
- 该法案旨在保护儿童免受网络伤害，但因可能侵犯个人选择和网络自由而受到批评。
- 政府正在考虑使用人脸识别技术的年龄验证系统，这引起了人们对隐私的关注。
- 该法案还包括网站屏蔽条款。
- 文章反对该法案，强调在不诉诸政府支持的审查制度的情况下保护儿童的重要性。
- 评论部分探讨了其他解决方案，以及由于网络技术的进步而对网站进行封锁或审查所面临的挑战。
- Michael Geist 撰写了这篇文章，分享了他对 C-11 法案以及与广播和电信立法审查小组报告相关的其他话题的担忧。

### [反应](https://news.ycombinator.com/item?id=38643369)

- 拟议的加拿大互联网法案 S-210 旨在限制访问色情内容并保护儿童。
- 批评者对缺乏民主倒逼、隐私、审查和滥用权力表示担忧。
- 辩论内容包括：为采取反 CAM 措施而进行的游说活动、针对特鲁多的请愿、年龄验证系统的有效性、过度封堵、家长控制、政府控制、过滤系统的局限性、规则的执行、社交媒体对青少年心理健康的影响以及家长指导和教育的重要性。

## [警察部门未能通过人体摄影机录像保持透明度](https://www.propublica.org/article/how-police-undermined-promise-body-cameras)

- 一些警察部门拒绝公开人体摄影机的录像，米格尔-理查兹（Miguel Richards）被纽约警察局枪杀一案就是一例。
