# 2023-11-21

## [YouTube 缩短火狐浏览器用户的视频加载时间，引发反垄断滥用担忧](https://old.reddit.com/r/youtube/comments/17z8hsz/youtube_has_started_to_artificially_slow_down/)

- r/youtube subreddit 上的用户提出了对 YouTube 的担忧，如火狐浏览器视频加载速度慢和潜在的反垄断滥用。
- 关于 "Web Integrity API "可能限制某些浏览器访问网站的讨论，引发了对 YouTube 进行反垄断监督和采取法律行动的呼声。
- 用户在该平台上遇到了延迟、缓冲和广告拦截器问题，促使一些用户寻找解决方案，而另一些用户则对 YouTube 的行为表示不满。

### [反应](https://news.ycombinator.com/item?id=38345858)

- 用户报告称，在使用 Firefox 浏览器时，YouTube 上的视频加载时间很慢，这可能是由于代码有意延迟视频加载以检测广告拦截器的使用情况。
- 关于某些网站功能背后的意图、加载时间的重要性以及编程中睡眠函数的使用，讨论仍在继续。
- 用户正在探索观看和下载视频的替代平台，表达对流媒体服务的不满，并对歧视和虚假广告表示担忧。
- 此外，还讨论了谷歌对浏览器的影响、谷歌平台在火狐浏览器上的性能和兼容性以及谷歌潜在的反竞争行为。

## [OpenAI 员工要求董事会辞职并威胁大规模出走](https://www.wired.com/story/openai-staff-walk-protest-sam-altman/)

- OpenAI 的 730 多名员工签署了一封公开信，要求这家初创公司的董事会辞职，并恢复被赶下台的首席执行官山姆-奥特曼（Sam Altman）的职务。
- 员工们认为，董事会解雇 Altman 和解除 Greg Brockman 职务的决定使公司的工作和使命面临风险。
- 信中还呼吁任命两名新的独立首席董事会成员，这些员工有可能加入奥特曼在微软的工作，奥特曼和布洛克曼已被任命领导一个新的高级人工智能研究部门。

### [反应](https://news.ycombinator.com/item?id=38347868)

- OpenAI 正面临着内部冲突，由于与董事会意见不一，可能会出现辞职的情况，这可能与微软的影响力和获取代码有关。
- 讨论了微软收购 OpenAI 以及 Quora 数据集的潜在用途，引起了人们对微软控制 OpenAI 资金和 Azure 信用点贬值的担忧。
- 讨论涉及人工通用智能（AGI）的未来、在推进 AGI 的过程中使用语言模型，以及大公司对小公司施加主导地位的道德影响。讨论还对 OpenAI 的领导力、可信度和声誉提出了质疑。

## [OpenAI 前首席执行官萨姆-阿尔特曼（Sam Altman）尽管被解雇，但仍在努力争取回归](https://www.theverge.com/2023/11/20/23969586/sam-altman-plotting-return-open-ai-microsoft)

- 如果解雇阿尔特曼的董事会成员让位，OpenAI 前首席执行官萨姆-阿尔特曼（Sam Altman）和联合创始人格雷格-布罗克曼（Greg Brockman）将寻求重返公司。
- 最近任命的新首席执行官埃米特-希尔（Emmett Shear）最初似乎断绝了奥特曼回归的机会，但支持奥特曼的员工和董事会成员伊利亚-苏茨克沃（Ilya Sutskever）不断向董事会施压。
- Altman 转投微软的事宜尚未最终敲定，双方仍在进行谈判，以确定未来合作的性质。OpenAI 内部出现了权力斗争，导致员工在公司会议上感到愤怒和不参与。Altman 和微软正在努力达成一项维护董事会尊严的解决方案。

### [反应](https://news.ycombinator.com/item?id=38352891)

- 在 700 名员工要求董事会辞职后，Y Combinator 前总裁、OpenAI 前首席执行官山姆-奥特曼（Sam Altman）正寻求重返董事会，担任首席执行官。
- OpenAI 董事会因重利轻安而受到批评，引发了一场关于国防科技伦理和美国在传播文化价值观方面的作用的辩论。
- 人们围绕 OpenAI 的未来、微软的参与以及相关人员的动机进行了猜测，同时也对法律责任和公司的潜在倒闭提出了担忧。此外，还讨论了 Altman 的离职对 OpenAI 员工和组织的影响。

## [波音公司拒绝付款，勒索软件组织泄露 45 GB 数据](https://www.itbrew.com/stories/2023/11/17/after-boeing-declines-to-pay-up-ransomware-group-leaks-45-gb-of-data)

- 波音公司是飞机行业的主要企业，它已成为与俄罗斯有联系的 LockBit 勒索软件组织的受害者。
- LockBit 要求波音公司支付赎金，在波音公司拒绝支付赎金后，该组织在网上发布了约 45 GB 的公司敏感数据。
- 泄露的信息包括公司电子邮件和云计算公司 Citrix 的文件。由于波音公司与 2006 年收购的零部件分销商 Aviall 公司进行了整合，专家们担心可能会产生更广泛的后果。

### [反应](https://news.ycombinator.com/item?id=38352484)

- 讨论的重点是对波音公司的勒索软件攻击、其对国家安全的影响以及泄露的数据。
- 我们就供应商合同谈判中的成本透明度以及飞机安全评估所面临的挑战进行了讨论。
- 此外，还讨论了勒索软件集团的可信度、支付赎金的合法性和后果，以及集团的动机和策略。

## [Typst：通过增强布局和格式简化科学论文写作](https://typst.app/)

- 它可以实现无缝协作，提供用于自动格式化内容的预设计模板，并提供出色的排版输出。

- 它因其专门的 LaTeX 过渡指南和自定义样式的能力而备受赞誉。
- 一些用户不确定期刊和会议是否会采用 Typst，但认为它是 LaTeX 的一个很好的替代品，具有实时协作功能。

## [Roc：高效代码生成和执行的通用编程语言](https://www.roc-lang.org/)

- Roc 是一种编程语言，旨在提高构建和运行代码的效率，支持命令行界面、网络服务器和嵌入其他语言。
- 尽管 Roc 还处于早期开发阶段，但它已显示出对各种任务的可用性，并通过模式匹配和字符串插值等功能优先考虑用户友好性。
- Roc 可以生成机器代码或 WebAssembly，并支持数据库访问，它还得到了企业赞助商和个人贡献者的支持，这些赞助商和个人贡献者致力于它的持续开发和改进。

### [反应](https://news.ycombinator.com/item?id=38346862)

- 讨论深入探讨了 Roc 编程语言的功能和优化，以及它与 Elm 和 Haskell 等语言的相似之处。
- 它还涉及大鹏社区的行为，并提到了 F#、Eff 和 HVM 等其他编程语言。
- 此外，还讨论了类型注解的重要性、宏的使用以及调试器在编程中的作用。
