# 2023-09-03

## [我认识的最糟糕的程序员](https://dannorth.net/2023/09/02/the-worst-programmer/)

- Dan North 的博文探讨了衡量开发人员生产率的复杂问题，并强调了从单个任务完成情况到累积业务影响的关键视角转变。
- 这篇文章分享了蒂姆-麦金农（Tim Mackinnon）的故事，他是一名程序员，由于专注于集体解决问题而不是单独完成任务，因此在传统的生产率指标上一直表现不佳。
- 尽管管理者施压要开除蒂姆，但诺斯还是为自己在提高整个团队的效率和生产力方面所发挥的关键作用进行了辩护，他强调，在复杂的系统中，生产力应该通过切实的业务影响来衡量。

### [反应](https://news.ycombinator.com/item?id=37361947)

- 这篇文章探讨了软件开发行业的重要话题，如知识共享、管理批评和办公室政治，以及了解观念如何影响职业发展。
- 它讲述了行业中的特定角色和方法，包括用于项目管理的 Scrum，以及初级/高级工程师、技术负责人、产品负责人和产品经理的角色。
- 文章还强调了有效沟通和协作的重要性，以及营造支持性工作环境的重要性。文章讨论了错综复杂的公司政治、工会利弊以及代码贡献的挑战。

## [报税公司：9000 万美元游说反对免费报税](https://www.opensecrets.org/news/2023/09/tax-prep-companies-lobbying-against-free-file-face-scrutiny-from-lawmakers/)

- 领先的报税公司，如 TurboTax 的所有者 Intuit 和 H&R Block，因游说反对政府管理的免费报税系统而受到审查。
- 自 2003 年以来，这些公司总共花费了 9000 多万美元进行游说，以阻止美国国税局开发自己的报税软件。
- 截至 2019 年 12 月，免费报税联盟谅解备忘录的附录消除了这一障碍，使国税局能够制定 2024 年免费报税服务的计划。一些国会民主党人呼吁对报税行业的游说活动进行调查。

### [反应](https://news.ycombinator.com/item?id=37363616)

- 美国报税公司斥资 9000 万美元进行游说，反对免费报税选项，引发了关于税收制度复杂性和报税公司作用的讨论。
- 会议讨论了游说势力的影响，特别是 Intuit 等公司的影响，行业混乱的可能性，以及政府管理的报税系统的潜在优势。
- 此外，还讨论了将报税公司国有化的建议、简化税法的影响以及最高法院对 "罗氏诉韦德 "案裁决的相关性和影响等话题。

## [公益广告：不要把你的业务建立在 Discord 上。7 年账户因发布 ASN 而被封禁](https://news.ycombinator.com/item?id=37364605)

- 作者强调了他们与 Discord 的不利互动，透露了他们的 7 岁账户曾因一个小错误而被突然封禁。
- Discord 以前的问题包括因未成年参与者而威胁删除作者服务器的问题。
- 有鉴于此，作者建议在考虑在 Discord 上开展业务时要慎之又慎。

### [反应](https://news.ycombinator.com/item?id=37364605)

- 这场辩论凸显了利用 Discord 开展业务的缺点和潜在危险，其中包括对账户封禁、管理政策和支持不足的担忧。
- 与会者敦促需要替代平台和备份战略，强调依赖第三方平台的相关风险。
- 大家一致认为，完全依赖 Discord 会带来巨大的商业风险。

## [特斯拉和 Rivian 假意支持 "维修权](https://www.techdirt.com/2023/09/01/tesla-rivian-put-on-fake-show-of-support-for-right-to-repair/)

- 特斯拉和 Rivian 因似乎在倡导 "维修权 "运动，但又不承诺制定重要的新法律而受到抨击。
- 这两家公司都因车辆维修费用高昂且不方便而饱受批评。
- 文中还提到了其他各种与科技相关的话题，包括 Valve/Steam 的质量控制、播客剧集、谷歌关闭程序、像素手机和法律问题。

### [反应](https://news.ycombinator.com/item?id=37361462)

- 讨论围绕维修权运动和电动汽车（特别是特斯拉）维修的复杂性展开。重点讨论了使用套件将旧车改装成电动车的实用性、制造商通过非标准组件和软件使维修复杂化等问题。
- 讨论的其他主题包括电动汽车（EV）维修文件和零件的可用性、电动汽车在启动其他汽车时的局限性，以及太阳能屋顶和可更换电池的实用性。
- 与会者对解决方案有不同的看法；一些人主张制定维修权立法，而另一些人则认为，通过理解和修复没有结构图的组件，可以使任务变得更容易。

## [关于工艺](https://www.drcathicks.com/post/on-craft)

- 作者反思了祖父对工匠精神的包容性和实用性的影响，并与软件工程行业中的排他性和精英主义形成对比。
- 作者提出了 "所有劳动都是熟练劳动 "的概念，并列举了自己在创客空间中被排斥的经历，认为手工艺应优先考虑包容性和努力的价值。
- 作者将手工艺视为交流、合作和创造的媒介，而不是评判的工具或武器，这表明在改善团队环境和工作空间时，需要从他们祖父的视角出发。

### [反应](https://news.ycombinator.com/item?id=37361094)

- 这篇文章强调了在软件工程行业中作为 "局外人 "的感觉，强调了包容的必要性，并赞扬了这些 "局外人 "程序员意想不到的成功故事。
- 它提出了 "固定 "的概念，强调了手工艺的重要性以及对自己作品的自豪感。
- 文章深入探讨了工艺、质量和行业中计算机使用之间的关系，提出了工艺对软件质量的影响。

## [递归总结实现了 LLM 的长期对话记忆](https://arxiv.org/abs/2308.15022)

- 本文提出了一种策略，通过使用大型语言模型（LLM）递归创建摘要或记忆，增强开放域对话系统中的长期记忆。
- 这种方法鼓励语言学习者保留和回忆较小的对话语境，然后利用先前的记忆和随后的语境创建新的记忆。
- 实验结果表明，该方法生成的长语境对话中的回答更加一致。作者计划稍后发布代码和脚本。

### [反应](https://news.ycombinator.com/item?id=37363362)

- 辩论的重点是语言模型的制约因素和可能性，主要是它们在推理和数学任务中的能力。
- 讨论内容包括人工智能模型的能力，质疑是否有必要使用特定工具或系统来提高性能。
- 此外，还提出了对话式人工智能系统中长期记忆的作用和递归总结方法的效率，以及替代策略和流行的实现方法。

## [我将所有代码从 PyPI 镜像到 GitHub 并进行了分析](https://py-code.org/stats)

- 作者开发了一套自动系统，用于将 PyPI 中的所有代码镜像到 Github，从而方便进行实时软件包扫描、搜索索引和综合语言分析。
- 这项计划允许任何人在短时间内下载和评估 PyPI 上发布的所有代码。
- 这项创新预计将使用户能够利用 Python 代码库开发各种新应用。

### [反应](https://news.ycombinator.com/item?id=37364290)

- 作者研究了 GitHub 上的 Python 软件包仓库 PyPI 中的代码，发现在 8 年时间里，软件包的数量迅速超过了人类。
- 调查深入了解了 Python 的使用情况、代码中嵌入的证书，以及数据分析的复杂性、代码归档和语言功能的普及等其他各个方面。
- 作者还讨论了 PyPI 面临的挑战，如创建账户、搜索软件包时遇到的挫折，以及与许可有关的问题，特别是微软在没有许可的情况下使用软件包。

## [暴风雨过后，火人节进退两难](https://www.sfgate.com/travel/article/no-access-after-burning-man-rain-storm-18344393.php)
