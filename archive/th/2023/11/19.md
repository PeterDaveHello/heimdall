# 2023-12-19

## [Figma และ Adobe ละทิ้งการควบรวมกิจการหลังจากการตรวจสอบกฎระเบียบ](https://www.figma.com/blog/figma-adobe-abandon-proposed-merger/)

- Figma และ Adobe ได้ยกเลิกการควบรวมกิจการตามแผนหลังจากกระบวนการตรวจสอบกฎระเบียบที่ยาวนาน
- Figma จะรักษาสถานะอิสระและเปิดกว้างในการสํารวจโอกาสในการเป็นพันธมิตรกับ Adobe
- Figma มีความก้าวหน้าอย่างมากในช่วง 15 เดือนที่ผ่านมาโดยมุ่งเน้นที่การลดความซับซ้อนของการออกแบบและพัฒนาผลิตภัณฑ์ดิจิทัล ซีอีโอ Dylan Field แสดงความเห็นในแง่ดีต่ออนาคตของ บริษัท และขอบคุณสําหรับการสนับสนุนของชุมชน

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38681861)

- การควบรวมกิจการที่อาจเกิดขึ้นระหว่าง Figma และ Adobe ถูกยกเลิกซึ่งนําไปสู่การอภิปรายเกี่ยวกับเหตุผลที่อยู่เบื้องหลังและผลกระทบต่อพนักงานของ Figma
- ข้อดีและข้อ จํากัด ของเครื่องมือออกแบบเช่น Figma ได้รับการสํารวจในการสนทนารวมถึงการหยุดชะงักที่อาจเกิดขึ้นของ AI ในอุตสาหกรรมการออกแบบ
- การอภิปรายยังกล่าวถึงความกังวลเกี่ยวกับการรวมกิจการและการเสนอขายหุ้น IPO ในวัฒนธรรมการเริ่มต้นการกํากับดูแลความยั่งยืนในระยะยาวและบทบาทของกฎหมายต่อต้านการผูกขาดในตลาดดิจิทัล

## [Wasm3 เข้าสู่ขั้นตอนการบํารุงรักษาท่ามกลางการรุกรานของรัสเซีย](https://github.com/wasm3/wasm3)

## [ผู้เขียน MacBook ล็อคทิมคุกปลดล็อคมัน](https://www.tokyodev.com/articles/unbricking-my-macbook-took-an-email-to-tim-cook)

- MacBook ของผู้เขียนถูกส่งคืนให้พวกเขาด้วยการล็อคการเข้าใช้เครื่องเพื่อป้องกันการเข้าถึงอุปกรณ์
- ในตอนแรก Apple ปฏิเสธที่จะปลดล็อก MacBook แต่ผู้เขียนติดต่อ Tim Cook ทางอีเมลและในที่สุดก็ปลดล็อกได้
- MacBook ถูกลบและรายงานว่าสูญหายโดยบัญชี iCloud ที่สร้างขึ้นใหม่โดยแนะนําว่าอาจถูกนําไปที่ร้านค้ารีเซ็ตและรายงานว่าสูญหายเพื่อรีดไถเงิน
- ผู้เขียนเน้นย้ําถึงความจําเป็นที่ Apple ต้องแก้ไขเวิร์กโฟลว์การตั้งค่าเพื่อสื่อสารผลที่ตามมาของการไม่ตั้งค่า FindMy ได้ดีขึ้น เนื่องจากร้านค้าที่ไร้ยางอายสามารถใช้ประโยชน์จากสถานการณ์นี้ได้
- ผู้เขียนรายงานปัญหานี้ต่อการวิจัยด้านความปลอดภัยของ Apple แต่ถูกไล่ออก

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38691025)

## [โมเดล Word2Vec ใหม่มีประสิทธิภาพเหนือกว่าแนวทางก่อนหน้า](https://openreview.net/forum?id=idpCdOWtqXd60)

- บทสรุปเน้นความสําคัญของความชัดเจนและความเข้มงวดในการวิจัยเน้นอคติในการตรวจสอบกระดาษและสนับสนุนสภาพแวดล้อมการวิจัยที่เปิดกว้างและทํางานร่วมกันมากขึ้น

## [ความคืบหน้าสู่คอมไพเลอร์ Rust ที่รวมเข้ากับ GCC](https://lwn.net/SubscriberLink/954787/41470c731eda02a4/)

- โครงการ gccrs กําลังทํางานเพื่อสร้างคอมไพเลอร์ Rust เพื่อรวมไว้ในรุ่น GCC 14 โดยกําหนดเป้าหมายไปที่ Rust เวอร์ชัน 1.49
- ปัจจุบันโครงการนี้มุ่งเน้นไปที่การสนับสนุนการรวบรวมลังหลักและลังอัลลอค แต่ต้องเผชิญกับความท้าทายด้วยความละเอียดมาโครและมาโครมัณฑนากร
- โครงการนี้มีจุดมุ่งหมายเพื่อให้โค้ด Rust สามารถคอมไพล์โดยใช้ GCC ซึ่งมีส่วนช่วยในการพัฒนา Rust และรองรับระบบฝังตัว โครงการขั้นสูงอีกโครงการหนึ่งคือ rustc_codegen_gcc ช่วยให้ Rust สามารถคอมไพล์สําหรับ Linux ได้โดยไม่ต้องมีแพตช์และการอภิปรายรวมถึงการรวม Polonius ในคอมไพเลอร์ Rust และประโยชน์ของการใช้คอมไพเลอร์ Rust ที่ใช้ GCC เช่นการเปิดรับบั๊กและการบูตสแตรป

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38684102)

- มีการถกเถียงกันเกี่ยวกับการมีการใช้งาน Rust หลายครั้งโดยบางคนแย้งว่าสามารถช่วยค้นหาข้อบกพร่องและปรับปรุงภาษาได้ในขณะที่คนอื่น ๆ กังวลเกี่ยวกับความเข้ากันได้และการทําซ้ํา
