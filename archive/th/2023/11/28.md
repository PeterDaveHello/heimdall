# 2023-11-28

## [ค่าหัว $ 20k ของ Prettier ได้รับการอ้างสิทธิ์เพิ่มประสิทธิภาพและความเข้ากันได้](https://prettier.io/blog/2023/11/27/20k-bounty-was-claimed)

- Prettier ซึ่งเป็นตัวจัดรูปแบบโค้ด JavaScript ได้เสนอเงินรางวัล 20,000 ดอลลาร์สําหรับโครงการ Rust ที่ผ่าน 95% ของชุดทดสอบ ซึ่งโครงการ Biome อ้างสิทธิ์และเสร็จสมบูรณ์เรียบร้อยแล้ว
- การแข่งขันนี้นําไปสู่การปรับปรุงประสิทธิภาพและความเข้ากันได้ใน Prettier ด้วยการแก้ไขข้อบกพร่องที่กระตุ้นโดยเงินรางวัล
- Prettier ได้รับเงินบริจาคจํานวนมากจาก บริษัท และบุคคลทั่วไปทําให้สามารถพัฒนาได้อย่างต่อเนื่องในขณะที่ Open Collective ได้อํานวยความสะดวกในการทําธุรกรรมทางการเงินและการจัดการภาษีสําหรับโครงการ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38434613)

- ทีม Prettier เสนอเงินรางวัล 20,000 ดอลลาร์เพื่อส่งเสริมการพัฒนาโครงการฟอร์แมตเตอร์รหัสใหม่ใน Rust เนื่องจากพวกเขาเชื่อว่ามีการแข่งขันเพียงเล็กน้อยในพื้นที่นี้
- บทความนี้สํารวจค่าใช้จ่ายฟรีแลนซ์ที่เกิดขึ้นอัตรารายชั่วโมงสําหรับโปรแกรมเมอร์และปัจจัยที่มีอิทธิพลต่ออัตราการเรียกเก็บเงิน
- การอภิปรายครอบคลุมถึงแรงจูงใจของผู้ดูแลโอเพ่นซอร์สและความสําคัญของมุมมองที่หลากหลายในโครงการพัฒนาตลอดจนการอภิปรายเกี่ยวกับประสิทธิภาพของตัวจัดรูปแบบรหัสพริตเทียร์และทางเลือก

## [ผู้ใช้ Google ไดรฟ์ประสบปัญหาข้อมูลสูญหายเนื่องจากไฟล์หายไปอย่างลึกลับ](https://www.theregister.com/2023/11/27/google_drive_files_disappearing/)

- ผู้ใช้ Google ไดรฟ์กําลังประสบปัญหาไฟล์ที่หายไปจากบริการทําให้เกิดความหงุดหงิดและตื่นตระหนกสําหรับผู้ที่สูญเสียงานไปหลายเดือน
- ปัญหาดูเหมือนจะเชื่อมโยงกับปัญหาการซิงโครไนซ์โดยผู้ใช้บางคนสามารถกู้คืนไฟล์บางส่วนได้โดยจัดการไฟล์แคช
- Google ยังไม่ได้แก้ไขปัญหาหรือเสนอวิธีแก้ปัญหาทําให้เกิดความกังวลเกี่ยวกับความน่าเชื่อถือและความปลอดภัยของที่เก็บข้อมูลบนคลาวด์สําหรับข้อมูลสําคัญ

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38431743)

- ผู้ใช้แสดงความไม่พอใจและความกังวลเกี่ยวกับการสูญหายของข้อมูลไฟล์ที่หายไปและซ้ํากันและปัญหาความน่าเชื่อถือของ Google ไดรฟ์และแพลตฟอร์มที่เก็บข้อมูลบนคลาวด์อื่น ๆ
- ขอแนะนําให้ใช้ตัวเลือกการสํารองข้อมูลทางเลือกเช่นแพลตฟอร์มที่โฮสต์ด้วยตนเองและฮาร์ดไดรฟ์ภายนอกโดยมีการอภิปรายเกี่ยวกับประสิทธิภาพและค่าใช้จ่าย
- มีการหารือเกี่ยวกับการปกป้องความเป็นส่วนตัวและความปลอดภัยของข้อมูลโดยเน้นที่การเข้ารหัสข้อมูลในเครื่อง ความกังวลเกี่ยวกับผลิตภัณฑ์ OneDrive และ SharePoint ของ Microsoft ก็ถูกยกขึ้นเช่นกัน
- ผู้ใช้เน้นย้ําถึงความสําคัญของการสํารองข้อมูลส่วนบุคคลและแนะนําเครื่องมือและแพลตฟอร์มต่างๆ การเก็งกําไรเกี่ยวกับสาเหตุของการสูญหายของข้อมูลรวมถึงการซิงค์ข้อบกพร่องของโปรแกรมและมัลแวร์
- ความกังวลเกี่ยวกับการสนับสนุนของ Google และความต้องการตัวเลือกพื้นที่เก็บข้อมูลอื่น ๆ เป็นที่แพร่หลาย มีการสํารวจวิธีการต่างๆในการสํารองข้อมูลรวมถึงเซิร์ฟเวอร์ภายในและกระบวนการสํารองข้อมูลอัตโนมัติ
- มีการถกเถียงกันเกี่ยวกับความน่าเชื่อถือและความปลอดภัยของบริการจัดเก็บข้อมูลบนคลาวด์ความเสี่ยงของการพึ่งพาผู้ให้บริการรายเดียวและความต้องการความรับผิดชอบส่วนบุคคลในการรักษาการสํารองข้อมูล
- การอภิปรายเกี่ยวกับแนวทางปฏิบัติในการจ้างงานคุณภาพของซอฟต์แวร์และอิทธิพลการจัดการของ Google นอกจากนี้ยังมีการกล่าวถึงสิทธิ์ความเป็นเจ้าของความรับผิดของผู้ให้บริการคลาวด์และการเยียวยาทางกฎหมายสําหรับการสูญเสียทรัพย์สิน
- มีการสํารวจแนวคิดเรื่องทรัพย์สินทางปัญญาและการเป็นเจ้าของข้อมูลโดยเน้นถึงความแตกต่างในกฎหมายและข้อกําหนดในการให้บริการของ Google
- โดยรวมแล้วการอภิปรายสะท้อนให้เห็นถึงความกังวลความผิดหวังและข้อเสนอแนะที่เกี่ยวข้องกับความน่าเชื่อถือความปลอดภัยและความเป็นเจ้าของข้อมูลบนแพลตฟอร์มที่เก็บข้อมูลบนคลาวด์เช่น Google ไดรฟ์

- ความจําเป็นในการปรับปรุงการขนส่งทางรถไฟประสิทธิภาพที่เพิ่มขึ้นและความสามารถในการจ่ายจะถูกเน้นตลอดการอภิปราย

## [ผู้ใช้ GitHub แสดงความไม่พอใจต่อข้อ จํากัด การค้นหาใหม่อ้างว่ามันบ่อนทําลายหลักการโอเพ่นซอร์ส](https://github.com/orgs/community/discussions/77046)

- GitHub ได้ใช้นโยบายใหม่ที่กําหนดให้ผู้ใช้ต้องเข้าสู่ระบบเพื่อค้นหารหัสซึ่งนําไปสู่ความไม่พอใจในหมู่ผู้ใช้
- ผู้ใช้บางคนโต้แย้งว่าการเปลี่ยนแปลงนี้ไม่จําเป็นและเป็นปฏิปักษ์ต่อหลักการโอเพ่นซอร์ส
- GitHub แสดงให้เห็นถึงข้อกําหนดการเข้าสู่ระบบเป็นมาตรการรักษาความปลอดภัยเพื่อปกป้องเซิร์ฟเวอร์ของพวกเขาจากการโอเวอร์โหลดที่ไม่ระบุชื่อ แต่ผู้ใช้บางคนเชื่อว่าอิทธิพลของ Microsoft อาจเป็นปัจจัยหนึ่ง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=38432261)

- ตอนนี้ GitHub กําหนดให้ผู้ใช้เข้าสู่ระบบเพื่อค้นหาโค้ดโดยอ้างถึงลักษณะที่ใช้ทรัพยากรมากของคุณลักษณะการค้นหาโค้ดใหม่
- ผู้ใช้บางคนคาดเดาว่าการเปลี่ยนแปลงนี้อาจเป็นกลยุทธ์ในการผลักดันการลงทะเบียนและเพิ่มจํานวนผู้ใช้
- มีความคิดเห็นที่หลากหลายเกี่ยวกับการเปลี่ยนแปลงโดยบางคนปฏิเสธข้อร้องเรียนว่าเป็นความเกียจคร้านหรือไร้ความสามารถในขณะที่คนอื่น ๆ ทําให้เกิดความกังวลเกี่ยวกับปัญหาทางเศรษฐกิจและสังคมที่อาจเกิดขึ้นการเข้าถึงโดยไม่ได้รับอนุญาตความเสี่ยงด้านความปลอดภัยการป้องกันรหัสการขูดโดย บริษัท AI การโฮสต์โครงการโอเพ่นซอร์สและการเข้าซื้อกิจการ GitHub ของ Microsoft การอภิปรายยังรวมถึงแพลตฟอร์มทางเลือกและวิธีการค้นหา
