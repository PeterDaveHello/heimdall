## [ไดรเวอร์ M1 GPU ตัวแรกที่สอดคล้อง](https://rosenzweig.io/blog/first-conformant-m1-gpu-driver.html)

- ไดรเวอร์เหล่านี้ได้รับการยอมรับจากหน่วยงานมาตรฐาน Khronos หลังจากผ่านการทดสอบมาตรฐานอุตสาหกรรม
- การใช้งานของ Asahi Linux รวมถึงการสนับสนุนการประมวลผลเฉดสีและอะตอมบนภาพแม้ว่าจะไม่มีคําแนะนําฮาร์ดแวร์สําหรับอะตอมของภาพบน M1 GPU ก็ตาม

## [วัฒนธรรมการวัดที่ดีที่ตัวเลขไม่ได้แทนที่สามัญสํานึก](https://blog.promaton.com/how-to-avoid-kpi-psychosis-in-your-organization-5ffc83967f2b)

- โรคจิต KPI (ตัวบ่งชี้ประสิทธิภาพหลัก) เป็นปัญหาที่แพร่หลายใน บริษัท เทคโนโลยีซึ่งการตัดสินใจขึ้นอยู่กับข้อมูลและตัดการเชื่อมต่อจากความเป็นจริงเท่านั้น
- บริษัทต่างๆ พึ่งพาข้อมูลอย่างมากเพื่อเอาชนะอคติของมนุษย์ แต่การใช้ KPI เป็นพื้นฐานเพียงอย่างเดียวสําหรับการตัดสินใจมีข้อจํากัด
- บทความนี้แนะนําโซลูชันที่รวมสัญชาตญาณและข้อมูลสะท้อนให้เห็นถึงความน่าเชื่อถือของ KPI อย่างต่อเนื่องและมุ่งเน้นไปที่วัตถุประสงค์ที่แท้จริงมากกว่าเพียงแค่ KPI เอง

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37220667)

- การพึ่งพาเมตริกและ KPI เพียงอย่างเดียวเพื่อวัดความสําเร็จและประสิทธิภาพในองค์กรมีข้อจํากัด
- สิ่งสําคัญคือต้องรักษาสมดุลระหว่างเมตริกและสามัญสํานึกในการประเมินประสิทธิภาพ
- วิธีการทางเลือกเช่นการสื่อสารและข้อเสนอแนะเป็นประจําจะมีประสิทธิภาพนอกเหนือจากเมตริก

## [GPT-3.5 Turbo ปรับแต่งอย่างละเอียดและอัปเดต API](https://openai.com/blog/gpt-3-5-turbo-fine-tuning-and-api-updates)

- OpenAI ได้เปิดตัวคุณสมบัติใหม่ที่เรียกว่าการปรับแต่งอย่างละเอียดสําหรับรุ่น GPT-3.5 Turbo ซึ่งช่วยให้นักพัฒนาสามารถปรับแต่งสําหรับกรณีการใช้งานเฉพาะได้
- การทดสอบในช่วงต้นแสดงให้เห็นว่า GPT-3.5 Turbo รุ่นที่ปรับแต่งมาอย่างดีสามารถทํางานได้ดีกว่ารุ่น GPT-4 พื้นฐานในบางงานซึ่งบ่งบอกถึงประสิทธิภาพที่ดีขึ้น
- การปรับแต่งอย่างละเอียดให้ประโยชน์เช่นความสามารถในการบังคับเลี้ยวที่เพิ่มขึ้นการจัดรูปแบบเอาต์พุตที่เชื่อถือได้และความสามารถในการปรับแต่งโทนเสียง โมเดลที่อัปเดตยังรองรับการจัดการโทเค็น 4k และช่วยประหยัดต้นทุนผ่านการลดขนาดที่รวดเร็ว
- OpenAI ได้เปิดเผยรายละเอียดราคาสําหรับการปรับแต่งอย่างละเอียดและประกาศความพร้อมใช้งานของรุ่น GPT-3 ที่อัปเดต
- OpenAI ได้ใช้ระบบการกลั่นกรองเพื่อให้แน่ใจว่ามีการปรับใช้โมเดลที่ปรับแต่งอย่างปลอดภัยและมีแผนที่จะเปิดตัวอินเทอร์เฟซผู้ใช้ที่ปรับแต่งอย่างละเอียดในไม่ช้า

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37227139)

- OpenAI ได้ทําการอัปเดตโมเดล GPT-3.5 Turbo อย่างละเอียดและ API ทําให้โมเดลสามารถเรียนรู้รูปแบบและข้อมูลเฉพาะได้
- การปรับแต่งอย่างละเอียดมีข้อ จํากัด และอคติที่อาจเกิดขึ้นดังนั้น OpenAI จึงแนะนําให้ใช้การดึงข้อมูลสําหรับงานบางอย่างแทน
- การปรับแต่งอย่างละเอียดอาจมีประสิทธิภาพน้อยกว่า แต่ถูกกว่าวิธีการแบบเดิม ๆ แต่อาจไม่ให้ผลลัพธ์ที่ดีกว่าเสมอไปและสามารถแนะนําอคติได้ OpenAI วางแผนที่จะเปิดตัวการปรับแต่ง GPT4 ในอนาคต

- แพคเกจสามารถรวมเข้ากับแอปพลิเคชัน Python เพื่อวัตถุประสงค์ในการปรับแต่งและการพัฒนาส่วนบุคคล

## [Experian ปรับ $ 650K โดย FTC และ DOJ สําหรับอีเมลขยะโดยไม่มีการเลือกไม่รับ](https://www.theverge.com/2023/8/22/23841646/experian-fined-spam-emails-ftc-doj)

- Experian ซึ่งเป็นหน่วยงานรายงานเครดิตถูกปรับ 650,000 ดอลลาร์โดยรัฐบาลสหรัฐฯ เนื่องจากละเมิดกฎหมายสแปมและไม่ได้ให้ตัวเลือกการเลือกไม่รับอีเมลการตลาด
- Federal Trade Commission (FTC) พบว่า Experian ส่งอีเมลการตลาดที่หลอกลวงไปยังลูกค้าที่มีสมาชิกตรวจสอบเครดิตฟรีขาดการแจ้งเตือนที่ชัดเจนและกลไกการเลือกไม่รับ
- เป็นผลให้กระทรวงยุติธรรม (DOJ) ได้รับคําสั่งถาวรต่อ Experian ห้ามข้อความบางประเภทและต้องการตัวเลือกการเลือกไม่ใช้อย่างชัดเจนในอีเมลการตลาด

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37227172)

- Experian ซึ่งเป็น บริษัท รายงานเครดิตถูกปรับ $ 650,000 โดย FTC และ DOJ สําหรับการส่งอีเมลสแปมที่ไม่มีตัวเลือกไม่เข้าร่วม
- นักวิจารณ์เชื่อว่าค่าปรับนั้นน้อยเกินไปเมื่อเทียบกับรายได้ของ Experian ซึ่งชี้ให้เห็นว่าบทลงโทษที่มากขึ้นเป็นสิ่งจําเป็นเพื่อกีดกันการกระทําที่คล้ายคลึงกันโดย บริษัท อื่น ๆ
- การไม่มีตัวเลือกการเลือกไม่ใช้ในอีเมลขยะเป็นการละเมิดกฎระเบียบซึ่งนําไปสู่การจัดเก็บค่าปรับโดยหน่วยงานกํากับดูแล

## [การบันทึกแบบมีโครงสร้างด้วยสโลแกน](https://go.dev/blog/slog)

- แพคเกจ log/slog ใหม่ใน Go 1.21 แนะนําการบันทึกที่มีโครงสร้างไปยังไลบรารีมาตรฐาน
- บันทึกที่มีโครงสร้างใช้คู่คีย์-ค่าเพื่อการแยกวิเคราะห์ การกรอง การค้นหา และการวิเคราะห์ที่ง่ายดาย
- แพคเกจนี้มีจุดมุ่งหมายเพื่อสร้างเฟรมเวิร์กมาตรฐานสําหรับการบันทึกแบบมีโครงสร้างในระบบนิเวศ Go โดยมีระดับบันทึกที่แตกต่างกันและตัวเลือกรูปแบบเอาต์พุตที่ปรับแต่งได้

### [ปฏิกิริยา](https://news.ycombinator.com/item?id=37224651)

- การอภิปรายมุ่งเน้นไปที่การบันทึกแบบมีโครงสร้างใน Go และวิธีการต่างๆที่ใช้โดยการบันทึกไลบรารี
- ประโยชน์ของการบันทึกที่มีโครงสร้างและความสําคัญของรูปแบบการบันทึกมาตรฐานและ API จะถูกเน้น
- ผู้เข้าร่วมแบ่งปันมุมมองของพวกเขาเกี่ยวกับการจัดการข้อผิดพลาดและความสําคัญของกองการโทร มีการกล่าวถึงเครื่องมือที่มีอยู่บางส่วนสําหรับการสร้างภาพบันทึกพร้อมกับคําแนะนําในการเพิ่มประสิทธิภาพการบันทึก
