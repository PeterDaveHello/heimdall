## [בדיקות WebGPU פתוחות כעת בתצוגה מקדימה של טכנולוגיית Safari](https://webkit.org/blog/14879/webgpu-now-available-for-testing-in-safari-technology-preview/)

- WebGPU, ממשק API חדש המאפשר גרפיקה תלת-ממדית וחישובים בעלי ביצועים גבוהים באינטרנט, זמין כעת לבדיקה בתצוגה מקדימה של טכנולוגיית Safari.
- WebGPU מאפשר להשתמש בפונקציונליות GPU בתוכן אינטרנט וכתוב ב- JavaScript.
- כדי להפעיל את WebGPU, דגלי תכונות מסוימים צריכים להיות מופעלים בהעדפות Safari.
- הגישה ל-WebGPU API מתבצעת באמצעות JavaScript, בדומה ל-WebGL.
- רכיבים שונים כגון התקני GPU, הקשר בד ציור, צינורות עיבוד ושיחות ציור משמשים ליצירה ולהגדרה של פלטי גרפיקה.
- WebGPU מציג גם שפת הצללה חדשה בשם WGSL.
- אנו מעודדים את המשתמשים לנסות את WebGPU ולדווח על באגים או בעיות שהם נתקלים בהם.

### [תגובות](https://news.ycombinator.com/item?id=38737028)

- הדיון מכסה מגוון נושאים של טכנולוגיית אינטרנט כגון יישום WebGPU בספארי, שימוש בפורמטים בינאריים ב- WebGPU ושילוב תמיכה ב- WGSL ב- Xcode.
- הוא כולל גם דיונים על הפעלת מודלי שפה בטלפונים ובדפדפנים, חוויות משתמש עם Safari ו-Chrome, והביצועים ונתח השוק של Safari.
- יש ספקולציות לגבי המהלכים האסטרטגיים של אפל, תביעות פוטנציאליות ופעולות הגבלים עסקיים, וההשקעה שלהם בממשקי API אינטרנטיים. דפדפני האינטרנט Epiphany ו- Midori מוזכרים גם הם, יחד עם הבעיות שלהם.
