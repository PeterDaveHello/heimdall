## [Code Llama, מודל שפה גדול ומתקדם לקידוד](https://ai.meta.com/blog/code-llama-large-language-model-coding/)

- Code Llama הוא מודל שפה גדולה חדש (LLM) שתוכנן במיוחד עבור משימות קידוד.
- הוא מסוגל ליצור קוד ושפה טבעית על קוד מתוך קוד או הנחיות בשפה טבעית.
- Code Llama זמין בשלושה דגמים: Code Llama, Codel Llama - Python ו-Code Llama - Instruct.
- זה עלה על LLMs אחרים הזמינים לציבור על משימות קידוד.
- הדגמים בנויים על גבי Llama 2 והם בחינם לשימוש מחקרי ומסחרי.
- ל-Code Llama יש פוטנציאל לשפר את זרימות העבודה של המפתחים ולהפוך את הקידוד לנגיש יותר.
- הוא תומך בשפות תכנות פופולריות וניתן להשתמש בו להשלמת קוד וניפוי באגים.
- מודגש שימוש בטוח ואחראי בקוד לאמה, והמודלים עברו הערכות בטיחות.
- שחרורו של Code Llama מעודד חדשנות ושיתוף פעולה בקהילת הבינה המלאכותית.

- הביצועים והרישוי של מודל שנקרא "קוד לא טבעי לאמה" נדונים, לצד ההשפעות האפשריות של התקדמות AI, כגון ביטחון תעסוקתי ושליטה אנושית.
- המשתתפים מביעים התרגשות ממודלים של שפה שמחוללים מהפכה בתעשייה, אך מכירים במגבלות, כולל חששות לגבי ניפוח ביצועים פוטנציאלי באמצעות נתוני אימונים.

## [Code Llama, מודל שפה גדול ומתקדם לקידוד](https://ai.meta.com/blog/code-llama-large-language-model-coding/)

- Code Llama הוא מודל שפה גדולה חדשני (LLM) שתוכנן במיוחד עבור משימות קידוד.
- זה יכול ליצור קוד ושפה טבעית על קוד מבוסס על הנחיות.
- ל-Code Llama שלושה מודלים: Code Llama (מודל הקוד הבסיסי), Code Llama - Python (המתמחה בפייתון) ו-Code Llama - Instruct (מכוונן היטב להוראות שפה טבעית).
- בבדיקות בנצ'מרק, Code Llama השיג ביצועים טובים יותר מבוגרי LLM אחרים הזמינים לציבור במשימות קוד.
- הוא תומך בשפות תכנות פופולריות וניתן להשתמש בו להשלמת קוד וניפוי באגים.
- ל-Code Llama יש גדלי דגם שונים כדי לספק דרישות השהיה ספציפיות.
- יש לו פוטנציאל לשפר את זרימות העבודה של הקידוד ולהפוך את הקידוד לנגיש יותר למתחילים.
- Code Llama משוחרר תחת רישיון קהילה, והמשתמשים חייבים לדבוק במדיניות השימוש המקובל.
- המודל עבר הערכות בטיחות וננקטו אמצעי זהירות כדי להפחית סיכונים.
- אנו מעודדים מפתחים להעריך את המודל באמצעות אמות מידה להערכה ספציפיות לקוד ולבצע מחקרי בטיחות.
- המטרה היא להמשיך לפתח בינה מלאכותית גנרטיבית לקידוד על ידי מינוף Llama 2 והשראה לאחרים ליצור כלים חדשניים.

## [Hugging Face מגייסת 235 מיליון דולר ממשקיעים כמו סיילספורס ואנבידיה](https://techcrunch.com/2023/08/24/hugging-face-raises-235m-from-investors-including-salesforce-and-nvidia/)

- Hugging Face, סטארט-אפ בתחום הבינה המלאכותית, השיג גיוס של 235 מיליון דולר בסבב D, בהשתתפות משקיעים בולטים כמו סיילספורס ואנבידיה (Nvidia).
- סבב הגיוס הכפיל את השווי של Hugging Face ל-4.5 מיליארד דולר מאז מאי 2022.
- Hugging Face מציעה כלי אירוח ופיתוח של מדעי הנתונים, כולל רכזת מאגר קוד AI, מודלים ומערכי נתונים, כמו גם אפליקציות אינטרנט ליישומים המופעלים על ידי בינה מלאכותית.
- החברה מספקת ספריות ופונקציות בתשלום כגון AutoTrain, Inference API ו- Infinity.
- הכספים שיגויסו ישמשו את Hugging Face להרחבת התמיכה שלה במחקר, יזמות וסטארט-אפים.

### [תגובות](https://news.ycombinator.com/item?id=37248895)

- Hugging Face, פלטפורמת אירוח מודלים של בינה מלאכותית, גייסה לאחרונה מימון של 235 מיליון דולר ממשקיעים כולל סיילספורס ואנבידיה.
- התוכניות העתידיות של החברה כוללות מונטיזציה של השירותים שלה, מה שעורר חששות לגבי סיכונים לאקוסיסטם של בינה מלאכותית והצורך להפחית את התלות ב-Hugging Face.
- מתקיימים דיונים לגבי אסטרטגיות מונטיזציה פוטנציאליות, השוואות לפלטפורמות אחרות והקיימות של משאבים חינמיים.
- ישנם ויכוחים סביב המודל העסקי של מכירת AI/ML ובלבול לגבי ההצעות שמספקת Hugging Face.
- החברה מתכוונת להשתמש במימון כדי להרחיב את הצוות שלה ולהמשיך לפתח את הפלטפורמה שלה.

## [עקיפת Bitlocker באמצעות מנתח לוגיקה זול במחשב נייד של Lenovo](https://www.errno.fr/BypassingBitlocker.html)

- המחבר מציג שיטה לעקיפת הצפנת BitLocker במחשב נייד של Lenovo באמצעות מנתח לוגיקה בעלות נמוכה.
- הארכיטקטורה של BitLocker והאחסון של מפתח ההצפנה ב- TPM מוסברים.
- תהליך הלכידה והפענוח של חילופי TPM לאחזור מפתח ההצפנה מפורט, יחד עם מגבלות השיטה והמלצות לשיפור האבטחה.

### [תגובות](https://news.ycombinator.com/item?id=37249623)

- הדיון מתמקד בפגיעויות ובמגבלות של הצפנת Bitlocker של מיקרוסופט במחשבים ניידים של Lenovo.
- משתמשים מביעים חששות לגבי האבטחה של TPM והפוטנציאל להתקפות.
- הנושאים כוללים גם את הגדרות ברירת המחדל של Bitlocker, את החשיבות של מפתחות שחזור גיבוי ואת ההיתכנות של יירוט מפתחות הצפנה.
- מערכות הצפנה אחרות כמו fTPM ו- LUKS מוזכרות.
- הדיונים נוגעים לשיטות עיבוד ופענוח אותות, כמו גם למגבלות השימוש ב- TPM נפרד.
- השיחה עוסקת גם בהצפנה מבוססת קושחה SSD, אישורי חומרה ודרישות TPM במערכות הפעלה כמו Windows 11.

## [FreeBSD על Firecracker](https://www.usenix.org/publications/loginonline/freebsd-firecracker)

- המחבר מספר על ניסיונם בהסבה מוצלחת של FreeBSD להפעלה על צג המכונה הווירטואלית Firecracker.
- למרות שהתמודדו עם אתגרים, הם הצליחו להתגבר עליהם ולהתקדם משמעותית באופטימיזציה של FreeBSD כדי לשפר את זמן האתחול שלה ב- Firecracker.
- המחבר מזכיר גם תוכניות עתידיות, כולל הפרדת התמיכה ב-Xen והסבה פוטנציאלית של Firecracker כך שירוץ על FreeBSD.

### [תגובות](https://news.ycombinator.com/item?id=37253035)

- FreeBSD פועלת ביעילות ובמהירות על פלטפורמת המיקרו-VM של Firecracker.
- Firecracker מציע את היתרונות של מכונה שלמה וסביבת פיתוח יעילה.
- המאמר בוחן את השימוש ב-gvisor וב-Hypervisors, אופטימיזציה של ליבת לינוקס עבור מחזורי חיים קצרים של VM, ואת היתרונות של טכנולוגיות כמו Lambda ו-Firecracker בהשוואה לשיטות מסורתיות.

## [הגנה מפני הוכחת עבודה עבור שירותי בצל](https://blog.torproject.org/introducing-proof-of-work-defense-for-onion-services/)

- Tor יישמה הגנת הוכחת עבודה (PoW) עבור שירותי בצל כדי למנוע התקפות מניעת שירות (DoS).
- קשרי לקוחות נכנסים נדרשים כדי לפתור חידה, להוכיח את האותנטיות שלהם ולהרתיע תוקפים.
- מנגנון PoW מתעדף תעבורה אמיתית והופך התקפות בקנה מידה גדול ללא מעשיות, מה שמשפר את האבטחה והאמינות של רשת Tor.

### [תגובות](https://news.ycombinator.com/item?id=37255079)

- הדיון מתמקד בשימוש בהוכחת עבודה (PoW) ברשת Tor כדי להגן על שירותי בצל מפני התקפות.
- חששות סביבתיים, אנונימיות ופתרונות פוטנציאליים כמו הוכחות עבודה הקשורות לזהות CPU נדונים.
- בחינת האפשרות להשתמש ב-Tor כרשת אספקת תוכן ולמנף אלגוריתמים של PoW כדי להגן על אתרים.
