## 25 березня 2023 року

### GitHub оновив ключ хоста RSA SSH після витоку даних

- GitHub оновив ключ хоста RSA SSH після витоку приватного ключа в публічному сховищі
- Клієнтам, які використовують ключі ECDSA або Ed25519, не потрібно вживати ніяких заходів, але тим, хто використовує RSA, необхідно оновити свій файл ~/.ssh/known_hosts
- GitHub ще не запровадив механізм відкликання SSH-ключів, що може стати потенційною проблемою в разі їх компрометації
- Наявність реальної моделі ризиків/загроз має вирішальне значення для побудови розумних засобів контролю безпеки
- Автори Git'у повинні підписувати свої комміти власним приватним ключем для забезпечення цілісності коду, а не довіряти хостинг-провайдерам
- GitHub випадково опублікував свій ключ, підірвавши принцип довіри і поставивши під загрозу всі комунікації з моменту заснування
- GitHub заохочує вдосконалення сертифікатів, що використовуються для входу в сховища коду
- GitHub та GitLab підтримують двофакторну автентифікацію (2FA) через свої інтерфейси командного рядка (CLI)
- Нещодавній інцидент висвітлює потенційні недоліки безпеки в багатьох фінтех-компаніях, включаючи сліпу довіру до ключів хостів GitHub
- GitHub ненадовго розкрив приватний ключ через нещасний випадок, але вжив проактивних заходів для відкликання та перерозподілу нових ключів і сертифікатів, а також запровадив посилений контроль.
- [Обговорити на Discussion Service](http://news.ycombinator.com/item?id=35285390) або [Читати оригінальний текст](https://github.blog/2023-03-23-we-updated-our-rsa-ssh-host-key/).

### Суд США заборонив програму книговидачі Інтернет-архіву

- Інтернет-архів (IA) порушив закон про авторське право, надаючи цифрові копії книг, захищених авторським правом, без дозволу, згідно з рішенням американського судді.
- Інтернет-архів стверджував, що не підпадає під дію закону через "добросовісне використання", але суддя не знайшов нічого трансформуючого в копіюванні та наданні книг напрокат.
- ІА зобов'язали припинити розповсюдження книг, захищених авторським правом, без дозволу
- Судовий позов видавництв Hachette, HarperCollins та Penguin Random House проти практики сканування та цифрової позички ІА
- IA виконає рішення суду та припинить програму кредитування
- Рішення може вплинути на користувачів, які покладаються на МА для доступу до текстів, недоступних в інших місцях, особливо на тих, хто вважає традиційні бібліотеки недоступними.
- [Обговорення на Discussion Service](http://news.ycombinator.com/item?id=35297117) або [Читати оригінальний текст](https://file770.com/judge-decides-against-internet-archive/).

### Продовольча інфляція у Великобританії досягла найвищого рівня з 1977 року, що викликає занепокоєння

- Продовольча інфляція у Великобританії в лютому зросла до 18,2%, що є найвищим показником з 1977 року через підвищення вартості овочів, зростання цін на зерно, деякі види білків, яйця та олію.
- Британські супермаркети борються з дефіцитом продуктів харчування, високими цінами на енергоносії та глобальними проблемами в ланцюжку поставок, що призводить до порожніх полиць і зростання цін.
- Зростання цін на продукти харчування викликає занепокоєння у вразливих груп населення, оскільки деякі люди намагаються придбати достатню кількість їжі.
- Зростання цін на енергоносії та перебої в ланцюгах постачання є одними з факторів, які називають основними причинами інфляційного тиску, що спричиняє зростання цін на продукти харчування в багатьох країнах.
- Першопричиною інфляції є підвищення цін на енергоносії, оскільки енергія впливає на виробництво і транспортування, що в свою чергу впливає на ціни на продукти харчування.

### Витік і злом даних в ChatGPT і GPT-3 від OpenAI - досвід витоку і злому

- OpenAI також зазнав порушення безпеки, що призвело до витоку даних клієнтів, включаючи імена та прізвища, адреси електронної пошти та налаштування використання API.

- OpenAI вжила заходів для усунення інцидентів, включаючи виправлення помилки в клієнтській бібліотеці Redis, вдосконалення своїх систем, зміну всіх ключів і токенів API, а також поліпшення процесів тестування, тверджень, оповіщення в журналах і налагодження.
